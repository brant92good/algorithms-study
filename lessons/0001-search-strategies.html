Lesson 1 · Ch6 Searching Strategies

The Four Search Strategies

Exam goal: hand-trace a search and name the strategy. All four below are the same loop with one knob turned — which open node do I expand next? Master that knob and you can trace any of them, and tell them apart in a short-answer question.

0 · What are we even searching?

Start with nothing assumed. A hard problem is one where you can't just compute the answer in one shot — you have to try possibilities. The standard trick is to turn the problem into a tree of partial decisions and search it.

That tree is called a solution tree (or state-space tree): each node is one state of the problem — a snapshot of "here's what I've decided so far." The root is "nothing decided yet." Going down one level means making one more decision, which produces a new state. A leaf at the bottom is a fully-decided candidate. A goal node is a state that solves the problem; solving = finding a goal node. We turn hard problems into trees because once it's a tree, the only question left is the easy-sounding "which node do I look at next?" — and that single choice is the whole game.

Two pieces of vocabulary you'll see everywhere:

  • Expanding a node = take a node, generate all its children (the states reachable by making one more decision), and put those children into your to-do list. "Expand A" means "produce A's children."
  • The open list = the to-do list of nodes that have been discovered but not yet expanded. The search loop keeps pulling one node off this list, checking if it's the goal, and if not, expanding it (which adds its children back to the open list).
The evaluation function f(n). Some strategies need to judge how promising a node looks. f(n) is a number we attach to each node n that estimates how good it is — here, lower means more promising (think "estimated distance still to go"). It's only an estimate, computed cheaply from the state; the strategy uses it to decide which open node to pick.

A concrete problem: the 8-puzzle

The classic example is the 8-puzzle: a 3×3 board holding 8 numbered tiles (1–8) and one blank square. A move slides a tile that's next to the blank into the blank's spot (equivalently, the blank swaps with an adjacent tile). You start from some scrambled board and want to reach a fixed goal arrangement.

start goal 2 8 3 1 2 3 1 _ 4 → 8 _ 4 7 6 5 7 6 5 (_ = blank; slide a neighbour of the blank into it)

Each board is a state; expanding a board produces the boards you can reach in one slide. For the 8-puzzle the evaluation function is

f(n) = d(n) + w(n) d(n) = depth of n = number of slides made to reach this board w(n) = number of tiles sitting in the wrong place vs the goal

So f rewards being close to the goal (few misplaced tiles) without having wandered too far (shallow depth). Lower f = more promising. This is the exact f(n) you'll see hill-climbing and best-first use below.

1 · Everything is a tree walk

Now the unifying picture. Whatever the problem — SAT, the 8-puzzle, a tour — it becomes that same solution tree: the root is "nothing decided," each level fixes one more decision, each leaf is a full candidate, and solving means reaching a goal node. Every strategy keeps the open list of discovered-but-not-yet-expanded nodes and repeats:

repeat: pick one open node n ← THE KNOB if n is a goal → stop else expand n: add its children to the open list

The four strategies differ only in "pick one open node."

2 · The knob

Key contrast for the exam: hill climbing only ever looks at the children of where it is now — so it can march into a dead end and get stuck. Best-first keeps every open node on the table and always jumps to the globally most promising one, so it recovers. Both use the estimate f(n); only their scope differs.

3 · Trace it yourself

Tree below. Numbers in (…) are f(n) — lower = more promising. Children are listed left-to-right. Goal = G.

A ┌─────┼─────┐ B(5) C(2) D(4) ┌─┐ ┌─┐ E(6)F(3) G(1)H(7) ← G is the goal

Work each out on paper, then open the answer. Give the order nodes are expanded.

DFS (deepest first, left-to-right)

A → B → E → F → C → D → G ✓
Dives down the leftmost branch fully (B,E,F) before ever touching C or D. Ignores the f-values entirely.

BFS (shallowest first)

A → B → C → D → E → F → G ✓
Clears a whole level before the next. Also ignores f-values. Finds G after finishing level 1.

Hill climbing (greedy, children only)

A → C … stuck.
From A the best child is C(2). Expand C — it's a leaf and not the goal. Hill climbing has no other open nodes in view, so it dead-ends (must backtrack/restart). The greedy step toward C(2) was a local trap.

Best-first (global best open node)

A → C → D → G ✓
Expand A → open {B5, C2, D4}; take C2. C is a leaf, not goal → open {B5, D4}; take D4 → adds G1,H7 → open {B5,G1,H7}; take G1 = goal. It explored C just like hill climbing, but kept B and D on the list and recovered. Fewest expansions here.
