The Four Search Strategies四種搜尋策略

Exam goal: hand-trace a search and name the strategy. All four below are the same loop with one knob turned — which open node do I expand next? Master that knob and you can trace any of them, and tell them apart in a short-answer question.考試會考你：用手追一遍搜尋過程，然後說出它是哪一種策略。其實下面這四種根本是同一個迴圈，只是轉了一個旋鈕而已——下一步我要展開哪個 open node？把這個旋鈕搞懂，你就能追任何一種，簡答題也能一眼分出來。

0 · What are we even searching?0 · 我們到底在搜尋什麼？

Start with nothing assumed. A hard problem is one where you can't just compute the answer in one shot — you have to try possibilities. The standard trick is to turn the problem into a tree of partial decisions and search it.先什麼都別假設。所謂難題（hard problem），就是你沒辦法一步把答案算出來，只能一個一個可能去試的那種問題。最常見的做法是：把問題拆成一棵「一步一步做決定」的樹，再去搜尋這棵樹。

That tree is called a solution tree (or state-space tree): each node is one state of the problem — a snapshot of "here's what I've decided so far." The root is "nothing decided yet." Going down one level means making one more decision, which produces a new state. A leaf at the bottom is a fully-decided candidate. A goal node is a state that solves the problem; solving = finding a goal node. We turn hard problems into trees because once it's a tree, the only question left is the easy-sounding "which node do I look at next?" — and that single choice is the whole game.這棵樹叫做 solution tree（解答樹），也叫 state-space tree（狀態空間樹）。樹上每個 node 就是問題的一個 state（狀態），你可以想成是「我目前為止做了哪些決定」的一張快照。root 就是「什麼都還沒決定」。往下走一層，就是再多做一個決定，得到一個新的 state。最底下的 leaf 是一個整個決定完的候選解。goal node 則是真正能解掉問題的 state；換句話說，解題 = 找到一個 goal node。為什麼要硬把難題弄成一棵樹？因為一旦變成樹，剩下要煩惱的就只有一句聽起來超簡單的話：「我下一步要看哪個 node？」——而這一個選擇，就是整場遊戲的全部。

Two pieces of vocabulary you'll see everywhere:有兩個詞你之後會一直看到：

  • Expanding a node = take a node, generate all its children (the states reachable by making one more decision), and put those children into your to-do list. "Expand A" means "produce A's children."Expanding a node（展開一個 node）＝拿一個 node 出來，把它所有的 children（也就是「再多做一個決定」能走到的 states）全部生出來，再丟進你的待辦清單。簡單講，「expand A」就是「把 A 的 children 生出來」。
  • The open list = the to-do list of nodes that have been discovered but not yet expanded. The search loop keeps pulling one node off this list, checking if it's the goal, and if not, expanding it (which adds its children back to the open list).open list＝就是那份待辦清單，裡面放的是已經被發現、但還沒展開的 nodes。搜尋迴圈就是不停地從清單裡撈一個 node 出來，看看它是不是 goal；不是的話就展開它，而展開又會把它的 children 加回 open list。這樣一直循環下去。
The evaluation function f(n). Some strategies need to judge how promising a node looks. f(n) is a number we attach to each node n that estimates how good it is — here, lower means more promising (think "estimated distance still to go"). It's only an estimate, computed cheaply from the state; the strategy uses it to decide which open node to pick.評估函數 f(n)。有些策略需要判斷一個 node 看起來「多有搞頭」。f(n) 就是我們幫每個 node n 貼上的一個數字，用來估計它有多好——這裡的規則是越小越有希望（你可以想成「估計還要走多遠才到」）。注意它只是個估計值，是從 state 很快算出來的，不保證準。策略就靠這個數字來決定要挑哪個 open node。

A concrete problem: the 8-puzzle舉個實際的例子：8-puzzle

The classic example is the 8-puzzle: a 3×3 board holding 8 numbered tiles (1–8) and one blank square. A move slides a tile that's next to the blank into the blank's spot (equivalently, the blank swaps with an adjacent tile). You start from some scrambled board and want to reach a fixed goal arrangement.最經典的就是 8-puzzle：一個 3×3 的盤面，上面擺了 8 個有編號的方塊（1–8），再加一個blank（空格）。所謂走一步，就是把 blank 旁邊的某個方塊滑到 blank 的位置（其實就是 blank 跟旁邊的方塊互換）。你從一個被打亂的盤面開始，目標是把它喬成固定的 goal 排列。

start goal 2 8 3 1 2 3 1 _ 4 → 8 _ 4 7 6 5 7 6 5 (_ = blank; slide a neighbour of the blank into it)

Each board is a state; expanding a board produces the boards you can reach in one slide. For the 8-puzzle the evaluation function is每個盤面就是一個 state；展開一個盤面，就是把「再滑一步能走到的所有盤面」都生出來。對 8-puzzle 來說，評估函數長這樣：

f(n) = d(n) + w(n) d(n) = depth of n = number of slides made to reach this board w(n) = number of tiles sitting in the wrong place vs the goal

So f rewards being close to the goal (few misplaced tiles) without having wandered too far (shallow depth). Lower f = more promising. This is the exact f(n) you'll see hill-climbing and best-first use below.所以 f 喜歡的是「離 goal 很近（放錯的方塊很少）」、又「沒繞太遠（深度淺）」這種盤面。f 越小就越有希望。等一下 hill-climbing 跟 best-first 用的，就是這同一個 f(n)。

1 · Everything is a tree walk1 · 說穿了全都是在走一棵樹

Now the unifying picture. Whatever the problem — SAT, the 8-puzzle, a tour — it becomes that same solution tree: the root is "nothing decided," each level fixes one more decision, each leaf is a full candidate, and solving means reaching a goal node. Every strategy keeps the open list of discovered-but-not-yet-expanded nodes and repeats:現在來看那張把全部串起來的大圖。不管你碰到的是什麼問題——SAT、8-puzzle、還是跑一趟 tour——最後都會被整理成同一棵 solution tree：root 是「什麼都還沒決定」，每往下一層就釘死多一個決定，每個 leaf 都是一個完整的候選解，而解題就是走到某個 goal node。每一種策略都會維護那份「已發現、但還沒展開」的 open list，然後重複做這幾件事：

repeat: pick one open node n ← THE KNOB if n is a goal → stop else expand n: add its children to the open list

The four strategies differ only in "pick one open node."重點是：這四種策略唯一的差別，就只在「pick one open node（挑一個 open node）」這一步而已。

One wording note: the loop goal-tests every node the moment it is picked, and only nodes that fail the test get expanded (children generated). So when a trace below says a goal like G is "expanded" or "stop/done," it is really being goal-tested and found — the final goal node never gets its children generated.補一個用語上的細節：迴圈是在挑出每個 node 的當下就先做 goal-test，只有沒通過測試的 node 才會被展開（生 children）。所以下面的 trace 寫某個 goal（例如 G）被「展開」或「stop/done」時，它其實是被goal-test 到、然後找到了——最後那個 goal node 並不會真的去生 children。

2 · The knob2 · 那個旋鈕

In the View column below, local means the strategy only looks at a small nearby part of the open list (e.g. just the current node's children), while global means it compares across all open nodes at once. The note under the table unpacks why that difference matters.下面那欄 View 講的是「視野」：local 是說這個策略只看 open list 附近一小塊（例如只看目前 node 的 children），global 則是把所有 open nodes 一起拿來比。為什麼這個差別很關鍵，表格底下那則 note 會說清楚。

BFSshallowest open nodequeuelocal

Key contrast for the exam: hill climbing only ever looks at the children of where it is now — so it can march into a dead end and get stuck. Best-first keeps every open node on the table and always jumps to the globally most promising one, so it recovers. Both use the estimate f(n); only their scope differs.考試最愛考的對比：hill climbing 從頭到尾就只看「我現在所在位置的 children」——所以它很容易一路衝進死巷然後卡住。Best-first 不一樣，它把每一個 open node 都留在檯面上，永遠跳去全域裡最有希望的那個，所以它有辦法繞回來。兩個用的都是同一個估計值 f(n)，差就差在它們看的範圍（scope）不一樣。

3 · Trace it yourself3 · 自己動手追一遍

Tree below. Numbers in (…) are f(n) — lower = more promising. Children are listed left-to-right. Goal = G.樹在下面。(…) 裡的數字就是 f(n)——越小越有希望。Children 是由左到右排的。Goal = G。

A ┌─────┼─────┐ B(5) C(2) D(4) ┌─┐ ┌─┐ E(6)F(3) G(1)H(7) ← G is the goal

These f-values are supplied directly for the exercise (not computed from any 8-puzzle board) — so don't try to match them to f(n)=d(n)+w(n). The root A has no f because it is always the starting node you expand first, so nothing ever needs to compare it.這裡的 f 值是直接給你練習用的（不是從哪個 8-puzzle 盤面算出來的），所以別硬要把它們對到 f(n)=d(n)+w(n)。root A 沒有 f，是因為它永遠是你第一個展開的起點，根本不會有人需要拿它去跟別人比。

Work each out on paper, then open the answer. Give the order nodes are expanded.每一種都先自己在紙上算一遍，再打開答案對。要寫出的是 nodes 被展開的順序。

DFS (deepest first, left-to-right)DFS（最深優先，由左到右）

A → B → E → F → C → D → G ✓
Dives down the leftmost branch fully (B,E,F) before ever touching C or D. Ignores the f-values entirely. Why left-to-right from a stack? A stack pops the last thing pushed, so to make the leftmost child come out first you push the children in reverse order (rightmost first): push D, then C, then B — now B is on top and pops next.它會先沿著最左邊那條分支一路鑽到底（B、E、F），鑽完才回頭碰 C 跟 D。f 值它根本不看。那為什麼用 stack 還會是由左到右？因為 stack 是後進先出，最晚推進去的最先被拿出來，所以你要把 children 反過來推（最右邊的先推）：先推 D、再推 C、最後推 B——這樣 B 在最上面，下一個就會先彈出 B。

BFS (shallowest first)BFS（最淺優先）

A → B → C → D → E → F → G ✓
Clears a whole level before the next. Also ignores f-values. G is a child of D, so it sits at level 2 (A=level 0; B,C,D=level 1; E,F,G,H=level 2). BFS finds G only after finishing level 1 and reaching level 2 — G is the last node expanded.它是把一整層清完才往下一層走。一樣不理 f 值。G 是 D 的 child，所以它在第 2 層（A 是第 0 層；B、C、D 是第 1 層；E、F、G、H 是第 2 層）。所以 BFS 要先把第 1 層做完、再走到第 2 層才會找到 G——G 是最後一個被展開的 node。

Hill climbing (greedy, children only)Hill climbing（貪婪，只看 children）

A → C … stuck.
From A the best child is C(2). Expand C — it's a leaf and not the goal. Hill climbing has no other open nodes in view, so it dead-ends (must backtrack/restart). The greedy step toward C(2) was a local trap.從 A 看出去，最好的 child 是 C(2)，那就展開 C 吧——結果 C 是個 leaf，又不是 goal。偏偏 hill climbing 視野裡又沒有別的 open nodes 可以退，於是直接卡死（只能 backtrack 或重來）。當初貪心地往 C(2) 走，其實就踩到一個局部（local）陷阱了。

Best-first (global best open node)Best-first（全域最佳的 open node）

A → C → D → G ✓
Expand A → open {B5, C2, D4}; take C2. C is a leaf, not goal → open {B5, D4}; take D4 → adds G1,H7 → open {B5,G1,H7}; take G1 = goal. It explored C just like hill climbing, but kept B and D on the list and recovered. Fewest expansions here. Note that B(5) stayed on the open list the whole time and at the last step was compared against the newly-added G(1) and H(7) — which are one level deeper. Best-first ranks purely by f, regardless of depth: G(1) wins over the leftover B(5) even though B sits higher up. That's why it can "jump" to a node out of left-to-right order.展開 A → open {B5, C2, D4}，挑 C2。C 是 leaf、不是 goal → open {B5, D4}，挑 D4 → 把 G1、H7 加進來 → open {B5,G1,H7}，挑 G1 = goal。注意它跟 hill climbing 一樣有去碰 C，但因為它把 B 跟 D 留在清單上，所以卡住之後還繞得回來。這題它展開的次數最少。再注意一點：B(5) 從頭到尾都留在 open list 上，到最後一步還被拿去跟剛加進來、深了一層的 G(1)、H(7) 一起比。Best-first 純粹照 f 排，不管深度：就算 B 在上面那層，G(1) 還是贏過沒被挑走的 B(5)。這就是為什麼它能不照由左到右的順序、直接「跳」去某個 node。
