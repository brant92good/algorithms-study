0 · What is this even about?0 · 這章到底在講什麼？

This chapter is a vocabulary for sorting problems into easy and seemingly-hard. Before any of the famous letters (P, NP, …) mean anything, you need five plain ideas. Read this section slowly — everything after it is just these ideas dressed up.這章是一套把問題分成容易與看起來很難的詞彙。在那些有名的字母（P、NP、…）有意義之前，你需要先懂五個樸素的概念。這一節慢慢讀 — 後面所有東西都只是這些概念換上盛裝而已。

Decision problem vs. optimization problem. A decision problem is any question whose answer is just Yes or No. An optimization problem asks for the best value — the smallest, cheapest, or largest something. Optimization is harder, because "best" implies you also know the answer is achievable. The whole NP theory is built only on decision problems, so we always rephrase an optimization problem as a Yes/No one.Decision problem 與 optimization problem。一個 decision problem（判定問題）是任何答案只有 Yes 或 No 的問題。一個 optimization problem（最佳化問題）則要求最好的值 — 最小、最便宜或最大的某個東西。最佳化比較難，因為「最好」意味著你還要知道這個答案是可達成的。整套 NP 理論只建立在 decision 問題上，所以我們總是把最佳化問題改寫成 Yes/No 的形式。

Concrete example — the Travelling Salesperson Problem (TSP): a salesperson must visit every city exactly once and return to the start; each leg between two cities has a cost (distance), and a full round-trip is called a tour.具體例子 — Travelling Salesperson Problem (TSP)：一位業務員必須恰好造訪每個城市一次再回到起點；兩城市之間的每一段路有一個 cost（距離），而一趟完整的環遊稱為一個 tour。

Flavour類型The question it asks它問的問題
OptimizationWhat is the cheapest tour? (give me the number / the route)最便宜的 tour 是哪一條？（給我數字／路線）
DecisionIs there a tour of total cost ≤ C? (just Yes or No)存在總 cost ≤ C 的 tour 嗎？（只要 Yes 或 No）

Polynomial time = "efficient/tractable". When we say an algorithm runs in polynomial time, we mean its running time grows like a fixed power of the input size n — n, n², n³, … As n doubles, the cost goes up by a constant factor. This is what we informally call efficient or tractable: it scales. By contrast, exponential time means the cost grows like 2ⁿ — every extra element doubles the work, so it blows up and becomes hopeless even for modest n. "Hard" in this chapter means "seems to need exponential time."Polynomial time（多項式時間）=「有效率／可處理」。當我們說一個 algorithm 跑在 polynomial time，意思是它的執行時間像輸入大小 n 的某個固定次方那樣成長 — n、n²、n³、…。n 變成兩倍時，成本只乘上一個常數倍。這就是我們口語說的 efficient（有效率）或 tractable（可處理）：它能擴展。相對地，exponential time（指數時間）表示成本像 2ⁿ 那樣成長 — 每多一個元素工作量就翻倍，所以它會爆炸，連 n 不大時也沒救。這章裡的「難」指的就是「看起來需要 exponential time」。

n = input size polynomial n² exponential 2ⁿ 10 100 1,024 20 400 1,048,576 40 1,600 1,099,511,627,776 ← blows up

P — problems a normal algorithm solves efficiently. P is the family of decision problems that an ordinary, step-by-step (deterministic) algorithm can solve in polynomial time. "Deterministic" just means the usual kind of algorithm: at each step exactly one thing happens. P = the genuinely easy problems.P — 普通 algorithm 就能有效率解掉的問題。P 是一族 decision 問題：一個普通、一步一步的（deterministic，確定型）algorithm 能在 polynomial time 內解掉。「Deterministic」就是指一般那種 algorithm：每一步恰好發生一件事。P = 真正容易的問題。

NP — problems you can check efficiently given the answer. A decision problem is in NP if, when someone hands you a candidate answer — called a certificate (e.g. an actual tour, or an actual truth assignment) — you can verify in polynomial time whether it really works. You might have no fast way to find the answer, but you can check a proposed one fast. The catchy phrase: NP = "guess, then check."NP — 給你答案後你能有效率檢查的問題。一個 decision 問題屬於 NP，是指當有人遞給你一個候選答案 — 稱為 certificate（例如一條實際的 tour，或一組實際的 truth assignment）— 你能在 polynomial time 內驗證它是否真的成立。你也許沒有快速找出答案的方法，但你能快速檢查別人提出的答案。那句好記的話：NP = 「先猜，再檢查（guess, then check）」。

Nondeterministic algorithm = magically guess, then verify. This is the formal version of "guess then check." A nondeterministic algorithm runs in two stages: a guessing stage that magically picks a candidate (imagine an oracle that always guesses a working answer if one exists), and a checking stage that verifies it like a normal algorithm. If the checking stage is polynomial, the problem is in NP. (The "magic" is a thought-experiment, not a real machine — it is what the "N" in NP stands for: nondeterministic.)Nondeterministic algorithm = 神奇地猜，再驗證。這是「先猜再檢查」的正式版本。一個 nondeterministic algorithm（非確定型）分兩個階段：一個 guessing（猜測）階段神奇地挑出一個候選（想像一個 oracle，只要存在可行答案它總是猜中），以及一個 checking（檢查）階段，像普通 algorithm 那樣驗證它。如果檢查階段是 polynomial 的，這問題就在 NP。（那個「神奇」是一個思想實驗，不是真實機器 — 它就是 NP 裡那個「N」的由來：nondeterministic。）

Reduction A → B = "B is at least as hard as A". A reduction from problem A to problem B (written A → B) means: if I had a fast solver for B, I could use it as a subroutine to solve A fast too. So a fast B gives a fast A — which makes B at least as hard as A (B can't be easier, or A would be easy via B). Getting this direction right is the single most-tested idea in the chapter, so reread it: the arrow points from the easier-or-equal problem to the at-least-as-hard one.Reduction A → B =「B 至少和 A 一樣難」。從問題 A 到問題 B 的一個 reduction（寫成 A → B）意思是：如果我有一個快速的 B 解法，我就能把它當成 subroutine，也快速地解掉 A。所以一個快的 B 給出一個快的 A — 這使得 B 至少和 A 一樣難（B 不可能更容易，否則 A 透過 B 就會變容易）。把這個方向弄對是這章最常考的觀念，所以再讀一次：箭頭從較容易或相等的問題指向至少一樣難的那個。

The payoff (read once, it ties everything together): some decision problems are so central that every problem in NP reduces to them — these are NP-hard. The ones that are NP-hard and themselves sit in NP are NP-complete: the hardest problems that are still "checkable." Cook's theorem found the very first such problem — SAT — and everything else is proven hard by reducing SAT (or its descendants) to it. The rest of this lesson formalizes exactly these terms.回報（讀一次，它把一切串起來）：有些 decision 問題太核心了，以致 NP 裡每一個問題都能 reduce 到它們 — 這些就是 NP-hard。那些既是 NP-hard 又自己落在 NP 裡的，就是 NP-complete：仍然「可檢查」的問題中最難的。Cook's theorem 找到了第一個這樣的問題 — SAT — 其他一切的「難」都是靠把 SAT（或它的後代）reduce 到它來證明的。這一課剩下的部分就是把這些名詞正式化。

1 · The four classes (do not swap them)1 · 四個類別（不要互相搞混）

Class類別Definition定義
PSolvable by a deterministic polynomial-time algorithm.能被一個 deterministic 的 polynomial-time algorithm 解掉。
NPDecision problems solvable by a nondeterministic poly-time algorithm = guess a certificate, then check it in poly time.能被一個 nondeterministic poly-time algorithm 解掉的 decision 問題 = 猜一個 certificate，再在 poly time 內檢查它。
NP-hardProblems to which every NP problem reduces. (Need not be in NP.)每一個 NP 問題都能 reduce 到它的問題。（不一定要在 NP 裡。）
NP-complete (NPC)NP-hard and in NP — the hardest problems that are still NP.既是 NP-hard 又在 NP 裡 — 仍屬於 NP 的問題中最難的。
The big theorem: if any one NPC problem is in P, then P = NP (all NP problems become poly-time). No one has found such an algorithm, so NPC problems "seem" to need exponential time in the worst case. Note: not every NP problem is hard — MST is in NP yet poly-time solvable.大定理：如果任何一個 NPC 問題在 P 裡，那麼 P = NP（所有 NP 問題都變成 poly-time）。沒有人找到這樣的 algorithm，所以 NPC 問題在最壞情況下「看起來」需要 exponential time。注意：不是每個 NP 問題都難 — MST 在 NP 裡卻能 poly-time 解掉。

2 · Reduction — and its direction2 · Reduction — 以及它的方向

Write A → B ("A reduces to B"): A can be solved by a deterministic poly-time algorithm that calls, as a subroutine, a poly-time algorithm for B.寫成 A → B（「A reduce 到 B」）：A 能被一個 deterministic poly-time algorithm 解掉，而這個 algorithm 把一個 B 的 poly-time algorithm 當成 subroutine 來呼叫。

Direction is the trap. A → B means B is at least as hard as A — "A is no harder than B." If you can solve B fast, you get A for free. To prove a new problem A is NP-hard you show a known hard problem reduces to it (known → A), pinning A's hardness from below. (If A, B are both NPC then A → B and B → A — all NPC problems are equally hard.)方向就是陷阱。A → B 表示 B 至少和 A 一樣難 —「A 不會比 B 難。」如果你能快速解 B，A 就免費到手。要證明一個新問題 A 是 NP-hard，你要證明一個已知的難問題 reduce 到它（known → A），從下方釘住 A 的難度。（如果 A、B 都是 NPC，則 A → B 且 B → A — 所有 NPC 問題一樣難。）

3 · Decision vs. optimization — and why decision is enough3 · Decision 與 optimization — 以及為何 decision 就夠了

NP-completeness is defined only on decision problems (Yes/No answers); optimization is harder (§0). Recall the two flavours of TSP:NP-completeness 只定義在 decision 問題上（Yes/No 答案）；optimization 比較難（§0）。回想 TSP 的兩種類型：

Version版本Question問題
OptimizationFind the shortest tour.找出最短的 tour。
DecisionIs there a tour of total length ≤ C?存在總長度 ≤ C 的 tour 嗎？

You can get the optimum from a decision oracle: binary-search the bound C. Range is 1 … K where K = sum of all edge weights. Each test is one decision call, so O(log K) calls give the optimum — an efficient decision algorithm yields an efficient optimization algorithm.你可以從一個 decision oracle 得到最佳解：對界限 C 做 binary-search。範圍是 1 … K，其中 K = 所有 edge weight 的總和。每次測試是一次 decision 呼叫，所以 O(log K) 次呼叫就能得到最佳解 — 一個有效率的 decision algorithm 能產出一個有效率的 optimization algorithm。

4 · Nondeterministic algorithms — guess then check4 · Nondeterministic algorithms — 先猜再檢查

A nondeterministic algorithm has two stages: a guessing phase (magically pick a candidate via Choice) and a checking phase (verify it deterministically). The problem is in NP iff the check runs in polynomial time.一個 nondeterministic algorithm 有兩個階段：一個 guessing 階段（透過 Choice 神奇地挑出一個候選）和一個 checking 階段（deterministically 地驗證它）。當且僅當這個檢查跑在 polynomial time 內，這問題才在 NP。

Nondeterministic SAT — guess each variable, then evaluate the formula:Nondeterministic SAT — 猜每個變數，再評估這個 formula：

for i = 1 to n do // GUESS xᵢ ← Choice(true, false) if E(x₁, x₂, …, xₙ) is true // CHECK then Success else Failure

Guessing all n variables is O(n) Choice calls; evaluating the CNF formula E is polynomial. So SAT is an NP problem. (Same shape as nondeterministic searching: j ← Choice(1:n); if A(j)=x then Success.)猜全部 n 個變數是 O(n) 次 Choice 呼叫；評估 CNF formula E 是 polynomial 的。所以 SAT 是一個 NP 問題。（和 nondeterministic searching 同樣的形狀：j ← Choice(1:n); if A(j)=x then Success。）

Caution — NP-hard ≠ NP-complete. The halting problem ("will this program terminate?") is NP-hard (every NP problem reduces to it) but it is undecidable, so it is not in NP. NP-hard is a strictly bigger set than NP-complete.注意 — NP-hard ≠ NP-complete。Halting problem（「這個程式會不會終止？」）是 NP-hard（每個 NP 問題都能 reduce 到它）但它是 undecidable（不可判定）的，所以它不在 NP 裡。NP-hard 是比 NP-complete 嚴格更大的集合。

5 · Cook's Theorem5 · Cook's Theorem

The seed of the whole theory:整套理論的種子：

Cook's Theorem: NP = P iff SAT ∈ P. Consequences: SAT is NP-complete — the first NPC problem — and every NP problem reduces to SAT.Cook's Theorem：NP = P iff SAT ∈ P。推論：SAT 是 NP-complete — 第一個 NPC 問題 — 而且每個 NP 問題都能 reduce 到 SAT。

Why? Every NP problem has a nondeterministic poly-time algorithm, and that algorithm can be transformed in poly time into a SAT instance that is satisfiable iff the original answer is "Yes." So every NP problem → SAT, making SAT NP-hard; SAT is also in NP (Section 4) ⇒ NP-complete.為什麼？每個 NP 問題都有一個 nondeterministic poly-time algorithm，而那個 algorithm 能在 poly time 內被轉換成一個 SAT instance，使得它可滿足（satisfiable）若且唯若原問題的答案是「Yes」。所以每個 NP 問題 → SAT，使 SAT 成為 NP-hard；SAT 也在 NP 裡（第 4 節）⇒ NP-complete。

Recipe: how to prove a new problem A is NP-complete食譜：如何證明一個新問題 A 是 NP-complete

(I) Show A ∈ NP — give a guess-and-poly-check algorithm.
(II) Show some known B ∈ NPC reduces to A, i.e. B → A (A is at least as hard as B ⇒ NP-hard).
(I) + (II) ⇒ A is NP-complete. That second step is Lesson 10's reduction chain, all seeded by SAT.(I) 證明 A ∈ NP — 給一個 guess-and-poly-check 的 algorithm。
(II) 證明某個已知的 B ∈ NPC reduce 到 A，也就是 B → A（A 至少和 B 一樣難 ⇒ NP-hard）。
(I) + (II) ⇒ A 是 NP-complete。那第二步就是第 10 課的 reduction chain，全都從 SAT 起源。
