0 · What is this even about?0 · 這章到底在幹嘛？

This chapter is a vocabulary for sorting problems into easy and seemingly-hard. Before any of the famous letters (P, NP, …) mean anything, you need five plain ideas. Read this section slowly — everything after it is just these ideas dressed up.說穿了，這章就是一套用來把問題分成容易跟看起來很難的詞彙。那些有名的字母（P、NP、…）在你看懂之前都是空殼，所以你得先把五個最基本的概念搞懂。這一節慢慢看 — 後面講的東西其實都只是這五個概念換個包裝而已。

Decision problem vs. optimization problem. A decision problem is any question whose answer is just Yes or No. An optimization problem asks for the best value — the smallest, cheapest, or largest something. Optimization is harder, because "best" implies you also know the answer is achievable. The whole NP theory is built only on decision problems, so we always rephrase an optimization problem as a Yes/No one.Decision problem 跟 optimization problem。Decision problem（判定問題）就是答案只有 Yes 或 No 的問題。Optimization problem（最佳化問題）則是要你找最好的那個值 — 最小、最便宜、或最大的某個東西。最佳化比較難，因為要講「最好」，你等於還要知道這答案真的做得到。整套 NP 理論只蓋在 decision 問題上，所以我們習慣把最佳化問題改寫成 Yes/No 的版本。

Concrete example — the Travelling Salesperson Problem (TSP): a salesperson must visit every city exactly once and return to the start; each leg between two cities has a cost (distance), and a full round-trip is called a tour.舉個具體例子 — Travelling Salesperson Problem (TSP)：一個業務員要把每個城市剛好走過一次，最後再回到起點；任兩個城市之間的那段路都有一個 cost（距離），而走完一整圈就叫做一個 tour。

Flavour類型The question it asks它在問什麼
OptimizationWhat is the cheapest tour? (give me the number / the route)最便宜的 tour 是哪一條？（要的是數字或路線）
DecisionIs there a tour of total cost ≤ C? (just Yes or No)有沒有一條 tour 的總 cost ≤ C？（只要回答 Yes 或 No）

Polynomial time = "efficient/tractable". When we say an algorithm runs in polynomial time, we mean its running time grows like a fixed power of the input size n — n, n2, n3, … As n doubles, the cost goes up by a constant factor. This is what we informally call efficient or tractable: it scales. By contrast, exponential time means the cost grows like 2n — every extra element doubles the work, so it blows up and becomes hopeless even for modest n. "Hard" in this chapter means "seems to need exponential time."Polynomial time（多項式時間）=「有效率、跑得動」。當我們說一個 algorithm 跑在 polynomial time，意思是它的執行時間大概像輸入大小 n 的某個固定次方那樣長 — n、n2、n3、…。n 變兩倍，成本也只是乘上一個固定倍數而已。這就是我們平常講的 efficient（有效率）或 tractable（跑得動）：規模變大它撐得住。反過來看，exponential time（指數時間）是成本像 2n 在長 — 每多一個元素，工作量就翻一倍，所以很快就爆掉，n 才大一點點就沒救了。這章講的「難」，意思就是「看起來非得 exponential time 不可」。

n = input size polynomial n2 exponential 2n 10 100 1,024 20 400 1,048,576 40 1,600 1,099,511,627,776 ← blows up

P — problems a normal algorithm solves efficiently. P is the family of decision problems that an ordinary, step-by-step (deterministic) algorithm can solve in polynomial time. "Deterministic" just means the usual kind of algorithm: at each step exactly one thing happens. P = the genuinely easy problems.P — 普通 algorithm 就能輕鬆解掉的問題。P 裡面是這樣的 decision 問題：拿一個普通的、一步一步走的（deterministic，確定型）algorithm，就能在 polynomial time 內解掉。所謂「deterministic」其實就是你平常寫的那種 algorithm：每一步就是固定發生一件事，沒有岔路。簡單講，P = 真正算簡單的那些問題。

NP — problems you can check efficiently given the answer. A decision problem is in NP if, when someone hands you a candidate answer — called a certificate (e.g. an actual tour, or an actual truth assignment — a choice of true/false for each variable of a Boolean formula, introduced with SAT in §4) — you can verify in polynomial time whether it really works. You might have no fast way to find the answer, but you can check a proposed one fast. The catchy phrase: NP = "guess, then check."NP — 答案丟給你，你能快速檢查的問題。一個 decision 問題屬於 NP，意思是：當有人把一個候選答案遞到你面前 — 這個東西叫 certificate（比方說一條實際的 tour，或一組實際的 truth assignment — 也就是幫 Boolean formula 裡每個變數各挑一個 true/false，這個東西到 §4 講 SAT 時會正式登場）— 你能在 polynomial time 內驗證它到底成不成立。重點是，你可能根本沒辦法快速找出答案，但只要有人給你一個，你就能快速檢查對不對。一句話記起來：NP = 「先猜，再檢查（guess, then check）」。

Nondeterministic algorithm = magically guess, then verify. This is the formal version of "guess then check." A nondeterministic algorithm runs in two stages: a guessing stage that magically picks a candidate (imagine an oracle that always guesses a working answer if one exists), and a checking stage that verifies it like a normal algorithm. If the checking stage is polynomial, the problem is in NP. (The "magic" is a thought-experiment, not a real machine — it is what the "N" in NP stands for: nondeterministic.)Nondeterministic algorithm = 用神力猜一個，再去驗證。這就是「先猜再檢查」的正式講法。一個 nondeterministic algorithm（非確定型）拆成兩段：先是 guessing（猜測）這段，像開外掛一樣挑出一個候選（你可以想成有個 oracle，只要存在可行答案它就一定猜中），接著是 checking（檢查）這段，就跟普通 algorithm 一樣去驗證它對不對。只要檢查這段是 polynomial 的，這問題就落在 NP。（那個「神力」只是個思想實驗，不是真有這種機器 — 它正是 NP 裡那個「N」的由來：nondeterministic。）

Reduction A → B = "B is at least as hard as A". A reduction from problem A to problem B (written A → B) means: if I had a fast solver for B, I could use it as a subroutine to solve A fast too. So a fast B gives a fast A — which makes B at least as hard as A (B can't be easier, or A would be easy via B). Getting this direction right is the single most-tested idea in the chapter, so reread it: the arrow points from the easier-or-equal problem to the at-least-as-hard one.Reduction A → B =「B 至少跟 A 一樣難」。從問題 A 到問題 B 做一個 reduction（寫成 A → B）的意思是：只要我手上有一個跑得快的 B 解法，我就能把它當 subroutine 來呼叫，順便也把 A 快速解掉。換句話說，B 一快，A 就跟著快 — 所以 B 至少跟 A 一樣難（B 不可能比 A 簡單，不然 A 借 B 一用也變簡單了）。這個方向是全章最愛考的點，所以再看一次：箭頭是從比較簡單（或一樣難）的那個，指向至少一樣難的那個。

The payoff (read once, it ties everything together): some decision problems are so central that every problem in NP reduces to them — these are NP-hard. The ones that are NP-hard and themselves sit in NP are NP-complete: the hardest problems that are still "checkable." Cook's theorem found the very first such problem — SAT — and everything else is proven hard by reducing SAT (or its descendants) to it. The rest of this lesson formalizes exactly these terms.收網（看一次，它會把前面全部串起來）：有些 decision 問題實在太核心，核心到 NP 裡每一個問題都能 reduce 到它們身上 — 這些就叫 NP-hard。而那些既是 NP-hard 又自己也待在 NP 裡的，就是 NP-complete：在還「檢查得了」的問題裡面最難的一群。Cook's theorem 就是找到了第一個這種問題 — SAT — 之後其他問題要證明「難」，都是靠把 SAT（或它衍生出來的那些）reduce 過去。這一課接下來，就是把這幾個名詞講清楚、正式化。

1 · The four classes (do not swap them)1 · 四個類別（別搞混了）

Class類別Definition定義
PSolvable by a deterministic polynomial-time algorithm.用一個 deterministic 的 polynomial-time algorithm 就解得掉。
NPDecision problems solvable by a nondeterministic poly-time algorithm = guess a certificate, then check it in poly time.能用 nondeterministic poly-time algorithm 解掉的 decision 問題 = 猜一個 certificate，再在 poly time 內檢查它對不對。
NP-hardProblems to which every NP problem reduces. (Need not be in NP.)NP 裡每一個問題都 reduce 得到它身上的那種問題。（它自己不一定要在 NP 裡。）
NP-complete (NPC)NP-hard and in NP — the hardest problems that are still NP.既是 NP-hard 又在 NP 裡 — 還待在 NP 裡的問題中最難的那群。
The big theorem: if any one NPC problem is in P, then P = NP (all NP problems become poly-time). No one has found such an algorithm, so NPC problems "seem" to need exponential time in the worst case. Note: not every NP problem is hard — MST is in NP yet poly-time solvable.關鍵大定理：只要隨便一個 NPC 問題落在 P 裡，那 P = NP 就成立（所有 NP 問題瞬間全部變 poly-time）。問題是到現在都沒人找到這種 algorithm，所以 NPC 問題在最壞情況下「看起來」就是非 exponential time 不可。但要注意：不是每個 NP 問題都很難 — 像 MST 雖然在 NP 裡，卻照樣能 poly-time 解掉。

2 · Reduction — and its direction2 · Reduction — 以及它的方向

Write A → B ("A reduces to B"): A can be solved by a deterministic poly-time algorithm that calls, as a subroutine, a poly-time algorithm for B.寫成 A → B（「A reduce 到 B」）：意思是 A 可以用一個 deterministic poly-time algorithm 解掉，而這個 algorithm 過程中會去呼叫一個 B 的 poly-time algorithm 當 subroutine 來用。

Direction is the trap. A → B means B is at least as hard as A — "A is no harder than B." If you can solve B fast, you get A for free. To prove a new problem A is NP-hard you show a known hard problem reduces to it (known → A), pinning A's hardness from below. (If A, B are both NPC then A → B and B → A — all NPC problems are equally hard.)方向就是最大的坑。A → B 表示 B 至少跟 A 一樣難 — 也就是「A 不會比 B 難」。只要你能快速解 B，A 就等於白白送你。所以要證明一個新問題 A 是 NP-hard，你得拿一個已知很難的問題去 reduce 到 A（known → A），這樣才能從下面把 A 的難度撐住。（如果 A、B 兩個都是 NPC，那 A → B 跟 B → A 都成立 — 所有 NPC 問題其實一樣難。）

3 · Decision vs. optimization — and why decision is enough3 · Decision 對上 optimization — 為什麼光看 decision 就夠

NP-completeness is defined only on decision problems (Yes/No answers); optimization is harder (§0). Recall the two flavours of TSP:NP-completeness 只定義在 decision 問題上（答案是 Yes/No）；optimization 本來就比較難（見 §0）。回頭看一下 TSP 的兩種版本：

Version版本Question它問什麼
OptimizationFind the shortest tour.找出最短的那條 tour。
DecisionIs there a tour of total length ≤ C?有沒有一條 tour 的總長度 ≤ C？

The same recasting works for any familiar problem. Sorting, for instance, has a decision form too: "is there a permutation of the array whose cost (e.g. number of out-of-order pairs) is below C?" — a plain Yes/No question you can verify in poly time by checking a given permutation. (You'll see this exact phrasing in Q1 below.) Back to TSP: you can get the optimum from a decision oracle by binary-searching the bound C. Range is 1 … K where K = sum of all edge weights. Each test is one decision call, so O(log K) calls give the optimum — an efficient decision algorithm yields an efficient optimization algorithm.這種改寫法對任何你熟悉的問題都適用。就拿 Sorting 來說，它也有 decision 版：「有沒有一種把 array 重排的方式（permutation），它的 cost（例如逆序對的數量）小於 C？」 — 這就是一個單純的 Yes/No 問題，而且給你一個 permutation，你能在 poly time 內驗證它行不行。（下面 Q1 用的就是這個講法。）回到 TSP：其實只要有 decision oracle，你就能把最佳解逼出來：對界限 C 做 binary-search就好。範圍是 1 … K，這裡 K = 所有 edge weight 加起來的總和。每問一次就是一次 decision 呼叫，所以 O(log K) 次呼叫就把最佳解找出來了 — 這就是為什麼，有了有效率的 decision algorithm，就等於有了有效率的 optimization algorithm。

4 · Nondeterministic algorithms — guess then check4 · Nondeterministic algorithms — 先猜再檢查

A nondeterministic algorithm has two stages: a guessing phase (magically pick a candidate via Choice) and a checking phase (verify it deterministically). The problem is in NP iff the check runs in polynomial time.一個 nondeterministic algorithm 分成兩段：guessing 這段（用 Choice 開外掛挑出一個候選）跟 checking 這段（用 deterministic 的方式去驗證它）。只有當這個檢查能在 polynomial time 內跑完，這問題才算在 NP 裡。

What is SAT? SAT (satisfiability) asks: given a Boolean formula in CNF (conjunctive normal form) — an AND of clauses, where each clause is an OR of variables that may be negated (a variable xi or its negation NOT xi is called a literal) — is there an assignment of true/false to the variables that makes the whole formula true? A formula is satisfiable if at least one such assignment exists. Tiny example: (x1 OR NOT x2) AND (x2 OR x3) is satisfiable — e.g. x1=true, x2=false, x3=anything makes both clauses true.SAT 是什麼？SAT（satisfiability，可滿足性）在問：給你一個 CNF（conjunctive normal form）的 Boolean formula — 它是一堆 clause 用 AND 串起來，而每個 clause 又是一堆變數（可以加上 NOT 變成反向）用 OR 接起來（一個變數 xi 或它的反向 NOT xi 就叫一個 literal）— 那有沒有辦法幫這些變數各填一個 true/false，讓整個 formula 變成 true？只要存在至少一組這樣的填法，這個 formula 就叫 satisfiable（可滿足）。舉個小例子：(x1 OR NOT x2) AND (x2 OR x3) 是 satisfiable 的 — 例如 x1=true, x2=false, x3=隨便，兩個 clause 就都成立了。

Nondeterministic SAT — guess each variable, then evaluate the formula:拿 Nondeterministic SAT 來看 — 每個變數都猜一次，再把整個 formula 算出來：

for i = 1 to n do // GUESS xi ← Choice(true, false) if E(x1, x2, …, xn) is true // CHECK then Success else Failure

Guessing all n variables is O(n) Choice calls; evaluating the CNF formula E is polynomial. So SAT is an NP problem. (Same shape as nondeterministic searching: j ← Choice(1:n); if A(j)=x then Success.)把 n 個變數全猜一遍是 O(n) 次 Choice 呼叫；把 CNF formula E 算出來則是 polynomial 的。所以 SAT 就是個 NP 問題。（這跟 nondeterministic searching 長得一模一樣：j ← Choice(1:n); if A(j)=x then Success。）

Caution — NP-hard ≠ NP-complete. The halting problem ("will this program terminate?") is NP-hard (every NP problem reduces to it) but it is undecidable, so it is not in NP. Undecidable means no algorithm exists that always halts and gives the correct Yes/No answer for every input — so there's nothing to verify, which is exactly why it can't be in NP (NP needs a poly-time check). NP-hard is a strictly bigger set than NP-complete.小心 — NP-hard ≠ NP-complete。Halting problem（「這支程式到底會不會停？」）是 NP-hard（每個 NP 問題都 reduce 得到它），但它同時是 undecidable（不可判定）的，所以它根本不在 NP 裡。Undecidable 的精確意思是：不存在任何 algorithm 能對所有輸入都一定停下來、又給出正確的 Yes/No 答案 — 既然連解都解不出來，自然沒東西好驗證，這也正是它不可能在 NP 裡的原因（NP 需要有個 poly-time 的檢查）。簡單講，NP-hard 是個比 NP-complete 嚴格更大的集合。

5 · Cook's Theorem5 · Cook's Theorem

The seed of the whole theory:整套理論的起點就在這：

Cook's Theorem: NP = P iff SAT ∈ P. Consequences: SAT is NP-complete — the first NPC problem — and every NP problem reduces to SAT.Cook's Theorem：NP = P iff SAT ∈ P。由此可以推出：SAT 是 NP-complete — 也就是第一個 NPC 問題 — 而且每個 NP 問題都 reduce 得到 SAT。

Why? Every NP problem has a nondeterministic poly-time algorithm, and that algorithm can be transformed in poly time into a SAT instance that is satisfiable iff the original answer is "Yes." So every NP problem → SAT, making SAT NP-hard; SAT is also in NP (Section 4) ⇒ NP-complete.為什麼會這樣？因為每個 NP 問題都有一個 nondeterministic poly-time algorithm，而你能在 poly time 內把那個 algorithm 改寫成一個 SAT instance，讓它剛好在原問題答案是「Yes」的時候才 satisfiable（可滿足），反之亦然。這樣每個 NP 問題就都 → SAT，於是 SAT 變成 NP-hard；再加上 SAT 自己也在 NP 裡（第 4 節）⇒ 它就是 NP-complete。

Recipe: how to prove a new problem A is NP-completeSOP：怎麼證明一個新問題 A 是 NP-complete

(I) Show A ∈ NP — give a guess-and-poly-check algorithm.
(II) Show some known B ∈ NPC reduces to A, i.e. B → A (A is at least as hard as B ⇒ NP-hard).
(I) + (II) ⇒ A is NP-complete. That second step is Lesson 10's reduction chain, all seeded by SAT.(I) 先證 A ∈ NP — 給一個 guess-and-poly-check 的 algorithm 就好。
(II) 再找一個已知的 B ∈ NPC，把它 reduce 到 A，也就是 B → A（這代表 A 至少跟 B 一樣難 ⇒ NP-hard）。
(I) + (II) 湊齊 ⇒ A 就是 NP-complete。第二步那條路就是第 10 課的 reduction chain，全部都是從 SAT 一路接出來的。
