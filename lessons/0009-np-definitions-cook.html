Lesson 9 · Ch8 NP-Completeness I

P, NP, and Cook's Theorem

Exam goal: this chapter is short-answer / definition heavy. Win = state the four classes without swapping them, get the direction of a reduction right, and explain a nondeterministic algorithm as guess then check. Master those and the QA points are free.

0 · What is this even about?

This chapter is a vocabulary for sorting problems into easy and seemingly-hard. Before any of the famous letters (P, NP, …) mean anything, you need five plain ideas. Read this section slowly — everything after it is just these ideas dressed up.

Decision problem vs. optimization problem. A decision problem is any question whose answer is just Yes or No. An optimization problem asks for the best value — the smallest, cheapest, or largest something. Optimization is harder, because "best" implies you also know the answer is achievable. The whole NP theory is built only on decision problems, so we always rephrase an optimization problem as a Yes/No one.

Concrete example — the Travelling Salesperson Problem (TSP): a salesperson must visit every city exactly once and return to the start; each leg between two cities has a cost (distance), and a full round-trip is called a tour.

FlavourThe question it asks
OptimizationWhat is the cheapest tour? (give me the number / the route)
DecisionIs there a tour of total cost ≤ C? (just Yes or No)

Polynomial time = "efficient/tractable". When we say an algorithm runs in polynomial time, we mean its running time grows like a fixed power of the input size n — n, n², n³, … As n doubles, the cost goes up by a constant factor. This is what we informally call efficient or tractable: it scales. By contrast, exponential time means the cost grows like 2ⁿ — every extra element doubles the work, so it blows up and becomes hopeless even for modest n. "Hard" in this chapter means "seems to need exponential time."

n = input size polynomial n² exponential 2ⁿ 10 100 1,024 20 400 1,048,576 40 1,600 1,099,511,627,776 ← blows up

P — problems a normal algorithm solves efficiently. P is the family of decision problems that an ordinary, step-by-step (deterministic) algorithm can solve in polynomial time. "Deterministic" just means the usual kind of algorithm: at each step exactly one thing happens. P = the genuinely easy problems.

NP — problems you can check efficiently given the answer. A decision problem is in NP if, when someone hands you a candidate answer — called a certificate (e.g. an actual tour, or an actual truth assignment) — you can verify in polynomial time whether it really works. You might have no fast way to find the answer, but you can check a proposed one fast. The catchy phrase: NP = "guess, then check."

Nondeterministic algorithm = magically guess, then verify. This is the formal version of "guess then check." A nondeterministic algorithm runs in two stages: a guessing stage that magically picks a candidate (imagine an oracle that always guesses a working answer if one exists), and a checking stage that verifies it like a normal algorithm. If the checking stage is polynomial, the problem is in NP. (The "magic" is a thought-experiment, not a real machine — it is what the "N" in NP stands for: nondeterministic.)

Reduction A → B = "B is at least as hard as A". A reduction from problem A to problem B (written A → B) means: if I had a fast solver for B, I could use it as a subroutine to solve A fast too. So a fast B gives a fast A — which makes B at least as hard as A (B can't be easier, or A would be easy via B). Getting this direction right is the single most-tested idea in the chapter, so reread it: the arrow points from the easier-or-equal problem to the at-least-as-hard one.

The payoff (read once, it ties everything together): some decision problems are so central that every problem in NP reduces to them — these are NP-hard. The ones that are NP-hard and themselves sit in NP are NP-complete: the hardest problems that are still "checkable." Cook's theorem found the very first such problem — SAT — and everything else is proven hard by reducing SAT (or its descendants) to it. The rest of this lesson formalizes exactly these terms.

1 · The four classes (do not swap them)

ClassDefinition
PSolvable by a deterministic polynomial-time algorithm.
NPDecision problems solvable by a nondeterministic poly-time algorithm = guess a certificate, then check it in poly time.
NP-hardProblems to which every NP problem reduces. (Need not be in NP.)
NP-complete (NPC)NP-hard and in NP — the hardest problems that are still NP.
The big theorem: if any one NPC problem is in P, then P = NP (all NP problems become poly-time). No one has found such an algorithm, so NPC problems "seem" to need exponential time in the worst case. Note: not every NP problem is hard — MST is in NP yet poly-time solvable.

2 · Reduction — and its direction

Write A → B ("A reduces to B"): A can be solved by a deterministic poly-time algorithm that calls, as a subroutine, a poly-time algorithm for B.

Direction is the trap. A → B means B is at least as hard as A — "A is no harder than B." If you can solve B fast, you get A for free. To prove a new problem A is NP-hard you show a known hard problem reduces to it (known → A), pinning A's hardness from below. (If A, B are both NPC then A → B and B → A — all NPC problems are equally hard.)

3 · Decision vs. optimization — and why decision is enough

NP-completeness is defined only on decision problems (Yes/No answers); optimization is harder (§0). Recall the two flavours of TSP:

VersionQuestion
OptimizationFind the shortest tour.
DecisionIs there a tour of total length ≤ C?

You can get the optimum from a decision oracle: binary-search the bound C. Range is 1 … K where K = sum of all edge weights. Each test is one decision call, so O(log K) calls give the optimum — an efficient decision algorithm yields an efficient optimization algorithm.

4 · Nondeterministic algorithms — guess then check

A nondeterministic algorithm has two stages: a guessing phase (magically pick a candidate via Choice) and a checking phase (verify it deterministically). The problem is in NP iff the check runs in polynomial time.

Nondeterministic SAT — guess each variable, then evaluate the formula:

for i = 1 to n do // GUESS xᵢ ← Choice(true, false) if E(x₁, x₂, …, xₙ) is true // CHECK then Success else Failure

Guessing all n variables is O(n) Choice calls; evaluating the CNF formula E is polynomial. So SAT is an NP problem. (Same shape as nondeterministic searching: j ← Choice(1:n); if A(j)=x then Success.)

Caution — NP-hard ≠ NP-complete. The halting problem ("will this program terminate?") is NP-hard (every NP problem reduces to it) but it is undecidable, so it is not in NP. NP-hard is a strictly bigger set than NP-complete.

5 · Cook's Theorem

The seed of the whole theory:

Cook's Theorem: NP = P iff SAT ∈ P. Consequences: SAT is NP-complete — the first NPC problem — and every NP problem reduces to SAT.

Why? Every NP problem has a nondeterministic poly-time algorithm, and that algorithm can be transformed in poly time into a SAT instance that is satisfiable iff the original answer is "Yes." So every NP problem → SAT, making SAT NP-hard; SAT is also in NP (Section 4) ⇒ NP-complete.

Recipe: how to prove a new problem A is NP-complete

(I) Show A ∈ NP — give a guess-and-poly-check algorithm.
(II) Show some known B ∈ NPC reduces to A, i.e. B → A (A is at least as hard as B ⇒ NP-hard).
(I) + (II) ⇒ A is NP-complete. That second step is Lesson 10's reduction chain, all seeded by SAT.
