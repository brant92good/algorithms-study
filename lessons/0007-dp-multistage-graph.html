0 · What is this all about?0 · 這一課在講什麼？

Dynamic programming (DP) is a way to solve a problem by viewing it as a sequence of decisions. The trick: break the problem into smaller subproblems, solve each one once, store the answer, and reuse it instead of recomputing it. The greedy method (the rival) just grabs the locally cheapest choice at each step and never looks back. DP is smarter: it does not commit to a choice that looks cheap right now — it weighs how that choice pays off later. That is why DP can find the true best answer where greedy gets fooled.Dynamic programming（DP）是一種把問題看成一連串決策（sequence of decisions）來求解的方法。訣竅在於：把問題拆成較小的 subproblem，每個只解一次，把答案存起來，之後重複利用而不是重算。對手 greedy 方法則是每一步都只抓當下最便宜的選擇，從不回頭。DP 更聰明：它不會急著承諾一個看起來當下便宜的選擇，而是衡量這個選擇之後會帶來多少代價。這就是為什麼在 greedy 被騙的地方，DP 仍能找到真正的最佳解。

Everything here lives on a graph. A few words, in plain terms:這裡的一切都建立在一張 graph 上。先用白話說幾個名詞：

  • A graph is just dots connected by lines.一張 graph 就是一些用線連起來的點。
  • Each dot is a vertex (also called a node).每個點是一個 vertex（也叫 node）。
  • Each line is an edge connecting two vertices.每條線是一條 edge，連接兩個 vertex。
  • A directed edge is a one-way arrow: u→v lets you go from u to v but not back.一條 directed edge 是單向箭頭：u→v 讓你能從 u 走到 v，但不能反過來。
  • Each edge has a number, its weight or cost. A path is a chain of edges; its path cost is the sum of the weights along it.每條 edge 都有一個數字，叫 weight（權重）或成本。一條 path 是一串相連的 edge；它的 path cost 就是沿途所有 weight 的總和。

A multistage graph is a special directed graph. Its vertices are grouped into ordered stages (stage 1, stage 2, …), and every edge only goes from one stage to the very next one — never sideways, never skipping ahead. There is exactly one start vertex (in the first stage) and one end vertex (in the last stage). The goal: find the cheapest path from start to end.一張 multistage graph 是一種特殊的有向圖。它的 vertex 被分組成有順序的 stage（stage 1、stage 2、…），而且每條 edge 只會從一個 stage 走到緊接著的下一個 stage——不會橫向走、也不會跳過。剛好有一個 start vertex（在第一個 stage）和一個 end vertex（在最後一個 stage）。目標：找出從 start 到 end 的最便宜路徑。

stage 1 stage 2 stage 3 stage 4 ┌── A ──┐ ┌── D ──┐ S ─────────┼── B ──┼────┼── E ──┼──── T └── C ──┘ └── F ──┘ (start) (end)

Our running example has 4 stages: S alone in stage 1; A, B, C in stage 2; D, E, F in stage 3; T alone in stage 4. We want the cheapest way to get from S to T. The exact edge weights appear in §2; the rest of the lesson is about computing that cheapest path the DP way (and seeing why greedy can't).我們的範例有 4 個 stage：stage 1 只有 S；stage 2 是 A, B, C；stage 3 是 D, E, F；stage 4 只有 T。我們要找從 S 到 T 最便宜的走法。確切的 edge weight 在 §2；本課其餘部分就是用 DP 的方式算出那條最便宜的路徑（並看清楚為什麼 greedy 辦不到）。

Intuition to hold onto: picking each step by "what's cheapest right now" can trap you. DP instead asks "from this vertex, what's the cheapest way to finish?" — solving small finish-from-here subproblems once and reusing them. That reuse is the whole point.要記住的直覺：每一步都只挑「現在最便宜」可能讓你掉進陷阱。DP 改問的是「從這個 vertex 出發，最便宜的走完全程方式是什麼？」——把這些「從這裡走到終點」的小 subproblem 解一次並重複利用。那個重複利用正是重點所在。

1 · Why greedy fails here1 · 為什麼 greedy 在這裡會失敗

Greedy "take the cheapest outgoing edge each stage" looks smart and is wrong. From S it grabs S→A (cost 1), then A→D (4), then D→T (18):Greedy 的「每個 stage 都選最便宜的出邊」看起來很聰明，其實是錯的。從 S 出發它會抓 S→A（成本 1），接著 A→D（4），再來 D→T（18）：

greedy: S →A→D→T = 1 + 4 + 18 = 23 optimum: S →C→F→T = 5 + 2 + 2 = 9

The true optimum starts with the most expensive-looking first edge, S→C = 5 — because C opens onto C→F=2 and F→T=2, dirt cheap downstream. A locally bad choice that pays off later: that is exactly the situation greedy cannot see and dynamic programming can.真正的最佳解卻是從看起來最貴的第一條 edge 開始，S→C = 5——因為 C 接出 C→F=2 和 F→T=2，後段超級便宜。一個局部很差、卻在後面回本的選擇：這正是 greedy 看不到、而 dynamic programming 看得到的情況。

2 · Multistage graph — the formal definition2 · Multistage graph——正式定義

Now the precise version of the idea from §0. A multistage graph G=(V,E) is a directed graph whose vertices split into k≥2 disjoint stages V₁,…,V_k, where every edge ⟨u,v⟩ jumps from one stage to the next (u∈Vᵢ, v∈V_{i+1}), and the first and last stages are single vertices: |V₁|=|V_k|=1. The lone source (start) is s, the lone sink (end) is t. The problem: find the minimum-cost s→t path.現在來看 §0 那個想法的精確版本。一張 multistage graph G=(V,E) 是一張有向圖，它的 vertex 被切成 k≥2 個不相交的 stage V₁,…,V_k，每條 edge ⟨u,v⟩ 都從一個 stage 跳到下一個（u∈Vᵢ, v∈V_{i+1}），而且第一個和最後一個 stage 各只有一個 vertex：|V₁|=|V_k|=1。唯一的 source（起點）是 s，唯一的 sink（終點）是 t。問題就是：找出成本最小的 s→t path。

3 · Backward reasoning — trace it yourself3 · 反向（backward）推理——自己推一遍

Let d(X,T) = cost of the shortest path from X to the sink T. Start at the stage just before T (the base cases d(D,T)=18, d(E,T)=13, d(F,T)=2) and work backward toward S. Compute d(C,T), d(A,T), d(B,T), then d(S,T) on paper before opening the answer.令 d(X,T) = 從 X 到 sink T 的最短路徑成本。從 T 前一個 stage 開始（base case 是 d(D,T)=18, d(E,T)=13, d(F,T)=2），一路反向推回 S。先在紙上算出 d(C,T), d(A,T), d(B,T)，再算 d(S,T)，然後才打開答案。

Backward computation (every min line)反向計算（每一條 min 算式）
d(C,T) = min{ 2 + d(F,T) } = 2 + 2 = 4 d(A,T) = min{ 4 + d(D,T), 11 + d(E,T) } = min{ 4 + 18, 11 + 13 } = min{ 22, 24 } = 22 d(B,T) = min{ 9 + d(D,T), 5 + d(E,T), 16 + d(F,T) } = min{ 9 + 18, 5 + 13, 16 + 2 } = min{ 27, 18, 18 } = 18 d(S,T) = min{ 1 + d(A,T), 2 + d(B,T), 5 + d(C,T) } = min{ 1 + 22, 2 + 18, 5 + 4 } = min{ 23, 20, 9 } = 9

Answer: d(S,T)=9, achieved by S→C→F→T. Each vertex's optimum is written in terms of vertices closer to the sink, then resolved from the sink back — that is backward reasoning.答案：d(S,T)=9，由 S→C→F→T 達成。每個 vertex 的最佳值都用更靠近 sink 的 vertex 來表示，再從 sink 往回求解——這就是 backward reasoning（反向推理）。

4 · Forward reasoning — same graph, other direction4 · 正向（forward）推理——同一張圖，另一個方向

Now let d(S,X) = cost of the shortest path from S to X. Start at the stage just after S and work forward. Base cases are the direct S-edges; then stage 3, then T.這次令 d(S,X) = 從 S 到 X 的最短路徑成本。從 S 後一個 stage 開始，一路正向往前推。base case 是 S 的直接出邊；接著 stage 3，最後 T。

d(S,A) = 1 d(S,B) = 2 d(S,C) = 5 d(S,D) = min{ d(S,A)+4, d(S,B)+9 } = min{ 1+4, 2+9 } = min{ 5, 11 } = 5 d(S,E) = min{ d(S,A)+11, d(S,B)+5 } = min{ 1+11, 2+5 } = min{ 12, 7 } = 7 d(S,F) = min{ d(S,B)+16, d(S,C)+2 } = min{ 2+16, 5+2 } = min{ 18, 7 } = 7 d(S,T) = min{ d(S,D)+18, d(S,E)+13, d(S,F)+2 } = min{ 5+18, 7+13, 7+2 } = min{ 23, 20, 9 } = 9

Same optimum d(S,T)=9, reached by expressing each vertex's cost via vertices closer to the source. Two directions, one answer — the recurrence is consistent either way.一樣的最佳解 d(S,T)=9，這次是用更靠近 source 的 vertex 來表示每個 vertex 的成本而得到。兩個方向、同一個答案——遞迴式不論哪個方向都一致。

Key exam insight: forward and backward are both correct and give the same value. They differ only in what the subproblem is: backward stores "cost from X to sink," forward stores "cost from source to X." Pick whichever the question's figure makes easier to fill — and always write the min{…} lines, that is where the marks are.考試關鍵洞見：forward 和 backward 兩者都正確，得到的值相同。它們只差在subproblem 是什麼：backward 存的是「從 X 到 sink 的成本」，forward 存的是「從 source 到 X 的成本」。題目的圖哪個比較好填就用哪個——而且一定要寫出 min{…} 算式，分數就在那裡。

5 · Principle of optimality5 · Principle of optimality（最佳化原則）

Statement. If a problem is solved by a sequence of decisions D₁,…,Dₙ and that sequence is optimal, then for any k the last k decisions must themselves be optimal (an optimal solution is built from optimal sub-solutions).敘述。如果一個問題是由一連串決策 D₁,…,Dₙ 解出，而這串決策是最佳的，那麼對任意 k，最後 k 個決策本身也必須是最佳的（最佳解是由最佳子解組成的）。

Shortest-path corollary. If i, i₁, i₂, …, j is a shortest path from i to j, then the suffix i₁, i₂, …, j must be a shortest path from i₁ to j. (If it weren't, swap in a cheaper suffix and the whole path gets cheaper — contradicting that it was shortest.)最短路徑推論。如果 i, i₁, i₂, …, j 是從 i 到 j 的最短路徑，那麼它的後段 i₁, i₂, …, j 也必須是從 i₁ 到 j 的最短路徑。（若不是，換上一段更便宜的後段，整條路徑就更便宜——和它原本是最短的矛盾。）

This corollary is what licenses the recurrence d(X,T)=min_v{w(X,v)+d(v,T)}: because every optimal suffix is itself optimal, we may reuse the already-computed d(v,T) instead of re-searching. Consequence: any problem expressible as a multistage graph can be solved by dynamic programming.正是這個推論允許了遞迴式 d(X,T)=min_v{w(X,v)+d(v,T)}：因為每段最佳後段本身也是最佳的，我們可以重複利用已經算好的 d(v,T)，不必重新搜尋。結論：任何能表示成 multistage graph 的問題，都能用 dynamic programming 解。
