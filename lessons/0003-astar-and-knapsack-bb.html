A* Search & 0/1-Knapsack Branch-and-BoundA* Search 與 0/1-Knapsack Branch-and-Bound

Exam goal: hand-trace an A* expansion with full g/h/f bookkeeping, and read the bounds off a knapsack branch-and-bound. Two optimization tools, one shared idea — a bound that never overestimates lets the first goal you pull off the list be the optimal one.考試目標：手動追蹤一次 A* 的展開過程，完整記錄 g/h/f，並從 knapsack branch-and-bound 中讀出界限（bounds）。兩個最佳化工具，共用一個想法——一個永遠不會高估的 bound，能讓你第一個從清單取出的 goal 就是最佳解。

0 · What are these two things?0 · 這兩個東西是什麼？

This lesson covers two unrelated-looking problems that share one trick: a bound that never lies in the optimistic direction lets you stop early and still be sure you found the best answer. Before any machinery, here is each problem in plain words.這一課涵蓋兩個看似無關、卻共用同一個訣竅的問題：一個永遠不會往樂觀方向說謊的 bound，讓你能提早停下、又確定自己找到了最佳解。在進入任何機制之前，先用白話說明每個問題。

A* — finding the cheapest route to a goal. Imagine a road map. You start at one town (the root) and want to reach a destination town (the goal node) along roads, where each road has a cost (miles, money, time). A least-cost path is the cheapest way to get there. A* is a method that explores the map outward, always working next on whichever partial route looks most promising, until it can prove it has the cheapest route to the goal.A*——找到通往 goal 的最便宜路徑。想像一張公路地圖。你從某個城鎮出發（root），沿著道路想抵達目的地城鎮（goal node），每條路都有成本（里程、金錢、時間）。least-cost path 就是抵達那裡最便宜的方式。A* 是一種方法，它往外探索地圖，每次都優先處理看起來最有希望的部分路徑，直到能證明自己握有通往 goal 的最便宜路徑。

"Most promising" is measured by one number per place you could be at, call that place a node n:「最有希望」是用一個數字來衡量每個你可能所在的位置，把那個位置稱為一個 node n：

f(n) = g(n) + h(n) g(n) = cost ALREADY SPENT getting from the start to n h(n) = a GUESS of the cost still to go, from n to the goal f(n) = total trip cost if that guess is right
Road-trip picture: you are partway through a drive. g = the miles already on the odometer. h = your guess of the miles left (e.g. straight-line distance to the destination on the map). f = g + h = your guess of the whole trip. A* always works next on the route with the smallest f — the one that looks cheapest end-to-end.公路旅行的畫面：你開車開到一半。g = 里程表上已經跑掉的里程。h = 你對剩餘里程的猜測（例如地圖上到目的地的直線距離）。f = g + h = 你對整趟旅程的猜測。A* 永遠優先處理 f 最小的路徑——也就是從頭到尾看起來最便宜的那條。

The guess h(n) is called a heuristic (an educated estimate). Write h*(n) for the true cheapest cost from n to the goal — what you'd get if you somehow knew the answer. A heuristic is called admissible when it never overestimates: h(n) ≤ h*(n) for every node. (Straight-line distance is admissible: real roads can only be longer than a straight line, never shorter.)這個猜測 h(n) 稱為 heuristic（有根據的估計）。用 h*(n) 表示從 n 到 goal 的真正最便宜成本——如果你不知怎地知道答案的話會得到的值。當一個 heuristic 永遠不會高估時，就稱它為 admissible：對每個 node 都有 h(n) ≤ h*(n)。（直線距離是 admissible 的：真實道路只會比直線長，不會更短。）

Why admissibility is the whole point. If h never overestimates, then f(n) is never bigger than the true best total through n — so f is an honest lower bound. Now suppose A* picks a goal node off the list because it has the smallest f. Every other route still waiting has an f at least this big, and its real cost can only be that high or higher. So nothing waiting can beat the goal we just pulled — the first goal you select for expansion is guaranteed optimal, and you can stop.為什麼 admissibility 是整個重點。如果 h 永遠不高估，那麼 f(n) 永遠不會大於通過 n 的真正最佳總成本——所以 f 是一個誠實的 lower bound。現在假設 A* 因為某個 goal node 的 f 最小而把它從清單取出。其他還在等待的路徑，每一條的 f 都至少這麼大，而它們的真實成本只會等於這個值或更高。所以等待中的任何路徑都贏不過我們剛取出的 goal——你第一個選來展開的 goal 保證是最佳解，你可以停下來了。

The 0/1 knapsack problem — packing a bag for maximum value. You have a bag (the knapsack) that can hold at most M units of weight. There are several items; each item i has a weight Wᵢ and a profit (value) Pᵢ. You must choose a subset of the items to pack so that their total weight stays within M and their total profit is as large as possible. The "0/1" part means each item is all-or-nothing: you either take the whole item (Xᵢ = 1) or leave it (Xᵢ = 0) — no cutting it in half.0/1 knapsack 問題——把袋子裝到價值最大。你有一個袋子（knapsack），最多能裝 M 單位的重量。有好幾件物品，每件物品 i 有一個 weight Wᵢ 和一個 profit（價值）Pᵢ。你必須挑出一個物品的子集（subset）來裝，使得總重量不超過 M、總 profit 越大越好。「0/1」的意思是每件物品要嘛全拿、要嘛不拿：你要嘛拿整件（Xᵢ = 1），要嘛不拿（Xᵢ = 0）——不能把它切一半。

Tiny example: bag holds M = 10 item A: weight 6, profit 9 item B: weight 5, profit 6 item C: weight 5, profit 5 A+B weighs 11 > 10 ✗ won't fit together Best legal pick: A + C = weight 11? no — A+C = 6+5 = 11 > 10 ✗ take A alone → profit 9 (weight 6) take B + C → profit 11 (weight 10) ← best So the answer is {B, C}: profit 11.

Notice you can't just grab the single most valuable item — you must weigh combinations against the capacity. That is what makes it hard.注意你不能只抓那件單獨最值錢的物品——你必須拿各種組合去跟容量權衡。這正是它困難的地方。

The fractional relaxation — an easier cousin that bounds the hard one. "Relax" means loosen a rule to make a problem easier. Here we drop the all-or-nothing rule and allow taking fractions of items (0 ≤ Xᵢ ≤ 1) — like pouring out part of a sack of grain. This fractional knapsack is easy: sort items by density Pᵢ/Wᵢ (profit per unit weight) and greedily take the best-value-per-pound items first, slicing the last one to fill the bag exactly. Because the relaxed version has strictly more freedom than the 0/1 version, its best profit is always at least as good — so it gives a bound (a guaranteed ceiling) on the all-or-nothing answer. That bound is exactly the "never-overestimate" tool A* uses, reused here to prune the search.fractional relaxation——一個界住難題的較簡單表親。「Relax」（鬆弛）意思是放寬一條規則，把問題變簡單。這裡我們拿掉「全拿或不拿」的規則，允許拿物品的分數（0 ≤ Xᵢ ≤ 1）——就像倒出一袋穀物的一部分。這個 fractional knapsack 很簡單：把物品按 density Pᵢ/Wᵢ（每單位重量的 profit）排序，貪心地先拿單位重量最划算的，最後一件切開好把袋子剛好裝滿。因為 relax 過的版本嚴格地比 0/1 版本有更多自由，它的最佳 profit 永遠至少一樣好——所以它給出全拿或不拿這個答案的一個 bound（保證的上限）。那個 bound 正是 A* 用的「永不高估」工具，在這裡被重複利用來剪枝（prune）搜尋。

1 · A* — best-first with a price tag1 · A*——帶價格標籤的 best-first

A* is plain best-first search (Lesson 1's global-view knob: keep all partial routes open and always expand the globally best-looking one) aimed at optimization. Every open node carries a number:A* 就是單純的 best-first search（第 1 課那個全域視角的旋鈕：把所有部分路徑都保持開啟，永遠展開全域看起來最好的那一條），瞄準的是最佳化（optimization）。每個 open node 都帶著一個數字：

f(n) = g(n) + h(n) g(n) = actual cost root → n (what you've spent) h(n) = estimated cost n → goal (what's left, a guess) h*(n) = the TRUE remaining cost n → goal

Expand the open node with the smallest f. The magic is the admissibility condition:展開 f 最小的 open node。神奇之處在於 admissibility condition：

Admissibility: if h(n) ≤ h*(n) for every node (the heuristic never overestimates), then f(n) = g(n)+h(n) ≤ g(n)+h*(n) = f*(n) — so f is a lower bound on the best solution through n. Consequence: the first goal node selected for expansion is optimal, and you can stop. Reaching a goal that merely sits in the list is not enough — it must be the smallest-f node you pick.Admissibility：如果對每個 node 都有 h(n) ≤ h*(n)（heuristic 永不高估），那麼 f(n) = g(n)+h(n) ≤ g(n)+h*(n) = f*(n)——所以 f 是通過 n 的最佳解的 lower bound。結果是：第一個被選來展開的 goal node 就是最佳解，你可以停下來。光是讓一個 goal 待在清單裡還不夠——它必須是你選中的、f 最小的 node。

A* expansion order & stopping nodeA* 的展開順序與停止節點

Open list always sorted by f; expand the front. Newly added children in bold:Open list 永遠按 f 排序；展開最前面的。新加入的子節點以粗體表示：

start open = {R} exp R → A(4) C(5) B(6) open = {A4, C5, B6} exp A → D(5) E(7) open = {C5, D5, B6, E7} exp C → F(6) G(10) open = {D5, F6, B6, E7, G10} exp D → H(10) I(7) open = {F6, B6, I7, E7, G10, H10} exp B → J(11) open = {F6, I7, E7, ...} exp F → K(11) L(8) open = {I7, E7, L8, ...} SELECT I (f = 7) — I is a goal → STOP, optimal

Expansion order: R → A → C → D → B → F, then the search terminates by selecting goal node I (f = g = 7, since h(I)=0). Note the correction: F is just the last node expanded — it is not the goal. I is the goal, and because f(I)=7 is the smallest open value when selected, that path is provably optimal (cost 7 = S→V₁→V₄→T = 2+2+3).展開順序：R → A → C → D → B → F，接著搜尋因選中 goal node I 而終止（f = g = 7，因為 h(I)=0）。注意這個修正：F 只是最後被展開的 node——它不是 goal。I 才是 goal，而因為 f(I)=7 是被選中時 open 中最小的值，那條路徑可被證明是最佳的（成本 7 = S→V₁→V₄→T = 2+2+3）。

The deck turns this into a minimization by negating profit (objective −Σ PᵢXᵢ) so the same lower-bound machinery as TSP/personnel applies — more negative is better. Branch on Xᵢ = 1 vs Xᵢ = 0 at level i.講義透過把 profit 取負（目標 −Σ PᵢXᵢ）將這題轉成最小化（minimization），好讓和 TSP／personnel 相同的 lower-bound 機制適用——越負越好。在第 i 層對 Xᵢ = 1 與 Xᵢ = 0 進行 branch。

Compute the two bounds (upper from feasible, lower from fractional)計算這兩個 bound（upper 來自可行解、lower 來自 fractional）

Upper bound — a feasible integer solution. Greedily take items 1 and 2:Upper bound——一個可行的整數解。貪心地拿物品 1 和 2：

X1 = X2 = 1, rest 0 weight = W1 + W2 = 10 + 19 = 29 ≤ 34 ✓ objective = −(P1 + P2) = −(6 + 10) = −16 ← UPPER BOUND

"Any solution higher than −16 cannot be optimal" — we already have something this good, so −16 caps the search.「任何比 −16 更高的解都不可能是最佳的」——我們已經握有這麼好的解，所以 −16 為搜尋封頂。

Lower bound — fractional (greedy) relaxation. Relax Xᵢ ∈ {0,1} to 0 ≤ Xᵢ ≤ 1; fill by density until M is exactly full, taking a fraction of the last item:Lower bound——fractional（貪心）relaxation。把 Xᵢ ∈ {0,1} 鬆弛成 0 ≤ Xᵢ ≤ 1；按 density 裝填直到 M 剛好滿，最後一件物品拿一個分數：

X1 = 1, X2 = 1, X3 = 5/8, rest 0 weight = 10 + 19 + (5/8)(8) = 10 + 19 + 5 = 34 = M ✓ objective = −(P1 + P2 + (5/8)P3) = −(6 + 10 + 2.5) = −18.5

The fractional optimum is at least as good as any integer solution (it has strictly more freedom), so −18.5 is a valid lower bound on the true objective. Since the real 0/1 objective is an integer, round inward to −18.fractional 最佳解至少和任何整數解一樣好（它嚴格地有更多自由），所以 −18.5 是真正目標值的一個有效 lower bound。由於真正的 0/1 目標值是整數，往內取整到 −18。

bracket: −18 ≤ optimal ≤ −16
