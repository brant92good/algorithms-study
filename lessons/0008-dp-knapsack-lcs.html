0/1 Knapsack & Longest Common Subsequence

Exam goal: write the recurrence and hand-fill the table. Both problems are the same DP move — split on one decision, write max{…}, fill a base, then solve. Master knapsack's gᵢ(y) and LCS's L[i,j] and you can produce the recurrence and trace it back to the answer.

1 · What are these problems? (start here)

This lesson solves two classic problems with the same tool — dynamic programming (DP). Before any formula, here is what each problem actually asks, assuming you have never seen them.

The 0/1 knapsack problem. You have a knapsack (a bag) that can hold a total weight of at most M — its capacity. There are n objects; object i has a weight wᵢ and a profit (value) pᵢ. For each object you make one yes/no choice: take the whole object or leave it — you may not take a fraction, hence "0/1". Goal: pick a subset whose total weight stays within M while the total profit is as large as possible.

Capacity M = 10 object 1: weight 10, profit 40 object 2: weight 3, profit 20 object 3: weight 5, profit 30 Take object 1 alone: weight 10 ✓ (fits), profit 40 Take objects 2 + 3 : weight 3+5 = 8 ✓ (fits), profit 20+30 = 50 ← better Take all three : weight 18 ✗ (over capacity, not allowed) Best = take 2 and 3, profit 50.

"Maximize profit while total weight ≤ capacity" is the whole problem. Notice you cannot just grab the most-profitable object (object 1, profit 40) — it eats all the room and blocks the better pair. That is why a careful method is needed.

What is a DP table / recurrence? A recurrence is a formula that defines the answer to a problem in terms of the answers to smaller versions of the same problem. A DP table is just a grid where we store each of those smaller answers once, so we never recompute them. The plain idea we will use for knapsack is: "what is the best profit I can get using only objects i…n if the remaining capacity is y?" Answer that for every (i, y), building up from tiny cases, and the full answer falls out.

What is a subsequence? A subsequence of a string is what you get by deleting zero or more letters while keeping the rest in their original order — you may skip letters, but you may not reorder them. A substring is stricter: its letters must be contiguous (a solid run, no gaps).

String: C O M P U T E R "CME" is a subsequence (kept order, skipped O, P, U, T, R) "CME" is NOT a substring (those letters aren't next to each other) "OMP" is a substring (a contiguous run) — and also a subsequence "TUP" is neither (out of order)

The Longest Common Subsequence (LCS) problem. Given two strings A and B, find the longest string that is a subsequence of both. We use the verified example A = b a c a d and B = a c c b a d c b; their LCS is a c a d (length 4). Check it: reading bacad and skipping the right letters gives a c a d; reading accbadcb and skipping gives a c a d too.

The one idea behind both: name the answer to a smaller subproblem, write it as a max over one decision, set a base case, and fill a table in dependency order. Knapsack decides "take this object or not"; LCS decides "do these two letters match." Everything below is that single move applied twice.

2 · 0/1 knapsack — split on the first decision

The whole problem is KNAP(1, n, M): maximize Σ pᵢxᵢ subject to Σ wᵢxᵢ ≤ M, each xᵢ ∈ {0,1}. DP starts by asking one question: do I take object 1 or not? That single decision splits the problem into two smaller ones.

y₁ = 0 (skip object 1) ⇒ solve KNAP(2, n, M) y₁ = 1 (take object 1) ⇒ solve KNAP(2, n, M − w₁)

By the principle of optimality (Lesson 7), whichever branch is optimal, the rest of its decisions must themselves be optimal for the smaller subproblem. So define

gᵢ(y) = optimal value of KNAP(i+1, n, y) (best profit using objects i+1 … n with capacity y)

Then the answer is g₀(M), and the recurrence just writes "skip vs take" as a max:

gᵢ(y) = max{ g_{i+1}(y), g_{i+1}(y − w_{i+1}) + p_{i+1} } │ └ skip object i+1 └ take object i+1: pay w_{i+1}, gain p_{i+1} base: gₙ(y) = 0 for all y answer: g₀(M)
The exam twist — direction: this is the forward formulation (decision xᵢ defined via the later objects x_{i+1}…xₙ). Forward-formulated relations are solved backward: start from the base gₙ(y)=0, compute g_{n-1}, then g_{n-2}, … up to g₀(M). Formulation direction and solution direction are always opposite.

3 · Longest Common Subsequence — the L-table

Recall (section 1): the LCS is the longest string that is a subsequence of both inputs, where a subsequence keeps order but may skip letters. Let L[i,j] = length of the LCS of prefixes a₁…aᵢ and b₁…bⱼ. Same DP move: the last cell depends on whether the two current letters match.

L[i,j] = L[i-1,j-1] + 1 if aᵢ = bⱼ (extend the diagonal) max{ L[i-1,j], L[i,j-1] } if aᵢ ≠ bⱼ (drop one letter, take the best) boundary: row 0 = col 0 = 0 answer: L[m,n] (bottom-right cell)

Verified example: A = b a c a d (rows), B = a c c b a d c b (columns).

Before you peek, fill two cells yourself. The top-left block is started for you; row 0 and column 0 are all 0.

A\B "" a c c b a d c b "" 0 0 0 0 0 0 0 0 0 b 0 0 0 0 ? … a 0 ? … ? cell 1 = L[b,b] (aᵢ=b, bⱼ=b → match?) ? cell 2 = L[a,a] (aᵢ=a, bⱼ=a → match?)
Reveal those two cells

L[b, b]: comparing A's b with B's 4th letter b — they match, so = L[prev diagonal] + 1 = 0 + 1 = 1.
L[a, a]: comparing A's a with B's 1st letter a — they match, so = 0 + 1 = 1.
A mismatch cell instead takes the larger of the neighbor above and the neighbor to the left.

Trace it back — recover the actual letters

Start at the bottom-right (value 4) and walk to the 0 region. On a match cell the value came from the diagonal — record that letter and step diagonally up-left; otherwise step to the larger neighbor. Following the circled chain 1 → 2 → 3 → 4 picks out the matches a, c, a, d.

LCS = a c a d, length 4. (Verify: acad is a subsequence of bacad and of accbadcb — read both off, skipping letters.)

Pattern to carry: every DP here is "name the subproblem value, split on one decision, write max, set a base, fill in dependency order." Knapsack fills along the index i; LCS fills the 2-D table row by row. The trace-back (which decision did each optimal cell take?) recovers the actual solution, not just its value.
