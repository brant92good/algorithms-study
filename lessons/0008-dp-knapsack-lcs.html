Exam goal: write the recurrence and hand-fill the table. Both problems are the same DP move — split on one decision, write max{…}, fill a base, then solve. Master knapsack's gᵢ(y) and LCS's L[i,j] and you can produce the recurrence and trace it back to the answer.考試目標：寫出 recurrence 並手算填表。兩個問題都是同一個 DP 招式——在一個決策上分支、寫出 max{…}、填好 base，再求解。掌握 knapsack 的 gᵢ(y) 與 LCS 的 L[i,j]，你就能寫出 recurrence，並且回溯出答案。

1 · What are these problems? (start here)1 · 這些是什麼問題？（從這裡開始）

This lesson solves two classic problems with the same tool — dynamic programming (DP). Before any formula, here is what each problem actually asks, assuming you have never seen them.這一課用同一個工具——Dynamic Programming（DP）——解兩個經典問題。在任何公式之前，先說明每個問題實際在問什麼，假設你從沒看過它們。

The 0/1 knapsack problem. You have a knapsack (a bag) that can hold a total weight of at most M — its capacity. There are n objects; object i has a weight wᵢ and a profit (value) pᵢ. For each object you make one yes/no choice: take the whole object or leave it — you may not take a fraction, hence "0/1". Goal: pick a subset whose total weight stays within M while the total profit is as large as possible.0/1 knapsack 問題。你有一個 knapsack（背包），最多能裝總重 M ——這是它的 capacity（容量）。共有 n 個物件；物件 i 有 weight（重量）wᵢ 與 profit（利潤／價值）pᵢ。對每個物件你做一個是／否的選擇：拿整個物件或不拿——不能拿一部分，所以叫「0/1」。目標：挑出一個子集，使總重不超過 M，同時總 profit 越大越好。

Capacity M = 10 object 1: weight 10, profit 40 object 2: weight 3, profit 20 object 3: weight 5, profit 30 Take object 1 alone: weight 10 ✓ (fits), profit 40 Take objects 2 + 3 : weight 3+5 = 8 ✓ (fits), profit 20+30 = 50 ← better Take all three : weight 18 ✗ (over capacity, not allowed) Best = take 2 and 3, profit 50.

"Maximize profit while total weight ≤ capacity" is the whole problem. Notice you cannot just grab the most-profitable object (object 1, profit 40) — it eats all the room and blocks the better pair. That is why a careful method is needed.「在總重 ≤ capacity 的前提下最大化 profit」就是整個問題。注意你不能直接抓 profit 最高的物件（物件 1，profit 40）——它吃光所有空間，擋掉更好的那一組。這就是為什麼需要一個謹慎的方法。

What is a DP table / recurrence? A recurrence is a formula that defines the answer to a problem in terms of the answers to smaller versions of the same problem. A DP table is just a grid where we store each of those smaller answers once, so we never recompute them. The plain idea we will use for knapsack is: "what is the best profit I can get using only objects i…n if the remaining capacity is y?" Answer that for every (i, y), building up from tiny cases, and the full answer falls out.什麼是 DP table / recurrence？一個 recurrence（遞迴關係式）是用同一問題較小版本的答案來定義問題答案的公式。一個 DP table 只是一張表格，把每個較小的答案各存一次，這樣就不會重算。我們對 knapsack 要用的樸素想法是：「如果只用物件 i…n、剩餘 capacity 是 y，我能拿到的最佳 profit 是多少？」對每個 (i, y) 都回答這個問題，從微小的情形往上堆疊，完整答案就自然浮現。

What is a subsequence? A subsequence of a string is what you get by deleting zero or more letters while keeping the rest in their original order — you may skip letters, but you may not reorder them. A substring is stricter: its letters must be contiguous (a solid run, no gaps).什麼是 subsequence？一個字串的 subsequence（子序列）是刪掉零個或多個字母、其餘保持原本順序後得到的結果——你可以跳過字母，但不能重新排序。substring（子字串）更嚴格：它的字母必須是 contiguous（連續的一段，沒有空隙）。

String: C O M P U T E R "CME" is a subsequence (kept order, skipped O, P, U, T, R) "CME" is NOT a substring (those letters aren't next to each other) "OMP" is a substring (a contiguous run) — and also a subsequence "TUP" is neither (out of order)

The Longest Common Subsequence (LCS) problem. Given two strings A and B, find the longest string that is a subsequence of both. We use the verified example A = b a c a d and B = a c c b a d c b; their LCS is a c a d (length 4). Check it: reading bacad and skipping the right letters gives a c a d; reading accbadcb and skipping gives a c a d too.Longest Common Subsequence（LCS）問題。給定兩個字串 A 與 B，找出同時是兩者 subsequence 的最長字串。我們用已驗證的例子 A = b a c a d 與 B = a c c b a d c b；它們的 LCS 是 a c a d（長度 4）。驗證一下：讀 bacad 並跳過正確的字母可得 a c a d；讀 accbadcb 並跳過也得到 a c a d。

The one idea behind both: name the answer to a smaller subproblem, write it as a max over one decision, set a base case, and fill a table in dependency order. Knapsack decides "take this object or not"; LCS decides "do these two letters match." Everything below is that single move applied twice.兩者背後的同一個想法：為較小的 subproblem 的答案命名、把它寫成對一個決策取 max、設定 base case，再依相依順序填表。Knapsack 決定「拿不拿這個物件」；LCS 決定「這兩個字母是否相符」。底下的一切都是同一招式用了兩次。

2 · 0/1 knapsack — split on the first decision2 · 0/1 knapsack——在第一個決策上分支

The whole problem is KNAP(1, n, M): maximize Σ pᵢxᵢ subject to Σ wᵢxᵢ ≤ M, each xᵢ ∈ {0,1}. DP starts by asking one question: do I take object 1 or not? That single decision splits the problem into two smaller ones.整個問題是 KNAP(1, n, M)：在 Σ wᵢxᵢ ≤ M、每個 xᵢ ∈ {0,1} 的條件下最大化 Σ pᵢxᵢ。DP 從問一個問題開始：我要不要拿物件 1？這一個決策就把問題分成兩個較小的問題。

y₁ = 0 (skip object 1) ⇒ solve KNAP(2, n, M) y₁ = 1 (take object 1) ⇒ solve KNAP(2, n, M − w₁)

By the principle of optimality (Lesson 7), whichever branch is optimal, the rest of its decisions must themselves be optimal for the smaller subproblem. So define根據 principle of optimality（最佳化原理，第 7 課），無論哪個分支是最佳的，它其餘的決策對較小的 subproblem 本身也必然是最佳的。所以定義

gᵢ(y) = optimal value of KNAP(i+1, n, y) (best profit using objects i+1 … n with capacity y)

Then the answer is g₀(M), and the recurrence just writes "skip vs take" as a max:那麼答案就是 g₀(M)，而 recurrence 只是把「跳過 vs 拿取」寫成一個 max：

gᵢ(y) = max{ g_{i+1}(y), g_{i+1}(y − w_{i+1}) + p_{i+1} } │ └ skip object i+1 └ take object i+1: pay w_{i+1}, gain p_{i+1} base: gₙ(y) = 0 for all y answer: g₀(M)
The exam twist — direction: this is the forward formulation (decision xᵢ defined via the later objects x_{i+1}…xₙ). Forward-formulated relations are solved backward: start from the base gₙ(y)=0, compute g_{n-1}, then g_{n-2}, … up to g₀(M). Formulation direction and solution direction are always opposite.考試陷阱——方向：這是 forward（前向）formulation（決策 xᵢ 透過後面的物件 x_{i+1}…xₙ 來定義）。Forward 寫法的關係式要 backward（反向）求解：從 base gₙ(y)=0 開始，算出 g_{n-1}，再 g_{n-2}，…一直到 g₀(M)。Formulation 方向與 solution 方向永遠相反。

3 · Longest Common Subsequence — the L-table3 · Longest Common Subsequence——L-table

Recall (section 1): the LCS is the longest string that is a subsequence of both inputs, where a subsequence keeps order but may skip letters. Let L[i,j] = length of the LCS of prefixes a₁…aᵢ and b₁…bⱼ. Same DP move: the last cell depends on whether the two current letters match.回想（第 1 節）：LCS 是同時為兩個輸入 subsequence 的最長字串，其中 subsequence 保持順序但可跳過字母。令 L[i,j] = 前綴 a₁…aᵢ 與 b₁…bⱼ 的 LCS 長度。同樣的 DP 招式：最後一格取決於目前兩個字母是否相符。

L[i,j] = L[i-1,j-1] + 1 if aᵢ = bⱼ (extend the diagonal) max{ L[i-1,j], L[i,j-1] } if aᵢ ≠ bⱼ (drop one letter, take the best) boundary: row 0 = col 0 = 0 answer: L[m,n] (bottom-right cell)

Verified example: A = b a c a d (rows), B = a c c b a d c b (columns).已驗證的例子：A = b a c a d（列），B = a c c b a d c b（欄）。

Before you peek, fill two cells yourself. The top-left block is started for you; row 0 and column 0 are all 0.偷看之前，先自己填兩格。左上角區塊已幫你起頭；第 0 列與第 0 欄全是 0。

A\B "" a c c b a d c b "" 0 0 0 0 0 0 0 0 0 b 0 0 0 0 ? … a 0 ? … ? cell 1 = L[b,b] (aᵢ=b, bⱼ=b → match?) ? cell 2 = L[a,a] (aᵢ=a, bⱼ=a → match?)
Reveal those two cells揭曉那兩格

L[b, b]: comparing A's b with B's 4th letter b — they match, so = L[prev diagonal] + 1 = 0 + 1 = 1.
L[a, a]: comparing A's a with B's 1st letter a — they match, so = 0 + 1 = 1.
A mismatch cell instead takes the larger of the neighbor above and the neighbor to the left.L[b, b]：拿 A 的 b 和 B 的第 4 個字母 b 比——它們相符，所以 = L[prev diagonal] + 1 = 0 + 1 = 1。
L[a, a]：拿 A 的 a 和 B 的第 1 個字母 a 比——它們相符，所以 = 0 + 1 = 1。
不相符的格子則取上方鄰格與左方鄰格中較大的那個。

Start at the bottom-right (value 4) and walk to the 0 region. On a match cell the value came from the diagonal — record that letter and step diagonally up-left; otherwise step to the larger neighbor. Following the circled chain 1 → 2 → 3 → 4 picks out the matches a, c, a, d.從右下角（值 4）出發，走向 0 的區域。在相符的格子上，這個值來自對角線——記下那個字母並斜向左上走一步；否則就走向較大的鄰格。沿著圈起來的鏈 1 → 2 → 3 → 4 就挑出相符的 a, c, a, d。

LCS = a c a d, length 4. (Verify: acad is a subsequence of bacad and of accbadcb — read both off, skipping letters.)LCS = a c a d，長度 4。（驗證：acad 是 bacad 和 accbadcb 的 subsequence——兩個都讀一遍、跳過字母即可。）

Pattern to carry: every DP here is "name the subproblem value, split on one decision, write max, set a base, fill in dependency order." Knapsack fills along the index i; LCS fills the 2-D table row by row. The trace-back (which decision did each optimal cell take?) recovers the actual solution, not just its value.要帶走的模式：這裡每個 DP 都是「為 subproblem 的值命名、在一個決策上分支、寫 max、設定 base、依相依順序填表」。Knapsack 沿著索引 i 填；LCS 一列一列填 2-D 表。回溯（每個最佳格子當初選了哪個決策？）還原的是實際的解，而不只是它的值。
