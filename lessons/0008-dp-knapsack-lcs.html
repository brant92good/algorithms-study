0/1 Knapsack & Longest Common Subsequence0/1 Knapsack 與 Longest Common Subsequence

Exam goal: write the recurrence and hand-fill the table. Both problems are the same DP move — split on one decision, write max{…}, fill a base, then solve. Master knapsack's gi(y) and LCS's L[i,j] and you can produce the recurrence and trace it back to the answer.考試要會的：寫出 recurrence，再用手把表填出來。這兩題其實是同一個 DP 套路——挑一個決策來分支、寫成 max{…}、把 base 填好，然後算。把 knapsack 的 gi(y) 跟 LCS 的 L[i,j] 搞懂，你就能寫出 recurrence，而且還能回溯把答案還原出來。

1 · What are these problems? (start here)1 · 這兩題到底在問什麼？（先看這裡）

This lesson solves two classic problems with the same tool — dynamic programming (DP). Before any formula, here is what each problem actually asks, assuming you have never seen them.這一課用同一個工具——Dynamic Programming（DP）——解兩個經典題目。先別管公式，假設你完全沒看過這兩題，我們先把它們各自在問什麼講清楚。

The 0/1 knapsack problem. You have a knapsack (a bag) that can hold a total weight of at most M — its capacity. There are n objects; object i has a weight wi and a profit (value) pi. For each object you make one yes/no choice: take the whole object or leave it — you may not take a fraction, hence "0/1". Goal: pick a subset whose total weight stays within M while the total profit is as large as possible.0/1 knapsack 問題。你有一個 knapsack（背包），最多只能裝總重 M——這就是它的 capacity（容量）。現在有 n 個物件，第 i 個的 weight（重量）是 wi、profit（利潤、也就是價值）是 pi。每個物件你只能二選一：整個拿走，或是完全不拿——不能只拿一半，所以才叫「0/1」。目標就是：挑一組物件出來，總重不要超過 M，但總 profit 越大越好。

Capacity M = 10 object 1: weight 10, profit 40 object 2: weight 3, profit 20 object 3: weight 5, profit 30 Take object 1 alone: weight 10 ✓ (fits), profit 40 Take objects 2 + 3 : weight 3+5 = 8 ✓ (fits), profit 20+30 = 50 ← better Take all three : weight 18 ✗ (over capacity, not allowed) Best = take 2 and 3, profit 50.

"Maximize profit while total weight ≤ capacity" is the whole problem. Notice you cannot just grab the most-profitable object (object 1, profit 40) — it eats all the room and blocks the better pair. That is why a careful method is needed.整題就一句話：「總重不超過 capacity 的情況下，把 profit 衝到最大。」重點是你不能傻傻地抓 profit 最高的物件（物件 1，profit 40）——它一個就把空間吃光，反而擋掉了更好的那一組。所以才需要一個比較有系統的方法。

What is a DP table / recurrence? A recurrence is a formula that defines the answer to a problem in terms of the answers to smaller versions of the same problem. A DP table is just a grid where we store each of those smaller answers once, so we never recompute them. The plain idea we will use for knapsack is: "what is the best profit I can get using only objects i…n if the remaining capacity is y?" Answer that for every (i, y), building up from tiny cases, and the full answer falls out.DP table 跟 recurrence 是什麼？recurrence（遞迴關係式）就是一條公式，它用同一個問題比較小的版本的答案，來定義整題的答案。DP table 則只是一張表，把那些小答案各存一次，這樣就不會重算。knapsack 的核心想法很單純：「假設我只能用物件 i…n、手上剩 y 的 capacity，那我最多能拿到多少 profit？」把每個 (i, y) 都回答出來，從最小的情況一路往上堆，整題的答案自然就跑出來了。

What is a subsequence? A subsequence of a string is what you get by deleting zero or more letters while keeping the rest in their original order — you may skip letters, but you may not reorder them. A substring is stricter: its letters must be contiguous (a solid run, no gaps).subsequence 又是什麼？一個字串的 subsequence（子序列），就是把其中零個或多個字母刪掉、剩下的維持原本順序所得到的東西——你可以跳過字母，但不能換位置。substring（子字串）就嚴格多了：字母必須是 contiguous（連在一起的一段，中間不能有空隙）。

String: C O M P U T E R "CME" is a subsequence (kept order, skipped O, P, U, T, R) "CME" is NOT a substring (those letters aren't next to each other) "OMP" is a substring (a contiguous run) — and also a subsequence "TUP" is neither (out of order)

The Longest Common Subsequence (LCS) problem. Given two strings A and B, find the longest string that is a subsequence of both. We use the verified example A = b a c a d and B = a c c b a d c b; their LCS is a c a d (length 4). Check it: reading bacad and skipping the right letters gives a c a d; reading accbadcb and skipping gives a c a d too.Longest Common Subsequence（LCS）問題。給你兩個字串 A 跟 B，要找出一個最長的字串，它同時是這兩個的 subsequence。我們用驗證過的例子：A = b a c a d、B = a c c b a d c b，它們的 LCS 是 a c a d（長度 4）。自己對一下：把 bacad 跳掉該跳的字母會剩 a c a d；把 accbadcb 跳一跳也會剩 a c a d。

The one idea behind both: name the answer to a smaller subproblem, write it as a max over one decision, set a base case, and fill a table in dependency order. Knapsack decides "take this object or not"; LCS decides "do these two letters match." Everything below is that single move applied twice.兩題共用的同一個想法：先幫某個比較小的 subproblem 的答案取個名字，把它寫成「對一個決策取 max」，設好 base case，再照相依順序把表填出來。Knapsack 要決定的是「這個物件拿不拿」，LCS 要決定的是「這兩個字母有沒有相符」。底下的東西，都是同一招用了兩次而已。

2 · 0/1 knapsack — split on the first decision2 · 0/1 knapsack——拿第一個決策來分支

The whole problem is KNAP(1, n, M): maximize Σ pixi subject to Σ wixi ≤ M, each xi ∈ {0,1}. DP starts by asking one question: do I take object 1 or not? That single decision splits the problem into two smaller ones.整題寫成 KNAP(1, n, M)：在 Σ wixi ≤ M、每個 xi ∈ {0,1} 的限制下，把 Σ pixi 最大化。DP 的起手式就是問一句：物件 1 到底拿不拿？光是這一個決策，就把整題拆成兩個比較小的問題。

y1 = 0 (skip object 1) ⇒ solve KNAP(2, n, M) y1 = 1 (take object 1) ⇒ solve KNAP(2, n, M − w1)

By the principle of optimality (Lesson 7), whichever branch is optimal, the rest of its decisions must themselves be optimal for the smaller subproblem. So define由 principle of optimality（最佳化原理，第 7 課）可知：不管最後是哪個分支勝出，它剩下那些決策，對那個比較小的 subproblem 來說也一定是最佳的。所以我們定義

gi(y) = optimal value of KNAP(i+1, n, y) (best profit using objects i+1 … n with capacity y)

Mind the shift in the subscript. Read gi as "objects 1…i are already decided, so gi decides about object i+1 next." So g0 has decided nothing yet and still has all n objects to consider (that is why the answer is g0(M)); g2 is the step that decides object 3, not object 2. A beginner naturally reads "g2 → object 2" — but the index is the count of objects behind you, so the object in play is always i+1.注意這個 subscript 的位移。gi 要這樣讀：「物件 1…i 已經決定完了，所以 gi 接下來要決定的是物件 i+1。」所以 g0 什麼都還沒決定、整 n 個物件都還在（這就是為什麼答案是 g0(M)）；而 g2 是在決定物件 3，不是物件 2。很多人會直覺把「g2 → 物件 2」配在一起——但下標其實是你背後已經處理掉的物件數，正在考慮的那個永遠是 i+1。

Then the answer is g0(M), and the recurrence just writes "skip vs take" as a max. One caveat on the take-branch: it only makes sense when object i+1 actually fits. If w_{i+1} > y then y − w_{i+1} < 0 — the object cannot fit, so the take-branch is invalid and you keep only the skip-branch g_{i+1}(y).這樣答案就是 g0(M)，而 recurrence 不過就是把「跳過還是拿」寫成一個 max。take 那一支有個前提：要物件 i+1 真的塞得進去才有意義。如果 w_{i+1} > y，那 y − w_{i+1} < 0——物件根本放不下，所以 take 那支不成立，這時只能留下 skip 那支 g_{i+1}(y)。

gi(y) = max{ g_{i+1}(y), g_{i+1}(y − w_{i+1}) + p_{i+1} } │ └ skip object i+1 └ take object i+1: pay w_{i+1}, gain p_{i+1} base: gn(y) = 0 for all y answer: g0(M)
The exam twist — direction. Why is this the forward formulation? Because each gi is written using a higher index, g_{i+1} (look at the recurrence above — its right side mentions g_{i+1}). The other choice, the backward formulation, would define gi over objects 1…i and write it using g_{i-1} (a lower index) — that is what "the other direction" even looks like.考試最愛考的陷阱——方向：這是 forward（前向）formulation（決策 xi 是靠後面那些物件 x_{i+1}…xn 來定義的）。重點來了：forward 寫出來的式子，要 backward（反向）去解——從 base gn(y)=0 開始，先算 g_{n-1}、再 g_{n-2}，一路算到 g0(M)。為什麼這叫 forward？因為每個 gi 都是用更大的下標 g_{i+1} 寫出來的（看上面的 recurrence，右邊出現的是 g_{i+1}）。另一種寫法 backward formulation，會把 gi 定義成物件 1…i、並用 g_{i-1}（更小的下標）來寫——這就是「另一個方向」長什麼樣子。

And what "solved backward" means, operationally: A DP value can only be computed once everything it depends on already exists — that is the whole reason direction matters here, not a technicality. So: because each gi is written using g_{i+1}, you cannot fill gi until the higher-index g_{i+1} already exists — so you must compute the high-index g's first. Start from the base gn(y)=0, then g_{n-1}, then g_{n-2}, … down to g0(M). That descending order is all "backward solving" means here. The rule it shows: a formula pointing forward (to g_{i+1}) forces you to fill the table backward (high index first) — formulation and solution order always run opposite.那「backward 去解」實際在做什麼：一個 DP 值，要等它依賴的東西全部都算好了才算得出來——這就是這裡「方向」之所以重要的原因，不是在鑽牛角尖。所以：因為每個 gi 都用到 g_{i+1}，你在算 gi 之前，下標更大的 g_{i+1} 一定要先存在——所以下標大的要先算。從 base gn(y)=0 開始，接著 g_{n-1}、g_{n-2}……一路算到 g0(M)。這個由大到小的順序，就是這裡所謂的「backward solving」。它示範的規則是：公式指向後面（g_{i+1}），就逼你反過來（下標大的先）填表——formulation 跟解的順序永遠相反。

3 · Longest Common Subsequence — the L-table3 · Longest Common Subsequence——L-table

Recall (section 1): the LCS is the longest string that is a subsequence of both inputs, where a subsequence keeps order but may skip letters. Let L[i,j] = length of the LCS of prefixes a1…ai and b1…bj, where m and n are the lengths of A and B (here m=5, n=8). Heads-up: this n is the length of string B and is unrelated to the knapsack's n (the object count) — same letter, different problem. Same DP move: the last cell depends on whether the two current letters match.複習一下（第 1 節）：LCS 就是那個同時是兩個輸入的 subsequence 的最長字串，而 subsequence 是順序要保留、但字母可以跳。我們令 L[i,j] = 前綴 a1…ai 跟 b1…bj 的 LCS 長度，其中 m、n 是 A 跟 B 的長度（這裡 m=5、n=8）。提醒一下：這個 n 是字串 B 的長度，跟前面 knapsack 的 n（物件數量）沒有關係——同一個字母、不同的題目，別搞混了。一樣的 DP 套路：最後一格填什麼，就看現在這兩個字母有沒有相符。

L[i,j] = L[i-1,j-1] + 1 if ai = bj (extend the diagonal) max{ L[i-1,j], L[i,j-1] } if ai ≠ bj (drop one letter, take the best) boundary: row 0 = col 0 = 0 answer: L[m,n] (bottom-right cell)

Verified example: A = b a c a d (rows), B = a c c b a d c b (columns).驗證過的例子：A = b a c a d（當列），B = a c c b a d c b（當欄）。

L[b, b]: comparing A's b with B's 4th letter b — they match, so = L[prev diagonal] + 1 = 0 + 1 = 1.
L[a, a]: comparing A's a with B's 1st letter a — they match, so = 0 + 1 = 1.
A mismatch cell instead takes the larger of the neighbor above and the neighbor to the left.L[b, b]：把 A 的 b 跟 B 的第 4 個字母 b 比——相符，所以 = L[prev diagonal] + 1 = 0 + 1 = 1。
L[a, a]：把 A 的 a 跟 B 的第 1 個字母 a 比——也相符，所以 = 0 + 1 = 1。
要是不相符，這格就改成取「上面」跟「左邊」兩個鄰格裡比較大的那個。

Start at the bottom-right (value 4) and walk to the 0 region. On a match cell the value came from the diagonal — record that letter and step diagonally up-left; on a mismatch step to the larger of {up, left}, and on a tie use the left neighbour (any fixed tie-rule yields a valid LCS; we pick left here). Below, "(row, col)" gives the actual cell — rows are b a c a d = 1…5, columns are the B-letters a c c b a d c b = 1…8.從右下角（值 4）開始，往 0 的那一區走。碰到相符的格子，代表這個值是從對角線來的——把那個字母記下來，然後斜斜往左上走一步；碰到不相符就往 {上, 左} 裡比較大的那個走，遇到平手就走左邊（任何固定的 tie 規則都會得到一個合法的 LCS，這裡選走左）。下面「(row, col)」標的是實際的格子——row 是 b a c a d = 1…5，column 是 B 的字母 a c c b a d c b = 1…8。

Pattern to carry: every DP here is "name the subproblem value, split on one decision, write max, set a base, fill in dependency order." Knapsack fills along the index i; LCS fills the 2-D table row by row. The trace-back (which decision did each optimal cell take?) recovers the actual solution, not just its value.要記住的套路：這裡每個 DP 都是同一句話——「幫 subproblem 的值取名、挑一個決策來分支、寫 max、設好 base、照相依順序填表」。Knapsack 是沿著索引 i 填，LCS 是一列一列把 2-D 表填滿。最後的回溯（看每個最佳格子當初是選了哪個決策）還原出來的是真正的解，而不只是那個數值。
