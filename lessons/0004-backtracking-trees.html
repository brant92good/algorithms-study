Backtracking on Solution Trees在解答樹上做 Backtracking

Exam goal: hand-trace the three classic Ch6 feasibility problems — SAT, sum-of-subsets, Hamiltonian circuit — as a walk over a solution tree. Lesson 1 gave you the abstract knob; here you turn it on real instances. The tangible win: draw the tree, prune dead subtrees, and read off the answer.考試目標：手動追蹤第 6 章的三個經典 feasibility 問題 — SAT、sum-of-subsets、Hamiltonian circuit — 把它們看成在一棵解答樹上的走訪。第 1 課給了你抽象的旋鈕；這裡我們把它套用到實際的 instance 上。實際收穫：畫出樹、剪掉死掉的子樹，然後讀出答案。

0 · What are these problems?0 · 這些問題是什麼？

Before any tree-drawing, here are the four ideas this lesson rests on — assume you have never met them.在畫任何樹之前，先看這一課依賴的四個概念 — 假設你從沒接觸過它們。

Backtracking. A way to search for a solution by building a candidate one piece at a time. Picture the choices as a tree: the root has chosen nothing, and each step down adds one more decision. You walk down a branch, and the moment a partial choice can no longer lead to a valid solution you abandon that branch — retreat back up and try the next option. That retreat is the "backtrack." It saves you from finishing candidates you already know are doomed.Backtracking。一種搜尋解答的方法，做法是一次一塊地建構候選解。把這些選擇想成一棵樹：root 什麼都還沒選，每往下一步就多做一個決定。你沿著一條分支往下走，一旦某個部分選擇再也不可能導向合法解，就放棄那條分支 — 退回上一層，試下一個選項。這個退回就是「backtrack（回溯）」。它讓你不必把早就注定失敗的候選解走完。

The Satisfiability problem (SAT). You are given a Boolean formula — a logic expression whose variables are each either true or false — and asked: is there a way to set the variables that makes the whole formula true? Some vocabulary, defined once and used throughout:Satisfiability 問題（SAT）。給你一個 Boolean formula — 一個邏輯運算式，其中每個變數不是 true 就是 false — 並問：有沒有辦法設定這些變數，使整個 formula 為真？以下是一些詞彙，定義一次後全程使用：

Term術語Plain meaning白話意思Example範例
literala variable, or its negation (NOT)一個變數，或它的否定（NOT）x₁ or ¬x₁
clausean OR of literals — true if any one literal in it is true若干 literal 的 OR — 只要其中任一個 literal 為真就成立x₂ ∨ x₅
CNF(conjunctive normal form) an AND of clauses — true only if every clause is true（conjunctive normal form）若干 clause 的 AND — 只有每個 clause 都為真才成立(x₂ ∨ x₅) ∧ x₃
satisfiablesome assignment of true/false makes the whole formula true存在某種 true/false 的指派使整個 formula 為真see below見下方

Tiny example: the CNF formula (x₁ ∨ x₂) ∧ (¬x₁) is satisfiable — set x₁ = false, x₂ = true and both clauses hold. By contrast (x₁) ∧ (¬x₁) is unsatisfiable: no value of x₁ can make both clauses true at once.小例子：CNF formula (x₁ ∨ x₂) ∧ (¬x₁) 是 satisfiable — 設 x₁ = false, x₂ = true，兩個 clause 都成立。相對地 (x₁) ∧ (¬x₁) 是 unsatisfiable：沒有任何 x₁ 的值能讓兩個 clause 同時為真。

The sum-of-subsets problem. Given a set of numbers and a target, is there a subset whose elements add up to exactly the target? Tiny example: from {3, 5, 8} with target 11, the subset {3, 8} works (3 + 8 = 11); with target 4, no subset hits it.sum-of-subsets 問題。給一組數字和一個 target，是否存在某個 subset，其元素加總剛好等於 target？小例子：從 {3, 5, 8} 且 target 11，subset {3, 8} 可行（3 + 8 = 11）；若 target 為 4，則沒有任何 subset 命中。

The Hamiltonian circuit problem. A Hamiltonian circuit is a cycle in a graph that visits every vertex exactly once and returns to where it started. The question: does a given graph contain one? Do not confuse it with an Euler circuit, which is a cycle that uses every edge exactly once (vertices may be revisited). The classic trap is swapping the two:Hamiltonian circuit 問題。Hamiltonian circuit 是圖中的一個環，恰好走過每個 vertex 一次並回到起點。問題是：給定的圖裡有沒有這樣的環？不要把它和 Euler circuit 搞混，後者是恰好用過每條 edge 一次的環（vertex 可以重複造訪）。經典陷阱就是把這兩個對調：

Hamiltonian = every vertex once. Euler = every edge once. In a square a-b-c-d-a, going round the rim a→b→c→d→a is both — but add a diagonal a-c and the rim is still Hamiltonian (all 4 corners once) yet no longer Euler (the diagonal edge is never used).Hamiltonian = 每個 vertex 一次。Euler = 每條 edge 一次。在一個正方形 a-b-c-d-a 中，沿著外圈走 a→b→c→d→a 兩者皆是 — 但加上一條對角線 a-c，外圈仍然是 Hamiltonian（4 個角各一次），卻不再是 Euler（那條對角 edge 從沒被用到）。

Why one technique fits all three: each asks "does a valid candidate exist?" (a feasibility question, not "find the cheapest"). Each candidate is built choice by choice, so each is a walk down a tree — exactly what backtracking searches, abandoning doomed branches early.為什麼同一種技巧適用於這三者：每個問題都在問「是否存在一個合法的候選解？」（一個 feasibility 問題，而不是「找最便宜的」）。每個候選解都是一個選擇接一個選擇建出來的，所以每個都是在一棵樹上往下走 — 正是 backtracking 搜尋的對象，並提早放棄注定失敗的分支。

1 · The same tree, three disguises1 · 同一棵樹，三種偽裝

Every problem below becomes a solution tree: the root decides nothing, each level fixes one more choice, each leaf is a full candidate. We walk it depth-first (deepest open node first → a stack, from Lesson 1) and backtrack — pop and retreat — the instant a partial choice can no longer succeed. That early abandon is pruning: skip a whole subtree without examining its leaves.下面每個問題都變成一棵解答樹：root 不做決定，每往下一層固定多一個選擇，每個 leaf 是一個完整候選解。我們以 depth-first 的方式走訪它（最深的未展開節點優先 → 一個 stack，來自第 1 課），並在某個部分選擇不再可能成功的瞬間 backtrack — pop 並退回。這種提早放棄就是 pruning（剪枝）：略過整棵子樹，不去檢查它的 leaf。

Key exam idea: these are feasibility problems — we want a solution (or "none exists"), so DFS stops at the first goal leaf. Pruning is what makes the 2ⁿ-leaf tree tractable: you may never need to examine all 2ⁿ assignments.關鍵考點：這些是 feasibility 問題 — 我們要的是一個解（或「不存在」），所以 DFS 在第一個 goal leaf 就停下。Pruning 正是讓這棵 2ⁿ 個 leaf 的樹變得可行的關鍵：你可能根本不需要檢查全部 2ⁿ 種指派。

2 · SAT: a depth-n binary tree2 · SAT：一棵 depth-n 的 binary tree

Given a Boolean formula in n variables, there are 2ⁿ truth assignments — a complete binary tree of depth n, branching xᵢ = T / xᵢ = F at level i. For n = 3, that is 2³ = 8 leaves.給定一個有 n 個變數的 Boolean formula，共有 2ⁿ 種真值指派 — 一棵完整的 depth n 的 binary tree，在第 i 層以 xᵢ = T / xᵢ = F 分支。當 n = 3 時就是 2³ = 8 個 leaf。

The slide instance (clauses that must all hold; ¬ = NOT, ∨ = OR):投影片上的 instance（這些 clause 必須全部成立；¬ = NOT，∨ = OR）：

(1) ¬x₁ (4) x₃ (2) x₁ (5) ¬x₂ (3) x₂ ∨ x₅

Walk the tree and prune the moment any clause is violated. At the very first level we already hit a wall:走訪這棵樹，並在任何 clause 被違反的瞬間剪枝。在最開始的第一層我們就已經撞牆：

Trace the root's two branches追蹤 root 的兩條分支

Branch x₁ = T → clause (1) ¬x₁ is falsified → prune that whole subtree.
Branch x₁ = F → clause (2) x₁ is falsified → prune that subtree too.
Both children of the root die immediately because clauses (1) and (2) directly contradict. So this instance is unsatisfiable — proven by pruning at depth 1, without ever building the other 2³ − 2 nodes. That is the whole point: a partial assignment can already reveal a contradiction.x₁ = T 分支 → clause (1) ¬x₁ 被否證 → 剪掉整棵子樹。
x₁ = F 分支 → clause (2) x₁ 被否證 → 那棵子樹也剪掉。
root 的兩個子節點都立刻死亡，因為 clause (1) 和 (2) 直接矛盾。所以這個 instance 是 unsatisfiable — 在 depth 1 就靠剪枝證明，根本不必建出其餘的 2³ − 2 個節點。這正是重點：一個部分指派就已經能揭露矛盾。

3 · Sum-of-subsets by DFS3 · 用 DFS 解 Sum-of-subsets

S = {7, 5, 1, 2, 10}, target = 9. Binary tree: at level i decide whether element i is in or out. A stack guides the DFS (push children, pop to backtrack). Prune when the running sum exceeds 9. Node labels below are the running sum; x = pruned (over target).S = {7, 5, 1, 2, 10}，target = 9。Binary tree：在第 i 層決定元素 i 要選入還是不選。一個 stack 引導 DFS（push 子節點，pop 來 backtrack）。當累積總和超過 9 時剪枝。下方節點標籤是累積總和；x = 已剪枝（超過 target）。

0 X1=7 / 7 X2=5 / X2=1 \ X2=2 \ 12(x) 8 9 ← Goal (7+2=9) X3=2/ X3=10\ 10(x) 18(x)

Decide the answer subset yourself before opening it.先自己想出答案 subset，再打開來看。

DFS trace and the answerDFS 追蹤與答案

Push root 0. Take x₁ = 7 → sum 7. From 7, try x₂ = 5 → 12 > 9 → prune, pop. Try x₂ = 1 → 8; from 8, x₃ = 2 → 10 (x) and x₃ = 10 → 18 (x), both over → backtrack. Try x₂ = 2 → 7 + 2 = 9 = target → goal node.
Answer subset: {7, 2}. The stack's LIFO order is exactly what makes the search dive down the 7 branch fully, then retreat.Push root 0。取 x₁ = 7 → 總和 7。從 7 試 x₂ = 5 → 12 > 9 → 剪枝、pop。試 x₂ = 1 → 8；從 8，x₃ = 2 → 10 (x) 與 x₃ = 10 → 18 (x)，兩者都超過 → backtrack。試 x₂ = 2 → 7 + 2 = 9 = target → goal node。
答案 subset：{7, 2}。stack 的 LIFO 順序正是讓搜尋先把 7 那條分支完整往下鑽、再退回的原因。

4 · Hamiltonian circuit4 · Hamiltonian circuit

A Hamiltonian circuit visits every vertex exactly once and returns to the start. Tree: each level chooses the next vertex to visit; a root-to-leaf path of length n that reuses no vertex and closes back to the start is a Hamiltonian circuit (Fig. 6-8). The example graph (Fig. 6-6, vertices 1–7) — adjacency you can walk by hand:Hamiltonian circuit 恰好造訪每個 vertex 一次並回到起點。樹：每一層選擇下一個要造訪的 vertex；一條長度為 n 的 root-to-leaf 路徑，若不重複使用任何 vertex 並收回到起點，就是一個 Hamiltonian circuit（Fig. 6-8）。範例圖（Fig. 6-6，vertex 1–7）— 可以手動走訪的 adjacency：

edges: 1-2 1-6 1-3 2-3 6-5 7-5 7-4 3-4 4-5 vertex 1 → 2, 6, 3 vertex 2 → 1, 3 vertex 3 → 1, 2, 4 vertex 4 → 7, 3, 5 vertex 5 → 6, 7, 4 vertex 6 → 1, 5 vertex 7 → 5, 4

Root the search at vertex 1; extend partial paths, mark dead ends X, backtrack on failure (children left→right). This is Fig. 6-8, the full search tree — node labels are the vertices visited, and every doomed branch is marked X:把搜尋的 root 設在 vertex 1；延伸部分路徑，把死路標 X，失敗就 backtrack（子節點由左到右）。這就是 Fig. 6-8，完整的搜尋樹 — 節點標籤是已造訪的 vertex，每條注定失敗的分支都標 X：

Fig. 6-8 — Search tree for a Hamiltonian circuit of the Fig. 6-6 graph. Each path from the root (vertex 1) down chooses the next vertex; branches that reuse a vertex or hit a missing edge dead-end at X; the two branches that reach a leaf 1 close the cycle.Fig. 6-8 — Fig. 6-6 圖的 Hamiltonian circuit 搜尋樹。從 root（vertex 1）往下的每條路徑都選擇下一個 vertex；重複使用某個 vertex 或撞上不存在的 edge 的分支會在 X 走到死路；兩條抵達 leaf 1 的分支收成環。

Which leaves are Hamiltonian circuits?哪些 leaf 是 Hamiltonian circuit？

Two root-to-leaf paths visit all 7 vertices once and close back to 1 (the two leaf nodes labeled 1 in Fig. 6-8): 1-2-3-4-5-7-… down the left subtree closing at 6-1, and 1-6-5-7-4-3-2-1 down the middle subtree. They are the same cycle traversed in opposite directions. Every other branch ends in X — a reused vertex, a missing edge, or a dead end with vertices still unvisited. Because this is feasibility, finding one closing path answers "yes, a Hamiltonian circuit exists." No edge weights appear: Hamiltonicity is feasibility, not optimization.有兩條 root-to-leaf 路徑造訪全部 7 個 vertex 各一次並收回到 1（Fig. 6-8 中那兩個標籤為 1 的 leaf 節點）：沿左子樹的 1-2-3-4-5-7-… 在 6-1 收尾，以及沿中間子樹的 1-6-5-7-4-3-2-1。它們是同一個環，只是以相反方向走訪。其餘每條分支都以 X 結束 — 重複的 vertex、不存在的 edge，或還有 vertex 未造訪的死路。因為這是 feasibility，找到一條收尾路徑就回答了「是，存在一個 Hamiltonian circuit」。沒有出現任何 edge weight：Hamiltonicity 是 feasibility，不是 optimization。
