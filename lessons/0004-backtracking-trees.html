Lesson 4 · Ch6 Searching Strategies

Backtracking on Solution Trees

Exam goal: hand-trace the three classic Ch6 feasibility problems — SAT, sum-of-subsets, Hamiltonian circuit — as a walk over a solution tree. Lesson 1 gave you the abstract knob; here you turn it on real instances. The tangible win: draw the tree, prune dead subtrees, and read off the answer.

0 · What are these problems?

Before any tree-drawing, here are the four ideas this lesson rests on — assume you have never met them.

Backtracking. A way to search for a solution by building a candidate one piece at a time. Picture the choices as a tree: the root has chosen nothing, and each step down adds one more decision. You walk down a branch, and the moment a partial choice can no longer lead to a valid solution you abandon that branch — retreat back up and try the next option. That retreat is the "backtrack." It saves you from finishing candidates you already know are doomed.

The Satisfiability problem (SAT). You are given a Boolean formula — a logic expression whose variables are each either true or false — and asked: is there a way to set the variables that makes the whole formula true? Some vocabulary, defined once and used throughout:

TermPlain meaningExample
literala variable, or its negation (NOT)x₁ or ¬x₁
clausean OR of literals — true if any one literal in it is truex₂ ∨ x₅
CNF(conjunctive normal form) an AND of clauses — true only if every clause is true(x₂ ∨ x₅) ∧ x₃
satisfiablesome assignment of true/false makes the whole formula truesee below

Tiny example: the CNF formula (x₁ ∨ x₂) ∧ (¬x₁) is satisfiable — set x₁ = false, x₂ = true and both clauses hold. By contrast (x₁) ∧ (¬x₁) is unsatisfiable: no value of x₁ can make both clauses true at once.

The sum-of-subsets problem. Given a set of numbers and a target, is there a subset whose elements add up to exactly the target? Tiny example: from {3, 5, 8} with target 11, the subset {3, 8} works (3 + 8 = 11); with target 4, no subset hits it.

The Hamiltonian circuit problem. A Hamiltonian circuit is a cycle in a graph that visits every vertex exactly once and returns to where it started. The question: does a given graph contain one? Do not confuse it with an Euler circuit, which is a cycle that uses every edge exactly once (vertices may be revisited). The classic trap is swapping the two:

Hamiltonian = every vertex once. Euler = every edge once. In a square a-b-c-d-a, going round the rim a→b→c→d→a is both — but add a diagonal a-c and the rim is still Hamiltonian (all 4 corners once) yet no longer Euler (the diagonal edge is never used).

Why one technique fits all three: each asks "does a valid candidate exist?" (a feasibility question, not "find the cheapest"). Each candidate is built choice by choice, so each is a walk down a tree — exactly what backtracking searches, abandoning doomed branches early.

1 · The same tree, three disguises

Every problem below becomes a solution tree: the root decides nothing, each level fixes one more choice, each leaf is a full candidate. We walk it depth-first (deepest open node first → a stack, from Lesson 1) and backtrack — pop and retreat — the instant a partial choice can no longer succeed. That early abandon is pruning: skip a whole subtree without examining its leaves.

ProblemEach level decides…Prune when…
SATxᵢ = T or xᵢ = Fa clause is already falsified
Sum-of-subsetselement i in or outpartial sum exceeds target
Hamiltonianwhich vertex to visit nextvertex reused / no edge / dead end
Key exam idea: these are feasibility problems — we want a solution (or "none exists"), so DFS stops at the first goal leaf. Pruning is what makes the 2ⁿ-leaf tree tractable: you may never need to examine all 2ⁿ assignments.

2 · SAT: a depth-n binary tree

Given a Boolean formula in n variables, there are 2ⁿ truth assignments — a complete binary tree of depth n, branching xᵢ = T / xᵢ = F at level i. For n = 3, that is 2³ = 8 leaves.

(root) X1=T / \ X1=F ( ) ( ) X2=T/ \X2=F X2=T/ \X2=F ( ) ( ) ( ) ( ) X3=T/\X3=F … … … ← 8 leaves at depth 3

The slide instance (clauses that must all hold; ¬ = NOT, ∨ = OR):

(1) ¬x₁ (4) x₃ (2) x₁ (5) ¬x₂ (3) x₂ ∨ x₅

Walk the tree and prune the moment any clause is violated. At the very first level we already hit a wall:

Trace the root's two branches

Branch x₁ = T → clause (1) ¬x₁ is falsified → prune that whole subtree.
Branch x₁ = F → clause (2) x₁ is falsified → prune that subtree too.
Both children of the root die immediately because clauses (1) and (2) directly contradict. So this instance is unsatisfiable — proven by pruning at depth 1, without ever building the other 2³ − 2 nodes. That is the whole point: a partial assignment can already reveal a contradiction.

3 · Sum-of-subsets by DFS

S = {7, 5, 1, 2, 10}, target = 9. Binary tree: at level i decide whether element i is in or out. A stack guides the DFS (push children, pop to backtrack). Prune when the running sum exceeds 9. Node labels below are the running sum; x = pruned (over target).

0 X1=7 / 7 X2=5 / X2=1 \ X2=2 \ 12(x) 8 9 ← Goal (7+2=9) X3=2/ X3=10\ 10(x) 18(x)

Decide the answer subset yourself before opening it.

DFS trace and the answer

Push root 0. Take x₁ = 7 → sum 7. From 7, try x₂ = 5 → 12 > 9 → prune, pop. Try x₂ = 1 → 8; from 8, x₃ = 2 → 10 (x) and x₃ = 10 → 18 (x), both over → backtrack. Try x₂ = 2 → 7 + 2 = 9 = target → goal node.
Answer subset: {7, 2}. The stack's LIFO order is exactly what makes the search dive down the 7 branch fully, then retreat.

4 · Hamiltonian circuit

A Hamiltonian circuit visits every vertex exactly once and returns to the start. Tree: each level chooses the next vertex to visit; a root-to-leaf path of length n that reuses no vertex and closes back to the start is a Hamiltonian circuit (Fig. 6-8). The example graph (Fig. 6-6, vertices 1–7) — adjacency you can walk by hand:

edges: 1-2 1-6 1-3 2-3 6-5 7-5 7-4 3-4 4-5 vertex 1 → 2, 6, 3 vertex 2 → 1, 3 vertex 3 → 1, 2, 4 vertex 4 → 7, 3, 5 vertex 5 → 6, 7, 4 vertex 6 → 1, 5 vertex 7 → 5, 4

Root the search at vertex 1; extend partial paths, mark dead ends X, backtrack on failure (children left→right). This is Fig. 6-8, the full search tree — node labels are the vertices visited, and every doomed branch is marked X:

Fig. 6-8 — Search tree for a Hamiltonian circuit of the Fig. 6-6 graph. Each path from the root (vertex 1) down chooses the next vertex; branches that reuse a vertex or hit a missing edge dead-end at X; the two branches that reach a leaf 1 close the cycle.

Which leaves are Hamiltonian circuits?

Two root-to-leaf paths visit all 7 vertices once and close back to 1 (the two leaf nodes labeled 1 in Fig. 6-8): 1-2-3-4-5-7-… down the left subtree closing at 6-1, and 1-6-5-7-4-3-2-1 down the middle subtree. They are the same cycle traversed in opposite directions. Every other branch ends in X — a reused vertex, a missing edge, or a dead end with vertices still unvisited. Because this is feasibility, finding one closing path answers "yes, a Hamiltonian circuit exists." No edge weights appear: Hamiltonicity is feasibility, not optimization.
