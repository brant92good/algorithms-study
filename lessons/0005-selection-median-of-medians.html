Linear-Time Selection線性時間 selection

Exam goal: hand-trace the 3-way partition to find a k-th smallest element, and recite why groups of 5 give O(n). Sorting finds the median in O(n log n) — this lesson's single tangible win is finding it in O(n), by never sorting the whole set, only pruning a guaranteed fraction each round.考試目標：你要能手動推一遍 3-way partition 來找出第 k 小的元素，還要講得出為什麼每 5 個一組就能做到 O(n)。用排序找 median 要 O(n log n)；這一課唯一實在的收穫，就是改成 O(n) 找到它——關鍵是從頭到尾都不把整組排序，每一輪只砍掉保證比例的元素。

1 · What are we even solving?1 · 我們到底在解什麼？

The selection problem: you're given a set of n numbers (in no particular order) and a rank k, and you must return the k-th smallest element — the value that would sit in position k if you sorted the set, but found without fully sorting. The median is the middle one: the special case k ≈ n/2 (precisely the ⌈n/2⌉-th smallest).Selection problem 是這樣：給你一組 n 個數字（沒有排好），再給你一個 rank k，要你回傳第 k 小的那個元素。換句話說，就是「如果把整組排序，會排在第 k 位」的那個值，但你得不靠完整排序就把它挑出來。Median 就是正中間那個，也就是 k ≈ n/2 的特例（精確講是第 ⌈n/2⌉ 小）。

S = {7, 2, 9, 4, 5} (n = 5) sorted → 2 4 5 7 9 ← we never actually do this 1 2 3 4 5 ← rank k k = 1 → 2 (smallest) k = 3 → 5 (the median) k = 5 → 9 (largest)

The obvious recipe is: sort the whole set, then read off position k. Sorting costs O(n log n). But we only want one element, not the whole order — so fully sorting is wasteful. The single win of this lesson is finding the answer in O(n) (linear — work proportional to n, no log n factor), by never sorting the whole set.最直覺的做法就是：把整組排序，再讀第 k 個位置。排序要 O(n log n)。但我們只要一個元素，又不是要整組的順序——所以完整排序根本是浪費。這一課的收穫，就是改用 O(n)（linear——工作量跟 n 成正比，沒有 log n 這個因子）找到答案，靠的就是從頭到尾都不把整組排序。

The idea behind that win — prune-and-search. Think of how binary search finds a value in a sorted array: each step throws away half the array (the half that can't contain the target) and repeats on the rest. Prune-and-search generalizes this to unsorted data: each round, prove that some constant fraction of the elements cannot be the answer, throw them away ("prune"), and repeat on what survives. Because each round discards a fixed slice, the survivors shrink geometrically and the total work stays small.這個收穫背後的想法——prune-and-search。 你想想 binary search 是怎麼在排好的陣列裡找一個值的：每一步都砍掉一半陣列（不可能藏著目標的那一半），再對剩下那半重複。Prune-and-search 就是把這招推廣到沒排序的資料：每一輪都證明某個固定比例的元素不可能是答案，直接砍掉（也就是 "prune"），再對活下來的繼續做。因為每一輪都固定砍掉一段，活下來的會以幾何級數越縮越小，所以總工作量壓得很小。

To prune, we need to split the set. We pick one element as the pivot p (a reference value we compare everything against) and do a partition — one linear sweep that drops every element into one of three buckets: those smaller than p, those equal to p, and those larger than p. Knowing how many fall below p tells us immediately which bucket the k-th smallest lives in, so we can discard the other two. Sections 2–3 make this precise; sections 4–5 show how to choose p so the pruning is guaranteed.要 prune，就得先把集合切開。我們挑一個元素當 pivot p（也就是拿來跟每個元素比大小的參考值），然後做一次 partition——一趟線性掃描，把每個元素丟進三個桶裡的其中一個：小於 p、等於 p、大於 p。只要知道有幾個落在 p 底下，馬上就能判斷第 k 小在哪個桶，剩下兩個桶就直接丟掉。第 2–3 節把細節講清楚；第 4–5 節談怎麼選 p 才能保證砍得夠多。

2 · The prune-and-search method2 · Prune-and-search 方法

Binary search runs in many iterations; each one throws away a fixed fraction f of the data and recurses on the rest. Prune-and-search generalizes that: prune a fraction f, recurse on the remaining (1−f)·n elements, until the survivor is small enough to solve directly. If each iteration's work is O(nᶜ), then:Binary search 會跑很多輪，每輪砍掉固定比例 f 的資料，再對剩下的遞迴。Prune-and-search 把這個想法推廣：每輪 prune 掉比例 f，對剩下的 (1−f)·n 個元素遞迴，直到活下來的數量小到可以直接解掉。假設每一輪的工作量是 O(nᶜ)，那麼：

T(n) = T((1 − f)·n) + O(nᶜ) ⇒ T(n) = O(nᶜ)
Heads-up on letters. The exponent c here means "work per iteration is O(nᶜ)" (for us c = 1, linear work each round). This c is unrelated to the rank k (the k-th smallest) used everywhere else in the lesson — same alphabet, totally different things. And the prune fraction f is just "some fixed fraction" for now; for this problem it turns out to be f = 1/20 (you'll see why in Section 6).提醒一下符號。 這裡的指數 c 是指「每一輪的工作量是 O(nᶜ)」（我們的情況是 c = 1，每輪 linear）。這個 c 跟本課其他地方一直用的 rank k（第 k 小）完全沒關係——只是剛好都是字母，意思天差地遠。至於 prune 比例 f，現在先當成「某個固定比例」就好；這題最後會算出 f = 1/20（第 6 節會看到為什麼）。
Why the first iteration dominates: unrolling gives a·nᶜ·(1 + (1−f)ᶜ + (1−f)2ᶜ + …). Since 1−f < 1, that bracket is a convergent geometric series — a constant. So the total cost is just a constant times the first iteration's nᶜ. The whole process is the same order as one pass.為什麼第一輪就決定一切： 把它展開來看會是 a·nᶜ·(1 + (1−f)ᶜ + (1−f)2ᶜ + …)。因為 1−f < 1，括號裡其實是一個會收斂的 geometric series，加起來就是個常數。所以總成本不過是某個常數乘上第一輪的 nᶜ。也就是說，整個流程跟掃一趟是同一個 order。

3 · The 3-way partition3 · 3-way partition

Pick a pivot p and split S into three buckets (the partition), then recurse into only one. |S1| means "the number of elements in S1".挑一個 pivot p，把 S 切成三個桶（這就是 partition），然後只挑其中一個桶遞迴下去。|S1| 就是「S1 裡有幾個元素」的意思。

Bucket桶Holds裝什麼
S1everything < p所有 < p 的
S2everything = p所有 = p 的
S3everything > p所有 > p 的

To find the k-th smallest, compare k against the bucket sizes:要找第 k 小，就拿 k 去跟各桶的大小比一比：

if |S1| ≥ k → answer is in S1; recurse on S1 with same k else if |S1|+|S2| ≥ k → answer is p itself; DONE else → answer is the (k − |S1| − |S2|)-th smallest of S3; recurse on S3

5 · Median of medians — the pivot trick5 · Median of medians — 選 pivot 的訣竅

A bad pivot prunes almost nothing (worst case O(n2)). We need a pivot guaranteed "central enough" regardless of input:如果 pivot 挑得爛，幾乎砍不掉東西（最壞情況會掉到 O(n2)）。我們要的是一個不管輸入長怎樣都保證「夠居中」的 pivot：

1. Split S into ⌈n/5⌉ groups of 5. 2. Sort each group of 5; its median is the 3rd element. 3. p = the median of those ⌈n/5⌉ group-medians, found by calling THIS SAME selection algorithm recursively.

Step 3 looks circular — "to find a median I have to find a median?" It isn't: we find it by calling this same selection algorithm recursively on just the n/5 group-medians. That is the recursive step, and because n/5 is far smaller than n each call shrinks fast and the recursion terminates. (Section 6 accounts for its cost as T(n/5).)步驟 3 看起來很像在繞圈圈——「要找 median 結果又得先找 median？」其實不會：我們是把這個 selection 演算法拿來遞迴呼叫自己，只對那 n/5 個 group-median 做。這就是遞迴的那一步，而因為 n/5 比 n 小很多，每次呼叫規模都縮得很快，遞迴會收掉。（第 6 節會把它的成本算成 T(n/5)。）

The grid below is slide 7, drawn with a concrete example: 5 groups of 5 (n = 25). Each column is one group, already sorted top-to-bottom (smallest on top). The columns are arranged left-to-right by their median, so the group-medians (middle row) increase as you go right. P = 13 is the median of those 5 group-medians — the pivot.下面的格子就是第 7 張投影片，這次用具體數字畫：5 組、每組 5 個（n = 25）。每一欄是一組，已經由上到下排好（最小的在最上面）。各欄又依照自己的 median 由左到右排，所以中間那一列的 group-median 往右會越來越大。P = 13 就是這 5 個 group-median 的 median，也就是 pivot。

Reading the shaded block — the count. Look at the boxed upper-left rectangle: the 3 columns whose median is ≤ P (here G1, G2, G3 — their medians 5, 8, 13 are all ≤ 13), and within each the top 3 cells (rows 1–3). Every one of those 3 × 3 = 9 cells is provably ≤ P. Count them in the grid: {2,4,5, 6,7,8, 9,11,13} — yes, 9 values, all ≤ 13. Symmetrically the lower-right block gives 9 cells that are ≥ P. So 9 of 25 ≈ n/4 are ≤ P and another ≈ n/4 ≥ P — that is the guarantee the figure makes visible.怎麼讀那塊陰影——把它數出來。 看左上角被框起來的長方形：那是 median ≤ P 的 3 欄（這裡是 G1、G2、G3，它們的 median 5、8、13 都 ≤ 13），再取每欄最上面 3 格（第 1–3 列）。這 3 × 3 = 9 格每一個都可以證明 ≤ P。直接在圖上數：{2,4,5, 6,7,8, 9,11,13}——對，9 個值，全都 ≤ 13。對稱地，右下那塊也給你 9 格 ≥ P。所以25 個裡有 9 個 ≈ n/4 是 ≤ P，另外 ≈ n/4 個 ≥ P——這就是這張圖讓你親眼看到的保證。
The guarantee — the full chain, with numbers.
(1) P is the median of the group-medians. By definition of median, at least half of the n/5 group-medians are ≤ P — that's about n/10 groups. (In the example: 3 of the 5 medians, namely 5, 8, 13, are ≤ 13.)
(2) In each such group of 5, the group's median plus the two cells above it are ≤ that median (a sorted column of 5 has positions 1, 2, 3 all ≤ the 3rd element), hence ≤ P. That's 3 elements per group.
(3) Multiply: (n/10 groups) × (3 each) = 3n/10 elements that are ≤ P. And 3n/10 ≥ n/4 (since 3/10 = 0.30 ≥ 0.25), so n/4 is a safe lower bound — we state the algorithm's guarantee as the round number n/4. Symmetrically ≥ n/4 are ≥ P.
So at least n/4 are pruned every iteration and at most 3n/4 survive.這個保證——把整條推理鏈連數字一起講清楚。
(1) P 是 group-median 的 median。照 median 的定義，那 n/5 個 group-median 裡至少有一半 ≤ P，也就是大約 n/10 組。（在例子裡：5 個 median 中有 3 個，5、8、13，都 ≤ 13。）
(2) 在每一個這種組裡，這組的 median 加上它上面那兩格都 ≤ 這個 median（排好的 5 格一欄，第 1、2、3 位都 ≤ 第 3 個），所以也 ≤ P。這樣一組就有 3 個。
(3) 乘起來：(n/10 組) × (每組 3 個) = 3n/10 個元素 ≤ P。而 3n/10 ≥ n/4（因為 3/10 = 0.30 ≥ 0.25），所以 n/4 是個安全的下界——我們就用比較好記的 n/4 來陳述這個保證。對稱地，也有 ≥ n/4 個 ≥ P。
所以每一輪至少砍掉 n/4，最多只剩 3n/4 活下來。

6 · Why it's O(n)6 · 為什麼是 O(n)

The 5-step algorithm: (1) group by 5, (2) sort each group, (3) recursively find the median of the n/5 medians = the pivot p, (4) partition into S1,S2,S3 around p, (5) the 3-way recurse on at most 3n/4 survivors. Steps 1, 2, 4 are each O(n); step 3 costs T(n/5); step 5 costs T(3n/4). Hence:這個演算法有 5 步：(1) 每 5 個分一組，(2) 把每組各自排序，(3) 遞迴找出那 n/5 個 median 的 median，也就是 pivot p，(4) 用 p partition 成 S1,S2,S3，(5) 對最多 3n/4 個活下來的做 3-way 遞迴。其中步驟 1、2、4 各是 O(n)，步驟 3 花 T(n/5)，步驟 5 花 T(3n/4)。所以：

T(n) = T(3n/4) + T(n/5) + O(n)

This has two recursive calls, so the clean one-call template from Section 2 doesn't apply directly — yet. Here's the bridge. The two subproblems' sizes sum to 3/4 + 1/5 = 19/20 of n, which is < n. So the two calls together do no more work than a single call on 19n/20 elements: T(3n/4) + T(n/5) ≤ T(19n/20) (plus lower-order terms). That collapses it back to one recursive call —這裡有兩個遞迴呼叫，所以第 2 節那個漂亮的「單一呼叫」模板沒辦法直接套——至少現在還不行。橋接的關鍵在這：兩個子問題的大小加起來是 n 的 3/4 + 1/5 = 19/20，比 n 小。所以這兩個呼叫合起來，做的工作不會比對 19n/20 個元素的單一呼叫還多：T(3n/4) + T(n/5) ≤ T(19n/20)（外加一些低階項）。這樣一來就縮回成一個遞迴呼叫——

T(n) ≤ T(19n/20) + O(n)

— which is exactly the Section-2 template with prune fraction f = 1/20 (since 1 − f = 19/20) and per-round work O(n) (c = 1). The geometric series argument fires and gives O(n). That slack under 1 is exactly why this stays linear.——這正好就是第 2 節的模板，prune 比例 f = 1/20（因為 1 − f = 19/20），每輪工作 O(n)（c = 1）。geometric series 的論證直接套上去，結果就是 O(n)。重點就是這個比 1 小一點的餘裕，正是它能維持 linear 的原因。

The substitution proof (algebra)代換證明（推一下代數）

Assume T is bounded by a polynomial. Comparing coefficients of the two recursive terms against T(19n/20): the linear parts match (3/4 + 1/5 = 19/20) and the quadratic part of the sum is smaller (9/16 + 1/25 = 241/400 < 361/400). So T(3n/4) + T(n/5) ≤ a0 + T(19n/20), collapsing the recurrence to a single term:先假設 T 被某個多項式 bound 住。拿兩個遞迴項的係數去跟 T(19n/20) 比：linear 部分剛好相等（3/4 + 1/5 = 19/20），而加總後的 quadratic 部分還更小（9/16 + 1/25 = 241/400 < 361/400）。所以 T(3n/4) + T(n/5) ≤ a0 + T(19n/20)，等於把遞迴式縮成一項：

T(n) ≤ cn + T(19n/20) ≤ cn·(1 + 19/20 + (19/20)2 + …) = cn · 1/(1 − 19/20) = cn · 20 = 20cn ∴ T(n) ≤ 20cn + b = O(n)

The geometric series with ratio 19/20 sums to 20. The constant 20 is fixed, while the sort's log n grows without bound — that's the asymptotic win.公比 19/20 的 geometric series 加起來是 20。這個 20 是固定的常數，反觀排序的 log n 會一直往上長沒有上限——這就是 asymptotic 上贏的地方。
