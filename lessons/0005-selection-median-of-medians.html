Linear-Time Selection線性時間 selection

Exam goal: hand-trace the 3-way partition to find a k-th smallest element, and recite why groups of 5 give O(n). Sorting finds the median in O(n log n) — this lesson's single tangible win is finding it in O(n), by never sorting the whole set, only pruning a guaranteed fraction each round.考試目標：能手動推演 3-way partition 來找出第 k 小的元素，並能說出為什麼以 5 個一組能得到 O(n)。排序找 median 要 O(n log n)——本課唯一具體的收穫，就是改用 O(n) 找到它：永遠不把整個集合排序，每一輪只 prune 掉保證比例的元素。

1 · What are we even solving?1 · 我們到底在解什麼？

The selection problem: you're given a set of n numbers (in no particular order) and a rank k, and you must return the k-th smallest element — the value that would sit in position k if you sorted the set, but found without fully sorting. The median is the middle one: the special case k ≈ n/2 (precisely the ⌈n/2⌉-th smallest).Selection problem：給你一組 n 個數字（沒有特定順序）和一個 rank k，你要回傳第 k 小的元素——也就是若把整個集合排序後會落在第 k 個位置的那個值，但要不完整排序就找出它。Median 是正中間那個：特例 k ≈ n/2（精確說是第 ⌈n/2⌉ 小）。

S = {7, 2, 9, 4, 5} (n = 5) sorted → 2 4 5 7 9 ← we never actually do this 1 2 3 4 5 ← rank k k = 1 → 2 (smallest) k = 3 → 5 (the median) k = 5 → 9 (largest)

The obvious recipe is: sort the whole set, then read off position k. Sorting costs O(n log n). But we only want one element, not the whole order — so fully sorting is wasteful. The single win of this lesson is finding the answer in O(n) (linear — work proportional to n, no log n factor), by never sorting the whole set.最直覺的作法是：把整個集合排序，然後讀出第 k 個位置。排序要 O(n log n)。但我們只想要一個元素，不是整個排序結果——所以完整排序是浪費。本課唯一的收穫，是改用 O(n)（linear——工作量正比於 n，沒有 log n 因子）找到答案，靠的是永遠不把整個集合排序。

The idea behind that win — prune-and-search. Think of how binary search finds a value in a sorted array: each step throws away half the array (the half that can't contain the target) and repeats on the rest. Prune-and-search generalizes this to unsorted data: each round, prove that some constant fraction of the elements cannot be the answer, throw them away ("prune"), and repeat on what survives. Because each round discards a fixed slice, the survivors shrink geometrically and the total work stays small.這項收穫背後的想法——prune-and-search。 想想 binary search 如何在已排序陣列中找一個值：每一步丟掉一半陣列（不可能含目標的那一半），對剩下的重複。Prune-and-search 把這套推廣到未排序的資料：每一輪都證明某個固定比例的元素不可能是答案，把它們丟掉（"prune"），再對存活的重複。因為每一輪都丟掉固定的一段，存活者以幾何級數縮減，總工作量維持很小。

To prune, we need to split the set. We pick one element as the pivot p (a reference value we compare everything against) and do a partition — one linear sweep that drops every element into one of three buckets: those smaller than p, those equal to p, and those larger than p. Knowing how many fall below p tells us immediately which bucket the k-th smallest lives in, so we can discard the other two. Sections 2–3 make this precise; sections 4–5 show how to choose p so the pruning is guaranteed.要 prune，就得把集合切開。我們挑一個元素當 pivot p（拿來和所有元素比較的參考值），做一次 partition——一趟線性掃描，把每個元素丟進三個桶之一：小於 p、等於 p、大於 p。知道有多少落在 p 之下，就立刻知道第 k 小的元素在哪一個桶，於是可以丟掉另外兩個。第 2–3 節把這講精確；第 4–5 節說明怎麼選 p 才能保證 prune 的效果。

2 · The prune-and-search method2 · Prune-and-search 方法

Binary search runs in many iterations; each one throws away a fixed fraction f of the data and recurses on the rest. Prune-and-search generalizes that: prune a fraction f, recurse on the remaining (1−f)·n elements, until the survivor is small enough to solve directly. If each iteration's work is O(nᵏ), then:Binary search 跑很多次迭代；每次丟掉固定比例 f 的資料，對剩下的遞迴。Prune-and-search 把這推廣：prune 掉比例 f，對剩下的 (1−f)·n 個元素遞迴，直到存活者小到能直接解。若每次迭代的工作量是 O(nᵏ)，則：

T(n) = T((1 − f)·n) + O(nᵏ) ⇒ T(n) = O(nᵏ)
Why the first iteration dominates: unrolling gives c·nᵏ·(1 + (1−f)ᵏ + (1−f)²ᵏ + …). Since 1−f < 1, that bracket is a convergent geometric series — a constant. So the total cost is just a constant times the first iteration's nᵏ. The whole process is the same order as one pass.為什麼第一次迭代主導： 展開後得到 c·nᵏ·(1 + (1−f)ᵏ + (1−f)²ᵏ + …)。因為 1−f < 1，括號裡是收斂的 geometric series——一個常數。所以總成本就是某個常數乘上第一次迭代的 nᵏ。整個過程和單趟掃描是同一個 order。

3 · The 3-way partition3 · 3-way partition

Pick a pivot p and split S into three buckets (the partition), then recurse into only one. |S₁| means "the number of elements in S₁".挑一個 pivot p，把 S 切成三個桶（partition），然後只對其中一個遞迴。|S₁| 代表「S₁ 中的元素個數」。

Bucket桶Holds裝什麼
S₁everything < p所有 < p 的元素
S₂everything = p所有 = p 的元素
S₃everything > p所有 > p 的元素

To find the k-th smallest, compare k against the bucket sizes:要找第 k 小，把 k 和各桶大小比較：

if |S₁| ≥ k → answer is in S₁; recurse on S₁ with same k else if |S₁|+|S₂| ≥ k → answer is p itself; DONE else → answer is the (k − |S₁| − |S₂|)-th smallest of S₃; recurse on S₃

5 · Median of medians — the pivot trick5 · Median of medians — 選 pivot 的訣竅

A bad pivot prunes almost nothing (worst case O(n²)). We need a pivot guaranteed "central enough" regardless of input:糟糕的 pivot 幾乎 prune 不掉東西（最壞情況 O(n²)）。我們需要一個不論輸入為何都保證「夠居中」的 pivot：

1. Split S into ⌈n/5⌉ groups of 5. 2. Sort each group of 5; its median is the 3rd element. 3. p = the median of those ⌈n/5⌉ group-medians.

The grid below is slide 7. Each column is one sorted group of 5 (smallest on top). The middle row M is the group-medians; P is their median.下面的格子是第 7 張投影片。每一欄是一組排好序的 5 個元素（最小的在上）。中間那列 M 是各組的 median；P 是它們的 median。

sorted ↓ each column = one sorted 5-element group ┌───────────────────────────┐ · │ · · · · │ · · · · │ · · · · │ · · · M → · · [·]──P · · │ · · · ← median row M · · · · │ · · · · · · · · │ · · · · └────────────────────┘ upper-left block: ≥ ¼ of S is ≤ P lower-right block: ≥ ¼ of S is ≥ P
The guarantee: about half the groups have median ≤ P; in each such group the median plus the two above it (3 of 5) are ≤ P. That forces at least n/4 elements ≤ P, and symmetrically at least n/4 ≥ P. So at least n/4 are pruned every iteration and at most 3n/4 survive.這個保證： 大約一半的組其 median ≤ P；在每一個這樣的組裡，median 加上它上面那兩個（5 個中的 3 個）都 ≤ P。這就逼出至少 n/4 個元素 ≤ P，對稱地也至少 n/4 個 ≥ P。所以每一輪至少 prune 掉 n/4，最多 3n/4 存活。

6 · Why it's O(n)6 · 為什麼是 O(n)

The 5-step algorithm: (1) group by 5, (2) sort each group, (3) recursively find the median of the n/5 medians = the pivot p, (4) partition into S₁,S₂,S₃ around p, (5) the 3-way recurse on at most 3n/4 survivors. Steps 1, 2, 4 are each O(n); step 3 costs T(n/5); step 5 costs T(3n/4). Hence:這個 5 步驟演算法：(1) 每 5 個分一組，(2) 把每組排序，(3) 遞迴找出那 n/5 個 median 的 median = pivot p，(4) 以 p partition 成 S₁,S₂,S₃，(5) 對最多 3n/4 個存活者做 3-way 遞迴。步驟 1、2、4 各是 O(n)；步驟 3 花 T(n/5)；步驟 5 花 T(3n/4)。因此：

T(n) = T(3n/4) + T(n/5) + O(n)

The two fractions sum to 3/4 + 1/5 = 19/20 < 1 — that slack under 1 is exactly why this stays linear.這兩個分數加起來是 3/4 + 1/5 = 19/20 < 1——這個在 1 以下的餘裕，正是它能維持 linear 的原因。

The substitution proof (algebra)代換證明（代數）

Assume T is bounded by a polynomial. Comparing coefficients of the two recursive terms against T(19n/20): the linear parts match (3/4 + 1/5 = 19/20) and the quadratic part of the sum is smaller (9/16 + 1/25 = 241/400 < 361/400). So T(3n/4) + T(n/5) ≤ a₀ + T(19n/20), collapsing the recurrence to a single term:假設 T 被某個多項式所 bound。把兩個遞迴項的係數和 T(19n/20) 比較：linear 部分相等（3/4 + 1/5 = 19/20），而總和的 quadratic 部分較小（9/16 + 1/25 = 241/400 < 361/400）。所以 T(3n/4) + T(n/5) ≤ a₀ + T(19n/20)，把遞迴式收縮成單一項：

T(n) ≤ cn + T(19n/20) ≤ cn·(1 + 19/20 + (19/20)² + …) = cn · 1/(1 − 19/20) = cn · 20 = 20cn ∴ T(n) ≤ 20cn + b = O(n)

The geometric series with ratio 19/20 sums to 20. The constant 20 is fixed, while the sort's log n grows without bound — that's the asymptotic win.公比 19/20 的 geometric series 加總為 20。常數 20 是固定的，而排序的 log n 會無上限地成長——這就是漸進上的勝利。
