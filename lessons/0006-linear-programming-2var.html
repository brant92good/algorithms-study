Two-Variable Linear Programming in O(n)兩變數的 Linear Programming，O(n) 求解

Exam goal: hand-trace the boundary envelope and explain the slope test. Linear programming sounds heavy, but in 2 variables it collapses to one picture — the lowest point of a convex curve — and prune-and-search finds it in linear time by throwing away a quarter of the constraints each round.考試重點：會用手畫出 boundary envelope，然後講清楚 slope test。Linear programming 聽起來好像很硬，但其實只要剩兩個變數，整題就濃縮成一張圖 — 一條凸曲線的最低點 — 而 prune-and-search 每一輪砍掉四分之一的 constraint，所以能在線性時間內把它找出來。

1 · What is linear programming?1 · linear programming 到底在做什麼？

Linear programming (LP) means: you have some quantities you can choose (the variables, say x and y), and you want to make one number as small (or large) as possible — that number is the objective, and it's linear, meaning it looks like ax + by (just variables times constants, added up — no squares, no products). But you can't choose freely: you must obey a list of constraints, each also linear, each an inequality like aᵢx + bᵢy ≤ cᵢ (read "≤" as "at most"). The job: find the (x, y) that obeys every constraint and makes the objective smallest. "Programming" here is an old word for planning, nothing to do with code.簡單講，linear programming (LP) 就是：你手上有幾個可以自己決定的量（variable，比如 x 跟 y），你想讓某個數字越小越好（或越大越好）— 這個數字叫 objective，而且它是 linear 的，也就是長得像 ax + by（變數乘常數再加起來，沒有平方、也沒有兩個變數相乘）。但你不能亂選：你得乖乖遵守一串 constraint，每條也都是 linear，長得像 aᵢx + bᵢy ≤ cᵢ 這種不等式（「≤」就讀成「最多」）。你的任務就是：找一組 (x, y)，它要滿足每一條 constraint，同時讓 objective 最小。對了，這裡的「Programming」是古早「規劃」的意思，跟寫程式一點關係都沒有。

A tiny example in words: a bakery makes x loaves and y cakes; flour and oven-time are limited (those are the ≤ constraints); we want to minimize cost ax + by. LP is everywhere — diets, shipping, scheduling — wherever you optimize under linear limits.舉個小例子你就懂了：一間麵包店做 x 條麵包、y 個蛋糕；麵粉和烤箱時間都是有限的（這些就是那些 ≤ constraint）；我們想把成本 ax + by 壓到最低。LP 到處都會用到 — 飲食規劃、運輸、排程 — 只要是在 linear 的限制下找最佳解，都跑不掉它。

What the constraints carve out這些 constraint 圈出了什麼範圍

Picture the plane of all possible (x, y). One inequality aᵢx + bᵢy ≤ cᵢ draws a straight line and keeps only the side that satisfies it — a half-plane (half the plane, on one side of a line). Stack up n such half-planes and only the points obeying all of them survive. That surviving overlap is the feasible region: every point in it is a legal choice (feasible = obeys all constraints; optimal = the best feasible one). It's a convex polygon-ish area — possibly unbounded. The optimum sits somewhere on its edge.你先想像一整片平面，上面是所有可能的 (x, y)。一條不等式 aᵢx + bᵢy ≤ cᵢ 會畫出一條直線，然後只留下滿足它的那一邊 — 這半邊就叫 half-plane（半平面，直線一側的那半）。把 n 條這種 half-plane 全部疊在一起，只有 每一條 都符合的點才活得下來。這些活下來的點重疊出的區域，就是 feasible region：裡面每個點都是合法的選擇（feasible = 滿足所有 constraint；optimal = feasible 之中最好的那個）。這塊區域長得像凸多邊形 — 而且有可能是無界的。最佳解就藏在它邊界上的某個地方。

2 · The problem is one picture2 · 整題其實就是一張圖

General two-variable LP minimizes ax + by subject to n half-plane constraints aᵢx + bᵢy ≤ cᵢ. By a change of variables it reduces to the easy-to-see simplified form:一般的兩變數 LP，就是在 n 條 half-plane constraint aᵢx + bᵢy ≤ cᵢ 之下把 ax + by 最小化。只要做個變數變換，它就能化簡成一眼就看得懂的 simplified form（簡化形式）：

Minimize y subject to y ≥ aᵢx + bᵢ , i = 1 … n

Where does this come from? Rename the objective itself as the new vertical variable. Set x′ = x and y′ = ax + by — that is, let y′ be the cost we are minimizing. Now "minimize ax + by" is literally "minimize y′": the objective becomes the new height, which is why the simplified form has just one objective, y, and no x-objective.這個簡化形式哪來的？ 其實就是把 objective 本身改名成新的「垂直變數」。令 x′ = x、y′ = ax + by — 也就是讓 y′ 直接 等於 我們要最小化的那個 cost。這樣一來，「最小化 ax + by」就字面變成「最小化 y′」：objective 變成新的高度，所以簡化形式只剩一個 objective y，沒有 x 方向的 objective。

Why the ≤ flips to ≥. Solve the rename for the old variables: from y′ = ax + by we get y = (y′ − ax)/b (take b > 0 for the picture). Substitute into a constraint aᵢx + bᵢy ≤ cᵢ, clear denominators, and collect y′ on one side: it becomes y′ ≥ (some line in x′), i.e. y′ ≥ aᵢ′x′ + bᵢ′. Concrete check: objective x + y (so a=b=1, hence y′ = x+y, y = y′ − x) with constraint x − y ≤ 2. Substitute: x − (y′ − x) ≤ 2 ⇒ 2x − y′ ≤ 2 ⇒ y′ ≥ 2x − 2 — a clean "y′ ≥ line". The original "stay below a line" (≤) turns into "stay above a line" (≥) once height is measured by the cost y′, because lowering the cost y′ now means moving down, so every constraint becomes a floor you must stay on or above. That is exactly the simplified form, with x′,y′ renamed back to x,y.為什麼 ≤ 會翻成 ≥。 把這個改名反解回舊變數：由 y′ = ax + by 得 y = (y′ − ax)/b（為了畫圖方便假設 b > 0）。把它代進某條 constraint aᵢx + bᵢy ≤ cᵢ，通分、把 y′ 移到一邊，就會變成 y′ ≥（某條 x′ 的線），也就是 y′ ≥ aᵢ′x′ + bᵢ′。實際驗算： objective 取 x + y（所以 a=b=1，即 y′ = x+y、y = y′ − x），constraint 取 x − y ≤ 2。代入：x − (y′ − x) ≤ 2 ⇒ 2x − y′ ≤ 2 ⇒ y′ ≥ 2x − 2 — 漂亮的「y′ ≥ 一條線」。原本「待在線下面」（≤）一旦改用 cost y′ 當高度，就翻成「待在線上面」（≥），因為現在要讓 cost y′ 變小就是往 下 移，每條 constraint 都變成一道「不能再低於它」的地板。這正好就是 simplified form，再把 x′,y′ 改名回 x,y 就好。

Each constraint y ≥ aᵢx + bᵢ says the feasible point must lie on or above that line. So the feasible region is everything above the highest line at each x. We want the lowest feasible y — the bottom of that region.每條 constraint y ≥ aᵢx + bᵢ 都在說：feasible 的點得待在那條線 上面或線上。所以 feasible region 就是在每個 x 那裡、最高 那條線以上的所有地方。而我們想要的，是 最低 的 feasible y — 也就是這塊區域的最底部。

3 · The boundary function F(x)3 · boundary function F(x)

In the simplified form, each constraint y ≥ aᵢx + bᵢ forces our point to sit on or above that line. To be feasible at a given x, the point must be above every line at once — so it must be above the highest one there. Define the boundary function F(x) as exactly that: at each x, the height of whichever line is on top.在 simplified form 裡，每條 constraint y ≥ aᵢx + bᵢ 都逼著我們的點待在那條線 上面或線上。想在某個 x feasible，這個點得同時壓在 每一條 線上面 — 換句話說，它得在那裡 最高 的那條線上面。boundary function F(x) 講的就是這件事：在每個 x，取最上面那條線的高度。

F(x) = max { aᵢx + bᵢ : i = 1 … n }

Tracing the top line across all x draws the upper envelope — the jagged ceiling formed by always taking the highest line. It is the lower edge of the feasible region: the lowest you're allowed to be at each x. Minimizing y means finding the lowest point on this ceiling.把「最上面那條線」沿著所有 x 一路描過去，就畫出 upper envelope（上包絡線） — 一條因為「永遠挑最高線」而形成的鋸齒狀天花板。它就是 feasible region 的下緣：在每個 x，你最低只能待到這條線。所以把 y 最小化，其實就是在 這片天花板上 找最低點。

Here is the key fact. Because F(x) is the pointwise maximum of straight lines, it is convex — it bends only upward, like the inside of a bowl. (Why: each line is straight; taking the top one at every x can only ever kink upward where a steeper line overtakes, never downward.) So F(x) is a single valley: it falls, hits one bottom, then rises — a V or bowl shape, never a W with two dips. The optimum is that bottom:重點來了。因為 F(x) 是一堆直線逐點取 最大值，所以它一定是 convex（凸） 的 — 它只會往上彎，就像碗的內側。（為什麼？每條線本身都是直的；在每個 x 挑最高那條，只會在某條更陡的線追上來的地方 往上 折一下，絕對不會往下折。）所以 F(x) 就是一個單一的山谷：先往下、碰到唯一的谷底、再往上 — 一個 V 或碗的形狀，絕不會是有兩個凹陷的 W。而最佳解就是那個谷底：

F(x₀) = min over all x of F(x) (Fig. 7-2)

Fig. 7-2. The lines are the constraints; F(x) is the highest one at each x (the jagged ceiling). The shaded feasible region sits above it; the convex valley bottoms out at the optimum (x₀, y₀) — the lowest point of the upper envelope.Fig. 7-2. 圖裡那些線都是 constraint；F(x) 就是每個 x 最高的那條（那條鋸齒狀天花板）。陰影的 feasible region 在它上方；這個 convex 山谷在最佳解 (x₀, y₀) 觸底 — 也就是 upper envelope 的最低點。

and (x₀, y₀) with y₀ = F(x₀) is the answer. Convexity is the whole engine: a valley has one bottom, so a single left/right test at any probe point tells you which way the bottom lies — exactly what makes the fast pruning below possible.所以 (x₀, y₀)（其中 y₀ = F(x₀)）就是答案。convexity 是整套方法的核心：山谷只有 一個 谷底，所以隨便挑一個探測點做一次「往左還是往右」的測試，就能知道谷底在哪一邊 — 這正是下面那個快速 pruning 行得通的原因。

Key exam line: F(x) = the upper envelope of the constraint lines (highest line at each x), and it is convex. The optimum is not the highest line — it's the lowest point of the highest-line-at-each-x curve.考試一定要會這句： F(x) = 那些 constraint 線的 upper envelope（每個 x 取最高的線），而且它是 convex 的。注意最佳解 不是 最高的那條線 — 而是「每個 x 取最高線」這條曲線本身的最低點。

Tiny example you can see — two lines, y ≥ x and y ≥ −x. The envelope is the higher of the two at each x: the falling line on the left, the rising line on the right, meeting in a V at the origin. Lowest point = (0, 0).舉個看得見的小例子 — 只有兩條線，y ≥ x 和 y ≥ −x。envelope 就是每個 x 取兩條中較高的那條：左半邊走下降線，右半邊走上升線，兩條在原點碰頭、形成一個 V。最低點就是 (0, 0)。

4 · The slope test, then prune4 · 先做 slope test，再 prune

Pick a probe abscissa xₘ and evaluate yₘ = F(xₘ) — find the line(s) active (on top) there. Read off the slope g of the active constraint:隨便挑一個探測點 xₘ，算出 yₘ = F(xₘ) — 也就是找出在那裡 active（被壓在最上面）的線。接著讀出這條 active constraint 的 slope g：

Active slope at xₘxₘ 的 active slopeEnvelope is…envelope 現在…So the bottom x₀ is…所以谷底 x₀ 在…
g > 0 (rising)（在上升）still descending to the left往左還在往下走left左邊: x₀ < xₘ
g < 0 (falling)（在下降）still descending to the right往右還在往下走right右邊: x₀ > xₘ
both signs present正負號同時出現at its bottom right here谷底就在這裡了xₘ itselfxₘ 自己 is x₀就是 x₀ ✓

Why the slope sign points to the bottom. Remember F is a convex valley — a V: it falls on the left wall, hits one bottom, rises on the right wall. The line under your probe is the wall you are standing on, and its slope g is the wall's tilt. If that wall slopes up to the right (g > 0) you are on the right wall, so the bottom must be to your left. If it slopes down to the right (g < 0) you are on the left wall, so the bottom is to your right. (That is what "still descending to the left/right" in the table means — keep walking downhill and you reach the bottom.)為什麼 slope 的正負就能指出谷底。 別忘了 F 是個 convex 山谷 — 一個 V：左半邊往下、碰到唯一谷底、右半邊往上。你探測點底下那條線，就是你現在踩著的那面牆，它的 slope g 就是這面牆的傾斜。如果這面牆 往右上斜（g > 0），代表你站在 右牆，那谷底一定在你 左邊；如果它 往右下斜（g < 0），你站在 左牆，谷底就在你 右邊。（這就是表格裡「往左／往右還在往下走」的意思 — 順著下坡一直走就會到谷底。）

When several lines meet at xₘ, use g_max/g_min = the max/min slope among the active lines: both positive ⇒ x₀ left; both negative ⇒ x₀ right; opposite signs (g_min < 0 < g_max) ⇒ xₘ is the optimum (Fig. 7-6).如果剛好有好幾條線都在 xₘ 交會，就改看 g_max 和 g_min = active 線裡最大跟最小的 slope：兩個都正 ⇒ x₀ 在左邊；兩個都負 ⇒ x₀ 在右邊；一正一負（g_min < 0 < g_max）⇒ xₘ 本身就是最佳解 (Fig. 7-6)。

How a pair gets pruned. Group the n constraints into n/2 pairs; intersect each pair, and let xₘ be the median of the n/2 intersection x-values. Suppose the test says x₀ < xₘ. Take any pair whose two lines cross at some x > xₘ. To the left of that crossing one line of the pair is strictly below the other — and since we only care about x ≤ xₘ < crossing, that lower line can never be on the upper envelope near x₀, so it is redundant and we delete it. Why exactly n/4? We took xₘ to be the median of the n/2 crossing x-values, and "median" means by definition half the values sit on each side — so exactly (n/2)/2 = n/4 of the pairs cross to the right of xₘ (the far side, since the test said x₀ < xₘ). Each such pair lets us delete its one lower line, that is one constraint per qualifying pair, giving (n/2) × (1/2) = n/4 constraints deleted every iteration.一對 constraint 是怎麼被砍掉的。 把 n 條 constraint 兩兩湊成 n/2 對；每一對算出交點，再讓 xₘ 等於這 n/2 個交點 x 值的 median。假設測試告訴你 x₀ < xₘ。現在隨便抓一對、它的兩條線交在某個 x > xₘ。在那個交點的左邊，這一對裡會有一條一直壓在另一條下面 — 而我們只在乎 x ≤ xₘ（也就是交點左邊）這一段，所以那條 比較低的線在 x₀ 附近根本不可能爬上 upper envelope，它就是多餘的，直接刪掉。為什麼剛好是 n/4？ 我們把 xₘ 取成那 n/2 個交點 x 值的 median，而 median 的定義就是兩側各一半 — 所以剛好 (n/2)/2 = n/4 對的交點落在 xₘ 的 右 側（也就是「另一側」，因為測試說 x₀ < xₘ）。每一對這樣的 pair 都能刪掉它比較低的那條線，也就是一對刪 一 條，合起來 (n/2) × (1/2) = 每一輪砍掉 n/4 條 constraint。

Why O(n): each iteration does O(n) work (pairing, one median, one F-evaluation) and discards a constant fraction (≈1/4), so T(n) = T(3n/4) + O(n) = O(n) — the same prune-and-search payoff that makes median-of-medians selection (Lesson 5) linear.為什麼是 O(n)： 每一輪只花 O(n)（配對、算一次 median、做一次 F 求值），又丟掉固定比例（大約 1/4）的 constraint，所以 T(n) = T(3n/4) + O(n) = O(n) — 這跟讓 median-of-medians selection（第 5 課）變線性的，是完全同一招 prune-and-search 的好處。

5 · Trace it yourself5 · 自己動手描一遍

Three constraints in simplified form. Find the upper envelope's lowest point.這裡有 simplified form 的三條 constraint。你來找出 upper envelope 的最低點。

L₁: y ≥ x + 1 (slope +1) L₂: y ≥ −x + 5 (slope −1) L₃: y ≥ 2 (slope 0, a floor)

Work out F(x) = max(L₁,L₂,L₃), find where it bottoms out, then reveal.先自己算 F(x) = max(L₁,L₂,L₃)，找出它在哪觸底，再展開看答案。

Where do the lines cross, and what is F(x)?這些線交在哪裡？F(x) 長怎樣？

L₁ = L₂: x+1 = −x+5 ⇒ x = 2, y = 3. At x=2 both L₁ and L₂ give 3, and the floor L₃ gives 2 — so the floor is below them and inactive. For x < 2 the falling line L₂ is on top; for x > 2 the rising line L₁ is on top. So the envelope is a clean V with its vertex at (2, 3).先讓 L₁ = L₂：x+1 = −x+5 ⇒ x = 2, y = 3。在 x=2，L₁ 跟 L₂ 都是 3，而 floor L₃ 只有 2 — 所以 floor 被壓在下面，是 inactive 的。x < 2 時，下降線 L₂ 在最上面；x > 2 時換上升線 L₁ 在最上面。所以整條 envelope 就是一個乾乾淨淨的 V，頂點在 (2, 3)。

Apply the slope test at the vertex x₀ = 2在頂點 x₀ = 2 跑一次 slope test

At xₘ = 2 the active lines are L₂ (slope −1) and L₁ (slope +1). g_min = −1 < 0 and g_max = +1 > 0 — opposite signs — so by the rule this point is the optimum. Answer: x₀ = 2, y₀ = F(2) = 3. (The floor L₃ = 2 never touches the envelope here, so it's a redundant constraint — exactly the kind prune-and-search would discard.)在 xₘ = 2，active 的線是 L₂（slope −1）跟 L₁（slope +1）。g_min = −1 < 0、g_max = +1 > 0 — 一正一負 — 所以照規則，這個點 就是 最佳解。答案：x₀ = 2, y₀ = F(2) = 3。（floor L₃ = 2 在這裡從頭到尾都沒碰到 envelope，所以它是多餘的 constraint — 正是 prune-and-search 會直接砍掉的那種。）

Sanity-check with a probe to the left, say xₘ = 0換左邊的探測點驗算一下，比如 xₘ = 0

F(0) = max(0+1, 5, 2) = 5, achieved by L₂ alone, slope −1 < 0 ⇒ envelope still descending to the right ⇒ x₀ > 0. Correct: the true bottom is at x₀ = 2, to the right. The single slope sign pointed the right way.F(0) = max(0+1, 5, 2) = 5，是 L₂ 一條線單獨撐起來的，slope −1 < 0 ⇒ envelope 往右還在下降 ⇒ x₀ > 0。果然沒錯：真正的谷底在 x₀ = 2，就在右邊。光看一個 slope 的正負號就指對方向了。
