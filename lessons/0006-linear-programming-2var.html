Two-Variable Linear Programming in O(n)兩變數的 Linear Programming，O(n) 求解

Exam goal: hand-trace the boundary envelope and explain the slope test. Linear programming sounds heavy, but in 2 variables it collapses to one picture — the lowest point of a convex curve — and prune-and-search finds it in linear time by throwing away a quarter of the constraints each round.考試目標：手動描出 boundary envelope 並解釋 slope test。Linear programming 聽起來很重，但在 2 個變數下它會收斂成一張圖 — 一條凸曲線的最低點 — prune-and-search 每一輪丟掉四分之一的 constraint，因此能在線性時間內找到它。

1 · What is linear programming?1 · 什麼是 linear programming？

Linear programming (LP) means: you have some quantities you can choose (the variables, say x and y), and you want to make one number as small (or large) as possible — that number is the objective, and it's linear, meaning it looks like ax + by (just variables times constants, added up — no squares, no products). But you can't choose freely: you must obey a list of constraints, each also linear, each an inequality like aᵢx + bᵢy ≤ cᵢ (read "≤" as "at most"). The job: find the (x, y) that obeys every constraint and makes the objective smallest. "Programming" here is an old word for planning, nothing to do with code.Linear programming (LP) 的意思是：你有一些可以自己選的量（variable，例如 x 和 y），你想讓某個數字盡量小（或盡量大）— 那個數字就是 objective，而且它是 linear 的，意思是它長得像 ax + by（只是變數乘上常數再相加 — 沒有平方、沒有乘積）。但你不能隨便選：你必須遵守一串 constraint，每一個也都是 linear，每一個都是像 aᵢx + bᵢy ≤ cᵢ 這樣的不等式（把「≤」讀成「至多」）。任務是：找出遵守每一個 constraint 又讓 objective 最小的 (x, y)。這裡的「Programming」是「規劃」的舊用法，跟寫程式無關。

A tiny example in words: a bakery makes x loaves and y cakes; flour and oven-time are limited (those are the ≤ constraints); we want to minimize cost ax + by. LP is everywhere — diets, shipping, scheduling — wherever you optimize under linear limits.用一個小例子來說：一間麵包店做 x 條麵包和 y 個蛋糕；麵粉和烤箱時間有限（那些就是 ≤ constraint）；我們想最小化成本 ax + by。LP 無處不在 — 飲食規劃、運輸、排程 — 只要是在 linear 限制下做最佳化的場合都用得到。

What the constraints carve out這些 constraint 切出了什麼

Picture the plane of all possible (x, y). One inequality aᵢx + bᵢy ≤ cᵢ draws a straight line and keeps only the side that satisfies it — a half-plane (half the plane, on one side of a line). Stack up n such half-planes and only the points obeying all of them survive. That surviving overlap is the feasible region: every point in it is a legal choice (feasible = obeys all constraints; optimal = the best feasible one). It's a convex polygon-ish area — possibly unbounded. The optimum sits somewhere on its edge.想像所有可能的 (x, y) 構成的平面。一個不等式 aᵢx + bᵢy ≤ cᵢ 畫出一條直線，只保留滿足它的那一側 — 一個 half-plane（半平面，直線一側的那半邊）。把 n 個這樣的 half-plane 疊起來，只有遵守 全部 的點才能存活。存活下來的重疊區域就是 feasible region：裡面每個點都是合法的選擇（feasible = 遵守所有 constraint；optimal = feasible 中最好的那個）。它是一塊類似凸多邊形的區域 — 也可能是無界的。最佳解就落在它邊界上的某處。

2 · The problem is one picture2 · 整個問題就是一張圖

General two-variable LP minimizes ax + by subject to n half-plane constraints aᵢx + bᵢy ≤ cᵢ. By a change of variables it reduces to the easy-to-see simplified form:一般的兩變數 LP 是在 n 個 half-plane constraint aᵢx + bᵢy ≤ cᵢ 之下最小化 ax + by。透過變數變換，它可以化簡成容易看懂的 simplified form（簡化形式）：

Minimize y subject to y ≥ aᵢx + bᵢ , i = 1 … n

Each constraint y ≥ aᵢx + bᵢ says the feasible point must lie on or above that line. So the feasible region is everything above the highest line at each x. We want the lowest feasible y — the bottom of that region.每個 constraint y ≥ aᵢx + bᵢ 都表示 feasible 的點必須落在那條線 之上或線上。所以 feasible region 就是在每個 x 處、最高 那條線以上的全部區域。我們要的是 最低 的 feasible y — 也就是那塊區域的底部。

3 · The boundary function F(x)3 · boundary function F(x)

In the simplified form, each constraint y ≥ aᵢx + bᵢ forces our point to sit on or above that line. To be feasible at a given x, the point must be above every line at once — so it must be above the highest one there. Define the boundary function F(x) as exactly that: at each x, the height of whichever line is on top.在 simplified form 中，每個 constraint y ≥ aᵢx + bᵢ 都迫使我們的點落在那條線 之上或線上。要在某個 x 處 feasible，這個點必須同時在 每一條 線之上 — 所以它必須在那裡 最高 的那條線之上。boundary function F(x) 的定義正是如此：在每個 x 處，取最上方那條線的高度。

F(x) = max { aᵢx + bᵢ : i = 1 … n }

Tracing the top line across all x draws the upper envelope — the jagged ceiling formed by always taking the highest line. It is the lower edge of the feasible region: the lowest you're allowed to be at each x. Minimizing y means finding the lowest point on this ceiling.把最上方那條線沿著所有 x 描過去，就畫出 upper envelope（上包絡線） — 那條由「永遠取最高線」形成的鋸齒狀天花板。它是 feasible region 的下緣：在每個 x 處你被允許達到的最低位置。最小化 y 就是在 這片天花板上 找最低點。

Here is the key fact. Because F(x) is the pointwise maximum of straight lines, it is convex — it bends only upward, like the inside of a bowl. (Why: each line is straight; taking the top one at every x can only ever kink upward where a steeper line overtakes, never downward.) So F(x) is a single valley: it falls, hits one bottom, then rises — a V or bowl shape, never a W with two dips. The optimum is that bottom:這裡是關鍵事實。因為 F(x) 是一堆直線的逐點 最大值，它是 convex（凸） 的 — 它只會往上彎，像碗的內側。（為什麼：每條線都是直的；在每個 x 取最上面那條，只會在較陡的線超過時 往上 出現轉折，絕不會往下。）所以 F(x) 是單一的山谷：先下降、碰到唯一的谷底、再上升 — 一個 V 或碗的形狀，絕不會是有兩個凹谷的 W。最佳解就是那個谷底：

F(x₀) = min over all x of F(x) (Fig. 7-2)

Fig. 7-2. The lines are the constraints; F(x) is the highest one at each x (the jagged ceiling). The shaded feasible region sits above it; the convex valley bottoms out at the optimum (x₀, y₀) — the lowest point of the upper envelope.Fig. 7-2. 那些線就是 constraint；F(x) 是每個 x 處最高的那條（鋸齒狀天花板）。陰影的 feasible region 落在它上方；這個 convex 山谷在最佳解 (x₀, y₀) 處觸底 — 也就是 upper envelope 的最低點。

and (x₀, y₀) with y₀ = F(x₀) is the answer. Convexity is the whole engine: a valley has one bottom, so a single left/right test at any probe point tells you which way the bottom lies — exactly what makes the fast pruning below possible.於是 (x₀, y₀)（其中 y₀ = F(x₀)）就是答案。convexity 是整個方法的引擎：山谷只有 一個 谷底，所以在任何探測點做一次「往左/往右」測試，就能判斷谷底在哪個方向 — 這正是下面那個快速 pruning 能成立的原因。

Key exam line: F(x) = the upper envelope of the constraint lines (highest line at each x), and it is convex. The optimum is not the highest line — it's the lowest point of the highest-line-at-each-x curve.考試關鍵句： F(x) = constraint 線的 upper envelope（每個 x 處最高的線），而且它是 convex 的。最佳解 不是 最高那條線 — 而是「每個 x 取最高線」這條曲線的最低點。

Tiny example you can see — two lines, y ≥ x and y ≥ −x. The envelope is the higher of the two at each x: the falling line on the left, the rising line on the right, meeting in a V at the origin. Lowest point = (0, 0).一個看得見的小例子 — 兩條線，y ≥ x 和 y ≥ −x。envelope 是每個 x 處兩者中較高的那條：左半邊是下降線，右半邊是上升線，在原點交會成一個 V。最低點 = (0, 0)。

y ↑ \ / F(x) = max(−x, x) \ \ / left half: y = −x (falling) \ \ / right half: y = x (rising) \ \ / bottom of the V → optimum \ \ / \ \ / ──────────●──────────→ x (0,0) ← lowest point of the envelope

4 · The slope test, then prune4 · slope test，然後 prune

Pick a probe abscissa xₘ and evaluate yₘ = F(xₘ) — find the line(s) active (on top) there. Read off the slope g of the active constraint:選一個探測橫座標 xₘ，算出 yₘ = F(xₘ) — 找出在那裡 active（在最上方）的線。讀出該 active constraint 的 slope g：

Active slope at xₘxₘ 處的 active slopeEnvelope is…envelope 的狀態…So the bottom x₀ is…所以谷底 x₀ 在…
g > 0 (rising)（上升）still descending to the left往左仍在下降left左邊: x₀ < xₘ
g < 0 (falling)（下降）still descending to the right往右仍在下降right右邊: x₀ > xₘ
both signs present正負號都出現at its bottom right here谷底就在這裡xₘ itselfxₘ 本身 is x₀就是 x₀ ✓

When several lines meet at xₘ, use g_max/g_min = the max/min slope among the active lines: both positive ⇒ x₀ left; both negative ⇒ x₀ right; opposite signs (g_min < 0 < g_max) ⇒ xₘ is the optimum (Fig. 7-6).當有好幾條線在 xₘ 交會時，用 g_max/g_min = active 線中的最大/最小 slope：兩者皆正 ⇒ x₀ 在左；兩者皆負 ⇒ x₀ 在右；正負相反（g_min < 0 < g_max）⇒ xₘ 就是最佳解 (Fig. 7-6)。

How a pair gets pruned. Group the n constraints into n/2 pairs; intersect each pair, and let xₘ be the median of the n/2 intersection x-values. Suppose the test says x₀ < xₘ. Take any pair whose two lines cross at some x > xₘ. To the left of that crossing one line of the pair is strictly below the other — and since we only care about x ≤ xₘ < crossing, that lower line can never be on the upper envelope near x₀, so it is redundant and we delete it. Half the pairs (those crossing on the far side of xₘ) qualify, so n/4 constraints die every iteration.一對 constraint 如何被 prune 掉。 把 n 個 constraint 分成 n/2 對；對每一對取交點，令 xₘ 為這 n/2 個交點 x 值的 median。假設測試說 x₀ < xₘ。取任何一對、其兩條線在某個 x > xₘ 處相交。在那個交點左側，這一對中有一條嚴格低於另一條 — 而既然我們只關心 x ≤ xₘ < 交點，那條 較低的線在 x₀ 附近絕不可能出現在 upper envelope 上，所以它是多餘的，可以刪掉。有一半的對（那些在 xₘ 遠側相交的）符合條件，所以 每一輪都有 n/4 個 constraint 被淘汰。

Why O(n): each iteration does O(n) work (pairing, one median, one F-evaluation) and discards a constant fraction (≈1/4), so T(n) = T(3n/4) + O(n) = O(n) — the same prune-and-search payoff that makes median-of-medians selection (Lesson 5) linear.為何是 O(n)： 每一輪做 O(n) 的工作（配對、一次 median、一次 F 求值），並丟掉一個固定比例（約 1/4），所以 T(n) = T(3n/4) + O(n) = O(n) — 這正是讓 median-of-medians selection（第 5 課）變成線性的同一個 prune-and-search 效益。

Three constraints in simplified form. Find the upper envelope's lowest point.simplified form 下的三個 constraint。找出 upper envelope 的最低點。

L₁: y ≥ x + 1 (slope +1) L₂: y ≥ −x + 5 (slope −1) L₃: y ≥ 2 (slope 0, a floor)

Work out F(x) = max(L₁,L₂,L₃), find where it bottoms out, then reveal.算出 F(x) = max(L₁,L₂,L₃)，找出它在哪裡觸底，再展開答案。

Where do the lines cross, and what is F(x)?這些線在哪裡相交，F(x) 又是什麼？

L₁ = L₂: x+1 = −x+5 ⇒ x = 2, y = 3. At x=2 both L₁ and L₂ give 3, and the floor L₃ gives 2 — so the floor is below them and inactive. For x < 2 the falling line L₂ is on top; for x > 2 the rising line L₁ is on top. So the envelope is a clean V with its vertex at (2, 3).L₁ = L₂：x+1 = −x+5 ⇒ x = 2, y = 3。在 x=2 處 L₁ 和 L₂ 都給出 3，而 floor L₃ 給出 2 — 所以 floor 在它們下方、是 inactive 的。對 x < 2，下降線 L₂ 在最上方；對 x > 2，上升線 L₁ 在最上方。所以 envelope 是一個漂亮的 V，頂點在 (2, 3)。

Apply the slope test at the vertex x₀ = 2在頂點 x₀ = 2 套用 slope test

At xₘ = 2 the active lines are L₂ (slope −1) and L₁ (slope +1). g_min = −1 < 0 and g_max = +1 > 0 — opposite signs — so by the rule this point is the optimum. Answer: x₀ = 2, y₀ = F(2) = 3. (The floor L₃ = 2 never touches the envelope here, so it's a redundant constraint — exactly the kind prune-and-search would discard.)在 xₘ = 2 處，active 的線是 L₂（slope −1）和 L₁（slope +1）。g_min = −1 < 0 且 g_max = +1 > 0 — 正負相反 — 所以依照規則，這個點 就是 最佳解。答案：x₀ = 2, y₀ = F(2) = 3。（floor L₃ = 2 在這裡從未碰到 envelope，所以它是多餘的 constraint — 正是 prune-and-search 會丟掉的那種。）

F(0) = max(0+1, 5, 2) = 5, achieved by L₂ alone, slope −1 < 0 ⇒ envelope still descending to the right ⇒ x₀ > 0. Correct: the true bottom is at x₀ = 2, to the right.
