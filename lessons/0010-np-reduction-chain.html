Lesson 10 · Ch8 NP-Completeness II

The Reduction Chain

Exam goal: prove a problem is NP-complete and reproduce a reduction by hand. This is the highest-stakes proof/QA topic. The single win: lock in the 2-step recipe, memorize the reduction map, and be able to redraw the two cleanest constructions — SAT → 3-SAT and sum of subsets → partition — from scratch.

1 · What is all this? (start here)

This whole topic is about which problems are "hard" in a precise sense. A few plain-words definitions first — everything later is built on these, so don't skip them.

Decision problem: a problem whose answer is just Yes or No (e.g. "is there a tour shorter than 100?"), as opposed to an optimization problem that asks for the best value. The whole theory is about decision problems.

NP (informally): the class of decision problems where, if someone hands you a candidate answer, you can check it is correct in polynomial time (fast). You may not be able to find the answer quickly, but you can verify a proposed one. (Formally: a "guess a candidate, then check it quickly" algorithm.)

NP-complete (NPC): the hardest problems inside NP. Two conditions: (a) the problem is in NP (answers are checkable fast), and (b) it is NP-hard — every NP problem can be transformed into it. So if you ever found a fast algorithm for one NP-complete problem, you'd get a fast algorithm for all of NP. Nobody has, which is why these are considered intractable.

What is a reduction? A reduction A → B is a recipe that turns any instance of problem A into an instance of problem B in polynomial time, so that the Yes/No answer is preserved (the A-instance is a "Yes" exactly when the built B-instance is a "Yes"). Intuition: if you can solve B, you can solve A by translating and asking B — so B is at least as hard as A. Read the arrow as "A reduces to B" = "B is at least as hard as A."

Why "known NP-complete B reduces to A" makes A NP-complete. Suppose B is already known to be NP-complete (the hardest kind), and you exhibit a reduction B → A. That says A is at least as hard as B — and since B is at least as hard as everything in NP, so is A. That makes A NP-hard. If you also show A ∈ NP, then A is NP-hard and in NP = NP-complete. The two-step recipe below is exactly this.

2 · How to prove A is NP-complete

Two steps, always (slide 35):

(I) show A ∈ NP — give a nondeterministic poly-time algorithm (guess + poly-time check) (II) show B → A — pick a KNOWN NPC problem B and reduce it TO A (B reduces to A)

Recall the reduction arrow: B → A means "B can be solved using a poly-time algorithm for A as a subroutine," i.e. A is at least as hard as B.

Why this suffices: (II) shows A is at least as hard as an already-hardest problem B, so A is NP-hard. (I) places A inside NP. NP-hard + in NP = NP-complete.

Direction trap (memorize): you reduce the known hard problem B → A (known → new). Reducing the other way (A → B) would only prove A is easy, not hard. Cook's theorem seeds the whole chain: every NP problem → SAT, so SAT is the first NPC problem.

4 · Meet SAT and 3-SAT

The first reduction is between two cousins, so define them before using them.

Literal: a variable or its negation — x₁ or ¬x₁ (read "not x₁"). Clause: a bunch of literals joined by OR (∨), e.g. x₁ ∨ ¬x₂ ∨ x₃ — true if any of its literals is true. CNF (conjunctive normal form): several clauses joined by AND (∧) — true only if every clause is true.

SAT (satisfiability): given a CNF formula, is there a True/False assignment to the variables that makes the whole formula true? If yes it's satisfiable. By Cook's theorem SAT is the first NP-complete problem.

tiny SAT instance: (x₁ ∨ x₂ ∨ x₃) ∧ (¬x₁) ∧ (¬x₂) assignment x₁=F, x₂=F, x₃=T → every clause true → SATISFIABLE ✓

3-SAT: exactly the same question, but with the restriction that every clause has exactly three literals. So (x₁ ∨ ¬x₂ ∨ x₃) ∧ (x₂ ∨ x₃ ∨ ¬x₄) is a legal 3-SAT formula; a 2-literal or 4-literal clause is not.

5 · Reduction in detail — SAT → 3-SAT

We show 3-SAT is NP-complete by reducing SAT (known NPC) to it. (I) 3-SAT is in NP (it's a restricted SAT: guess an assignment, check in poly time). (II) SAT → 3-SAT by padding each clause with fresh variables yᵢ so it ends up with exactly 3 literals, splitting by the clause's current size:

1 literal L₁ → 4 clauses, 2 new vars (all 4 sign combos): L₁ ∨ y₁ ∨ y₂ L₁ ∨ ¬y₁ ∨ y₂ L₁ ∨ y₁ ∨ ¬y₂ L₁ ∨ ¬y₁ ∨ ¬y₂ 2 literals L₁,L₂ → 2 clauses, 1 new var: L₁ ∨ L₂ ∨ y₁ L₁ ∨ L₂ ∨ ¬y₁ 3 literals → UNCHANGED k > 3 literals → chain with k−3 new vars y₁…y_{k−3}: L₁ ∨ L₂ ∨ y₁ L₃ ∨ ¬y₁ ∨ y₂ L₄ ∨ ¬y₂ ∨ y₃ ⋮ L_{k−1} ∨ Lₖ ∨ ¬y_{k−3}

Work the note's example before peeking. SAT instance: x₁ ∨ x₂  ·  ¬x₃  ·  x₁ ∨ ¬x₂ ∨ x₃ ∨ ¬x₄ ∨ x₅.

The transformed 3-SAT instance (slide 39)
x₁ ∨ x₂ ∨ y₁ (from the 2-literal clause) x₁ ∨ x₂ ∨ ¬y₁ ¬x₃ ∨ y₂ ∨ y₃ (from the 1-literal clause ¬x₃) ¬x₃ ∨ ¬y₂ ∨ y₃ ¬x₃ ∨ y₂ ∨ ¬y₃ ¬x₃ ∨ ¬y₂ ∨ ¬y₃ x₁ ∨ ¬x₂ ∨ y₄ (from the 5-literal clause, chained, k−3 = 2 new vars) x₃ ∨ ¬y₄ ∨ y₅ ¬x₄ ∨ x₅ ∨ ¬y₅
Why it works (correctness sketch)

S satisfiable ⇔ S′ satisfiable.

(⇒) Some original literal Lᵢ = T. In the k>3 chain, set the chain y's true for the clauses before Lᵢ's clause and false for those after — then every padding clause is covered by a y except Lᵢ's own clause, which Lᵢ = T already satisfies. (1- and 2-literal cases: the true literal carries over and the fresh y's can be set freely.)

(⇐) A satisfying assignment of S′ cannot ride on the y's alone: resolving the chain on all the yᵢ cancels the padding and yields back L₁ ∨ … ∨ Lₖ, so at least one original Lᵢ must be true ⇒ S is satisfiable.

Hence 3-SAT is NP-complete.

6 · Meet sum-of-subsets and partition

The second reduction is between two number problems. Define them first.

Sum-of-subsets: given a set of positive numbers A = {a₁,…,aₙ} and a target C, is there some subset of A whose numbers add up to exactly C?

tiny instance: A = {7, 5, 19, 1, 12, 8, 14}, C = 21 subset {7, 14} sums to 21 → YES ✓ (for C = 11, no subset works → NO)

Partition: given a set of positive numbers A, can you split it into two groups whose sums are equal (each group gets exactly half the total)?

tiny instance: A = {1, 3, 8, 4, 10} (total 26) {1, 8, 4} = 13 and {3, 10} = 13 → equal halves → YES ✓

7 · Reduction in detail — Sum of subsets → Partition

We show partition is NP-complete by reducing sum-of-subsets (known NPC, upstream in the map) to it. (I) partition is in NP (guess the split, add up both sides, check equal). (II) sum-of-subsets → partition by adding two numbers (slide 56). Let Tsum = Σ aᵢ:

bᵢ = aᵢ for 1 ≤ i ≤ n b_{n+1} = C + 1 b_{n+2} = Tsum + 1 − C B = A ∪ { C+1 , Tsum+1−C } Σ B = 2·Tsum + 2 ⇒ each side must sum to Tsum + 1

Try the correspondence yourself, then reveal.

The partition that matches a subset summing to C

A subset S ⊆ A with Σ_S = C exists iff B has a balanced partition:

side 1: S ∪ {b_{n+2}} → C + (Tsum+1−C) = Tsum + 1 ✓ side 2: (A∖S) ∪ {b_{n+1}} → (Tsum−C) + (C+1) = Tsum + 1 ✓
Why C+1, not C? (slide 57)

The +1 offsets force the two padding numbers onto different sides: b_{n+1} + b_{n+2} = (C+1)+(Tsum+1−C) = Tsum + 2 > Tsum + 1, so they cannot both sit on one side. That pins the structure so a balanced partition of B corresponds exactly to a subset of A summing to C. With plain C, the two padding numbers could land together and break the correspondence.

Map context: CN → exact cover → sum of subsets → partition is the chain that lands here, and SATY → CN supplies the construction upstream (a graph whose vertices are the literals xᵢ, ¬xᵢ, a y₁…yₙ palette clique, and a clause vertex per cⱼ joined to the literals not in it; satisfiable ⇔ (n+1)-colorable). We keep depth to the two reductions above — those are the cleanest to reproduce under exam pressure.
