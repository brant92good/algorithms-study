The Reduction ChainThe Reduction Chain（歸約鏈）

Exam goal: prove a problem is NP-complete and reproduce a reduction by hand. This is the highest-stakes proof/QA topic. The single win: lock in the 2-step recipe, memorize the reduction map, and be able to redraw the two cleanest constructions — SAT → 3-SAT and sum of subsets → partition — from scratch.考試目標：證明一個問題是 NP-complete，而且能自己手寫重現一個 reduction。這是最會考、分數最重的證明題。其實你只要拿下一件事就贏了：把那套兩步驟流程記熟、背下 reduction map，然後能從零開始重畫出兩個最乾淨的構造 — SAT → 3-SAT 跟 sum of subsets → partition。

1 · What is all this? (start here)1 · 這些到底在講什麼？（先看這裡）

This whole topic is about which problems are "hard" in a precise sense. A few plain-words definitions first — everything later is built on these, so don't skip them.整個主題其實就在問一件事：嚴格來說，到底哪些問題算「難」？先把幾個白話定義講清楚 — 後面所有東西都疊在這些上面，所以千萬別跳過。

Decision problem: a problem whose answer is just Yes or No (e.g. "is there a tour shorter than 100?"), as opposed to an optimization problem that asks for the best value. The whole theory is about decision problems.Decision problem（決策問題）：答案只有 Yes 或 No 的問題，例如「有沒有一條長度小於 100 的 tour？」。這跟要你算出最佳值的 optimization 問題不一樣。整套理論談的都是 decision problem。

NP (informally): the class of decision problems where, if someone hands you a candidate answer, you can check it is correct in polynomial time (fast). You may not be able to find the answer quickly, but you can verify a proposed one. (Formally: a "guess a candidate, then check it quickly" algorithm.)NP（白話講）：一類 decision problem，重點是 如果有人塞給你一個候選答案，你能在多項式時間內（很快）檢查它對不對。你不一定有辦法很快找出答案，但別人給你答案你能驗證。（正式講：就是「猜一個候選，再快速檢查」的演算法。）

NP-complete (NPC): the hardest problems inside NP. Two conditions: (a) the problem is in NP (answers are checkable fast), and (b) it is NP-hard — every NP problem can be transformed into it. So if you ever found a fast algorithm for one NP-complete problem, you'd get a fast algorithm for all of NP. Nobody has, which is why these are considered intractable.NP-complete (NPC)：NP 裡最難的那一批問題。要同時滿足兩個條件：(a) 它在 NP 裡（答案能快速檢查），(b) 它是 NP-hard — 每一個 NP 問題都能轉換成它。所以只要你對任何一個 NP-complete 問題找到快速演算法，你就等於一口氣搞定整個 NP。到現在沒人做到過，這就是為什麼大家把這些問題當成 intractable（難解的）。

What is a reduction? A reduction A → B is a recipe that turns any instance of problem A into an instance of problem B in polynomial time, so that the Yes/No answer is preserved (the A-instance is a "Yes" exactly when the built B-instance is a "Yes"). Intuition: if you can solve B, you can solve A by translating and asking B — so B is at least as hard as A. Read the arrow as "A reduces to B" = "B is at least as hard as A."什麼是 reduction？一個 reduction A → B 就是一套作法，能在多項式時間內把問題 A 的任何 instance 轉成問題 B 的 instance，而且Yes/No 答案不變（A-instance 是「Yes」，剛好就等於轉出來的 B-instance 也是「Yes」）。直覺上：只要你會解 B，那遇到 A 你就先轉成 B 再去問 B 就好 — 所以 B 至少跟 A 一樣難。箭頭就讀成「A reduces to B」，意思是「B 至少跟 A 一樣難」。

Why "known NP-complete B reduces to A" makes A NP-complete. Suppose B is already known to be NP-complete (the hardest kind), and you exhibit a reduction B → A. That says A is at least as hard as B — and since B is at least as hard as everything in NP, so is A. That makes A NP-hard. If you also show A ∈ NP, then A is NP-hard and in NP = NP-complete. The two-step recipe below is exactly this.為什麼「拿一個已知 NP-complete 的 B reduce 到 A」就能讓 A 變成 NP-complete？假設 B 已經知道是 NP-complete（最難的那種），而你做出了一個 reduction B → A。這就說明 A 至少跟 B 一樣難 — 既然 B 至少跟 NP 裡每一個問題一樣難，那 A 當然也是。這樣 A 就是 NP-hard 了。接著只要你順便證出 A ∈ NP，那 A 就同時是 NP-hard 又在 NP 裡 = NP-complete。下面那套兩步驟流程做的就是這件事。

2 · How to prove A is NP-complete2 · 怎麼證明 A 是 NP-complete

Two steps, always (slide 35):就兩步，永遠都這兩步（slide 35）：

(I) show A ∈ NP — give a nondeterministic poly-time algorithm (guess + poly-time check) (II) show B → A — pick a KNOWN NPC problem B and reduce it TO A (B reduces to A)

Recall the reduction arrow: B → A means "B can be solved using a poly-time algorithm for A as a subroutine," i.e. A is at least as hard as B.複習一下 reduction 箭頭：B → A 是說「B 可以把 A 的多項式時間演算法拿來當副程式來解」，換句話說 A 至少跟 B 一樣難。

Why this suffices: (II) shows A is at least as hard as an already-hardest problem B, so A is NP-hard. (I) places A inside NP. NP-hard + in NP = NP-complete.

Direction trap (memorize): you reduce the known hard problem B → A (known → new). Reducing the other way (A → B) would only prove A is easy, not hard. Cook's theorem seeds the whole chain: every NP problem → SAT, so SAT is the first NPC problem.為什麼這樣就夠了：(II) 證明 A 至少跟一個已知最難的問題 B 一樣難，所以 A 是 NP-hard；(I) 把 A 塞進 NP 裡。NP-hard ＋ 在 NP 裡 = NP-complete。

方向陷阱（一定要背）：是拿已知的難問題來做 B → A（known → new）。反過來做（A → B）只會證出 A 很簡單，根本沒證到難。整條鏈的源頭是 Cook's theorem：every NP problem → SAT，所以 SAT 是第一個 NPC 問題。

4 · Meet SAT and 3-SAT4 · 認識 SAT 與 3-SAT

The first reduction is between two cousins, so define them before using them.第一個 reduction 是在兩個近親之間做的，所以先把它們定義清楚再開始。

Literal: a variable or its negation — x₁ or ¬x₁ (read "not x₁"). Clause: a bunch of literals joined by OR (∨), e.g. x₁ ∨ ¬x₂ ∨ x₃ — true if any of its literals is true. CNF (conjunctive normal form): several clauses joined by AND (∧) — true only if every clause is true.Literal（文字）：一個變數，或它的否定 — x₁ 或 ¬x₁（讀成「not x₁」）。Clause（子句）：一堆 literal 用 OR（∨）串起來，例如 x₁ ∨ ¬x₂ ∨ x₃ — 裡面隨便一個 literal 為真，整個 clause 就為真。CNF (conjunctive normal form，合取範式)：好幾個 clause 用 AND（∧）串起來 — 要每個 clause 都為真，整串才為真。

SAT (satisfiability): given a CNF formula, is there a True/False assignment to the variables that makes the whole formula true? If yes it's satisfiable. By Cook's theorem SAT is the first NP-complete problem.SAT (satisfiability，可滿足性)：給你一個 CNF formula，問：能不能找到一組變數的 True/False 賦值，讓整個 formula 為真？找得到的話，它就是 satisfiable（可滿足的）。根據 Cook's theorem，SAT 是第一個 NP-complete 問題。

tiny SAT instance: (x₁ ∨ x₂ ∨ x₃) ∧ (¬x₁) ∧ (¬x₂) assignment x₁=F, x₂=F, x₃=T → every clause true → SATISFIABLE ✓

3-SAT: exactly the same question, but with the restriction that every clause has exactly three literals. So (x₁ ∨ ¬x₂ ∨ x₃) ∧ (x₂ ∨ x₃ ∨ ¬x₄) is a legal 3-SAT formula; a 2-literal or 4-literal clause is not.3-SAT：問的問題一模一樣，只是多了一條限制：每個 clause 都得剛好有三個 literal。所以 (x₁ ∨ ¬x₂ ∨ x₃) ∧ (x₂ ∨ x₃ ∨ ¬x₄) 是合法的 3-SAT formula；只有 2 個或多到 4 個 literal 的 clause 就不行。

5 · Reduction in detail — SAT → 3-SAT5 · Reduction 細節 — SAT → 3-SAT

We show 3-SAT is NP-complete by reducing SAT (known NPC) to it. (I) 3-SAT is in NP (it's a restricted SAT: guess an assignment, check in poly time). (II) SAT → 3-SAT by padding each clause with fresh variables yᵢ so it ends up with exactly 3 literals, splitting by the clause's current size:我們要證 3-SAT 是 NP-complete，作法就是把 SAT（已知 NPC）reduce 到 3-SAT。(I) 3-SAT 在 NP 裡（它就是加了限制的 SAT：猜一組賦值，多項式時間檢查就好）。(II) SAT → 3-SAT 的招數是用新變數 yᵢ 把每個 clause 補（pad）到剛好 3 個 literal，然後看 clause 現在有幾個 literal，分情況處理：

1 literal L₁ → 4 clauses, 2 new vars (all 4 sign combos): L₁ ∨ y₁ ∨ y₂ L₁ ∨ ¬y₁ ∨ y₂ L₁ ∨ y₁ ∨ ¬y₂ L₁ ∨ ¬y₁ ∨ ¬y₂ 2 literals L₁,L₂ → 2 clauses, 1 new var: L₁ ∨ L₂ ∨ y₁ L₁ ∨ L₂ ∨ ¬y₁ 3 literals → UNCHANGED k > 3 literals → chain with k−3 new vars y₁…y_{k−3}: L₁ ∨ L₂ ∨ y₁ L₃ ∨ ¬y₁ ∨ y₂ L₄ ∨ ¬y₂ ∨ y₃ ⋮ L_{k−1} ∨ Lₖ ∨ ¬y_{k−3}

Work the note's example before peeking. SAT instance: x₁ ∨ x₂  ·  ¬x₃  ·  x₁ ∨ ¬x₂ ∨ x₃ ∨ ¬x₄ ∨ x₅.先別偷看，自己動手做做看這個例子。SAT instance：x₁ ∨ x₂  ·  ¬x₃  ·  x₁ ∨ ¬x₂ ∨ x₃ ∨ ¬x₄ ∨ x₅。

The transformed 3-SAT instance (slide 39)轉換出來的 3-SAT instance（slide 39）
x₁ ∨ x₂ ∨ y₁ (from the 2-literal clause) x₁ ∨ x₂ ∨ ¬y₁ ¬x₃ ∨ y₂ ∨ y₃ (from the 1-literal clause ¬x₃) ¬x₃ ∨ ¬y₂ ∨ y₃ ¬x₃ ∨ y₂ ∨ ¬y₃ ¬x₃ ∨ ¬y₂ ∨ ¬y₃ x₁ ∨ ¬x₂ ∨ y₄ (from the 5-literal clause, chained, k−3 = 2 new vars) x₃ ∨ ¬y₄ ∨ y₅ ¬x₄ ∨ x₅ ∨ ¬y₅
Why it works (correctness sketch)為什麼這樣會對（正確性概要）

S satisfiable ⇔ S′ satisfiable.S satisfiable ⇔ S′ satisfiable。

(⇒) Some original literal Lᵢ = T. In the k>3 chain, set the chain y's true for the clauses before Lᵢ's clause and false for those after — then every padding clause is covered by a y except Lᵢ's own clause, which Lᵢ = T already satisfies. (1- and 2-literal cases: the true literal carries over and the fresh y's can be set freely.)(⇒) 假設原本有某個 literal Lᵢ = T。在 k>3 的 chain 裡，這樣設 chain 上的 y：排在 Lᵢ 那個 clause 之前的都設真、之後的都設假 — 這樣一來，除了 Lᵢ 自己那個 clause（已經被 Lᵢ = T 撐住）以外，其他每個填補 clause 都會被某個 y 蓋掉。（1 個跟 2 個 literal 的情況更簡單：那個為真的 literal 直接帶過去，新的 y 隨便設都行。）

(⇐) A satisfying assignment of S′ cannot ride on the y's alone: resolving the chain on all the yᵢ cancels the padding and yields back L₁ ∨ … ∨ Lₖ, so at least one original Lᵢ must be true ⇒ S is satisfiable.(⇐) 反過來，S′ 的滿足賦值不可能純靠 y 硬撐：把所有 yᵢ 沿著 chain 做 resolution，填補的部分會互相抵銷，最後又還原回 L₁ ∨ … ∨ Lₖ，所以一定至少有一個原本的 Lᵢ 是真的 ⇒ S 是 satisfiable。

Hence 3-SAT is NP-complete.因此 3-SAT 是 NP-complete。

6 · Meet sum-of-subsets and partition6 · 認識 sum-of-subsets 與 partition

The second reduction is between two number problems. Define them first.第二個 reduction 是在兩個數字問題之間做的。一樣先定義清楚。

Sum-of-subsets: given a set of positive numbers A = {a₁,…,aₙ} and a target C, is there some subset of A whose numbers add up to exactly C?Sum-of-subsets：給你一堆正數 A = {a₁,…,aₙ} 和一個目標 C，問：能不能從 A 裡挑某些數字出來，加起來剛好等於 C？

tiny instance: A = {7, 5, 19, 1, 12, 8, 14}, C = 21 subset {7, 14} sums to 21 → YES ✓ (for C = 11, no subset works → NO)

Partition: given a set of positive numbers A, can you split it into two groups whose sums are equal (each group gets exactly half the total)?Partition：給你一堆正數 A，問：能不能把它分成兩組，讓兩組加起來一樣多（也就是每組剛好拿到總和的一半）？

tiny instance: A = {1, 3, 8, 4, 10} (total 26) {1, 8, 4} = 13 and {3, 10} = 13 → equal halves → YES ✓

7 · Reduction in detail — Sum of subsets → Partition7 · Reduction 細節 — Sum of subsets → Partition

We show partition is NP-complete by reducing sum-of-subsets (known NPC, upstream in the map) to it. (I) partition is in NP (guess the split, add up both sides, check equal). (II) sum-of-subsets → partition by adding two numbers (slide 56). Let Tsum = Σ aᵢ:我們要證 partition 是 NP-complete，作法是把 sum-of-subsets（已知 NPC，在地圖的上游）reduce 到 partition。(I) partition 在 NP 裡（猜一種切法，把兩邊加起來，檢查相不相等就好）。(II) sum-of-subsets → partition 的招數是多塞兩個數字進去（slide 56）。令 Tsum = Σ aᵢ：

bᵢ = aᵢ for 1 ≤ i ≤ n b_{n+1} = C + 1 b_{n+2} = Tsum + 1 − C B = A ∪ { C+1 , Tsum+1−C } Σ B = 2·Tsum + 2 ⇒ each side must sum to Tsum + 1

Try the correspondence yourself, then reveal.先自己想想這個對應是怎麼對上的，再展開來看。

The partition that matches a subset summing to C跟「子集總和剛好 C」對上的那個 partition

A subset S ⊆ A with Σ_S = C exists iff B has a balanced partition:A 裡存在一個加起來剛好 Σ_S = C 的 subset S ⊆ A，剛好就等於 B 切得出一個兩邊平衡的 partition：

side 1: S ∪ {b_{n+2}} → C + (Tsum+1−C) = Tsum + 1 ✓ side 2: (A∖S) ∪ {b_{n+1}} → (Tsum−C) + (C+1) = Tsum + 1 ✓
Why C+1, not C? (slide 57)為什麼用 C+1，不是直接用 C？（slide 57）

The +1 offsets force the two padding numbers onto different sides: b_{n+1} + b_{n+2} = (C+1)+(Tsum+1−C) = Tsum + 2 > Tsum + 1, so they cannot both sit on one side. That pins the structure so a balanced partition of B corresponds exactly to a subset of A summing to C. With plain C, the two padding numbers could land together and break the correspondence.這個 +1 偏移的用意是逼那兩個填補數字非得分到不同邊不可：你看 b_{n+1} + b_{n+2} = (C+1)+(Tsum+1−C) = Tsum + 2 > Tsum + 1，兩個加起來已經超過一邊能裝的 Tsum+1，所以它們塞不進同一邊。這樣一來結構就被釘死了，B 的平衡 partition 就剛好對得上 A 裡總和為 C 的 subset。要是直接用 C，這兩個數字有可能跑到同一邊，對應就被破壞掉了。

Map context: CN → exact cover → sum of subsets → partition is the chain that lands here, and SATY → CN supplies the construction upstream (a graph whose vertices are the literals xᵢ, ¬xᵢ, a y₁…yₙ palette clique, and a clause vertex per cⱼ joined to the literals not in it; satisfiable ⇔ (n+1)-colorable). We keep depth to the two reductions above — those are the cleanest to reproduce under exam pressure.放回地圖看：落到這裡的那條鏈就是 CN → exact cover → sum of subsets → partition，而再上游的 SATY → CN 負責蓋出那個構造（一張圖：頂點是 literal xᵢ, ¬xᵢ、加一個 y₁…yₙ 的 palette clique，再幫每個 cⱼ 配一個 clause 頂點，連到那些不在這個 clause 裡的 literal；satisfiable ⇔ 可以 (n+1)-colorable）。我們只把功夫花在上面那兩個 reduction — 它們是考試一緊張時最乾淨、最好默出來的。
