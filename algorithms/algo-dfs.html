Depth-First SearchDepth-First Search

Search a tree of candidate solutions by diving as deep as possible down one branch, then backtracking the moment you hit a dead end — driven by a single stack.搜一棵候選解的樹：沿著一條分支盡量往深處鑽，一踩到死路就回溯——整個過程靠一個 stack 驅動。

📄 Source: Alg05 · The Searching Strategies §1–3 (solution-space trees → DFS guided by a stack).📄 出處：Alg05 · The Searching Strategies §1–3（解空間樹 → 用 stack 引導的 DFS）。

The idea核心觀念

Many hard yes/no problems share one shape: "can I build a complete object by making a sequence of choices?" Lay those choices out as a solution tree (also called a state-space tree). The root is the empty partial solution; each edge fixes one more decision; each node is the partial solution so far; a leaf is a complete candidate. A node we are happy with is a goal node. The whole job is to find a goal node somewhere in this tree.很多難的 yes/no 問題其實長得一樣：「我能不能靠一連串選擇，拼出一個完整的東西？」把這些選擇攤成一棵 solution tree（也叫 state-space tree）。root 是還什麼都沒選的空解；每條 edge 多定下一個決策；每個 node 是目前為止的部分解；leaf 則是一個完整的候選解。我們滿意的 node 就是 goal node。整件事就是要在這棵樹裡找到一個 goal node。

Depth-first search is one order for visiting that tree. The rule is simple: from the current node, walk down to its first child, then that node's first child, and keep going as deep as you can. When you reach a node that is a dead end — it violates a constraint, or it has no unexplored children — you backtrack: step back up to the most recent node that still has an untried child, and dive again. You only ever back up the minimum needed.Depth-first search 是拜訪這棵樹的其中一種順序。規則很簡單：從目前的 node 走到它的第一個 child，再走那個 node 的第一個 child，能多深就多深。等你走到一個死路——違反了某個條件，或是沒有還沒探索過的 child——就回溯：退回到最近一個還有 child 沒試過的 node，再往下鑽。每次都只退剛好需要的那麼多。

The bookkeeping is a stack (LIFO — last in, first out):記帳用的是一個 stack（LIFO——last in, first out）：

  1. Push the root onto the stack.把 root push 進 stack。
  2. Pop a node. If it is a goal node, stop — success.Pop 出一個 node。如果它是 goal node，就停——成功。
  3. Otherwise push its children, and repeat from step 2 until the stack is empty (failure).否則把它的 children push 進去，然後回到第 2 步重複，直到 stack 空了（失敗）。

Because the stack hands back the most recently pushed node first, the children you just added are explored before their cousins — that LIFO ordering is what produces the deep-first exploration order. (Swap the stack for a FIFO queue and the very same loop becomes breadth-first search, which sweeps level by level instead.)因為 stack 每次先還給你最近 push 進去的那個 node，所以你剛加進來的 children 會比它們的堂兄弟先被探索——就是這個 LIFO 順序造成了往深處走的探索順序。（把 stack 換成 FIFO 的 queue，同一個迴圈就變成 breadth-first search，改成一層一層掃。）

Why it is cheap. DFS does not need the whole tree in memory — only the current root-to-node path sits on the stack. And it can prune: the instant a partial solution already breaks a rule (a clause — one OR-group of a Boolean formula, like (x or not y) — is forced false, or a running sum overshoots the target), the entire subtree below it is abandoned without ever being built. That is why search beats brute-forcing all 2n candidates: with n independent yes/no choices there are 2 × 2 × … = 2n possible complete candidates, and pruning lets you skip whole blocks of them at once.為什麼它很省。DFS 不用把整棵樹放在記憶體裡——stack 上只有目前這條從 root 到 node 的路徑。而且它能剪枝：只要一個部分解已經違反了規則（某個 clause——布林公式裡的一個 OR 群組，例如 (x or not y)——被逼成 false，或目前的和超過了 target），它底下整個 subtree 就直接放掉，根本不用建出來。這就是為什麼用搜尋會贏過硬把 2n 個候選解全部試一遍：n 個各自獨立的 yes/no 選擇，會有 2 × 2 × … = 2n 種完整的候選解，而剪枝能讓你一次跳過一大塊。

A tiny illustration小範例

Here is the deck's sum-of-subsets tree (Fig. 6-11): from the ordered set S = (7, 5, 1, 2, 10), is there a subset summing to 9? Each node holds the running sum of the elements picked so far (we write it as sum=… so it is never confused with an element of the set). Each edge +x means "the next element you add is x".這是投影片裡的 sum-of-subsets 樹（Fig. 6-11）：從有順序的集合 S = (7, 5, 1, 2, 10) 裡，有沒有一個 subset 加起來等於 9？每個 node 記的是目前挑進來的元素的累計和（我們寫成 sum=…，這樣就不會跟集合裡的元素搞混）。每條 edge +x 代表「接下來要加進來的元素是 x」。

The branching rule (so you can regenerate the tree yourself): at any node you may add any element that comes after the last one you added, in the set's fixed order 7, 5, 1, 2, 10. (Forcing increasing position stops you from building the same subset twice, e.g. {7,2} and {2,7}.) A potential child is pruned the moment its running sum would exceed the target 9 — its whole subtree can only grow larger, so it can never reach 9. × marks such a pruned dead end. DFS starts at sum=0, adds 7, then dives.分支規則（這樣你就能自己重畫這棵樹）：在任何一個 node，你可以加進任何一個「位置排在你上一個加進來的元素後面」的元素，順序固定就是 7, 5, 1, 2, 10。（規定位置只能往後，是為了不要把同一個 subset 重複建兩次，例如 {7,2} 跟 {2,7}。）只要某個候選 child 的累計和會超過 target 9，它就會被剪掉——它底下的 subtree 只會越加越大，永遠回不到 9。× 標的就是這種被剪掉的死路。DFS 從 sum=0 開始，加進 7，然後往下鑽。

sum=0 (root, picked nothing yet) └─ +7 ──▶ sum=7 (still < 9, neither goal nor overshoot → expand its children) ├─ +5 ──▶ sum=12 × overshoot, backtrack ├─ +1 ──▶ sum=8 (still < 9 → expand) │ ├─ +2 ──▶ sum=10 × overshoot │ └─ +10 ─▶ sum=18 × overshoot ├─ +2 ──▶ sum=9 ◀ GOAL: {7, 2} └─ +10 ─▶ sum=17 × overshoot (not reached: search already stopped at the goal)

Walk it as the stack algorithm. Push the root sum=0, pop it, push its child sum=7. Pop sum=7: it is neither a goal (≠9) nor an overshoot (<9), so we push all of its children and keep diving. (One subtlety: a stack pops the last thing pushed first, so to make the children come out in the left-to-right tree order shown above, we push them in reverse — +10 first, then +2, +1, and +5 last — so +5 sits on top and pops first. The exploration order is the same either way; this just keeps the trace matching the picture.) The first child popped is sum=12 > 9 → overshoot, abandon it. Next the stack hands back sum=8 — again neither goal nor overshoot, so its children get pushed and explored first: sum=10 and sum=18, both overshoot. Now 8 has no untried children left, so the stack hands control back up to 7's next untried child, +2 → sum=9 — a goal. Answer: {7, 2}. Notice the order is forced entirely by the stack: a node that is neither goal nor dead end gets its children explored before any sibling, and we only back up to 7's +2 branch after every child of 8 has overshot. Because we stop at the first goal, 7's last branch +10 is never even tried — the search never had to look at element 10 on the winning path.用 stack 的角度走一遍。把 root sum=0 push 進去、pop 出來、push 它的 child sum=7。pop sum=7：它既不是 goal（≠9）也沒 overshoot（<9），所以把它所有 children push 進去、繼續往下鑽。（這裡有個小細節：stack 是後 push 的先 pop，所以要讓 children 照上面圖裡「由左到右」的順序跑出來，我們得反過來 push——先 push +10，再 +2、+1，最後才 push +5——這樣 +5 才會壓在最上面、先被 pop。其實不管哪種順序，探索到的東西都一樣，這只是讓追蹤的步驟跟圖對得起來。）第一個 pop 出來的 child 是 sum=12 > 9 → overshoot，放掉。接著 stack 還給我們 sum=8——一樣不是 goal 也沒 overshoot，所以換它的 children 先被 push、先被探索：sum=10 跟 sum=18，兩個都 overshoot。這時 8 已經沒有沒試過的 child 了，stack 就把控制權交回 7 的下一個沒試過的 child，+2 → sum=9——goal！答案：{7, 2}。注意這個順序完全是 stack 決定的：一個既不是 goal 也不是死路的 node，它的 children 會比任何兄弟節點先被探索；要等到 8 的每個 child 都 overshoot 之後，我們才繞回 7 的 +2 分支。又因為我們碰到第一個 goal 就停，7 的最後一條分支 +10 根本沒被試到——在這條成功的路徑上，搜尋從頭到尾都沒碰到元素 10。

One strategy, many trees. DFS itself is problem-agnostic — change only what the tree's nodes and "dead end" mean, and the identical dive-and-backtrack stack solves a whole family: SAT (given a Boolean formula, its tree branches on each variable being true vs. false, so a path from the root fixes one full true/false assignment), Hamiltonian circuits (branch on the next vertex to visit), sum-of-subsets (branch on in/out of each element). Same search, different solution trees.同一個策略，很多種樹。DFS 本身跟問題無關——只要改一下樹的 node 是什麼、「死路」是什麼意思，同一套往下鑽再回溯的 stack 就能解一整族問題：SAT（給一個布林公式，它的樹在每個變數「true 還是 false」上分支，所以從 root 走一條路就決定了一整組 true/false 指派）、Hamiltonian circuit（在下一個要走的 vertex 上分支）、sum-of-subsets（在每個元素的 in/out 上分支）。同一個搜尋，不同的 solution tree。

Problems it solves (in the slides)它能解的問題（投影片裡的）

  • Satisfiability (SAT) — given a Boolean formula in n variables, is there a true/false assignment making it true? (binary tree of 2n assignments)給一個 n 個變數的布林公式，有沒有一組 true/false 指派能讓它成立？（2n 種指派的二元樹）
  • Hamiltonian Circuit — does a graph contain a cycle that visits every vertex exactly once and returns to the start?一張圖裡有沒有一個環，剛好把每個 vertex 走過一次又繞回起點？
  • Sum-of-Subsets — given a set of numbers and a target, is there a subset that sums exactly to the target?給一堆數字跟一個 target，有沒有一個 subset 加起來剛好等於 target？
