Depth-First SearchDepth-First Search

Search a tree of candidate solutions by diving as deep as possible down one branch, then backtracking the moment you hit a dead end — driven by a single stack.搜一棵候選解的樹：沿著一條分支盡量往深處鑽，一踩到死路就回溯——整個過程靠一個 stack 驅動。

📄 Source: Alg05 · The Searching Strategies §1–3 (solution-space trees → DFS guided by a stack).📄 出處：Alg05 · The Searching Strategies §1–3（解空間樹 → 用 stack 引導的 DFS）。

The idea核心觀念

Many hard yes/no problems share one shape: "can I build a complete object by making a sequence of choices?" Lay those choices out as a solution tree (also called a state-space tree). The root is the empty partial solution; each edge fixes one more decision; each node is the partial solution so far; a leaf is a complete candidate. A node we are happy with is a goal node. The whole job is to find a goal node somewhere in this tree.很多難的 yes/no 問題其實長得一樣：「我能不能靠一連串選擇，拼出一個完整的東西？」把這些選擇攤成一棵 solution tree（也叫 state-space tree）。root 是還什麼都沒選的空解；每條 edge 多定下一個決策；每個 node 是目前為止的部分解；leaf 則是一個完整的候選解。我們滿意的 node 就是 goal node。整件事就是要在這棵樹裡找到一個 goal node。

Depth-first search is one order for visiting that tree. The rule is simple: from the current node, walk down to its first child, then that node's first child, and keep going as deep as you can. When you reach a node that is a dead end — it violates a constraint, or it has no unexplored children — you backtrack: step back up to the most recent node that still has an untried child, and dive again. You only ever back up the minimum needed.Depth-first search 是拜訪這棵樹的其中一種順序。規則很簡單：從目前的 node 走到它的第一個 child，再走那個 node 的第一個 child，能多深就多深。等你走到一個死路——違反了某個條件，或是沒有還沒探索過的 child——就回溯：退回到最近一個還有 child 沒試過的 node，再往下鑽。每次都只退剛好需要的那麼多。

The bookkeeping is a stack (LIFO — last in, first out):記帳用的是一個 stack（LIFO——last in, first out）：

  1. Push the root onto the stack.把 root push 進 stack。
  2. Pop a node. If it is a goal node, stop — success.Pop 出一個 node。如果它是 goal node，就停——成功。
  3. Otherwise push its children, and repeat from step 2 until the stack is empty (failure).否則把它的 children push 進去，然後回到第 2 步重複，直到 stack 空了（失敗）。

Because the stack hands back the most recently pushed node first, the children you just added are explored before their cousins — that LIFO ordering is what produces the deep-first exploration order. (Swap the stack for a FIFO queue and the very same loop becomes breadth-first search, which sweeps level by level instead.)因為 stack 每次先還給你最近 push 進去的那個 node，所以你剛加進來的 children 會比它們的堂兄弟先被探索——就是這個 LIFO 順序造成了往深處走的探索順序。（把 stack 換成 FIFO 的 queue，同一個迴圈就變成 breadth-first search，改成一層一層掃。）

Why it is cheap. DFS does not need the whole tree in memory — only the current root-to-node path sits on the stack. And it can prune: the instant a partial solution already breaks a rule (a clause is false, a running sum overshoots the target), the entire subtree below it is abandoned without ever being built. That is why search beats brute-forcing all 2ⁿ candidates.為什麼它很省。DFS 不用把整棵樹放在記憶體裡——stack 上只有目前這條從 root 到 node 的路徑。而且它能剪枝：只要一個部分解已經違反了規則（某個 clause 不成立、或目前的和超過了 target），它底下整個 subtree 就直接放掉，根本不用建出來。這就是為什麼用搜尋會贏過硬把 2ⁿ 個候選解全部試一遍。

Each node holds the running sum; going down the left edge of level i means "include element i". DFS starts at 0, includes 7, then dives. × marks a pruned dead end (sum already > 9).這是投影片裡的 sum-of-subsets 樹（Fig. 6-11）：從 S = {7, 5, 1, 2, 10} 裡，有沒有一個 subset 加起來等於 9？每個 node 記著目前的和；在第 i 層往左走代表「把第 i 個元素放進來」。DFS 從 0 開始，放入 7，然後往下鑽。× 標的是被剪掉的死路（和已經 > 9）。

0 (root, take nothing yet) └─ +7 ──▶ 7 ├─ +5 ──▶ 12 × overshoot, backtrack ├─ +1 ──▶ 8 │ ├─ +2 ──▶ 10 × overshoot │ └─ +10 ─▶ 18 × overshoot └─ +2 ──▶ 9 ◀ GOAL: {7, 2}

DFS dives 0 → 7 → 12, finds 12 > 9, backtracks to 7, tries +1 → 8, dives again to 10 and 18 (both overshoot, backtrack), pops back to 7, tries +2 → 9 — a goal. Answer: {7, 2}. The same machine never had to look at element 10 on the winning path.DFS 一路鑽 0 → 7 → 12，發現 12 > 9，回溯回 7，試 +1 → 8，再往下鑽到 10 跟 18（都超過，回溯），pop 回 7，試 +2 → 9——這就是 goal。答案：{7, 2}。在這條成功的路徑上，它從頭到尾根本沒碰到元素 10。

One strategy, many trees. DFS itself is problem-agnostic — change only what the tree's nodes and "dead end" mean, and the identical dive-and-backtrack stack solves a whole family: SAT (branch on each variable true/false), Hamiltonian circuits (branch on the next vertex), sum-of-subsets (branch on in/out of each element). Same search, different solution trees.同一個策略，很多種樹。DFS 本身跟問題無關——只要改一下樹的 node 是什麼、「死路」是什麼意思，同一套往下鑽再回溯的 stack 就能解一整族問題：SAT（在每個變數的 true/false 上分支）、Hamiltonian circuit（在下一個 vertex 上分支）、sum-of-subsets（在每個元素的 in/out 上分支）。同一個搜尋，不同的 solution tree。

Problems it solves (in the slides)它能解的問題（投影片裡的）

  • Satisfiability (SAT) — given a Boolean formula in n variables, is there a true/false assignment making it true? (binary tree of 2ⁿ assignments)給一個 n 個變數的布林公式，有沒有一組 true/false 指派能讓它成立？（2ⁿ 種指派的二元樹）
  • Hamiltonian Circuit — does a graph contain a cycle that visits every vertex exactly once and returns to the start?一張圖裡有沒有一個環，剛好把每個 vertex 走過一次又繞回起點？
  • Sum-of-Subsets — given a set of numbers and a target, is there a subset that sums exactly to the target?給一堆數字跟一個 target，有沒有一個 subset 加起來剛好等於 target？
