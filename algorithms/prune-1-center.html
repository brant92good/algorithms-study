Prune-and-Search

1-Center Problem

Find the one spot that is as close as possible to the farthest point — the centre of the smallest circle that covers every point.

What is it?

You are given n points scattered on a plane. The 1-center problem asks for a single centre position so that the distance to the point farthest from it is as small as possible. Equivalently: find the smallest circle that encloses all the points. The centre of that circle is the answer, and its radius is that smallest-possible farthest distance.

The key phrase is min-max: you minimise the maximum distance. You are not minimising the total or average distance to the points — only the single worst (largest) one matters, because that worst point is what forces the circle to be big.

Where it's used: facility location. If you must place one fire station, hospital, cell tower, or warehouse so that the worst-served customer is as close as possible, that is exactly the 1-center. Nobody should be more than the radius away.

The idea (prune-and-search). Prune-and-search is the strategy behind binary search: each round throw away a fixed fraction of the data, then repeat on what's left. Here the slides solve a constrained version first — force the centre onto a given line, which is solvable in O(n) — and use it to figure out which side the true optimum lies. A point sitting safely deep inside the current circle can never be the farthest point, so it can be pruned. Each round provably discards about n/16 of the points. Because a constant fraction vanishes every round, the work shrinks geometrically: T(n) = T(15n/16) + O(n) = O(n) — linear time.
