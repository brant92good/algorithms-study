The problem這題在問什麼

The 8-puzzle is a 3×3 board holding eight numbered tiles (1–8) and one empty square called the blank. The only legal move is to slide a tile that sits next to the blank into it — which is the same as moving the blank up, down, left, or right. Each slide produces a new board. The task: starting from a given start board, find a sequence of slides that reaches a fixed goal board.8-puzzle 是一個 3×3 的格子，上面有八個數字方塊（1–8），還有一個空格叫做 blank。你唯一能做的動作就是：把空格旁邊的方塊推進空格——其實也就是讓 blank 往上、下、左、右移一步。每滑一次就變成一個新盤面。任務很單純：給你一個起始盤面，找出一連串滑動，把它變成固定的目標盤面。

Searching for the right slides is a tree problem: each board is a node, and its children are the boards reachable in one slide. To decide which board to try next, we score every node with an evaluation function:找出正確的滑動順序，其實就是一個樹的搜尋問題：每個盤面是一個節點，它的子節點就是滑一步能到的那些盤面。那要怎麼決定下一個試哪個盤面？我們用一個 evaluation function（評估函數）來幫每個節點打分數：

f(n) = d(n) + w(n) d(n) = depth — how many slides you have already made to reach n w(n) = misplaced tiles — numbered tiles not yet on their goal square (the blank doesn't count)

Lower f is better. For the START board above: no slides made yet so d = 0; the tiles 2, 8 and 1 are off their goal squares so w = 3; hence f = 0 + 3 = 3. The whole question is just: in what order do we expand nodes of this tree? Two strategies below answer it differently — and that single difference decides whether the search gets stuck.f 越小越好。拿上面的 START 盤面來看：還沒滑過所以 d = 0；方塊 2、8、1 都沒在自己的目標位置上，所以 w = 3；算出來 f = 0 + 3 = 3。整題講白了就一個問題：我們要用什麼順序展開這棵樹的節點？下面兩種策略給的答案不一樣——而就是這一個差別，決定了搜尋會不會卡住。

① Hill Climbing — greedy, local view① Hill Climbing——貪婪、只看眼前

Expand the current board, look only at its own children, jump to the single lowest-f child, and forget the rest. The rule is simple: at every step, generate the children and move to the one with the smallest f (ties broken by a fixed child order). It keeps no open list and never backtracks. Note that f naturally tends to rise as you descend — each move adds 1 to d — so hill climbing picks the least-bad child (the smallest f), not necessarily one lower than the parent. Because it is greedy and memoryless, that one lowest-f step can walk it away from the goal (without lowering the misplaced count w), and on some boards it just oscillates between two states forever — with nothing saved, there is no way back to the other branch it ignored. Fast — but no global memory means it can wander or cycle and never recover.展開目前的盤面，只看它自己的子節點，跳到 f 最小的那一個，其他通通忘掉。規則很單純：每一步都把子節點生出來，走到 f 最小的那一個（打平時就照固定的子節點順序決定）。它不留 open list，也不回頭。注意 f 在往下走的時候本來就會越來越大——每走一步 d 就 +1——所以 hill climbing 挑的是最不爛的子節點（f 最小的那個），不一定要比父節點還小。正因為它貪婪又沒有記憶，這一步「f 最小」的選擇可能反而把它帶離目標（擺錯的數量 w 也沒變少），在有些盤面上它甚至會在兩個狀態之間來回無限循環——而它什麼都沒留，根本退不回去當初沒走的那條分支。很快——但沒有全域記憶，它就可能一直亂繞或繞圈，永遠回不來。

Why the lucky run below works even though f goes 3 → 4 → 4 → 4 → 4: the rule never required f to drop — it only requires picking the lowest-f child each step. Here that greedy choice also happened to keep making real progress (misplaced count w dropped 3 → 3 → 2 → 1 → 0), and the lowest-f path led straight to the goal, so the search was never trapped. The rising f is just d growing by 1 per move — perfectly normal. The stuck case below uses the same rule (pick the lowest-f child) but from a different start: there the greedy step does not reduce w, it walks toward a neighbour whose own lowest-f child is the board it just left — so the two boards cycle forever, and with no open list there is no way back.為什麼下面那個幸運跑法的 f 是 3 → 4 → 4 → 4 → 4，卻還是成功？因為規則從來沒要求 f 一定要變小——它只要求每一步都挑 f 最小的子節點。這裡這個貪婪的選擇剛好也一路有在進步（擺錯的數量 w 一路掉 3 → 3 → 2 → 1 → 0），而 f 最小的那條路又直接通到目標，所以它從頭到尾都沒被困住。f 會往上爬只是因為每走一步 d 就 +1——完全正常。下面那個卡住的例子用的是同一個規則（挑 f 最小的子節點），只是換了一個不同的起始盤面：那邊貪婪的這一步不會讓 w 變少，它走向的鄰居，其本身 f 最小的子節點又剛好是它剛離開的那塊盤面——於是兩塊盤面就這樣無限繞圈，而它又沒有 open list，根本退不回去。

② Best-First Search — global open list, recovers② Best-First Search——全域 open list，卡住能繞回來

Keep an open list of every discovered-but-unexpanded node across the whole tree, sort it by f, and always expand the globally cheapest node — so a sour branch is no trap: just jump back to a better node seen earlier.維護一個 open list，把整棵樹裡「已經發現但還沒展開」的節點全部記著，照 f 排序，每次都展開全域最便宜的那個——這樣一來，走進爛分支也不怕：直接跳回先前看過、比較好的節點就好。

The loop the captions follow: 1. take the lowest-f node off the open list; 2. if it is the goal, stop; 3. otherwise expand it, score its children, add them to the list; 4. sort the list by f; then repeat from 1. So a caption saying "Step 2" just means we are back at that step of this loop on a new node. In the Open list (by f) lane below, each entry is written node-id(f-value) — e.g. 7(4) means node 7 with f = 4.下面字幕跟著跑的迴圈：1. 從 open list 拿出 f 最小的節點；2. 如果它就是目標，停；3. 不然就展開它、幫子節點打分、全部加進 list；4. 把 list 照 f 重新排序；然後從第 1 步再來一次。所以字幕寫「Step 2」，意思就是換一個新節點、又回到這個迴圈的那一步。下面 Open list (by f) 那一排裡，每一項的寫法是 node-id(f-value)——舉例來說 7(4) 就是「node 7，f = 4」。
