The 8-Puzzle

Slide numbered tiles around a 3×3 board to reach a goal arrangement — and watch the same scoring function get solved two ways: a greedy climber that can freeze, and a global searcher that recovers.

📄 Source: Alg05 · Ch 6 §5–6 (Hill Climbing & Best-First Search), the 8-puzzle, Fig. 6-15 and Fig. 6-16.

🧠 Solved by: Hill Climbing and Best-First Search

The problem

The 8-puzzle is a 3×3 board holding eight numbered tiles (1–8) and one empty square called the blank. The only legal move is to slide a tile that sits next to the blank into it — which is the same as moving the blank up, down, left, or right. Each slide produces a new board. The task: starting from a given start board, find a sequence of slides that reaches a fixed goal board.

Searching for the right slides is a tree problem: each board is a node, and its children are the boards reachable in one slide. To decide which board to try next, we score every node with an evaluation function:

f(n) = d(n) + w(n) d(n) = depth — how many slides you have already made to reach n w(n) = misplaced tiles — numbered tiles not yet on their goal square (the blank doesn't count)

Lower f is better. For the START board above: no slides made yet so d = 0; the tiles 2, 8 and 1 are off their goal squares so w = 3; hence f = 0 + 3 = 3. The whole question is just: in what order do we expand nodes of this tree? Two strategies below answer it differently — and that single difference decides whether the search gets stuck.

Watch it solved

Both runs start from the same board and aim at the same goal. Misplaced tiles are tinted; each frame shows that board's f = d + w. The two strategies share the scoring — only the selection rule changes.

① Hill Climbing — greedy, local view

Expand the current board, look only at its own children, jump to the single lowest-f child, and forget the rest. Fast — until every child looks worse and it has nothing saved to fall back on.

② Best-First Search — global open list, recovers

Keep an open list of every discovered-but-unexpanded node across the whole tree, sort it by f, and always expand the globally cheapest node — so a sour branch is no trap: just jump back to a better node seen earlier.
