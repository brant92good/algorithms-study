Longest Common Subsequence

Find the longest string that hides inside both of two sequences — without rearranging anything — by filling one little table.

📄 Source: Alg07 · Ch 7 §11 (Longest Common Subsequence), slides 28–29.

What is it?

A subsequence is what you get by deleting zero or more symbols from a string while keeping the rest in their original order. From bacad you can pull out bcd, aca, or acad — you may skip letters, but you may never reorder them.

Watch the difference from a substring: a substring must be contiguous (a solid run, like aca taken from positions 2–4 of bacad). A subsequence is skip-but-keep-order, so it is more permissive — every substring is a subsequence, but not the reverse.

The Longest Common Subsequence (LCS) problem: given two strings A and B, find the longest sequence that is a subsequence of both. For A = bacad and B = accbadcb the answer is acad (length 4).

The idea. Build a table L where L[i,j] = length of the LCS of the first i letters of A and the first j letters of B. Row 0 and column 0 are all 0 (an empty prefix shares nothing). Then for every interior cell, compare the two current letters aᵢ and bⱼ:

  • Match (aᵢ = bⱼ): take the diagonal neighbour and add one — L[i,j] = L[i-1,j-1] + 1. The matched letter extends the best LCS of the two shorter prefixes.
  • Mismatch (aᵢ ≠ bⱼ): you can't use both letters, so keep the better of dropping one — L[i,j] = max(L[i-1,j], L[i,j-1]). No +1 here — nothing new was matched.

The bottom-right cell is the LCS length. To recover the actual string, trace back from that corner: step diagonally on each match (recording the letter), otherwise step toward the larger neighbour.
