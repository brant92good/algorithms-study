Branch-and-Bound

The master strategy for optimization: explore a tree of choices best-first, but compute a bound for every branch and throw away any branch that provably can't beat the best answer found so far.

📄 Source: Alg05 · Ch 6 §7–10 (Branch-and-Bound), TSP Fig 6-25/6-26.

What is it?

Plain tree searches like depth-first and breadth-first answer feasibility questions — "is there a solution?" — and stop at the first goal they hit. But many problems are optimization problems: we don't want any tour, we want the cheapest tour. The first goal found is rarely the best one, so those searches aren't enough.

Branch-and-bound fixes this with two ingredients:

  • Branch — split the set of all candidate solutions into smaller subsets, forming a tree. Here each split is "all tours that include edge X-Y" vs. "all tours that exclude edge X-Y".
  • Bound — for each node, compute a lower bound: a number that is guaranteed ≤ the cost of every solution in that subset. Because it's a guarantee, if a node's lower bound is ≥ the best complete tour we've already found, nothing in that whole subtree can win — so we prune it (never explore it).

What is a lower bound, concretely? For the Travelling Salesperson Problem (TSP — visit every city once and return home, minimizing total cost) the trick is the reduced cost matrix. Every tour must leave each city once, so it must pay at least the cheapest exit from each city. Subtract each row's minimum from that row (every row now has a 0), then subtract each column's minimum from any column still without a 0. The total amount subtracted is cost that any tour is forced to pay — so it is a valid lower bound. For the course's 7-city instance the reductions sum to 96.

Then we branch on edges. Including a zero-cost edge adds nothing (bound stays). Excluding an edge forces a more expensive choice, raising the bound. We pick the split that raises the "exclude" bound the most, keep expanding the most promising node, and prune the rest — until one complete tour is isolated and proven optimal.

The slides apply branch-and-bound to three problems — here is what each one actually asks:
  • Personnel Assignment — assign n people to n jobs where the jobs carry a precedence order (some jobs must be done before others); putting person i on job j costs Cij. Find the cheapest assignment that respects the order. (cost matrix Table 6-1)
  • Travelling Salesperson (shown here) — visit every city exactly once and return to the start, at minimum total cost (optimal tour cost 126).
  • 0/1 Knapsack — given items each with a weight and a profit, and a bag of fixed capacity, pick a subset (each item taken whole or not at all) that maximises profit without exceeding the capacity. (Fig 6-28: upper bound from a feasible packing, lower bound from the fractional relaxation.)
0/1 knapsack is also solved by dynamic programming in Ch 7 — see the 0/1 Knapsack card for that different approach.
