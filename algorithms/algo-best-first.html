Best-First Search

Keep one global list of all open nodes and always expand the single most promising one — anywhere in the tree. This global view recovers exactly where greedy hill climbing gets stuck.維護一份裝著所有 open 節點的全域 list，每次都展開最有希望的那一個——不管它在樹的哪裡。就是這個全域視野，讓它能在貪婪的 hill climbing 卡死的地方繞回來。

The idea核心想法

A tree-search strategy decides which open node to expand next. An open node is one that has been generated but not yet expanded (its children not yet produced). Different strategies pick differently:樹搜尋策略要決定的，就是下一個展開哪個 open 節點。所謂 open 節點，就是已經生出來、但還沒展開（子節點還沒生）的節點。不同策略挑法不一樣：

  • Depth-first (DFS) always dives into the most-recently-generated node (a stack).永遠鑽進最近才生出來的那個節點（用 stack）。
  • Breadth-first (BFS) expands strictly level by level (a queue).嚴格一層一層展開（用 queue）。
  • Hill climbing is greedy DFS: it expands the locally best child of the node it is currently at — and commits. If every child looks worse than where it just was, it stalls; it cannot reconsider a promising node it left behind on another branch.是貪婪版 DFS：它展開目前所在節點當下最好的子節點，而且一去不回頭。如果每個子節點看起來都比它剛剛待的地方更糟，它就卡住了；它沒辦法回頭去理會先前丟在別條分支、其實很有希望的節點。

Best-first search combines DFS and BFS. It keeps one list holding every open node from the whole tree, scores each with an estimation function f, and always expands the best-f node among all of them — no matter how deep or which branch it sits on. Because it can jump back to a promising node it generated earlier anywhere in the tree, best-first has a global view, whereas hill climbing only ever sees the children of its current node.把 DFS 和 BFS 揉在一起。它維護一份 list，裝著整棵樹所有的 open 節點，用估計函數 f 幫每個打分，每次都從全部裡面展開 f 最好的那個——不管它埋得多深、在哪條分支上。因為它隨時能跳回樹上任何地方、先前生出來的有希望節點，所以 best-first 有全域視野；而 hill climbing 從頭到尾只看得到目前節點的子節點。

Key intuition:關鍵直覺： hill climbing asks "which of my children is best?" and can dead-end. Best-first asks "which open node in the entire frontier is best?" — so a branch hill climbing abandoned is still in the list, ready to be picked up the moment it becomes the global best.hill climbing 問的是「我的子節點裡哪個最好？」，所以可能走進死路。Best-first 問的是「整個 frontier（邊界）裡哪個 open 節點最好？」——所以 hill climbing 放棄的那條分支還乖乖留在 list 裡，一旦它變成全域最好，馬上就會被挑出來。

The estimation (lower-is-better) function in the slides is the same one hill climbing uses on the 8-puzzle:投影片裡用的估計函數（越小越好）跟 hill climbing 解 8-puzzle 時用的是同一個：

  • f(n) = d(n) + w(n)
  • d(n) = depth of node n (moves made so far),＝節點 n 的 depth（到目前為止走了幾步），
  • w(n) = number of misplaced tiles in node n.＝節點 n 裡擺錯位置的方塊數。

The scheme (Step 1–Step 4)流程（Step 1–Step 4）

  1. Step 1. Form a one-element list consisting of the root node.先做一個只有一個元素的 list，裡面放根節點。
  2. Step 2. Remove the first element from the list. Expand it. If one of its descendants is a goal node, stop; otherwise add the descendants into the list.把 list 的第一個元素拿出來，展開它。如果它的某個後代是 goal 節點，就停；否則把這些後代加進 list。
  3. Step 3. Sort the entire list by the values of the estimation function.把整份 list 照估計函數的值排序。
  4. Step 4. If the list is empty, failure. Otherwise go to Step 2.如果 list 空了，就是失敗。否則回到 Step 2。

The sort in Step 3 over the whole list is the global view in action: it guarantees the "first element" pulled in Step 2 is always the globally best-looking open node. Tie-breaking: when two open nodes have the same f, we break the tie by node id (the order they were generated — lower id first), so the expansion order is fully reproducible. Complexity: like BFS, the list can grow to hold the whole frontier, so worst-case time and space are exponential in the depth — best-first trades memory for the ability to recover from a bad-looking branch.Step 3 對整份 list 排序，就是全域視野實際在運作的地方：它保證 Step 2 拿出的「第一個元素」永遠是全域看起來最好的 open 節點。同分怎麼辦：如果兩個 open 節點 f 一樣，就用 node id 決勝（也就是生出來的先後，id 小的先），這樣展開順序才會固定、可以重現。複雜度：跟 BFS 一樣，這份 list 可能膨脹到裝下整個 frontier，所以最差情況下時間和空間都是深度的指數級——best-first 是拿記憶體去換「能從爛分支繞回來」這個能力。

Problems it solves (in the slides)它能解的問題（投影片裡有）

  • 8-puzzle — slide tiles 1–8 (with one blank) on a 3×3 board from a start configuration to the goal. The same instance where hill climbing stalls: best-first keeps every generated board in one list and recovers by jumping to the globally best-f board, reaching the goal at node 10. Here root→2→7→9→10 is the goal's ancestor chain — the solution path you follow from the start to the goal. It is not the order nodes were expanded (that order was root, 1, 2, 7, 9): node 1 is expanded but is a dead-ish branch, so it never appears on the path TO the goal.——在 3×3 格子上滑動 1–8 的方塊（外加一個 blank），從起始排法推到目標。就是 hill climbing 會卡死的同一個例子：best-first 把每個生出來的盤面都放進同一份 list，靠跳到全域 f 最好的盤面繞回來，在 node 10 抵達目標。這裡的 root→2→7→9→10 是目標的祖先鏈，也就是從起點走到目標的解路徑。它不是節點被展開的順序（展開順序其實是 root、1、2、7、9）：node 1 有被展開，但那條分支沒走通，所以它不會出現在通往目標的路徑上。
