Best-First Search

Keep one global list of all open nodes and always expand the single most promising one — anywhere in the tree. This global view recovers exactly where greedy hill climbing gets stuck.

📄 Source: A05 · The Searching Strategies §6 (Best-First Search), Fig. 6-16.

The idea

A tree-search strategy decides which open node to expand next. An open node is one that has been generated but not yet expanded (its children not yet produced). Different strategies pick differently:

  • Depth-first (DFS) always dives into the most-recently-generated node (a stack).
  • Breadth-first (BFS) expands strictly level by level (a queue).
  • Hill climbing is greedy DFS: it expands the locally best child of the node it is currently at — and commits. If every child looks worse than where it just was, it stalls; it cannot reconsider a promising node it left behind on another branch.

Best-first search combines DFS and BFS. It keeps one list holding every open node from the whole tree, scores each with an estimation function f, and always expands the best-f node among all of them — no matter how deep or which branch it sits on. Because it can jump back to a promising node it generated earlier anywhere in the tree, best-first has a global view, whereas hill climbing only ever sees the children of its current node.

Key intuition: hill climbing asks "which of my children is best?" and can dead-end. Best-first asks "which open node in the entire frontier is best?" — so a branch hill climbing abandoned is still in the list, ready to be picked up the moment it becomes the global best.

The estimation (lower-is-better) function in the slides is the same one hill climbing uses on the 8-puzzle:

  • f(n) = d(n) + w(n)
  • d(n) = depth of node n (moves made so far),
  • w(n) = number of misplaced tiles in node n.

The scheme (Step 1–Step 4)

  1. Step 1. Form a one-element list consisting of the root node.
  2. Step 2. Remove the first element from the list. Expand it. If one of its descendants is a goal node, stop; otherwise add the descendants into the list.
  3. Step 3. Sort the entire list by the values of the estimation function.
  4. Step 4. If the list is empty, failure. Otherwise go to Step 2.

The sort in Step 3 over the whole list is the global view in action: it guarantees the "first element" pulled in Step 2 is always the globally best-looking open node. Complexity: like BFS, the list can grow to hold the whole frontier, so worst-case time and space are exponential in the depth — best-first trades memory for the ability to recover from a bad-looking branch.
