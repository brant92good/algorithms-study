Edit Distance (Levenshtein)

Find the fewest single-character edits that turn one word into another — and watch a table fill itself in to discover the answer.

What is it?

Take two words, say SUN and SAT. Edit distance is the smallest number of single-character edits needed to transform the first into the second. There are exactly three allowed edits:

  • Insert — add one character (e.g. SU → SUN).
  • Delete — remove one character (e.g. SUN → SU).
  • Substitute — swap one character for another (e.g. SUN → SAN).

The trick is to break the big question into subproblems: the edit distance between prefixes. We build a table dp where dp[i][j] = the edit distance between the first i letters of SUN and the first j letters of SAT. The empty prefix "" is included, which is why the table has an extra row and column.

Base cases: turning "" into a j-letter word costs j inserts; turning an i-letter word into "" costs i deletes. So the first row and column count up 0, 1, 2, 3.

Each interior cell dp[i][j] looks at three already-solved neighbors and takes the cheapest path:

  • left dp[i][j-1] + 1 — an insert,
  • up dp[i-1][j] + 1 — a delete,
  • diagonal dp[i-1][j-1] + cost — a match or substitute, where cost = 0 if the two current letters are equal, else 1.

The final answer sits in the bottom-right corner. For SUN → SAT it is 2 (substitute U→A, substitute N→T).

Watch it run

Rows are the prefixes of SUN ("", S, U, N); columns are the prefixes of SAT ("", S, A, T). The gold cell is the one we're filling; blue cells are the three neighbors it depends on.

Time: O(mn) Space: O(mn) Edits: insert · delete · substitute
