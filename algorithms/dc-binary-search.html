Binary Search

Find a value in a sorted list by halving the search range every step — the cleanest example of "divide and conquer".

What is it?

You have a list of items already sorted in increasing order, and a target value. Binary search finds the target (or reports it's absent) without scanning every item. The trick: look at the middle element. If it equals the target, done. If the target is smaller, it must be in the left half — throw the right half away. If larger, throw the left half away. Repeat on what's left. Each step halves the range, so even a million items take only ~20 looks.

We track two pointers, lo and hi (the current range), and mid = the middle index between them.
