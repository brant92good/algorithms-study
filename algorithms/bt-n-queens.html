Backtracking

N-Queens

Place N queens on an N×N board so none can attack another — by placing one per row, checking, and retreating the moment we get stuck.

What is it?

A chess queen attacks any square in the same row, the same column, or along either diagonal. The N-Queens problem asks: can you put N queens on an N×N board so that no two queens attack each other? Here we solve the classic 4×4 case — 4 queens, no two sharing a row, column, or diagonal.

Since two queens can never share a row, we place exactly one queen per row and only have to choose its column. That turns the puzzle into a search over column choices.

The technique is backtracking: a disciplined trial-and-error. Place a queen in the current row at the leftmost legal column; check it against every queen already placed; if it's safe, move on to the next row. If no column in this row is safe, we hit a dead end — so we backtrack: remove the queen from the previous row and slide it to its next column, then try again. We keep advancing and retreating until every row holds a queen.

Spotting a conflict is cheap. Two queens in rows r₁, r₂ and columns c₁, c₂ attack each other when they share a column (c₁ = c₂) or lie on a diagonal — which happens exactly when |r₁ − r₂| = |c₁ − c₂| (equal vertical and horizontal distance). We use cols[r] to store the column of the queen in row r, or -1 if that row is empty.
