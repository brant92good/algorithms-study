Dynamic Programming · Ch7

Multistage Graph — Shortest Path

Find the cheapest route across a layered graph by reasoning backward from the goal — the cleanest first taste of dynamic programming, and a problem where being greedy gets you burned.

What is it?

Dynamic programming (DP) is a way to solve a problem that is really a sequence of decisions. Instead of grabbing the locally cheapest option at each step (that's the greedy method), DP works out the best answer to small subproblems, stores it, and stitches those answers together — so the final route is guaranteed to be the best overall.

A multistage graph is the textbook setting for this. It's a directed graph whose nodes are split into ordered groups called stages. Every edge points from one stage to the very next stage — you can never skip ahead or go back. The first stage is a single source node S; the last is a single sink node T. Each edge has a cost (a weight). The job: find the minimum-cost path from S to T.

Our graph has 4 stages: S → then {A,B,C} → then {D,E,F} → then T. To get from S to T you make three decisions in a row: which stage-2 node, which stage-3 node, then on to T.

Why not just be greedy? Greedy says "take the cheapest edge you can see right now." From S the cheapest edge is S→A (cost 1), then A→D (cost 4), then the only option D→T (cost 18) — total 23. But the real cheapest route is S→C→F→T = 5+2+2 = 9. The first edge S→C looks expensive (cost 5), yet it opens a dirt-cheap finish. Greedy can't see that far ahead.

The DP idea — backward reasoning. Let d(X,T) = cost of the cheapest path from node X to the sink T. Start at the nodes nearest T (their answer is just the single edge to T), then move one stage back at a time. For each node, you only need: "the cost to step to each neighbor, plus that neighbor's already-known d(·,T); keep the smallest." This works because of the principle of optimality: if the whole S→T path is shortest, then the tail of it (from the second node onward) must itself be a shortest path. So an optimal route is built from optimal sub-routes — exactly what lets us reuse d(·,T) instead of re-exploring everything.
