Shortest Path with A*用 A* 找最短路徑

Given a graph of partial solutions with a start and a goal, find the least-cost path to a goal — guided by a guess of the cost still ahead that never overestimates.給你一張由部分解構成的 graph，有起點也有目標，要找出到目標成本最低的路徑。靠的是一個對「剩下還要花多少」的猜測，而且這個猜測絕不會高估。

🧠 Solved by:解這題的方法： A* Algorithm

The problem問題是什麼

You are given a graph (the slides build it as a search tree of partial solutions). One node is the start — call it R — and one or more nodes are goal nodes. Every edge carries a non-negative cost (a weight). The cost of a path is the sum of the edge weights along it.給你一張 graph（投影片把它畫成一棵由部分解構成的搜尋樹）。其中一個節點是起點，叫它 R，另外有一個或多個目標節點。每條邊都帶一個非負的成本（也就是權重），而一條路徑的成本就是沿路所有邊權重加起來。

The question: find a path from the start to some goal whose total cost is the smallest possible. Not just a path to a goal — the cheapest one. This is an optimization problem, and that is the catch: simple searches (depth-first, breadth-first, hill-climbing, plain best-first) stop at the first goal they reach, which need not be the cheapest.問題：從起點走到某個目標，找出總成本最小的那條路。注意不是隨便找一條到目標的路，而是要最便宜的那條。這是個最佳化問題，重點就在這：一般的搜尋（DFS、BFS、hill-climbing、單純的 best-first）一碰到第一個目標就停了，但那不一定是最便宜的。

To steer the search, A* attaches three numbers to every node n:為了引導搜尋方向，A* 會在每個節點 n 上掛三個數字：

  • g(n) — the cost already paid to reach n from the start (sum of edge weights so far). Known exactly.g(n)——從起點走到 n 已經花掉的成本（目前走過的邊權重總和）。這個是確定知道的。
  • h(n) — a heuristic guess of the cost still remaining from n to a goal. Just an estimate.h(n)——對「從 n 到目標還要花多少」的 heuristic 猜測。只是估計值。
  • h*(n) — the true cheapest remaining cost from n to a goal. Usually unknown; it is what h approximates.h*(n)——從 n 到目標真正最便宜的剩餘成本。通常不知道，h 就是在逼近它。

Where do the h values come from? For this problem they are simply given — precomputed and handed to you, like the edge costs (e.g. h(A)=2, h(D)=1, h(G)=5). You do not derive them here; you just use them. The only thing you should be able to check is that they are admissible (never overestimate). Quick check on A: the true cheapest route from A to goal I is A→D→I with cost 2+3 = 5, so h*(A)=5; the given h(A)=2 ≤ 5 ✓. Likewise D: cheapest is D→I = 3, so h*(D)=3 and h(D)=1 ≤ 3 ✓. Every node here satisfies h ≤ h*, which is exactly what makes A*'s answer optimal.那這些 h 值是哪來的？這題裡它們就是直接給你的——事先算好、跟邊成本一樣直接給（像是 h(A)=2、h(D)=1、h(G)=5）。你不用自己推，拿來用就好。唯一你該會驗證的，是它們有沒有 admissible（會不會高估）。拿 A 來檢查：從 A 到目標 I 真正最便宜的走法是 A→D→I，成本 2+3 = 5，所以 h*(A)=5；給的 h(A)=2 ≤ 5 ✓。D 也一樣：最便宜是 D→I = 3，所以 h*(D)=3，而 h(D)=1 ≤ 3 ✓。這裡每個節點都符合 h ≤ h*，這正是 A* 給的答案保證最佳的關鍵。

A* ranks nodes by f(n) = g(n) + h(n) — an estimate of the total cost of the best path through n — and always expands the open node with the smallest f. The heuristic must be admissible: h(n) ≤ h*(n) for every node (it never overestimates). When that holds, f(n) = g(n) + h(n) ≤ g(n) + h*(n) = f*(n), so f is a lower bound on the true best total cost through n — and the first goal A* selects for expansion is guaranteed optimal.A* 用 f(n) = g(n) + h(n) 來幫節點排序——這是經過 n 的最佳路徑總成本的估計值——然後每次都展開 open list 裡 f 最小的那個節點。這個 heuristic 必須是 admissible 的：每個節點都要 h(n) ≤ h*(n)（也就是絕不高估）。只要這條件成立，f(n) = g(n) + h(n) ≤ g(n) + h*(n) = f*(n)，所以 f 是經過 n 真正最佳總成本的下界——這樣一來，A* 第一個選中來展開的目標就保證是最佳解。

A tiny concrete instance. Suppose from the start R there are two children: A with edge cost 2 and heuristic h=2 (so f = 2+2 = 4), and C with edge cost 3 and h=2 (so f = 3+2 = 5). A* expands A first because f(A)=4 < f(C)=5 — it pursues the route that looks cheapest overall, not merely the one with the smallest step.舉個小例子。假設從起點 R 出發有兩個子節點：A 的邊成本是 2、heuristic h=2（所以 f = 2+2 = 4），C 的邊成本是 3、h=2（所以 f = 3+2 = 5）。A* 會先展開 A，因為 f(A)=4 < f(C)=5——它追的是看起來整體最便宜的路，而不是單看哪一步走得最短。

What if two open nodes tie on f? Then A* may pick either one — any tie-break still leads to an optimal goal, so the rule is just a convention. In the trace below we break ties by node label (alphabetical order) and apply it the same way every time: when C and D are both open at f=5, we expand C before D simply because C comes first. Watch for this — the same f=5 tie shows up twice, and we resolve it the same way both times.那如果兩個 open 節點 f 一樣大呢？這時候 A* 挑哪一個都行——不管怎麼選，最後都還是會走到最佳目標，所以這只是個約定俗成的規則。下面的 trace 裡，我們固定用節點代號（字母順序）來打破平手，而且每次都這樣做：當 C 和 D 同時 open 且 f=5 時，我們先展開 C 再展開 D，純粹因為 C 字母排在前面。注意看——同一個 f=5 的平手會出現兩次，而我們兩次都用一樣的方式解決。
