A* · Ch6

Shortest Path with A*

Given a graph of partial solutions with a start and a goal, find the least-cost path to a goal — guided by a guess of the cost still ahead that never overestimates.

📄 Source: Alg05 · Ch 6 §11 (A*), Fig. 6-36.

🧠 Solved by: A* Algorithm

The problem

You are given a graph (the slides build it as a search tree of partial solutions). One node is the start — call it R — and one or more nodes are goal nodes. Every edge carries a non-negative cost (a weight). The cost of a path is the sum of the edge weights along it.

The question: find a path from the start to some goal whose total cost is the smallest possible. Not just a path to a goal — the cheapest one. This is an optimization problem, and that is the catch: simple searches (depth-first, breadth-first, hill-climbing, plain best-first) stop at the first goal they reach, which need not be the cheapest.

To steer the search, A* attaches three numbers to every node n:

  • g(n) — the cost already paid to reach n from the start (sum of edge weights so far). Known exactly.
  • h(n) — a heuristic guess of the cost still remaining from n to a goal. Just an estimate.
  • h*(n) — the true cheapest remaining cost from n to a goal. Usually unknown; it is what h approximates.

A* ranks nodes by f(n) = g(n) + h(n) — an estimate of the total cost of the best path through n — and always expands the open node with the smallest f. The heuristic must be admissible: h(n) ≤ h*(n) for every node (it never overestimates). When that holds, f(n) = g(n) + h(n) ≤ g(n) + h*(n) = f*(n), so f is a lower bound on the true best total cost through n — and the first goal A* selects for expansion is guaranteed optimal.

A tiny concrete instance. Suppose from the start R there are two children: A with edge cost 2 and heuristic h=2 (so f = 2+2 = 4), and C with edge cost 3 and h=2 (so f = 3+2 = 5). A* expands A first because f(A)=4 < f(C)=5 — it pursues the route that looks cheapest overall, not merely the one with the smallest step.

A* is best-first search plus the admissible bound f = g + h, h ≤ h*, which turns it into an optimization-safe strategy.

The method: see A* Algorithm for how f = g + h and admissibility work in general. Sibling problem: Channel Routing — the same A* engine minimizes the number of wiring tracks, with g = tracks committed so far and h = maximal local density.
