Pack a knapsack of capacity M with whole items (take it or leave it) to maximize profit — searching a tree of "in / out" decisions, but using bounds to prune most of it away.用容量為 M 的背包裝整件物品（要嘛全拿、要嘛不拿），目標是讓利潤最大。我們會搜一棵「拿／不拿」的決策樹，但靠下界／上界（bounds）來剪枝，把大部分分支直接砍掉。

The problem問題是什麼

You have n items. Each item i has a weight Wᵢ and a profit Pᵢ (both positive integers). You have a knapsack that can carry a total weight of at most M (the capacity). For every item you make one yes/no choice — a decision variable Xᵢ ∈ {0, 1}: Xᵢ = 1 means "put item i in the bag," Xᵢ = 0 means "leave it out." You cannot take a fraction of an item — it is whole-or-nothing (that is the "0/1" in the name).你有 n 件物品。每件物品 i 有重量 Wᵢ 和利潤 Pᵢ（都是正整數）。背包最多只能裝總重 M（容量）。對每件物品你都做一個是／否的選擇——決策變數 Xᵢ ∈ {0, 1}：Xᵢ = 1 代表「把物品 i 放進去」，Xᵢ = 0 代表「不放」。你不能只拿一件物品的一部分，要嘛整件拿、要嘛不拿（名字裡的「0/1」就是這個意思）。

The goal is to pick the subset of items that maximizes total profit without overloading the bag:目標是挑一組物品，在不超重的前提下讓總利潤最大：

maximize   P₁X₁ + P₂X₂ + … + PₙXₙ   (total profit)
subject to  W₁X₁ + W₂X₂ + … + WₙXₙ ≤ M   (fits in the bag)
with each   Xᵢ = 0 or 1

One quirk of these slides:投影片的一個怪招： branch-and-bound is naturally a minimization machine (it prunes nodes whose bound is too high). So the deck negates the profit and minimizes −Σ PᵢXᵢ. A profit of 17 becomes the objective value −17; "more profit" means "more negative," and the best (optimal) value is the smallest (most negative) one. Keep that sign flip in mind — every bound below is a negative number.branch-and-bound 天生是做最小化的機器（它砍掉下界太高的節點）。所以這份投影片把利潤取負號，改去最小化 −Σ PᵢXᵢ。利潤 17 就變成目標值 −17；「利潤越多」等於「越負」，最佳（optimal）值就是最小（最負）的那個。記住這個正負號翻轉——下面每個 bound 都是負數。

Watch it solved看它怎麼解

Branch-and-bound builds a binary tree (Fig. 6-27): at level i the left edge is Xᵢ = 1 (take item i), the right edge is Xᵢ = 0 (skip it). At each node it computes two numbers, then prunes:branch-and-bound 會建一棵二元樹（Fig. 6-27）：在第 i 層，左邊那條邊是 Xᵢ = 1（拿物品 i），右邊是 Xᵢ = 0（跳過）。每個節點都算兩個數字，然後剪枝：

  • U.B. (upper bound) = the value of some feasible whole-item packing — a solution we could actually accept. The best U.B. seen so far is our "best yet."（upper bound）＝某個可行的整件裝法的值——一個我們真的能接受的解。目前看過最好的 U.B. 就是我們的「目前最佳」。
  • L.B. (lower bound) = the value of the fractional relaxation (allow 0 ≤ Xᵢ ≤ 1, fill greedily by density, take a fraction of the last item). Because fractions can only help, this L.B. is the best the subtree could ever reach — no integer packing under this node can beat it.（lower bound）＝fractional relaxation 的值（允許 0 ≤ Xᵢ ≤ 1，照密度貪婪地裝，最後一件可以只拿一部分）。因為能拿分數只會更好，所以這個 L.B. 是整棵子樹最好能到的程度——這個節點底下任何整數裝法都贏不過它。

The strategy:策略： Branch-and-Bound — branch the solution space into a tree, bound each subtree, prune the hopeless ones.把解空間 branch 成一棵樹，替每棵子樹算 bound，把沒救的砍掉。
Same problem, other method:同一題、另一種解法： 0/1 Knapsack by Dynamic Programming (Ch7) — instead of bounding a search tree, it fills a profit/weight table and reads the answer off the bottom-right.（第七章）——不是去 bound 一棵搜尋樹，而是填一張利潤／重量表，從右下角讀出答案。
