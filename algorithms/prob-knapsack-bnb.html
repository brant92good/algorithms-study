Pack a knapsack of capacity M with whole items (take it or leave it) to maximize profit — searching a tree of "in / out" decisions, but using bounds to prune most of it away.用容量為 M 的背包裝整件物品（要嘛全拿、要嘛不拿），目標是讓利潤最大。我們會搜一棵「拿／不拿」的決策樹，但靠下界／上界（bounds）來剪枝，把大部分分支直接砍掉。

🧠 Solved by:用什麼解： Branch-and-Bound (also by Dynamic Programming)（也可以用 Dynamic Programming）

The problem問題是什麼

You have n items. Each item i has a weight Wi and a profit Pi (both positive integers). You have a knapsack that can carry a total weight of at most M (the capacity). For every item you make one yes/no choice — a decision variable Xi ∈ {0, 1}: Xi = 1 means "put item i in the bag," Xi = 0 means "leave it out." You cannot take a fraction of an item — it is whole-or-nothing (that is the "0/1" in the name).你有 n 件物品。每件物品 i 有重量 Wi 和利潤 Pi（都是正整數）。背包最多只能裝總重 M（容量）。對每件物品你都做一個是／否的選擇——決策變數 Xi ∈ {0, 1}：Xi = 1 代表「把物品 i 放進去」，Xi = 0 代表「不放」。你不能只拿一件物品的一部分，要嘛整件拿、要嘛不拿（名字裡的「0/1」就是這個意思）。

The goal is to pick the subset of items that maximizes total profit without overloading the bag:目標是挑一組物品，在不超重的前提下讓總利潤最大：

maximize   P1X1 + P2X2 + … + PnXn   (total profit)
subject to  W1X1 + W2X2 + … + WnXn ≤ M   (fits in the bag)
with each   Xi = 0 or 1

The concrete instance (Fig. 6-28).實際例子（Fig. 6-28）。 n = 6 items, capacity M = 34. The items are listed in non-increasing profit density (Pi / Wi), which the bounding step relies on:n = 6 件物品，容量 M = 34。物品照利潤密度（Pi / Wi）由大到小排好，這正是 bounding 那一步要用到的：

One quirk of these slides:投影片的一個怪招： branch-and-bound is naturally a minimization machine (it prunes nodes whose bound is too high). So the deck negates the profit and minimizes −Σ PiXi. A profit of 17 becomes the objective value −17; "more profit" means "more negative," and the best (optimal) value is the smallest (most negative) one. Keep that sign flip in mind — every bound below is a negative number.branch-and-bound 天生是做最小化的機器（它砍掉下界太高的節點）。所以這份投影片把利潤取負號，改去最小化 −Σ PiXi。利潤 17 就變成目標值 −17；「利潤越多」等於「越負」，最佳（optimal）值就是最小（最負）的那個。記住這個正負號翻轉——下面每個 bound 都是負數。

Watch it solved看它怎麼解

Branch-and-bound builds a binary tree (Fig. 6-27): at level i the left edge is Xi = 1 (take item i), the right edge is Xi = 0 (skip it). At each node it computes two numbers, then prunes:branch-and-bound 會建一棵二元樹（Fig. 6-27）：在第 i 層，左邊那條邊是 Xi = 1（拿物品 i），右邊是 Xi = 0（跳過）。每個節點都算兩個數字，然後剪枝：

  • U.B. (upper bound) = the value of some feasible whole-item packing — a solution we could actually accept. The best U.B. seen so far is our "best yet."（upper bound）＝某個可行的整件裝法的值——一個我們真的能接受的解。目前看過最好的 U.B. 就是我們的「目前最佳」。
  • L.B. (lower bound) = the value of the fractional relaxation (allow 0 ≤ Xi ≤ 1, fill greedily by density, take a fraction of the last item). Because fractions can only help, this L.B. is the best the subtree could ever reach — no integer packing under this node can beat it.（lower bound）＝fractional relaxation 的值（允許 0 ≤ Xi ≤ 1，照密度貪婪地裝，最後一件可以只拿一部分）。因為能拿分數只會更好，所以這個 L.B. 是整棵子樹最好能到的程度——這個節點底下任何整數裝法都贏不過它。
Mind the signs (this is the #1 confusion).先搞清楚正負號（這裡最容易搞混）。 Because we minimize −profit, "upper" and "lower" are about the number line, not about which is "bigger profit." The feasible value U.B. = −16 is the upper (less-negative) end and the relaxation L.B. = −18 is the lower (more-negative) end, so the optimum is boxed in: −18 ≤ opt ≤ −16. Here higher = worse: −16 is a worse objective than −17, because −16 means less profit. The relaxation L.B. is the dreamier (better, more-negative) target nothing can pass; the feasible U.B. is a real packing we can already accept.因為我們是在最小化 −profit，「upper／lower」講的是數線上的位置，不是「誰利潤比較大」。可行值 U.B. = −16 是上（比較不負）那端，relaxation L.B. = −18 是下（比較負）那端，所以最佳解被夾在 −18 ≤ opt ≤ −16 之間。重點是這裡越高＝越爛：−16 比 −17 還爛，因為 −16 代表利潤比較少。L.B. 是那個誰都超不過、比較夢幻（比較負、比較好）的目標；U.B. 則是一個我們現在就能收下的真實裝法。
"Round inward" means round toward zero.「Round inward（往內取整）」就是往 0 的方向取整。 Every real 0/1 packing earns an integer profit, so a fractional bound like −18.5 can be tightened to the nearest integer that does not claim more than is actually possible. The true profit can't exceed 18, so the objective can't beat −18 — we move −18.5 inward toward zero to −18 (not outward to −19). That tightened −18 is still a valid lower bound, just a sharper one.每個真實的 0/1 裝法賺到的利潤一定是整數，所以像 −18.5 這種分數 bound，可以收緊到最接近、又不會誇大成做不到的那個整數。真實利潤不可能超過 18，所以目標值也不可能比 −18 更好——我們把 −18.5 往 0 的方向（往內）移到 −18（不是往外變 −19）。收緊後的 −18 仍然是有效的下界，只是更銳利。

How we walk the tree (so the steps aren't magic).我們怎麼走這棵樹（這樣每一步就不是憑空冒出來的）。 We do a depth-first search and always try take-first: at item i we dive down the Xi = 1 (left) edge before the Xi = 0 (right) edge, fixing one more variable at every step. So the very first leaf we reach is the path X1=1, X2=0, X3=0, X4=1, X5=1, X6=0. Instead of the slide's jumpy node numbers, the viz labels each node by the choices fixed so far — that is all a node really is — and re-computes its U.B. (a feasible packing) and L.B. (the fractional relaxation) from the items still free. Watch where the root bounds come from, how the dive finds the winning packing, and how the L.B.s prove it optimal.我們做的是depth-first（DFS）搜尋，而且固定先拿（take-first）：在物品 i 這層，先往 Xi = 1（左）那條邊鑽，再考慮 Xi = 0（右），每一步多固定一個變數。所以我們碰到的第一個葉子，就是路徑 X1=1, X2=0, X3=0, X4=1, X5=1, X6=0。視覺化不用投影片那種跳來跳去的節點編號，而是直接用目前固定了哪些選擇來標每個節點——一個節點本來就只是這樣而已——再從還沒決定的物品重算它的 U.B.（一個可行裝法）和 L.B.（fractional relaxation）。看著根節點 bounds 怎麼來、這趟下鑽怎麼找到勝出裝法，以及 L.B. 怎麼證明它最佳。

Reading the per-node numbers off the table (so nothing is magic).怎麼從表格把每個節點的數字算出來（這樣就不是憑空冒出來的）。 Every U.B. below is just "freeze the fixed choices, then greedily add whole free items by density until the next one won't fit, and read off the profit." Worked out for the dive nodes: at [X1=1,X2=0] item 1 is in (weight 10, profit 6) and item 2 is out; greedy then takes items 3,4 wholly (weights 10+8+10 = 28 ≤ 34, item 5 would make 40) → profit 6+4+5 = 15, so U.B. = −15. At [X1=1,X2=0,X3=0] item 3 is now also out, so greedy fills the free items 4,5,6 instead: take 4 then 5 (weights 10+10+12 = 32 ≤ 34, item 6 would make 40) → packing {1,4,5}, profit 6+5+6 = 17, so U.B. = −17. Dropping the low-density item 3 freed room for the more profitable items 4,5, which is why the U.B. improves from −15 to −17.下面每個 U.B. 其實都只是：「先把已固定的選擇鎖住，再照密度把還沒決定的物品整件一個個塞進去，塞到下一件放不下為止，最後讀出利潤。」把下鑽路徑上的節點算一遍：在 [X1=1,X2=0]，物品 1 已放（重 10、利潤 6）、物品 2 不放；greedy 接著整件拿物品 3、4（重 10+8+10 = 28 ≤ 34，再加物品 5 就變 40）→ 利潤 6+4+5 = 15，所以 U.B. = −15。到 [X1=1,X2=0,X3=0]，物品 3 現在也排除了，所以 greedy 改去填還自由的物品 4、5、6：先拿 4 再拿 5（重 10+10+12 = 32 ≤ 34，再加物品 6 就變 40）→ 裝法 {1,4,5}、利潤 6+5+6 = 17，所以 U.B. = −17。把低密度的物品 3 丟掉，反而騰出空間給利潤更高的物品 4、5，這就是為什麼 U.B. 會從 −15 變好到 −17。
The X1=0 branch: don't confuse its −16 with its −17.X1=0 這條分支：別把它的 −16 跟 −17 搞混。 This node reports two different numbers, and that is correct — they measure different things. Its U.B. = −16 is a feasible whole-item packing (with item 1 banned, greedy takes items 2,4: weights 19+10 = 29 ≤ 34, profit 10+5 = 16). Its L.B. = −17 is the fractional relaxation, which is allowed to slice the last item: take item 2 (w 19), item 4 (w 19+10 = 29), then 5 capacity is left, so take 5/12 of item 5 → profit 10+5+(5/12)·6 = 10+5+2.5 = 17.5, objective −17.5, rounded inward to −17. The L.B. (−17) is always at least as good as (≤) the U.B. (−16), because fractions can only help. So in Frame 5 it is the fractional −17, not the feasible −16, that decides pruning.這個節點會報兩個不一樣的數字，而且是對的——它們量的東西不同。它的 U.B. = −16 是一個可行的整件裝法（物品 1 被禁，greedy 拿物品 2、4：重 19+10 = 29 ≤ 34、利潤 10+5 = 16）。它的 L.B. = −17 則是fractional relaxation，可以切最後一件：拿物品 2（重 19）、物品 4（重 19+10 = 29），這時還剩 5 的容量，所以拿物品 5 的 5/12 → 利潤 10+5+(5/12)·6 = 10+5+2.5 = 17.5，目標值 −17.5，往內取整成 −17。L.B.（−17）一定至少跟 U.B.（−16）一樣好（≤），因為拿分數只會更好。所以 Frame 5 拿來決定要不要剪枝的，是那個分數的 −17，不是可行的 −16。
