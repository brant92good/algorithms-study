Searching Strategies · Ch6

Breadth-First Search

Search the very same solution tree as DFS — but sweep it level by level, shallowest nodes first, steered by a queue instead of a stack.

What is it?

Lots of hard problems can be rephrased as "search a tree of partial answers." Each node is a half-built decision; its children are the ways to extend it one more step. The job of a search strategy is to pick the order in which you visit those nodes.

We use the sum-of-subsets problem as our tree. Given a set S = {7, 5, 1, 2, 10} and a target of 9, find a subset whose elements add up to 9. We build a tree by deciding, one element at a time, whether to add it to a running sum. Each node holds that running sum; a node equal to 9 is a goal, and a node already over 9 is a dead end (prune it).

Depth-first search (DFS) dives straight down one branch to the bottom, then backtracks — it is guided by a stack (last-in, first-out). Breadth-first search (BFS) is its mirror image: it explores every node at one depth before touching the next depth, so it fans out across the whole tree like a ripple. The only machinery that changes is the container: BFS uses a queue — FIFO, first-in-first-out. You take nodes off the front in the same order you added them, which is exactly what makes shallow nodes come out before deep ones.

The BFS loop: put the root in the queue; repeatedly remove the front node, and if it is a goal stop, otherwise add its children to the back. Because children always go to the back, a whole level finishes before the next begins.
