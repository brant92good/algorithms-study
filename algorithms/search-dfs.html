Depth-First Search

Hunt for a solution by diving deep down one branch of a decision tree, then backtracking the moment a branch goes bad — guided by a stack.

📄 Source: Alg05 · Ch 6 Searching Strategies §3–4 (Sum-of-Subsets / DFS), Fig 6-11.

What is it?

Many problems can be solved by building a solution tree (also called a state-space tree): a tree where each node is a partial solution and each edge is one decision you make. A root-to-leaf path is one complete candidate answer. To solve the problem you search this tree for a node that is a goal (it satisfies what you wanted).

Depth-first search (DFS) is one way to explore that tree. The idea is in the name: go as deep as you can down one branch before trying any sibling branch. When the current branch is a dead end, you backtrack — undo the last decision and try the next option. DFS is driven by a stack, a "last-in, first-out" (LIFO) container: the node you pushed most recently is the next one you pop and explore. Pushing children and always popping the newest is exactly what makes the search dive deep first. (Contrast: a queue, first-in-first-out, would explore level by level — that is breadth-first search.)

Tiny concrete example — the sum-of-subsets problem. Given a set S = {7, 5, 1, 2, 10}, find a subset whose elements add up to a target of 9. We build a binary tree: at level i we decide whether element i is in the subset or out. Each node holds the running sum of the "in" choices so far. If a running sum ever exceeds 9, that whole branch is hopeless (all numbers are positive, so it can only grow) — so we backtrack. DFS dives down the "include" edges, overshoots, backs up, and lands on the subset {7, 2} = 9.

The slides use DFS to search the solution trees of three problems — SAT (§1), the Hamiltonian circuit (§2, Fig 6-8), and sum-of-subsets (shown here). Same strategy, different trees.
