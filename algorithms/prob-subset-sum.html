Sum of Subsets

Given a set of numbers and a target, decide whether some subset of them adds up to exactly the target — and find it.

📄 Source: Alg05 · Ch 6 Searching Strategies §3 (Sum-of-Subsets), Fig. 6-11.

🧠 Solved by: Depth-First Search (also BFS, same tree)

The problem

You are given a set of numbers S and a single number called the target. A subset of S is any selection of its elements — you take some of them and leave the rest (the empty selection and the whole set both count). The sum of a subset is just its elements added together.

The question: is there a subset of S whose sum is exactly the target? If yes, name it. This is a decision / feasibility problem — we want a subset that works, not the biggest or cheapest one.

The concrete instance we'll solve. Let

  • S = {7, 5, 1, 2, 10}
  • target = 9

We need a subset of {7, 5, 1, 2, 10} that sums to 9. Some attempts: {7} sums to 7 (too small), {7, 5} sums to 12 (too big), {10} already overshoots. The subset that works is {7, 2}, because 7 + 2 = 9. The rest of this page shows how a search finds that subset systematically instead of by guessing.

How do we search without trying all 2ⁿ subsets blindly? Lay the choices out as a binary decision tree: at level i we decide whether element i is in the subset or out. Each node holds the running sum of the "in" choices so far; a root-to-leaf path is one complete subset. Because every number is positive, the moment a running sum exceeds 9 that whole branch is hopeless (sums only grow) — so we can abandon it early. A tree-search strategy walks this tree until it lands on a node whose sum equals the target.

The same tree appears under "breadth-first search" in the deck — BFS is the FIFO counterpart of the DFS shown here (same tree, different expansion order).

The strategies that solve this: Depth-First Search dives deep with a stack and backtracks on overshoot (shown above); Breadth-First Search walks the identical decision tree level by level with a queue. Both halt at the first goal node found.
