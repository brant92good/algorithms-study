Sum of SubsetsSum of Subsets

Given a set of numbers and a target, decide whether some subset of them adds up to exactly the target — and find it.給你一堆數字跟一個目標值，這題在問：能不能從裡面挑一個 subset（子集合），加起來剛好等於目標？能的話就把它找出來。

📄 Source: Alg05 · Ch 6 Searching Strategies §3 (Sum-of-Subsets), Fig. 6-11.📄 出處：Alg05 · 第6章 Searching Strategies §3（Sum-of-Subsets），Fig. 6-11。

🧠 Solved by: Depth-First Search (also BFS, same tree)🧠 用什麼解：Depth-First Search（BFS 也行，同一棵樹）

The problem這題在問什麼

You are given a set of numbers S and a single number called the target. A subset of S is any selection of its elements — you take some of them and leave the rest (the empty selection and the whole set both count). The sum of a subset is just its elements added together.給你一個數字的集合 S，再給一個數字叫 target。S 的 subset 就是隨便挑一些元素出來——拿一部分、剩下的不拿（空集合跟整個集合都算）。一個 subset 的和就是把它裡面的元素加起來而已。

The question: is there a subset of S whose sum is exactly the target? If yes, name it. This is a decision / feasibility problem — we want a subset that works, not the biggest or cheapest one.問題是：有沒有一個 S 的 subset，它的和剛好等於 target？有的話就把它指出來。這是個 decision / feasibility 問題——我們只要找到一個湊得出來的 subset，不是要最大或最便宜的那個。

The concrete instance we'll solve. Let我們要解的具體例子。設

  • S = {7, 5, 1, 2, 10}
  • target = 9target = 9

We need a subset of {7, 5, 1, 2, 10} that sums to 9. Some attempts: {7} sums to 7 (too small), {7, 5} sums to 12 (too big), {10} already overshoots. The subset that works is {7, 2}, because 7 + 2 = 9. The rest of this page shows how a search finds that subset systematically instead of by guessing.我們要從 {7, 5, 1, 2, 10} 挑一個 subset，加起來等於 9。試幾個看看：{7} 是 7（太小）、{7, 5} 是 12（太大）、{10} 一個就超過了。湊得出來的是 {7, 2}，因為 7 + 2 = 9。這頁接下來要講的，就是怎麼用搜尋有系統地把這個 subset 找出來，而不是用猜的。

How do we search without trying all subsets blindly? Fix an order for the elements — we'll use the list order 7, 5, 1, 2, 10 — and build a search tree of "what to add next". The root is the empty subset, running sum 0. The generating rule for a node's children is: look at the last element you added, and branch on each element that comes after it in the list — adding that element to the subset. (You only ever add an element to the right of the last one, so no subset is built twice.) Each node holds the running sum of the elements added on the path down to it. Because every number is positive, the moment a running sum exceeds 9 that whole branch is hopeless — the sum can only grow — so we prune it (skip generating that child at all). A tree-search strategy walks this tree until it lands on a node whose sum equals the target.那要怎麼搜，才不用傻傻地把所有 subset 都試一遍？先把元素固定一個順序——這裡就用 list 的順序 7, 5, 1, 2, 10——然後建一棵「接下來要加哪個元素」的搜尋樹。root 是空集合，目前的和是 0。一個 node 的 children 怎麼長出來？看你上一個加進來的元素是哪個，然後對排在它後面的每一個元素各分一條岔路——把那個元素加進 subset。（你只會往上一個元素的右邊加，所以同一個 subset 不會被建兩次。）每個 node 記著從 root 一路加下來的那些元素的和。因為這些數字都是正的，只要目前的和超過 9，那整條分支就沒救了（和只會愈加愈大）——所以直接剪掉（那個 child 根本不長出來）。一個樹搜尋策略就是這樣走這棵樹，直到踩到某個 node 的和剛好等於 target。

Let's read the tree you'll see below straight off that rule. From the root we add the first element, 7 → node with sum 7. The last added element is 7, so its children branch on the elements after 7 in 7, 5, 1, 2, 10, namely 5, 1, 2, 10: adding 5 gives 7+5 = 12, adding 1 gives 8, adding 2 gives 9, adding 10 gives 17. Since 17 > 9 the "+10" child is pruned away, so node 7 shows exactly the three children {12, 8, 9} (the "+5", "+1", "+2" branches). Now take the node with sum 8 (path 7, 1): its last added element is 1, so its children branch on what comes after 1, namely 2, 10 — giving 8+2 = 10 and 8+10 = 18. Both overshoot 9, so both are dead ends. That is why node 8's children are exactly {10, 18}. Nothing in the tree is arbitrary; each branch is "add the next available element," and overshoots get pruned.用這條規則，下面那棵樹你可以直接讀出來。從 root 先加第一個元素 7 → 得到一個和是 7 的 node。它上一個加的是 7，所以 children 就對 7, 5, 1, 2, 10 裡排在 7 後面的元素 5, 1, 2, 10 各分一條：加 5 是 7+5 = 12、加 1 是 8、加 2 是 9、加 10 是 17。因為 17 > 9，「+10」那條直接被剪掉，所以 node 7 剛好就三個 children {12, 8, 9}（也就是 +5、+1、+2 三條）。再看和是 8 的 node（路徑 7, 1）：它上一個加的是 1，所以 children 就對排在 1 後面的 2, 10 分岔——得到 8+2 = 10 跟 8+10 = 18。兩個都超過 9，都是死路，所以 node 8 的 children 剛好是 {10, 18}。樹裡沒有任何一條是憑空冒出來的：每條分支都是「加下一個還能加的元素」，超過 9 的就被剪掉。

Type: decision / feasibility類型：decision / feasibility Full tree: up to 2ⁿ subsets · pruned to a few完整的樹：最多 2ⁿ 個 subset，剪枝後剩沒幾個 Prune when: sum > target何時剪枝：和 > target 時 Answer here: {7, 2} = 9這題的答案：{7, 2} = 9

Why only ~7 nodes, not 2⁵ = 32? The full "add-next" tree does have up to 2ⁿ leaves (one per possible subset) — including subsets that don't contain 7, like {5, 2}, which would hang off a sibling branch where the first element added is 5 instead of 7. The picture above only draws the part DFS actually visited: it dives into the "+7" branch first, and two things keep it tiny — pruning chops every branch whose sum passes 9 (that's why "+10" off node 7, and node 8's children, get cut), and DFS stops at the very first goal it reaches ({7, 2} = 9), so it never goes back to explore the "+5…" or "+1…" sibling branches at all. The other ~25 subsets exist in the full tree; they just never had to be looked at.為什麼只有大概 7 個 node，而不是 2⁵ = 32？那棵完整的「加下一個」樹確實最多有 2ⁿ 個 leaf（每個可能的 subset 一個）——包括不含 7 的 subset，像 {5, 2}，它會掛在另一條兄弟分支上，那條的第一個加的元素是 5 而不是 7。上面這張圖只畫出 DFS 真正走過的部分：它先鑽進「+7」那條，而有兩件事讓它一直很小——剪枝把每條和超過 9 的分支砍掉（所以 node 7 的「+10」、還有 node 8 的 children 都被剪了），而且 DFS 一找到第一個 goal 就停（{7, 2} = 9），所以它根本沒繞回去走「+5…」或「+1…」那些兄弟分支。其他大約 25 個 subset 在完整的樹裡都還在，只是這次沒必要去看它們而已。
