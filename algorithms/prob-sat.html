Satisfiability (SAT)Satisfiability（SAT）

Given a Boolean formula, is there any true/false assignment to its variables that makes the whole thing true? Search the tree of 2ⁿ assignments — and prune the instant a clause is broken.給你一個布林公式，問的是：有沒有任何一組變數的 true/false 指派，能讓整個公式成立？我們要搜一棵 2ⁿ 種指派的樹，而且只要有 clause 被弄壞，就馬上把那一塊砍掉。

📄 Source: Alg05 · The Searching Strategies §1 (the SAT problem; tree of 2ⁿ assignments; partial tree with early pruning).📄 出處：Alg05 · The Searching Strategies §1（SAT 問題；2ⁿ 種指派的樹；提早剪枝的 partial tree）。

🧠 Solved by: Depth-First Search (and BFS)🧠 用什麼解：Depth-First Search（也可以用 BFS）

The problem這題在問什麼

A Boolean variable is one that is either true or false. A literal is a variable or its negation — e.g. x₁ or ¬x₁ (read "not x₁"). A clause is an OR of literals, e.g. (x₂ ∨ x₅); it is satisfied if at least one of its literals is true. A formula is in conjunctive normal form (CNF) when it is an AND of clauses — so it is true only when every clause is satisfied at once.先講幾個名詞。Boolean variable（布林變數）就是只能是 true 或 false 的變數。literal（文字）是一個變數或它的否定，例如 x₁ 或 ¬x₁（唸作「not x₁」）。clause（子句）是一堆 literal 用 OR 串起來，像 (x₂ ∨ x₅)；只要裡面至少有一個 literal 是 true，這個 clause 就成立。整個公式如果是一堆 clause 用 AND 串起來，就叫做 conjunctive normal form（CNF）——所以它要成立，必須每一個 clause 同時都成立才行。

The satisfiability problem asks: given such a CNF formula in n variables x₁ … xₙ, does there exist a truth assignment that makes the formula true? If yes, the formula is satisfiable; if no assignment works, it is unsatisfiable.satisfiability problem 問的就是：給你這樣一個有 n 個變數 x₁ … xₙ 的 CNF 公式，到底有沒有一組指派能讓公式變成 true？有的話這公式就是 satisfiable（可滿足）；如果怎麼指派都不行，就是 unsatisfiable（不可滿足）。

The tiny instance (from the slide). Five clauses over variables x₁ … x₅, all of which must hold simultaneously:投影片上的小例子。變數 x₁ … x₅ 上的五個 clause，全部都要同時成立：

(1) ¬x₁ ← x₁ must be false (2) x₁ ← x₁ must be true (3) x₂ ∨ x₅ (4) x₃ (5) ¬x₂

Look at clauses (1) and (2): one demands x₁ = false, the other demands x₁ = true. They flatly contradict, so no assignment can satisfy both — this instance is unsatisfiable. The whole point of §1 is that we can discover this without enumerating all assignments: the very first variable we branch on already breaks a clause on both sides.看 clause (1) 跟 (2)：一個要 x₁ = false，另一個要 x₁ = true。這兩個直接打架，所以沒有任何指派能同時滿足它們——這個例子就是 unsatisfiable。§1 的重點就在這裡：我們不用把所有指派列出來也能發現這件事，因為光是第一個拿來分支的變數，兩邊都會弄壞某個 clause。

With n variables there are 2ⁿ possible assignments. We lay them out as a binary tree of depth n: at level i we branch on xᵢ = T (left) versus xᵢ = F (right). Every root-to-leaf path is one complete assignment; the leaves are the 2ⁿ rows of the truth table.n 個變數總共有 2ⁿ 種指派。我們把它們攤成一棵深度為 n 的二元樹：在第 i 層就針對 xᵢ = T（左）跟 xᵢ = F（右）分支。每一條從 root 到 leaf 的路徑就是一組完整的指派，而那些 leaf 就是真值表的 2ⁿ 列。

Watch it solved看它怎麼解

We run depth-first search down the 2ⁿ-assignment tree, checking after each choice whether any clause is already falsified by the partial assignment. The moment one is, that whole subtree is dead — prune it and back up. Step through.我們用 depth-first search 往這棵 2ⁿ 指派的樹下鑽，每做一個選擇就檢查一下：目前這個部分指派有沒有已經讓哪個 clause 不成立了？只要有，那整個 subtree 就死了——直接剪掉，退回上一步。一步一步看下去。

SAT is the historical seed of NP-completeness (Cook–Levin).主要出處：Alg05 · The Searching Strategies §1——SAT 問題、「Tree Representation of Eight Assignments」（完整的 2³ 真值表樹），以及「A Partial Tree to Determine the Satisfiability Problem」，它要講的就是：我們不一定要把全部 2ⁿ 種指派都看過——只要一個部分指派違反了某個 clause，整個 subtree 就直接死掉。SAT 是 NP-completeness 的歷史起點（Cook–Levin）。

The method: this tree is explored by Depth-First Search (stack / dive-and-backtrack) or Breadth-First Search (queue / level-by-level) — same tree, different visiting order. Sibling problems in this chapter: Hamiltonian Circuit and Sum of Subsets — both also cast as searching a solution-space tree with early pruning of dead branches.怎麼解：這棵樹可以用 Depth-First Search（stack／往下鑽再回溯）或 Breadth-First Search（queue／一層一層）來走——同一棵樹，只是拜訪順序不同。同章的姊妹題：Hamiltonian Circuit 跟 Sum of Subsets——這兩題一樣是把問題變成在一棵解空間樹上搜尋，並且提早把死路砍掉。
