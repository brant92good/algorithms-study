Satisfiability (SAT)Satisfiability（SAT）

Given a Boolean formula, is there any true/false assignment to its variables that makes the whole thing true? Search the tree of 2n assignments — and prune (discard that whole branch of possibilities) the instant a clause is broken.給你一個布林公式，問的是：有沒有任何一組變數的 true/false 指派，能讓整個公式成立？我們要搜一棵 2n 種指派的樹，而且只要有 clause 被弄壞，就馬上prune（把那一整段可能性砍掉）。

🧠 Solved by: Depth-First Search (and BFS)🧠 用什麼解：Depth-First Search（也可以用 BFS）

The problem這題在問什麼

A Boolean variable is one that is either true or false. A literal is a variable or its negation — e.g. x1 or ¬x1 (read "not x1"). A clause is an OR of literals, e.g. (x2 ∨ x5); it is satisfied if at least one of its literals is true. A formula is in conjunctive normal form (CNF) when it is an AND of clauses — so it is true only when every clause is satisfied at once.先講幾個名詞。Boolean variable（布林變數）就是只能是 true 或 false 的變數。literal（文字）是一個變數或它的否定，例如 x1 或 ¬x1（唸作「not x1」）。clause（子句）是一堆 literal 用 OR 串起來，像 (x2 ∨ x5)；只要裡面至少有一個 literal 是 true，這個 clause 就成立。整個公式如果是一堆 clause 用 AND 串起來，就叫做 conjunctive normal form（CNF）——所以它要成立，必須每一個 clause 同時都成立才行。

The satisfiability problem asks: given such a CNF formula in n variables x1 … xn, does there exist a truth assignment that makes the formula true? If yes, the formula is satisfiable; if no assignment works, it is unsatisfiable.satisfiability problem 問的就是：給你這樣一個有 n 個變數 x1 … xn 的 CNF 公式，到底有沒有一組指派能讓公式變成 true？有的話這公式就是 satisfiable（可滿足）；如果怎麼指派都不行，就是 unsatisfiable（不可滿足）。

The tiny instance (from the slide). Five clauses over variables x1 … x5, all of which must hold simultaneously:投影片上的小例子。變數 x1 … x5 上的五個 clause，全部都要同時成立：

(1) ¬x1 ← x1 must be false (2) x1 ← x1 must be true (3) x2 ∨ x5 (4) x3 (5) ¬x2

Look at clauses (1) and (2): one demands x1 = false, the other demands x1 = true. They flatly contradict, so no assignment can satisfy both — this instance is unsatisfiable. The whole point of §1 is that we can discover this without enumerating all assignments: the very first variable we branch on already breaks a clause on both sides.看 clause (1) 跟 (2)：一個要 x1 = false，另一個要 x1 = true。這兩個直接打架，所以沒有任何指派能同時滿足它們——這個例子就是 unsatisfiable。§1 的重點就在這裡：我們不用把所有指派列出來也能發現這件事，因為光是第一個拿來分支的變數，兩邊都會弄壞某個 clause。

With n variables there are 2n possible assignments. We lay them out as a binary tree of depth n: at level i we branch on xi = T (true, left branch) versus xi = F (false, right branch) — from here on, T always means true and F means false. Every root-to-leaf path is one complete assignment; the leaves are the 2n rows of the truth table.n 個變數總共有 2n 種指派。我們把它們攤成一棵深度為 n 的二元樹：在第 i 層就針對 xi = T（true，左分支）跟 xi = F（false，右分支）分支——之後 T 一律代表 true、F 代表 false。每一條從 root 到 leaf 的路徑就是一組完整的指派，而那些 leaf 就是真值表的 2n 列。

Watch it solved看它怎麼解

We run depth-first search (DFS) down the 2n-assignment tree. Depth-first means we dive all the way down one branch first — here, fix x1=T and go deeper — before backing up to try the alternative (x1=F); the chip's "stack" is just the bookkeeping that remembers where to back up to (last branch in, first to undo). After each choice we check whether any clause is already falsified by the partial assignment. The moment one is, that whole subtree is dead — prune it and back up. Note: the full tree would keep branching on x2…x5 below each node, but we never draw those deeper levels — x1 alone already kills both halves, so there is nothing left to expand. Step through.我們用 depth-first search（DFS）往這棵 2n 指派的樹下鑽。depth-first 的意思是：先沿著一條分支一路鑽到底——這裡就是先設 x1=T 一直往下——走不下去再退回來試另一條（x1=F）；chip 上講的「stack」其實就是用來記「該退回哪裡」的小帳本（最後進去的分支，最先被退掉）。每做一個選擇就檢查一下：目前這個部分指派有沒有已經讓哪個 clause 不成立了？只要有，那整個 subtree 就死了——直接剪掉，退回上一步。提醒：完整的樹本來會在每個節點底下繼續對 x2…x5 分支，但這裡我們不畫那些更深的層——光是 x1 就把左右兩半都殺光了，根本沒東西好展開。一步一步看下去。

The method: this tree is explored by Depth-First Search (stack / dive-and-backtrack) or Breadth-First Search (queue / level-by-level) — same tree, different visiting order. Sibling problems in this chapter: Hamiltonian Circuit and Sum of Subsets — both also cast as searching a solution-space tree with early pruning of dead branches.怎麼解：這棵樹可以用 Depth-First Search（stack／往下鑽再回溯）或 Breadth-First Search（queue／一層一層）來走——同一棵樹，只是拜訪順序不同。同章的姊妹題：Hamiltonian Circuit 跟 Sum of Subsets——這兩題一樣是把問題變成在一棵解空間樹上搜尋，並且提早把死路砍掉。
