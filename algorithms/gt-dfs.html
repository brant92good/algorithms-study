Graph Traversal

Depth-First Search

Explore a graph by diving as deep as you can down one path, then backtracking when you hit a dead end — like exploring a maze always taking the next unexplored door.

What is it?

A graph is a set of nodes (here, circles labelled A–F) joined by edges (lines). A traversal visits every reachable node exactly once. Depth-First Search (DFS) is one way to do that.

The idea: stand on a node, mark it visited, then walk to one of its unvisited neighbours (a node directly joined by an edge) and repeat — always pushing deeper. When the current node has no unvisited neighbours left (a dead end), you backtrack: step back to the most recent node that still has an unexplored neighbour, and continue from there.

DFS remembers the path back using a stack — a list where you only ever add or remove at the top, called LIFO (last-in, first-out, like a stack of plates). You push a node when you arrive, and pop it when it's fully explored. (The same bookkeeping happens automatically if you write DFS as a recursive function — the call stack is the stack.)

Contrast with BFS. Breadth-First Search (Lesson 1 in the finals lessons covers this same knob) uses a queue (FIFO, first-in-first-out) and explores in rings, level by level outward from the start. Swap the queue for a stack and you get DFS: instead of spreading out evenly, you plunge down one branch first. The only structural difference between the two is queue vs. stack.

Tiny example: from A with neighbours B then C, DFS goes A → B → (all of B's branch) → and only comes back to C at the very end. BFS would visit A, then B and C immediately.
