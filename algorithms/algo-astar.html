A* Algorithm

Best-first search that is guaranteed to return the optimal solution — by ranking nodes with f(n) = g(n) + h(n) and using a heuristic that never overestimates.一種保證會回傳最佳解的 best-first search——靠的是用 f(n) = g(n) + h(n) 幫節點排序，再搭配一個絕不高估的 heuristic。

📄 Source: Lee–Tseng–Chang–Tsai, Introduction to the Design and Analysis of Algorithms, Ch. 6 §11 (A\*) and §12 (channel routing).

The idea核心想法

Many optimization problems can be drawn as a search tree: the root is the empty/start state, and each path down to a goal node is one candidate solution. We want the cheapest goal. Plain searches — depth-first, breadth-first, hill-climbing, best-first — all stop at the first goal they reach, and the first goal is not necessarily the cheapest one. A* fixes this.很多最佳化問題都可以畫成一棵搜尋樹：root 是空的／起始狀態，往下走到某個目標節點的每一條路都是一個候選解。我們要的是最便宜的那個目標。一般的搜尋——DFS、BFS、hill-climbing、best-first——通通都是碰到第一個目標就停，但第一個目標不一定最便宜。A* 就是來解決這件事的。

A* is best-first search (keep a list of all open nodes, always expand the most promising one) with a carefully chosen cost estimate. To expand a node means: generate its children (successors), put those children into the open list, and move the node itself out of open (it is now "expanded"). For every node n it computesA* 就是 best-first search（維護一份所有 open 節點的清單，每次展開最有希望的那個），再配上一個精心挑過的成本估計。所謂展開一個節點（expand）就是：把它的子節點（successors）生出來，放進 open 清單，然後把這個節點本身從 open 裡拿掉（它現在算「已展開」了）。對每個節點 n 它會算出

  • g(n) — the actual cost already spent getting from the root to n (cost so far);g(n)——從 root 走到 n 實際已經花掉的成本（目前花費）；
  • h(n) — an estimate of the cost from n onward to a goal;h(n)——從 n 往後走到目標的成本估計值；
  • f(n) = g(n) + h(n) — the estimated cost of the cheapest full solution that passes through n.f(n) = g(n) + h(n)——經過 n 的最便宜完整解，它估計的成本。

The list of open nodes is sorted by f, and each step expands the open node with the smallest f, adding its children back into the list.open 節點清單會照 f 排序，每一步展開 f 最小的那個 open 節點，再把它的子節點放回清單裡。

The admissibility condition. Let h*(n) be the true minimum remaining cost from n to a goal. The heuristic h is admissible when it never overestimates:admissibility 條件。令 h*(n) 是從 n 到目標真正最小的剩餘成本。當 heuristic h 絕不高估時，它就是 admissible 的：

h(n) ≤ h*(n)  ⟹  f(n) = g(n) + h(n) ≤ g(n) + h*(n) = f*(n)

So an admissible h makes f(n) a genuine lower bound on the best total cost of any solution through n. That is the whole trick: because no node is ever rated better than it really is, the first goal node selected for expansion (the one with smallest f) is guaranteed optimal — and A* stops there.所以 admissible 的 h 會讓 f(n) 成為「經過 n 的任何解，其最佳總成本」的貨真價實下界。整個訣竅就在這：因為沒有任何節點會被評得比它實際更好，所以第一個被選中展開的目標節點（也就是 f 最小的那個）就保證是最佳解——A* 就停在那裡。

Checking admissibility on the example below. In the tree below, the goals are the nodes that can finish (h=0), namely I and L. Take node A: it has h(A)=2. The true cheapest remaining cost from A to a goal is via A→D→I, i.e. h*(A) = g(I) − g(A) = 7 − 2 = 5. Since 2 ≤ 5, A is admissible. Every node's h stays at or below its real distance-to-go — that is what "admissible" means here, shown rather than just claimed.用下面的範例驗一下 admissibility。下面那棵樹裡，能收尾（h=0）的節點就是目標，也就是 I 和 L。拿節點 A 來看：它的 h(A)=2。從 A 到目標真正最便宜的剩餘成本是走 A→D→I，也就是 h*(A) = g(I) − g(A) = 7 − 2 = 5。因為 2 ≤ 5，所以 A 是 admissible 的。C 也一樣：h(C)=2，而它到最近目標 L 的真實剩餘成本是 h*(C) = g(L) − g(C) = 8 − 3 = 5，2 ≤ 5 ✓。每個節點的 h 都壓在它真正還要走的距離以下——這就是這裡 admissible 的意思，是做給你看，不是嘴上講講。

Key intuition: A* is best-first search where the ranking is a lower bound (g known cost + h optimistic estimate). The instant a goal node becomes the cheapest node in the list, every other open node has f ≥ that goal's cost, so none of them can lead anywhere cheaper — the goal must be optimal. (Caution: reaching a goal isn't enough; it must be the node actually selected, i.e. the smallest-f node.)關鍵直覺：A* 是用「下界」當排序依據的 best-first search（g 是已知成本 + h 是樂觀估計）。當某個目標節點變成清單裡最便宜的那一刻，其他每個 open 節點的 f ≥ 這個目標的成本，所以它們誰都通不到更便宜的地方——這個目標一定是最佳的。（注意：只是碰到目標還不夠；它必須是真的被選中的那個節點，也就是 f 最小的那個。）

Two goals, why I wins. Both I (f=7) and L (f=8) have h=0, so both are goals — paths R→A→D→I (cost 7) and R→C→F→L (cost 8). A* pops I first because its f is smaller, and stops; L sits in the list with a larger f and is never reached. That is exactly why A* returns the cheaper goal. Tie-breaking: when two nodes share the same f (e.g. C and D both at f=5 below) A* may pop either one first — the order is arbitrary and does not change the final optimal cost.有兩個目標，為什麼 I 勝出。I（f=7）和 L（f=8）的 h 都是 0，所以兩個都是目標——對應路徑 R→A→D→I（成本 7）和 R→C→F→L（成本 8）。A* 因為 I 的 f 比較小，先把 I 取出來就停了；L 帶著比較大的 f 一直留在清單裡，根本沒被碰到。這正是 A* 會回傳比較便宜那個目標的原因。平手怎麼辦：當兩個節點 f 一樣（例如下面 C 和 D 都是 f=5），A* 先取哪個都行——順序是隨意的，不會改變最後的最佳成本。
