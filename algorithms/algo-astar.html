A* Algorithm

Best-first search that is guaranteed to return the optimal solution — by ranking nodes with f(n) = g(n) + h(n) and using a heuristic that never overestimates.

📄 Source: Lee–Tseng–Chang–Tsai, Introduction to the Design and Analysis of Algorithms, Ch. 6 §11 (A\*) and §12 (channel routing).

The idea

Many optimization problems can be drawn as a search tree: the root is the empty/start state, and each path down to a goal node is one candidate solution. We want the cheapest goal. Plain searches — depth-first, breadth-first, hill-climbing, best-first — all stop at the first goal they reach, and the first goal is not necessarily the cheapest one. A* fixes this.

A* is best-first search (keep a list of all open nodes, always expand the most promising one) with a carefully chosen cost estimate. For every node n it computes

  • g(n) — the actual cost already spent getting from the root to n (cost so far);
  • h(n) — an estimate of the cost from n onward to a goal;
  • f(n) = g(n) + h(n) — the estimated cost of the cheapest full solution that passes through n.

The list of open nodes is sorted by f, and each step expands the open node with the smallest f, adding its children back into the list.

The admissibility condition. Let h*(n) be the true minimum remaining cost from n to a goal. The heuristic h is admissible when it never overestimates:

h(n) ≤ h*(n)  ⟹  f(n) = g(n) + h(n) ≤ g(n) + h*(n) = f*(n)

So an admissible h makes f(n) a genuine lower bound on the best total cost of any solution through n. That is the whole trick: because no node is ever rated better than it really is, the first goal node selected for expansion (the one with smallest f) is guaranteed optimal — and A* stops there.

Key intuition: A* is best-first search where the ranking is a lower bound (g known cost + h optimistic estimate). The instant a goal node becomes the cheapest node in the list, every other open node has f ≥ that goal's cost, so none of them can lead anywhere cheaper — the goal must be optimal. (Caution: reaching a goal isn't enough; it must be the node actually selected, i.e. the smallest-f node.)

Problems it solves (in the slides)

  • A* on a graph (Fig. 6-36) — find the least-cost path from a source to a goal node, expanding nodes by f = g + h until the goal is popped.
  • Channel routing — wire all the nets in a VLSI channel using the fewest horizontal tracks, with g = tracks used so far and h = maximal local density (a lower bound on tracks still needed).
