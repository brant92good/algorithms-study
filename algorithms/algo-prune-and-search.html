Prune-and-Search

A design strategy: each round throw away a guaranteed fraction of the input, then recurse on what's left. Because the work shrinks geometrically, the first round dominates — and many problems that look like O(n log n) collapse to O(n).一種設計策略：每一輪都保證丟掉一定比例的輸入，然後對剩下的遞迴。因為工作量是幾何級數縮小，第一輪就佔了大部分——很多看起來像 O(n log n) 的問題，就這樣塌成 O(n)。

📄 Source: A06 · Prune-and-Search §1 (general method) → §2–5 (selection), §6 (linear programming), §7 (1-center).

The idea核心想法

Think about how binary search works. To find a target in a sorted array, you look at the middle, decide the target is in the left half or the right half, and discard the other half forever. Each step prunes half the data and searches what remains. That is the whole strategy, abstracted:想想 binary search 怎麼跑。要在排好序的陣列裡找目標，你看正中間，判斷目標在左半還右半，然後把另一半永遠丟掉。每一步都修剪掉一半資料，再搜尋剩下的。把它抽象出來，就是整個策略：

  • Prune.修剪。 In each iteration, do some O(nᵏ) work — where k ≥ 1 is a fixed constant (for all three example problems on this page k = 1, i.e. each round does linear O(n) work) — that lets you guarantee a fixed fraction f (with 0 < f < 1) of the current input can never contain the answer. Throw it away.每一輪花 O(nᵏ) 的工——這裡 k ≥ 1 是固定常數（本頁三個範例題目都是 k = 1，也就是每輪只做線性的 O(n) 工）——換來一個保證：目前輸入裡有固定比例 f（0 < f < 1）絕不可能藏著答案。把它丟掉。
  • Search.搜尋。 Recurse on the surviving (1−f)·n elements with the same algorithm.對活下來的 (1−f)·n 個元素，用同一個演算法遞迴下去。
  • Stop.停止。 Once the survivors shrink to a constant size, solve directly in O(1).當剩下的縮到常數大小，就直接 O(1) 解掉。

The key requirement — and the hard part of every prune-and-search algorithm — is choosing the pruning step so that the fraction f is guaranteed every round, on any input. A pruning rule that only sometimes discards a fraction (e.g. always pivoting on the first element) can degrade to removing one element at a time, blowing the bound to O(n2).關鍵要求——也是每個 prune-and-search 演算法最難的地方——就是要把修剪步驟設計成讓比例 f 每一輪、對任何輸入都保證砍得掉。如果某個修剪規則只是有時候砍掉一塊（例如總是拿第一個元素當 pivot），就可能爛成一次只砍一個，把界限炸成 O(n2)。

So how do you guarantee it? The trick (used by selection's median-of-medians) is to spend a little O(n) work picking a pivot that can't be near the extremes. Break the n items into groups of 5, find each group's median, then take the median of those medians as the pivot. By construction, roughly half the group-medians are smaller than it, and each of those sits above 2 more items in its own group — so the pivot is provably larger than about 3/10 of all items, and by the same argument smaller than about another 3/10. Whichever side the answer falls on, the other ≈3/10 can be thrown away no matter how adversarial the input is. That guaranteed fraction is exactly the f the analysis above needs; the rounding in real algorithms lands it near f ≈ ¼.那要怎麼「保證」？訣竅（selection 的 median-of-medians 就是這樣）是花一點 O(n) 的工，挑一個不可能太靠近極端的 pivot。把 n 個元素每 5 個一組，先算每組的中位數，再取「這些中位數的中位數」當 pivot。這樣一來，大約一半的組中位數比它小，而那些組中位數在自己組裡又各壓著另外 2 個更小的元素——所以可以證明這個 pivot 比大約 3/10 的元素都大，同樣的論證也讓它比另外大約 3/10 的元素都小。不管答案落在哪一邊，另一邊的 ≈3/10 都能砍掉，而且不管輸入多刁鑽都成立。這個保證砍掉的比例，正是上面分析需要的 f；實際演算法取整之後大約落在 f ≈ ¼ 附近。

Why it's linear (the headline result)為什麼是線性（重頭戲結論）

Let T(n) be the running time and suppose one iteration costs O(nᵏ). Then:令 T(n) 是執行時間，假設一輪花 O(nᵏ)。那麼：

T(n) = T((1−f)·n) + O(nᵏ)

Why a geometric series? The first round costs c·nᵏ. The next round runs on input (1−f)·n, so it costs c·((1−f)·n)ᵏ = (1−f)ᵏ · c·nᵏ — that is, (1−f)ᵏ times the previous term. Every following round shrinks the input by the same factor, so each successive cost is again (1−f)ᵏ times the one before. That fixed ratio is exactly what makes it a geometric series, with ratio (1−f)ᵏ < 1:為什麼是幾何級數？第一輪花 c·nᵏ。下一輪的輸入是 (1−f)·n，所以它花 c·((1−f)·n)ᵏ = (1−f)ᵏ · c·nᵏ——也就是前一項的 (1−f)ᵏ 倍。再往下每一輪輸入都縮同樣的倍數，所以每一項的成本又是前一項的 (1−f)ᵏ 倍。就是這個固定的比例，讓它變成 ratio 為 (1−f)ᵏ < 1 的幾何級數：

c·nᵏ · ( 1 + (1−f)ᵏ + (1−f)2ᵏ + … ) = c·nᵏ · (a constant)

That "constant" is just the sum of an infinite geometric series with ratio (1−f)ᵏ, which has the closed form 1 / (1 − (1−f)ᵏ). For the worked example below (f = ¼, k = 1) the ratio is ¾, so the constant is 1 / (1 − ¾) = 4 — meaning the entire multi-round process does only about 4·c·n work, just four times the first round.這個「常數」其實就是一個 ratio 為 (1−f)ᵏ 的無窮幾何級數的和，它有 closed form 1 / (1 − (1−f)ᵏ)。以下面的範例（f = ¼、k = 1）來說，ratio 是 ¾，所以這個常數就是 1 / (1 − ¾) = 4——也就是說整個多輪過程總共大約只做 4·c·n 的工，差不多就是第一輪的四倍而已。

The series converges, so the sum is just a constant times the first term:這個級數會收斂，所以總和就只是常數乘上第一項：

T(n) = O(nᵏ)

The punchline:結論一句話： the whole multi-round process costs the same order as a single round. The first iteration dominates — every later round works on an exponentially smaller input, so all of them together add only a constant factor. This is why selection, 2-variable LP, and the 1-center problem all come out O(n) even though each does O(n) work per round.整個多輪的過程，和單獨一輪是同一個 order。第一輪就佔了大部分——後面每一輪處理的輸入都指數級變小，全部加起來也只多一個常數倍。這就是為什麼 selection、2-variable LP、1-center 即使每輪都做 O(n) 的工，最後都還是 O(n)。

See the shrinkage看它怎麼縮

This is the engine of the whole strategy, made concrete. Take n = 100, a pruning fraction f = ¼ (the rate the real algorithms achieve), and k = 1 (linear O(n) work per round, like all three examples). With k = 1 the general ratio (1−f)ᵏ becomes simply (1−¼)1 = ¾ — so each round's work is ¾ of the last. Each bar is the input size at the start of a round; gold is the work that round does, dim is what gets pruned away. Watch the input — and so the work — collapse geometrically.這就是整個策略的引擎，具體跑給你看。拿 n = 100、修剪比例 f = ¼（真實演算法做得到的比例），再加上 k = 1（每輪做線性的 O(n) 工，跟三個範例一樣）。當 k = 1，通用的 ratio (1−f)ᵏ 就簡化成 (1−¼)1 = ¾——所以每一輪的工是上一輪的 ¾。每一根長條是某一輪開頭的輸入大小；金色是那輪做的工，變暗的是被砍掉的。看著輸入——連帶工作量——一路幾何級數塌下去。

Problems it solves (in the slides)它能解的問題（投影片裡的）

All three pick a clever pivot/median so a constant fraction (≈¼) is provably pruned each round, turning an O(n log n)-or-worse problem into linear time:這三題都靠巧妙挑 pivot／median，讓每輪都能證明砍掉固定比例（≈¼），把原本 O(n log n) 甚至更慢的問題變成線性時間：

  • 🎯 Selection — k-th smallestSelection — 第 k 小 — given n unsorted numbers and a rank k, return the value that would sit at position k if sorted (the median when k = ⌈n/2⌉). Solved in O(n) via the median-of-medians pivot.——給 n 個亂序數字和名次 k，回傳排好序後站在第 k 位的值（k = ⌈n/2⌉ 時就是中位數）。用 median-of-medians 當 pivot，在 O(n) 解出來。
  • 📐 2-variable linear programming2-variable linear programming — minimize y subject to n half-plane constraints y ≥ aix + bi; find the lowest point of their upper envelope in O(n) by pairing constraints and testing the median intersection.——在 n 條 half-plane 限制 y ≥ aix + bi 下最小化 y；把限制兩兩配對、測試交點的中位數，在 O(n) 找出它們 upper envelope 的最低點。
  • ⭕ 1-center — smallest enclosing circle1-center — 最小包覆圓 — given n planar points, find the smallest circle that covers them all, in O(n) by pairing points and pruning on median bisector slopes.——給 n 個平面上的點，找出能蓋住全部的最小圓；把點兩兩配對，靠中位 bisector 斜率來修剪，在 O(n) 解出來。

Contrast it with divide-and-conquer.跟 divide-and-conquer 比一比。 Divide-and-conquer (mergesort, etc.) keeps both halves and recombines their results — that recombination is why it lands at O(n log n). Prune-and-search keeps only one piece and never recombines, so the work decays geometrically to O(n). The three problems above — selection, 2-variable LP, and 1-center — are the canonical instances.divide-and-conquer（像 mergesort）會把兩半都留下，再把結果合回去——就是這個合併讓它落在 O(n log n)。Prune-and-search 只留一塊、從不合併，所以工作量幾何級數衰減到 O(n)。上面那三題——selection、2-variable LP、1-center——就是經典範例。
