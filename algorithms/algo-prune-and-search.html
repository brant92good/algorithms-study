Prune-and-Search

A design strategy: each round throw away a guaranteed fraction of the input, then recurse on what's left. Because the work shrinks geometrically, the first round dominates — and many problems that look like O(n log n) collapse to O(n).一種設計策略：每一輪都保證丟掉一定比例的輸入，然後對剩下的遞迴。因為工作量是幾何級數縮小，第一輪就佔了大部分——很多看起來像 O(n log n) 的問題，就這樣塌成 O(n)。

📄 Source: A06 · Prune-and-Search §1 (general method) → §2–5 (selection), §6 (linear programming), §7 (1-center).

The idea核心想法

Think about how binary search works. To find a target in a sorted array, you look at the middle, decide the target is in the left half or the right half, and discard the other half forever. Each step prunes half the data and searches what remains. That is the whole strategy, abstracted:想想 binary search 怎麼跑。要在排好序的陣列裡找目標，你看正中間，判斷目標在左半還右半，然後把另一半永遠丟掉。每一步都修剪掉一半資料，再搜尋剩下的。把它抽象出來，就是整個策略：

  • Prune.修剪。 In each iteration, do some O(nᵏ) work that lets you guarantee a fixed fraction f (with 0 < f < 1) of the current input can never contain the answer. Throw it away.每一輪花 O(nᵏ) 的工，換來一個保證：目前輸入裡有固定比例 f（0 < f < 1）絕不可能藏著答案。把它丟掉。
  • Search.搜尋。 Recurse on the surviving (1−f)·n elements with the same algorithm.對活下來的 (1−f)·n 個元素，用同一個演算法遞迴下去。
  • Stop.停止。 Once the survivors shrink to a constant size, solve directly in O(1).當剩下的縮到常數大小，就直接 O(1) 解掉。

The key requirement — and the hard part of every prune-and-search algorithm — is choosing the pruning step so that the fraction f is guaranteed every round, on any input. A pruning rule that only sometimes discards a fraction (e.g. always pivoting on the first element) can degrade to removing one element at a time, blowing the bound to O(n²).關鍵要求——也是每個 prune-and-search 演算法最難的地方——就是要把修剪步驟設計成讓比例 f 每一輪、對任何輸入都保證砍得掉。如果某個修剪規則只是有時候砍掉一塊（例如總是拿第一個元素當 pivot），就可能爛成一次只砍一個，把界限炸成 O(n²)。

Why it's linear (the headline result)為什麼是線性（重頭戲結論）

Let T(n) be the running time and suppose one iteration costs O(nᵏ). Then:令 T(n) 是執行時間，假設一輪花 O(nᵏ)。那麼：

T(n) = T((1−f)·n) + O(nᵏ)

Unrolling, the per-round cost forms a geometric series with ratio (1−f)ᵏ < 1:把它展開，每輪的成本構成一個 ratio 為 (1−f)ᵏ < 1 的幾何級數：

c·nᵏ · ( 1 + (1−f)ᵏ + (1−f)²ᵏ + … ) = c·nᵏ · (a constant)

The series converges, so the sum is just a constant times the first term:這個級數會收斂，所以總和就只是常數乘上第一項：

T(n) = O(nᵏ)

The punchline:結論一句話： the whole multi-round process costs the same order as a single round. The first iteration dominates — every later round works on an exponentially smaller input, so all of them together add only a constant factor. This is why selection, 2-variable LP, and the 1-center problem all come out O(n) even though each does O(n) work per round.整個多輪的過程，和單獨一輪是同一個 order。第一輪就佔了大部分——後面每一輪處理的輸入都指數級變小，全部加起來也只多一個常數倍。這就是為什麼 selection、2-variable LP、1-center 即使每輪都做 O(n) 的工，最後都還是 O(n)。

Problems it solves (in the slides)它能解的問題（投影片裡的）

All three pick a clever pivot/median so a constant fraction (≈¼) is provably pruned each round, turning an O(n log n)-or-worse problem into linear time:這三題都靠巧妙挑 pivot／median，讓每輪都能證明砍掉固定比例（≈¼），把原本 O(n log n) 甚至更慢的問題變成線性時間：

  • 🎯 Selection — k-th smallestSelection — 第 k 小 — given n unsorted numbers and a rank k, return the value that would sit at position k if sorted (the median when k = ⌈n/2⌉). Solved in O(n) via the median-of-medians pivot.——給 n 個亂序數字和名次 k，回傳排好序後站在第 k 位的值（k = ⌈n/2⌉ 時就是中位數）。用 median-of-medians 當 pivot，在 O(n) 解出來。
  • 📐 2-variable linear programming2-variable linear programming — minimize y subject to n half-plane constraints y ≥ aᵢx + bᵢ; find the lowest point of their upper envelope in O(n) by pairing constraints and testing the median intersection.——在 n 條 half-plane 限制 y ≥ aᵢx + bᵢ 下最小化 y；把限制兩兩配對、測試交點的中位數，在 O(n) 找出它們 upper envelope 的最低點。
  • ⭕ 1-center — smallest enclosing circle1-center — 最小包覆圓 — given n planar points, find the smallest circle that covers them all, in O(n) by pairing points and pruning on median bisector slopes.——給 n 個平面上的點，找出能蓋住全部的最小圓；把點兩兩配對，靠中位 bisector 斜率來修剪，在 O(n) 解出來。

Contrast it with divide-and-conquer.跟 divide-and-conquer 比一比。 Divide-and-conquer (mergesort, etc.) keeps both halves and recombines their results — that recombination is why it lands at O(n log n). Prune-and-search keeps only one piece and never recombines, so the work decays geometrically to O(n). The three problems above — selection, 2-variable LP, and 1-center — are the canonical instances.divide-and-conquer（像 mergesort）會把兩半都留下，再把結果合回去——就是這個合併讓它落在 O(n log n)。Prune-and-search 只留一塊、從不合併，所以工作量幾何級數衰減到 O(n)。上面那三題——selection、2-variable LP、1-center——就是經典範例。
