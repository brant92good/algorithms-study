Graph Traversal

Topological Sort

Order tasks so every job comes after the things it depends on — using Kahn's algorithm and the "in-degree 0" rule.

What is it?

A graph is a set of nodes (dots) joined by edges (lines). When the edges have a direction — an arrow from u to v — we call it a directed graph. If you can never follow the arrows in a loop back to where you started, it has no cycles: it's a DAG (Directed Acyclic Graph).

Think of course prerequisites: an arrow A→C means "you must finish A before you can take C". A topological order is a single line-up of all the courses such that every arrow points forward — you never take a course before something it requires. The same idea drives build systems (compile a library before the program that uses it) and task scheduling.

The key quantity is a node's in-degree: how many arrows point into it — i.e. how many unfinished prerequisites it still has. Kahn's algorithm: any node with in-degree 0 has no remaining prerequisites, so it's safe to do now. Output it, then "remove" it by decrementing the in-degree of every node it pointed to. That may free up new in-degree-0 nodes. Repeat until every node is placed.

Tiny example: with arrows A→C and B→C, both A and B start at in-degree 0 (do either first); only after both are done does C drop to 0.

Watch it run

DAG below — arrows are prerequisites (A→C means A before C). The badge on each node is its current in-degree. Each step we pick a node showing 0, mark it green (done), append it to the output order, and decrease its successors' badges.

Time: O(V + E) Space: O(V) Needs: a DAG (no cycles)
