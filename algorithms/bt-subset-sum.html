Backtracking

Subset Sum

Pick some numbers from a set so they add up to a target — by trying choices one at a time and backing out of dead ends.

What is it?

You have a set of numbers and a target total. The question: is there a subset (some of the numbers, in any combination) that adds up exactly to the target? A subset just means "pick any of them — each number is either in or out."

The core idea is backtracking: walk through the numbers one at a time and, for each, make a binary choice — include it or leave it out. Every choice splits the search into two branches, so all the possibilities form a decision tree: the root is "nothing decided yet," each level decides one number, and each leaf at the bottom is one complete subset. We explore the tree depth-first (go all the way down one branch, then back up and try the next).

The smart part is pruning. We keep a running sum of the numbers we've included so far. The moment that running sum exceeds the target, this branch is hopeless — adding more numbers only makes it bigger — so we abandon it immediately instead of exploring its children. That cut-off is what makes backtracking faster than blindly listing every subset.

Tiny example: with the set {3, 5, 6, 7} and target 13, including 6 and 7 gives 6 + 7 = 13. But the branch that includes 3, 5, and 6 reaches 3 + 5 + 6 = 14, which is already over 13 — so we prune it.
