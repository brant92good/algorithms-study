Fibonacci & Memoization

The same small problems keep coming back — so solve each one once and write the answer down. That single idea turns an exponential algorithm into a linear one.

What is it?

The Fibonacci numbers are 0, 1, 1, 2, 3, 5, 8, … — each one is the sum of the two before it: fib(n) = fib(n−1) + fib(n−2), with the base cases fib(0)=0 and fib(1)=1.

The obvious way to compute fib(5) is to recurse: a function that calls itself. To get fib(5) you ask for fib(4) and fib(3); each of those asks for two smaller ones, and so on down to the base cases. The problem: those smaller calls overlap. Computing fib(5) recomputes fib(3) twice and fib(2) three times — entirely from scratch each time. These repeated calls are called overlapping subproblems, and the wasted work doubles roughly every level: that is exponential time, O(2ⁿ).

Memoization is the fix. "Memo" as in a note to yourself: keep a small lookup table (a cache) of answers you've already computed. Before doing the work for fib(k), check the cache — if it's there, return it instantly in O(1) and skip the whole subtree. Each subproblem is now solved exactly once, so the total work is just one entry per value: linear time, O(n).
