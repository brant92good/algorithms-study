Multi-Stage Graph Search — Branch & BoundMulti-Stage Graph 搜尋 — Branch & Bound

Given a layered graph with one source and one sink, find the cheapest path from source to sink — by searching the paths with a lower bound and pruning, not by a DP recurrence.給你一張分層的圖，只有一個起點和一個終點，找出從起點到終點最便宜的一條路——這裡是用下界一邊搜尋一邊剪枝來解，而不是用 DP 遞迴。

The problem問題是什麼

A multi-stage graph (also called a layered graph) has its vertices grouped into ordered stages. Every edge goes from one stage to the next — never sideways, never backward. There's a single source in the first stage and a single sink in the last. Each edge carries a weight (a cost). A path from source to sink picks exactly one edge out of each stage it touches; its cost is the sum of the weights along it.Multi-stage graph（也叫分層圖）會把節點分成一個個有先後順序的 stage。每條邊都是從某一層連到下一層——不會往旁邊、也不會往回走。第一層只有一個起點（source），最後一層只有一個終點（sink）。每條邊都有一個權重（cost）。一條從起點到終點的 path，就是在每一層各挑一條邊接下去，成本就是沿路權重的總和。

The task:任務： find the path from source to sink with the smallest total cost. This is the example the slides use to introduce branch-and-bound — a tiny graph where you can watch the bound do its job.找出從起點到終點總成本最小的那條 path。這正是投影片拿來介紹 branch-and-bound 的例子——圖很小，剛好可以看清楚下界是怎麼運作的。

The concrete instance (Fig. 6-17).實際例子（Fig. 6-17）。 Four stages. Stage 0 is just the source V₀. Stage 1 has V₁,₁ V₁,₂ V₁,₃. Stage 2 has V₂,₁ V₂,₂ V₂,₃. Stage 3 is just the sink V₃. The twelve edge weights:總共四層。Stage 0 只有起點 V₀；Stage 1 有 V₁,₁ V₁,₂ V₁,₃；Stage 2 有 V₂,₁ V₂,₂ V₂,₃；Stage 3 只有終點 V₃。十二條邊的權重如下：

V₀ → V₁,₁ = 1 V₁,₁ → V₂,₁ = 5 V₂,₁ → V₃ = 4 V₀ → V₁,₂ = 3 V₁,₁ → V₂,₃ = 3 V₂,₂ → V₃ = 1 V₀ → V₁,₃ = 2 V₁,₂ → V₂,₁ = 4 V₂,₃ → V₃ = 1 V₁,₂ → V₂,₂ = 3 V₁,₃ → V₂,₂ = 2 V₁,₃ → V₂,₃ = 7

The cheapest source-to-sink path costs 5 — for example V₀→V₁,₁→V₂,₃→V₃ = 1+3+1, or equally V₀→V₁,₃→V₂,₂→V₃ = 2+2+1. The interesting part is how branch-and-bound finds it while skipping branches that clearly can't win.最便宜的起點到終點 path 成本是 5——例如 V₀→V₁,₁→V₂,₃→V₃ = 1+3+1，或同樣是 5 的 V₀→V₁,₃→V₂,₂→V₃ = 2+2+1。有趣的是 branch-and-bound 怎麼找到它，而且能跳過那些明顯贏不了的分支。

The key idea — best-first with a bound.關鍵想法——帶著下界的 best-first。 Each partial path so far has a cost (the weights you've already committed to). That cost is a valid lower bound: extending the path can only add more. Branch-and-bound keeps all the open partial paths and always expands the one with the smallest bound (best-first). When a full source-to-sink path is completed, its cost becomes the best-so-far; any open node whose bound is ≥ best-so-far gets pruned — it can't possibly do better. The x > k notation: on a pruned leaf, x means "the eventual cost of any full path that goes through this node", and the printed number k is that node's own bound. Since x ≥ k ≥ best-so-far, the whole branch is hopeless. So x>6 on a node simply reads "every completion here costs at least 6, which already loses to our 5". Pruning ties is safe too: we only need one cheapest path, and we already hold one — so discarding a node whose bound merely ties best-so-far loses nothing.每條走到一半的 path 都有一個目前成本（你已經選定的那些權重）。這個成本就是一個有效的下界：再接下去只會更貴、不會更便宜。Branch-and-bound 把所有開著的半成品 path 都留著，每次都挑下界最小的那個來展開（best-first）。一旦有一條完整的起點到終點 path 走完，它的成本就成為目前最佳（best-so-far）；任何下界 ≥ best-so-far 的節點就直接剪掉——它再怎麼走都不可能更好。那個 x > k 標記是什麼意思：在被剪掉的葉子上，x 代表「任何經過這個節點的完整 path 最後會花的成本」，印出來的數字 k 就是這個節點自己的下界。因為 x ≥ k ≥ best-so-far，整條分支都沒救了。所以節點上的 x>6 就是在講「從這裡走完至少要 6，已經輸給我們手上的 5 了」。就算下界剛好打平也照剪沒問題：我們只需要一條最便宜的 path，而且手上已經有一條了——所以把下界只是跟 best-so-far 打平的節點丟掉，不會少掉任何東西。

Same graph, different method — the DP version.同一張圖，不同解法——DP 版本。 The very same layered-graph idea is solved by dynamic programming in Ch7 — see Multistage Graph (DP). The difference in one line: branch-and-bound searches the paths with a bound, pruning branches that can't win; dynamic programming fills in each subproblem's optimum (the cheapest cost from each vertex to the sink) and reads the answer off the table. Same problem shape, two complementary mindsets.完全一樣的分層圖想法，在第七章是用 dynamic programming 解的——見 Multistage Graph (DP)。一句話講差別：branch-and-bound 是帶著下界搜尋各條 path，把贏不了的分支砍掉；dynamic programming 則是把每個子問題的最佳解填出來（每個節點到終點的最便宜成本），再從表上讀答案。題型一樣，但是兩種互補的思路。
