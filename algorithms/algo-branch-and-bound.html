Branch-and-Bound

Search a tree of partial solutions best-first, but for an optimization problem — and use a lower bound at every node to throw away whole branches that cannot possibly beat the best solution you've already found.用 best-first 的方式搜尋一棵部分解的樹，但這次是為了解最佳化（optimization）問題——並在每個節點用一個下界（lower bound），把那些絕不可能贏過你手上最佳解的整條分支直接丟掉。

📄 Source:出處： A05 · The Searching Strategies §7 (branch-and-bound), §8–10 (the three problems it solves).

The idea核心概念

Plain tree searches — depth-first, breadth-first, hill climbing, best-first — all stop at the first goal node they reach. That's fine for a decision problem ("does a solution exist?"), but it's useless for an optimization problem ("find the cheapest solution"): the first goal found is almost never the best one. Branch-and-bound is the fix.一般的樹搜尋——depth-first、breadth-first、hill climbing、best-first——都是碰到第一個 goal 節點就停。這對 decision 問題（「有沒有解？」）沒問題，但對 optimization 問題（「找最便宜的解」）就沒用了：第一個找到的 goal 幾乎不會是最好的。Branch-and-bound 就是來補這個洞。

It has two ingredients, hence the name:它有兩個成分，名字就是這麼來的：

  • Branch — split the solution space into subsets, each a subtree. (For a tour: "tours that use arc 4→6" vs "tours that don't." For knapsack: "take item i" vs "leave it.") This is ordinary tree search.把解空間切成幾塊，每塊是一棵子樹。（以 tour 為例：「用 arc 4→6 的 tour」對「不用的 tour」；以 knapsack 為例：「拿物品 i」對「不拿」。）這部分就是普通的樹搜尋。
  • Bound — for every node, compute a lower bound: a number that is guaranteed to be ≤ the cost of any complete solution living in that subtree. It's an optimistic estimate — the best the subtree could conceivably do.對每個節點算一個下界：一個保證 ≤ 那棵子樹裡任何完整解成本的數字。它是個樂觀的估計——這棵子樹最理想能做到的程度。

You expand nodes best-first (always open the live node with the smallest lower bound, using the Step 1–4 best-first scheme). When several live nodes are tied for smallest bound, you may break the tie by diving toward a complete solution first — that gets you an incumbent (a real best) early, which is what powers the pruning. You keep the best complete solution found so far — call its cost best. Then the whole payoff is one rule:你用 best-first 展開節點（永遠先打開下界最小的那個存活節點，照 Step 1–4 best-first 流程）。如果有好幾個存活節點下界一樣小（平手），你可以先挑一條往下鑽到完整解——這樣能早點拿到一個 incumbent（真正的 best），而剪枝靠的就是它。你會一直記著目前找到的最佳完整解——把它的成本叫做 best。整個威力就濃縮成一條規則：

The prune rule.剪枝規則。 If a node's lower bound ≥ best, kill the entire subtree. Reason: every solution down there costs at least the bound, which is already no better than a solution you already have in hand — so nothing in that subtree can win. No need to look.如果一個節點的下界 ≥ best，就把整棵子樹砍掉。理由：那裡面每個解的成本至少都是這個下界，而這已經不比你手上已經有的解更好——所以那棵子樹裡不可能有贏家，根本不用看。

Why a lower bound (and why ≥, not >)? Because we're minimizing. The bound is a promise: "you can't do better than this here." If that promise is already as bad as your current champion, the branch is hopeless. (Maximization problems flip the signs — the slides actually turn the knapsack into a minimization by minimizing negative profit, so the same lower-bound machinery applies.)為什麼是下界（又為什麼是 ≥ 而不是 >）？因為我們在最小化。下界等於一個保證：「這裡你不可能比它更好。」如果這個保證已經跟你現在的冠軍一樣爛，這條分支就沒希望了。（最大化問題只要翻正負號——投影片其實就是把 knapsack 改成最小化負利潤，這樣同一套下界機制照樣能用。）

A common way to get the lower bound is a relaxation: solve an easier version whose answer can only be optimistic. The mechanism depends on the problem, so the formula is worked out on each problem's own page:拿下界常見的招數是 relaxation（鬆弛）：解一個比較簡單、答案只會偏樂觀的版本。用哪種機制要看是哪個問題，所以實際公式都放在各自的問題頁面裡細講：

  • For the multi-stage graph below, the relaxation is the simplest one: "cost so far + the cheapest edge in each remaining stage." We work it out fully right here.下面這個 multi-stage graph 用的是最簡單的版本：「目前成本 ＋ 之後每個 stage 最便宜的邊」。這個我們等下會完整算給你看。
  • For assignment and TSP it's the reduced cost matrix (subtract a constant from each row/column until each has a zero; the total subtracted can't be avoided, so it's a lower bound) — worked through on the assignment and TSP pages.對 assignment 和 TSP 來說是 reduced cost matrix（每列每行各減一個常數直到都有 0；減掉的總量躲不掉，所以是個下界）——詳細算法在 assignment 和 TSP 頁面。
  • For knapsack it's the fractional knapsack (allow taking part of an item), solved greedily — see the knapsack page.對 knapsack 來說是 fractional knapsack（允許只拿一件物品的一部分），用貪婪解——看 knapsack 頁面。

Complexity:複雜度： worst case still exponential — these are NP-complete problems (TSP, knapsack) and B&B does not change that. Its value is pruning: a good bound chops away enormous subtrees, so in practice it explores far fewer nodes than brute force. A tighter bound prunes more.最壞情況還是指數級——這些是 NP-complete 問題（TSP、knapsack），B&B 改變不了這點。它的價值在剪枝：一個好的 bound 能砍掉超大片的子樹，所以實務上它探索的節點數遠少於硬爆搜。bound 越緊，剪掉的越多。

How we get the lower bound here. The bound is a relaxation: "cost committed so far + the cheapest edge in each stage still ahead." That's optimistic (no real path can spend less than the cheapest edge per stage), so it's a legitimate lower bound. The cheapest edge in stage 0 is 1, in stage 1 is 1, in stage 2 is 1. Worked examples you can check against the graph:這裡的下界怎麼來的。這個 bound 用的是 relaxation（鬆弛）：「目前已付出的成本 ＋ 之後每一個還沒走的 stage 裡最便宜的那條邊」。這很樂觀（沒有任何真實路徑能花得比「每個 stage 最便宜的邊」還少），所以它是合法的下界。stage 0 最便宜的邊是 1、stage 1 是 1、stage 2 是 1。你可以對著上面的圖驗算：

  • bound(e = V2,1) = cost(V0→V1,1→V2,1)=1+6=7, + 1 = 8. (Costly first edge → loose-but-honest bound, here ≥ best so it gets pruned.)bound(e = V2,1) ＝ cost(V0→V1,1→V2,1)=1+6=7，＋ 1 ＝ 8。（第一條邊就很貴 → 下界雖鬆但誠實，這裡 ≥ best 就被剪掉。）

The true cheapest path turns out to cost 5 via V0→V1,1→V2,3→V3 (1+1+3). We expand best-first (smallest bound first) and prune any node whose bound is already ≥ the best complete path found so far. Notice the bounds are concrete integers, not "> something" — every one is a single number you can recompute. Step through.真正最便宜的路徑成本是 5，走 V0→V1,1→V2,3→V3（1+1+3）。我們用 best-first 展開（下界最小的先開），並把任何下界已經 ≥ 目前找到的最佳完整路徑的節點剪掉。注意這些下界都是確定的整數，不是「> 某個值」——每一個都是你能自己重算出來的單一數字。一步步看。

Sibling strategy:姊妹策略： A\* is branch-and-bound's close cousin — best-first search with f(n) = g(n) + h(n). Here g(n) = cost actually paid to reach node n (the "cost so far"), h*(n) = the true cheapest remaining cost from n to a goal, and h(n) = an optimistic estimate of that remaining cost. When the heuristic is admissible — meaning h(n) ≤ h*(n), it never over-estimates — then f(n) is exactly a lower bound on any complete solution through n, so the first goal selected for expansion is guaranteed optimal. (In our multi-stage example, g = cost committed, and h = "sum of the cheapest edge per remaining stage" — that's the relaxation we used.)是 branch-and-bound 的近親——它是用 f(n) = g(n) + h(n) 的 best-first search。這裡 g(n) ＝ 真正走到節點 n 已經付的成本（也就是「cost so far」）、h*(n) ＝ 從 n 到 goal 真正最便宜的剩餘成本、h(n) ＝ 對這段剩餘成本的樂觀估計。當這個 heuristic 是 admissible（也就是 h(n) ≤ h*(n)，永遠不會高估）時，f(n) 剛好就是「經過 n 的任何完整解」的下界，所以第一個被挑出來展開的 goal 保證是最佳解。（在我們的 multi-stage 例子裡，g ＝ 已付出的成本，h ＝「每個剩餘 stage 最便宜的邊加起來」——就是我們用的那個 relaxation。）
