Longest Common SubsequenceLongest Common Subsequence

Given two strings, find the longest sequence of letters that hides inside both — in order, but not necessarily side by side.給你兩個字串，找出同時藏在兩邊的最長字母序列——順序要對，但不用連在一起。

📄 Source: Alg07 · Ch 7 §11 (Longest Common Subsequence), slides 28–29.

🧠 Solved by: Dynamic Programming

The problem這題在問什麼

Start with the one word you must understand first: subsequence. A subsequence of a string is what is left after you delete zero or more letters while keeping the survivors in their original order. You may skip letters, but you may never reorder them. From bacad you can pull out bcd, aca, or acad — but not dab, because that reshuffles the order.先搞懂一個關鍵字：subsequence（子序列）。簡單講，就是把一個字串刪掉幾個字母（刪幾個都行，包括不刪），剩下的字母維持原本的順序排好。你可以跳過字母，但絕對不能調換順序。從 bacad 裡你可以挑出 bcd、aca 或 acad，但不能是 dab，因為那把順序打亂了。

Do not confuse this with a substring. A substring must be contiguous — a solid uninterrupted run, like aca taken from positions 2–4 of bacad. A subsequence is skip-but-keep-order, so it is strictly more permissive: every substring is a subsequence, but not the reverse. acad is a subsequence of bacad (drop the b), yet it is not a substring of it (its letters aren't consecutive).別跟 substring（子字串）搞混。substring 一定要連在一起，是一段不間斷的字，像 bacad 第 2–4 位的 aca。subsequence 則是可以跳、但要保持順序，所以條件比較寬鬆：每個 substring 都是 subsequence，反過來不一定。acad 是 bacad 的 subsequence（把 b 拿掉），但不是它的 substring（這幾個字母沒有連在一起）。

A common subsequence of two strings A and B is a string that is a subsequence of both at once. The Longest Common Subsequence (LCS) problem asks:兩個字串 A 和 B 的 common subsequence（共同子序列），就是同時是兩邊 subsequence 的字串。Longest Common Subsequence (LCS) 這題要問的是：

Given: two strings A = a₁a₂…a_m and B = b₁b₂…b_n.
Find: the longest string that is a subsequence of A and a subsequence of B (return both the string and its length).給定：兩個字串 A = a₁a₂…a_m 和 B = b₁b₂…b_n。
求：同時是 A 和 B 的 subsequence、而且最長的那個字串（要回傳字串本身和它的長度）。

A tiny concrete instance. Take A = bacad and B = accbadcb. Both contain the letters a, c, a, d in that order (skip the rest of each string), and you cannot do better — so the answer is acad, length 4. That is the exact instance we solve below.來個小例子。取 A = bacad、B = accbadcb。兩邊都依序含有 a, c, a, d 這幾個字母（其他跳過就好），而且沒辦法更長了——所以答案是 acad，長度 4。下面就用這個例子一步步解。

Watch it solved看它怎麼解

The algorithm fills a table L where L[i,j] = the LCS length of the first i letters of A and the first j letters of B. Row 0 and column 0 are all 0 (an empty prefix shares nothing). For each interior cell it compares the current letters aᵢ and bⱼ: on a match it takes the diagonal neighbour +1; on a mismatch it keeps the larger of the up/left neighbours (no +1). The bottom-right cell is the answer length; a trace-back from that corner spells the string.演算法會填一張表 L，其中 L[i,j] = A 的前 i 個字母和 B 的前 j 個字母的 LCS 長度。第 0 列和第 0 欄全部是 0（空的前綴跟誰都沒有共同字母）。每個內部格子比一下當下的字母 aᵢ 和 bⱼ：如果對上了，就拿左上對角那格 +1；如果沒對上，就拿上面和左邊兩格的較大值（不 +1）。右下角那格就是答案長度；從那個角落往回追溯（trace-back）就能拼出字串。

Why these rules? When aᵢ == bⱼ, that shared letter can sit at the end of the LCS, so the best you can do is that one letter plus the LCS of the two shorter prefixes (everything before aᵢ and before bⱼ) — and that already-solved answer is exactly the diagonal cell L[i-1,j-1], hence diagonal + 1. When aᵢ ≠ bⱼ, those two letters can't both be the matched ending, so at least one of them is unused: either you drop aᵢ (look up at L[i-1,j]) or you drop bⱼ (look left at L[i,j-1]), and you keep whichever is larger — no +1, because no new letter was matched.為什麼是這兩條規則？當 aᵢ == bⱼ，這個共同字母可以放在 LCS 的結尾，所以最好的情況就是「這個字母 + 兩段更短前綴的 LCS」（也就是 aᵢ 之前和 bⱼ 之前的部分）——而那個早就算好的答案，剛好就放在對角格 L[i-1,j-1]，所以是 對角 + 1。當 aᵢ ≠ bⱼ，這兩個字母不可能同時當作配對的結尾，至少有一個沒用到：要嘛丟掉 aᵢ（往上看 L[i-1,j]），要嘛丟掉 bⱼ（往左看 L[i,j-1]），取較大的那個——而且不 +1，因為沒有配對到新字母。

Why the neighbours are always ready. Each rule reads only the cell's diagonal, up, and left neighbours. If you fill the table row by row from top to bottom, and left to right within each row, then by the time you reach any cell those three neighbours have already been computed (the diagonal and up cells sit in an earlier row, the left cell earlier in the same row) — so the answer it needs is guaranteed to be waiting.為什麼鄰居一定都算好了？每條規則只會讀那格的對角、上、左三個鄰居。只要你按照由上往下一列一列填、每一列再由左往右的順序來填，那麼當你走到任何一格時，這三個鄰居都早就算過了（對角和上面那格在更上面的列，左邊那格在同一列的前面）——所以它要用的值一定已經在那裡等著了。

Why O(mn) time and space. The table has m rows and n columns, so there are m × n cells; each one is filled in O(1) work (just compare two letters and read three neighbours), giving O(mn) time. And because we store every cell so the trace-back can read them later, we keep all m × n values around — hence O(mn) space.為什麼時間和空間都是 O(mn)？這張表有 m 列、n 欄，所以總共 m × n 格；每一格只做 O(1) 的事（比兩個字母、讀三個鄰居），加起來就是 O(mn) 時間。又因為我們把每一格都存下來，好讓後面的 trace-back 能回頭讀，所以這 m × n 個值都要留著——空間就是 O(mn)。

Below is the table for A = bacad (rows) and B = accbadcb (columns). The gold cell is the one being computed; amber marks the dependency it reads (diagonal on a match, left/up on a mismatch). The final frames light the green trace-back that spells acad.下面是 A = bacad（列）和 B = accbadcb（欄）的表格。金色格是正在算的那格；琥珀色標出它要讀的來源（對上時看對角，沒對上時看左／上）。最後幾格會亮起綠色的 trace-back，拼出 acad。

How to walk the trace-back yourself. Start at the bottom-right corner and move until you reach row 0 or column 0. At each cell, check its row letter and column letter: if they match, step diagonally (up-left) and record that letter; if they don't match, step toward whichever neighbour is larger — up if the up-neighbour ≥ the left-neighbour, otherwise left (so ties go up). Only the diagonal steps on a match contribute a letter. Because you walk from the corner backward, the letters come out reversed — here you record d, a, c, a — so you flip them to read the LCS in order: a, c, a, d, i.e. top-to-bottom along the green cells.怎麼自己走這條 trace-back。從右下角出發，一路走到第 0 列或第 0 欄為止。每到一格，看它的列字母和欄字母：如果對上了，就往左上對角走一步，並把這個字母記下來；如果沒對上，就往較大的那個鄰居走——上面的鄰居 ≥ 左邊的就往上，否則往左（平手時往上）。只有對上時的對角步才會貢獻字母。因為你是從角落往回走，記下來的字母順序是反的——這裡會記成 d, a, c, a——所以要反轉過來，才會是正確順序的 LCS：a, c, a, d，也就是沿著綠色格子由上往下讀。

Time: O(mn) Space: O(mn) Answer: acad (4)
