Longest Common Subsequence

Given two strings, find the longest sequence of letters that hides inside both — in order, but not necessarily side by side.

📄 Source: Alg07 · Ch 7 §11 (Longest Common Subsequence), slides 28–29.

🧠 Solved by: Dynamic Programming

The problem

Start with the one word you must understand first: subsequence. A subsequence of a string is what is left after you delete zero or more letters while keeping the survivors in their original order. You may skip letters, but you may never reorder them. From bacad you can pull out bcd, aca, or acad — but not dab, because that reshuffles the order.

Do not confuse this with a substring. A substring must be contiguous — a solid uninterrupted run, like aca taken from positions 2–4 of bacad. A subsequence is skip-but-keep-order, so it is strictly more permissive: every substring is a subsequence, but not the reverse. acad is a subsequence of bacad (drop the b), yet it is not a substring of it (its letters aren't consecutive).

A common subsequence of two strings A and B is a string that is a subsequence of both at once. The Longest Common Subsequence (LCS) problem asks:

Given: two strings A = a₁a₂…a_m and B = b₁b₂…b_n.
Find: the longest string that is a subsequence of A and a subsequence of B (return both the string and its length).

A tiny concrete instance. Take A = bacad and B = accbadcb. Both contain the letters a, c, a, d in that order (skip the rest of each string), and you cannot do better — so the answer is acad, length 4. That is the exact instance we solve below.

Watch it solved

The algorithm fills a table L where L[i,j] = the LCS length of the first i letters of A and the first j letters of B. Row 0 and column 0 are all 0 (an empty prefix shares nothing). For each interior cell it compares the current letters aᵢ and bⱼ: on a match it takes the diagonal neighbour +1; on a mismatch it keeps the larger of the up/left neighbours (no +1). The bottom-right cell is the answer length; a trace-back from that corner spells the string.

Below is the table for A = bacad (rows) and B = accbadcb (columns). The gold cell is the one being computed; amber marks the dependency it reads (diagonal on a match, left/up on a mismatch). The final frames light the green trace-back that spells acad.

Time: O(mn) Space: O(mn) Answer: acad (4)
