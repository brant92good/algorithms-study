Travelling Salesperson — Held–Karp DP

Visit every city exactly once and come home — for the cheapest total. Dynamic programming turns the factorial-many tours into a table you fill subset by subset.

📄 Source: Alg07 · Ch 7 §10 (TSP by DP / Held–Karp), slides 22–25.

🧠 Solved by: Dynamic Programming (contrast Branch-and-Bound)

The problem

You are given a set of cities and a cost matrix c(i,j) = the price to travel from city i to city j. The graph is directed, so going one way can cost more than coming back (and ∞ means there is no road that direction). You must find the cheapest tour: a route that starts at city 1, visits every other city exactly once, and returns to city 1. "Cheapest" means the sum of the edge costs along the route is as small as possible.

Why is this hard? With n cities there are (n−1)! possible orderings of the cities you could visit — that explodes fast (10 cities ≈ 360,000 tours; 20 cities ≈ 10¹⁷). Checking every tour separately is hopeless. We need to share work between tours that overlap.

The DP idea (Held–Karp). Two half-finished tours that (a) sit at the same current city and (b) still have the same set of cities left to visit have identical futures — the cheapest way to finish is the same for both. So we don't care how we arrived, only where we are and what's left. That pair is the DP state.

Define g(i, S) = the length of the shortest path that starts at city i, visits every city in the set S exactly once, and ends back at city 1. The recurrence picks the next city j to step to:

g(i, S) = minj∈S { c(i,j) + g(j, S − {j}) },   with base case g(i, ∅) = c(i,1).

The answer is g(1, {all other cities}). We build the table up by subset size: first all the |S|=0 values, then |S|=1, and so on — each level only needs values from the level below it.
