Breadth-First Search

Explore a solution tree level by level — all the shallowest nodes first — using a FIFO queue. It is the queue-driven twin of depth-first search.

📄 Source: A05 — Searching Strategies §4 (Breadth-First Search), with the sum-of-subsets tree of §3 (Fig. 6-11) as the shared illustration.

The idea

Many problems are solved by laying out their candidate (partial) solutions as a solution tree (a.k.a. state-space tree): the root is "nothing decided yet," and each level commits one more choice. A searching strategy only decides the order in which we expand nodes of that tree. Breadth-first search (BFS) expands the tree level by level: every node at depth d is visited before any node at depth d+1. It always works on the shallowest unexpanded node next.

The mechanism is a queue (FIFO — first in, first out):

  1. Put the root in the queue.
  2. Remove the node at the front. If it is a goal, stop.
  3. Otherwise append all its children to the back of the queue.
  4. Repeat from step 2 until the queue is empty (failure) or a goal is found.

Because children go to the back and we always serve from the front, a node's children only get expanded after every node already waiting (all of which are at the same or shallower depth). That is exactly what produces the level-by-level order.

BFS vs DFS — same tree, one data structure apart. DFS uses a stack (LIFO): it pushes children on top and pops the newest, so it dives down one branch as deep as possible, then backtracks. BFS swaps the stack for a queue (FIFO) and so fans out across each level instead of plunging. DFS finds a deep goal fast with little memory (one root-to-frontier path); BFS finds the shallowest goal first but must store a whole frontier (which grows wide). Neither, on its own, guarantees an optimal-cost goal — they stop at the first goal reached; that is what branch-and-bound and A* add later in the chapter.

A tiny illustration

Take the sum-of-subsets tree (Fig. 6-11): the set is S = {7, 5, 1, 2, 10} and we want a subset summing to 9. The root is the empty sum 0; at level i we branch on whether element i is in (left, value added) or out. Below, each node is annotated ①②③… with the order BFS would expand it — strictly increasing with depth. Read it top band, then next band, never jumping down a branch early.

depth 0:                0 ①
                       / +7 \ skip
depth 1:        7 ②          0 ③
              / +5 \ skip      ( … )
depth 2:   12✗ ④   7 ⑤
                    / +1 \ skip
depth 3:        8 ⑥    7 ⑦
                   / +2 \ skip
depth 4:      9 ★ ⑧   8 ⑨

BFS expansion order ①→⑨ sweeps each depth band fully before descending. The goal 9 ★ (the subset {7, 2}) sits at depth 4 and is reached at step ⑧. A DFS on the same tree would instead drive straight down the +7, +5 branch first, hit the dead end 12 ✗ (exceeds 9), backtrack, and only then try the next branch — same nodes, different visiting order.
