Breadth-First SearchBreadth-First Search

Explore a solution tree level by level — all the shallowest nodes first — using a FIFO queue. It is the queue-driven twin of depth-first search.用一個 FIFO queue，把 solution tree 一層一層走過——最淺的 node 全部先處理。它是 depth-first search 的雙胞胎，只是改用 queue 驅動。

The idea核心觀念

Many problems are solved by laying out their candidate (partial) solutions as a solution tree (a.k.a. state-space tree): the root is "nothing decided yet," and each level commits one more choice. A searching strategy only decides the order in which we expand nodes of that tree. Breadth-first search (BFS) expands the tree level by level: every node at depth d is visited before any node at depth d+1. It always works on the shallowest unexpanded node next.很多問題的解法，是把它們的候選（部分）解攤成一棵 solution tree（又叫 state-space tree）：root 是「什麼都還沒決定」，每往下一層就多定一個選擇。所謂 searching strategy 只是決定我們用什麼順序展開這棵樹的 node。Breadth-first search（BFS）是一層一層展開：深度 d 的每個 node 都先走完，才會碰任何深度 d+1 的 node。它下一個處理的，永遠是還沒展開、最淺的那個 node。

The mechanism is a queue (FIFO — first in, first out):背後的機制是一個 queue（FIFO——first in, first out）：

  1. Put the root in the queue.把 root 放進 queue。
  2. Remove the node at the front. If it is a goal, stop.從前端拿走一個 node。如果它是 goal，就停。
  3. Otherwise append all its children to the back of the queue.否則把它所有的 children 接到 queue 的後端。
  4. Repeat from step 2 until the queue is empty (failure) or a goal is found.回到第 2 步重複，直到 queue 空了（失敗）或找到 goal。

Because children go to the back and we always serve from the front, a node's children only get expanded after every node already waiting (all of which are at the same or shallower depth). That is exactly what produces the level-by-level order.因為 children 都接到後端、而我們永遠從前端處理，所以一個 node 的 children 要等到前面在排隊的 node（它們的深度都一樣或更淺）全部處理完，才會輪到被展開。這就剛好造成了一層一層的順序。

BFS vs DFS — same tree, one data structure apart. DFS uses a stack (LIFO): it pushes children on top and pops the newest, so it dives down one branch as deep as possible, then backtracks. BFS swaps the stack for a queue (FIFO) and so fans out across each level instead of plunging. DFS finds a deep goal fast with little memory (one root-to-frontier path); BFS finds the shallowest goal first but must store a whole frontier (which grows wide). Neither, on its own, guarantees an optimal-cost goal — they stop at the first goal reached; that is what branch-and-bound and A* add later in the chapter.BFS vs DFS——同一棵樹，差在一個資料結構。DFS 用 stack（LIFO）：把 children push 到頂、pop 最新的，所以它沿一條分支盡量往深處鑽，再回溯。BFS 把 stack 換成 queue（FIFO），所以是在每一層橫向散開，而不是往下衝。DFS 能用很少的記憶體（只存一條 root 到 frontier 的路徑）很快找到某個深處的 goal；BFS 會先找到最淺的 goal，但得存整個 frontier（會愈長愈寬）。不過這兩個本身都不保證找到成本最佳的 goal——它們一碰到第一個 goal 就停；要保證最佳，那是這章後面 branch-and-bound 跟 A* 才補上的。

A tiny illustration小範例

Take the sum-of-subsets tree (Fig. 6-11): the set is S = {7, 5, 1, 2, 10} and we want a subset summing to 9. The root is the empty sum 0; at level i we branch on whether element i is in (left, value added) or out. Below, each node is annotated ①②③… with the order BFS would expand it — strictly increasing with depth. Read it top band, then next band, never jumping down a branch early.拿 sum-of-subsets 樹來看（Fig. 6-11）：集合是 S = {7, 5, 1, 2, 10}，我們要找一個 subset 加起來等於 9。root 是空集合、和為 0；在第 i 層分支，決定第 i 個元素要 in（往左，把值加進來）還是 out。下面每個 node 都標了 ①②③…，代表 BFS 展開它的順序——隨深度嚴格遞增。看的時候先讀最上面那一條，再讀下一條，絕不提早往某條分支衝下去。

depth 0:                0 ①
                       / +7 \ skip
depth 1:        7 ②          0 ③
              / +5 \ skip      ( … )
depth 2:   12✗ ④   7 ⑤
                    / +1 \ skip
depth 3:        8 ⑥    7 ⑦
                   / +2 \ skip
depth 4:      9 ★ ⑧   8 ⑨

BFS expansion order ①→⑨ sweeps each depth band fully before descending. The goal 9 ★ (the subset {7, 2}) sits at depth 4 and is reached at step ⑧. A DFS on the same tree would instead drive straight down the +7, +5 branch first, hit the dead end 12 ✗ (exceeds 9), backtrack, and only then try the next branch — same nodes, different visiting order.BFS 的展開順序 ①→⑨ 會把每一層整條掃完才往下走。goal 9 ★（也就是 subset {7, 2}）在深度 4，在第 ⑧ 步被找到。同一棵樹換成 DFS 的話，會先沿 +7, +5 那條一路衝下去，撞到死路 12 ✗（超過 9），回溯，然後才試下一條分支——一樣的 node，只是拜訪順序不同。
