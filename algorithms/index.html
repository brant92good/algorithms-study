🔎 Searching Strategies — Chapter 6: explore a solution tree

✓
Depth-First Search
Dive deep with a stack; backtrack on a dead end.
 ✓
Breadth-First Search
Same tree, a queue, level by level.
 ✓
Hill Climbing
Greedy local choice via f(n)=d(n)+w(n) — can get stuck.
 ✓
Best-First Search
Pick the best node among ALL open ones — global view.
 ✓
Branch-and-Bound
Add a bound that prunes losing branches → optimization (TSP).
 ✓
A* Algorithm
f=g+h with admissible h≤h* → first goal popped is optimal.

✂️ Prune-and-Search — discard a constant fraction each round → O(n)

✓
Selection (Median of Medians)
k-th smallest in linear time; groups of 5, ≥n/4 pruned.
 ✓
2-Variable Linear Programming
The F(x) upper envelope; slope test prunes constraints.
 ✓
1-Center Problem
Smallest enclosing circle; prune ~n/16 each round.

🧩 Dynamic Programming — Chapter 7: solve subproblems once, reuse them

✓
Multistage Graph
Shortest path by forward/backward reasoning; d(S,T)=9.
 ✓
0/1 Knapsack
Fill gᵢ(y) = max{ skip, take }.
 ✓
Resource Allocation
Split units across projects to maximize profit.
 ✓
TSP (Held–Karp)
State = (current city, visited set); O(n²·2ⁿ).
 ✓
Longest Common Subsequence
Match → diag+1, else max(up,left); "acad".
 ✓
Optimal Binary Search Tree
Try each key as root; frequent keys sit shallow.
