Prune-and-Search · Alg06

Selection — k-th Smallest

Given n unsorted numbers and a rank k, find the value that would land in position k if the list were sorted — without ever fully sorting it.

📄 Source: Alg06 · Prune-and-Search §2–5 (selection problem → three-way partition → median-of-medians → O(n)).

🧠 Solved by: Prune-and-Search

The problem

You are handed a set S of n numbers in no particular order, and an integer k between 1 and n. The selection problem asks for the k-th smallest element of S: the value that would occupy position k if you sorted S into non-decreasing order — but you must return it without relying on a full sort.

  • Given: a bag of n numbers S = {a₁, …, aₙ} and a rank k.
  • Asked: the element x ∈ S such that exactly k−1 elements of S are smaller than x.
  • Median: the special case k = ⌈n/2⌉ — the "middle" value, the one with as many elements below it as above.

Tiny concrete instance. Take S = [3, 5, 7, 12, 19] and k = 2. Sorted, that's [3, 5, 7, 12, 19], so the 2nd smallest is 5. Here k = ⌈5/2⌉ = 3 would give the median, 7. The trick is doing this on much larger lists without paying for a sort.

Why not just sort? Sorting S and reading off position k is correct but costs O(n log n). We can do strictly better — O(n), linear time — by throwing away a guaranteed fraction of the data each round instead of ordering all of it.

The idea is prune-and-search: pick a pivot p, split S into S₁ (< p), S₂ (= p), S₃ (> p), then recurse into only one pile. If |S₁| ≥ k the answer is in S₁ (same k); else if |S₁|+|S₂| ≥ k then p itself is the answer; else recurse into S₃ for rank k − |S₁| − |S₂|. The whole thing only stays linear if the pivot reliably splits the list — which is why we pick the median of medians (groups of 5), guaranteeing ≥ n/4 elements pruned every round.

Primary source: Alg06 Prune-and-Search §2–5; or CLRS, Introduction to Algorithms, Ch. 9 "Medians and Order Statistics" (§9.3 selection in worst-case linear time). The key guarantee — the median of medians prunes ≥ n/4 each round — is what turns three-way partitioning into a worst-case O(n) algorithm.

The strategy: Prune-and-Search — the general method (binary search is its simplest instance) that also solves 2-variable linear programming and the 1-center problem in linear time.
