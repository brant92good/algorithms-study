Selection — k-th SmallestSelection — 第 k 小

Given n unsorted numbers and a rank k, find the value that would land in position k if the list were sorted — without ever fully sorting it.給你 n 個沒排序的數字，還有一個名次 k，要你找出「如果整串排好序，第 k 個會是誰」——但不准真的把整串排完。

📄 Source: Alg06 · Prune-and-Search §2–5 (selection problem → three-way partition → median-of-medians → O(n)).

🧠 Solved by:解法： Prune-and-Search

The problem這題在問什麼

You are handed a set S of n numbers in no particular order, and an integer k between 1 and n. The selection problem asks for the k-th smallest element of S: the value that would occupy position k if you sorted S into non-decreasing order — but you must return it without relying on a full sort.你手上有一組亂序的數字 S（共 n 個），還有一個介於 1 到 n 之間的整數 k。selection 問題要的就是 S 裡的第 k 小：把 S 由小到大排好後，站在第 k 位的那個值——但你得在不排序的情況下把它找出來。

  • Given:給定： a bag of n numbers S = {a₁, …, aₙ} and a rank k.一堆數字 S = {a₁, …, aₙ}（共 n 個）和一個名次 k。
  • Asked:要找： the element x ∈ S such that exactly k−1 elements of S are smaller than x.那個 x ∈ S，讓 S 裡剛好有 k−1 個比它小。
  • Median:中位數： the special case k = ⌈n/2⌉ — the "middle" value, the one with as many elements below it as above.就是 k = ⌈n/2⌉ 這個特例——正中間那個值，上下兩邊的元素一樣多。

Tiny concrete instance.來個小例子。 Take S = [3, 5, 7, 12, 19] and k = 2. Sorted, that's [3, 5, 7, 12, 19], so the 2nd smallest is 5. Here k = ⌈5/2⌉ = 3 would give the median, 7. The trick is doing this on much larger lists without paying for a sort.拿 S = [3, 5, 7, 12, 19]、k = 2。排好序是 [3, 5, 7, 12, 19]，所以第 2 小是 5。如果 k = ⌈5/2⌉ = 3 就會拿到中位數 7。重點是：當資料量大很多時，要怎麼不花排序的代價就做到這件事。

Why not just sort?為什麼不直接排序就好？ Sorting S and reading off position k is correct but costs O(n log n). We can do strictly better — O(n), linear time — by throwing away a guaranteed fraction of the data each round instead of ordering all of it.把 S 排好再讀第 k 個當然對，但要花 O(n log n)。其實可以做得更好——O(n) 線性時間。訣竅是：每一輪都保證砍掉一定比例的資料，而不是把全部都排出順序。

The idea is prune-and-search: pick a pivot p, split S into S₁ (< p), S₂ (= p), S₃ (> p), then recurse into only one pile. If |S₁| ≥ k the answer is in S₁ (same k); else if |S₁|+|S₂| ≥ k then p itself is the answer; else recurse into S₃ for rank k − |S₁| − |S₂| (e.g. if k = 12, |S₁| = 7, |S₂| = 1, you recurse into S₃ looking for rank 12 − 7 − 1 = 4). The whole thing only stays linear if the pivot reliably splits the list — which is why we pick the median of medians (groups of 5).

How we get that pivot — and the hidden recursion. Break the n numbers into groups of 5, and take each group's median; that gives n/5 medians. The pivot we want is the median of those n/5 medians — but finding a median is itself a selection problem! So the algorithm calls itself recursively on the n/5 medians. That recursive call is exactly where the T(n/5) term in the recurrence comes from. (In the tiny example below there are only 3 medians, so you can eyeball it; in general it's a real recursion.)

Why ≥ n/4 gets pruned (the linchpin). The pivot is the median of the n/5 medians, so it is ≥ half of them, i.e. ≥ n/10 group medians. Each such group has 5 elements, of which 3 (the median and the two below it) are ≤ that group's median ≤ the pivot. That's about 3·(n/10) = 3n/10 elements guaranteed ≤ the pivot — and symmetrically ≥ 3n/10 are ≥ the pivot. So whichever side we prune, we throw away at least roughly n/4 every round. That's the guarantee that makes it linear.核心想法就是 prune-and-search：挑一個 pivot p，把 S 切成 S₁（< p）、S₂（= p）、S₃（> p），然後只往其中一堆遞迴。如果 |S₁| ≥ k，答案就在 S₁（k 不變）；不然如果 |S₁|+|S₂| ≥ k，那 p 本身就是答案；再不然就往 S₃ 找名次 k − |S₁| − |S₂|（舉例：如果 k = 12、|S₁| = 7、|S₂| = 1，就往 S₃ 找名次 12 − 7 − 1 = 4）。整件事要保持線性，pivot 一定要把整串切得夠均勻——所以我們挑 median of medians（每 5 個一組）。

這個 pivot 怎麼來的——還有藏起來的遞迴。把 n 個數每 5 個分一組，各組取中位數，這樣會得到 n/5 個中位數。我們要的 pivot 就是「這 n/5 個中位數的中位數」——但是「找中位數」本身就是一個 selection 問題啊！所以演算法會對這 n/5 個中位數遞迴呼叫自己。這個遞迴呼叫，正是遞迴式裡 T(n/5) 那一項的來源。（下面那個小例子只有 3 個中位數，用眼睛看就好；但一般情況下這是真正的遞迴。）

為什麼保證砍掉 ≥ n/4（整個證明的關鍵）。pivot 是「n/5 個中位數的中位數」，所以它至少 ≥ 其中一半，也就是 ≥ n/10 個組中位數。每一組有 5 個元素，其中有 3 個（該組中位數加上比它小的兩個）會 ≤ 該組中位數 ≤ pivot。算一下大概是 3·(n/10) = 3n/10 個元素保證 ≤ pivot；對稱地，也有 3n/10 個保證 ≥ pivot。所以不管砍哪一邊，每輪至少都丟掉大約 n/4 個。就是這個保證讓它變成線性。

What if we'd wanted the 12th smallest instead?那如果改成要找第 12 小呢？ Same pivot p = 9 gives |S₁| = 7 (< 9) and |S₂| = 1 (= 9). Now |S₁| = 7 < 12 and |S₁|+|S₂| = 8 < 12, so the answer is in S₃ (the bigger pile) — and we look there for rank 12 − |S₁| − |S₂| = 12 − 7 − 1 = 4. (Intuition: we've discarded the 8 elements that are ≤ 9, so the overall 12th smallest is the 4th smallest among what's left.) That's the S₃ branch in action.一樣用 pivot p = 9，會得到 |S₁| = 7（< 9）和 |S₂| = 1（= 9）。這時候 |S₁| = 7 < 12，而且 |S₁|+|S₂| = 8 < 12，所以答案落在 S₃（比較大的那堆）——要在那邊找名次 12 − |S₁| − |S₂| = 12 − 7 − 1 = 4。（直覺：我們已經砍掉 8 個 ≤ 9 的元素，所以整體第 12 小，就是剩下那堆裡的第 4 小。）這就是 S₃ 分支實際跑起來的樣子。
