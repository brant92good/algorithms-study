Backtracking

Generating Permutations

List every possible ordering of a set of items by fixing one position at a time and undoing each choice once you've explored it — the textbook example of backtracking.

What is it?

A permutation is one particular ordering of a set of items. For the three letters {A, B, C} the orderings are ABC, ACB, BAC, BCA, CAB, CBA — six of them. Our goal is to generate them all, with none missed and none repeated.

The core idea is "fix one position, then recurse on the rest". To build an ordering we decide the first letter — it can be any of the three. Once the first is chosen, that letter is used up; we then decide the second letter from whatever remains, then the third from the single leftover. When all positions are filled we've reached a complete permutation (a leaf of the tree) and print it.

The catch: after exploring everything that starts with one choice, we must backtrack — undo that last choice, marking the letter unused again, so its branch is free to be re-explored with a different prefix. Backtracking is just "step back up one level and try the next option." The whole process traces out a decision tree: each level picks one more position, each branch is one available letter, and the leaves are the finished permutations.

Watch it run

We permute {A, B, C}. The tree grows depth-first: the ● marker shows where we are right now, ✓ marks a completed permutation, and ↩ marks a backtrack (undoing the last fixed choice).

Count: n! permutations Time to list: O(n·n!) Space: O(n) recursion depth
