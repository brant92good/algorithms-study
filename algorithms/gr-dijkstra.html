Dijkstra's Shortest Path

Find the cheapest route from one node to every other node in a weighted graph — by always growing outward from the nearest node you haven't finished yet.

What is it?

A graph is a set of nodes (dots) joined by edges (lines). Here the graph is weighted: each edge carries a number — its cost or distance — and undirected, so you can walk an edge either way. The shortest path from a start node to some node is the route whose edge weights add up to the smallest total. (Fewest edges is not the same thing: three cheap hops can beat one expensive one.)

Dijkstra's algorithm finds the shortest path from one source to every node. It keeps a dist value for each node — the best total cost found so far — starting at 0 for the source and ∞ for everything else. Two key moves:

Relax an edge (u, v) with weight w: check whether going through u beats what we already had for v. If dist[u] + w < dist[v], we found a cheaper way in, so lower dist[v] to dist[u] + w. Otherwise leave it.

Settle a node: among all nodes not yet finished, pick the one with the smallest dist (the greedy choice), mark it done, and relax all its edges. The invariant that makes this work: once a node is settled, its dist is its true shortest distance and can never improve again. Why? Every other route to it would have to leave through some farther, not-yet-settled node — and with non-negative weights, a longer start can only get longer.

Time: O((V+E) log V) Space: O(V) Needs: non-negative weights
