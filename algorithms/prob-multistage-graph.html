The problem

A multistage graph is a directed graph whose nodes are partitioned into ordered groups called stages. Every edge points from one stage to the very next stage â€” you can never skip a stage or go backward. The first stage is a single source node S; the last stage is a single sink node T. The other direction â€” forward reasoning

The animation used backward reasoning: d(X,T) = cheapest cost from X to the sink, filled in from T back toward S â€” giving d(C,T)=4, d(A,T)=22, d(B,T)=18, and finally d(S,T)=9. Principle of optimality.
