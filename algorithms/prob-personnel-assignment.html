Personnel Assignment人員指派問題

Hand n jobs to n people at minimum total cost — but the jobs come with a precedence order, so not every assignment is legal. Search the legal ones cleverly with a cost lower bound that prunes whole subtrees.把 n 個工作分給 n 個人，要讓總成本最小。但工作之間有先後順序（precedence order）的限制，所以不是每種分法都合法。我們會用成本的下界（lower bound）來聰明地搜尋合法解，順便把整棵不可能更好的子樹直接砍掉。

🧠 Solved by:用什麼解： Branch-and-Bound

The problem問題是什麼

You have n people and n jobs, and you must give each person exactly one job and each job exactly one person (a one-to-one assignment). Assigning person Pᵢ to job Jⱼ costs Cᵢⱼ — these costs are given in a table. The naive goal would be "pick the cheapest one-to-one assignment," but here there is a twist.你有 n 個人和 n 個工作，每個人剛好分到一個工作、每個工作剛好分給一個人（一對一的 assignment）。把 Pᵢ 指派去做 Jⱼ 的成本是 Cᵢⱼ，這些成本都列在表裡。直覺上你會想「挑總成本最小的一對一分法就好」，但這題有個轉折。

The jobs carry a partial order — a precedence relation. Some jobs must come before others; some pairs of jobs are unrelated (incomparable). The people, by contrast, are linearly ordered: P₁ < P₂ < … < Pₙ. The rule that makes an assignment feasible is that it must respect the job order:這些工作之間有 partial order（偏序），也就是一種先後關係：有些工作一定要排在別的工作前面，有些工作之間則沒關係（無法比較）。相對地，人是線性排好的：P₁ < P₂ < … < Pₙ。一個分法要算合法（feasible），就得遵守工作的順序：

If person Pᵢ comes before person Pⱼ (i.e. Pᵢ < Pⱼ), then the job given to Pᵢ must not come strictly after Pⱼ's job in the precedence order. Concretely: a job that must come later (J3 after J1, J4 after J2) may not be handed to an earlier person than its required predecessor. Incomparable jobs carry no constraint — J1 and J2 are unrelated, so either may go to the earlier person; same for J3 and J4. The clean way to say it: reading the jobs down the person list P₁, P₂, …, Pₙ must give a topological order of the precedence relation.如果 Pᵢ 排在 Pⱼ 前面（也就是 Pᵢ < Pⱼ），那 Pᵢ 拿到的工作在 precedence 上就不能排在 Pⱼ 的工作之後。講具體一點：一個一定要排比較後面的工作（J3 要在 J1 後、J4 要在 J2 後），不能發給比它的前置工作還早的人。互相無關（incomparable）的工作就完全沒限制——J1 跟 J2 沒關係，誰先誰後都行；J3 跟 J4 也一樣。最乾淨的講法：你照 P₁, P₂, …, Pₙ 由上往下讀他們拿到的工作，這個序列要剛好是 precedence 的一個 topological order（拓樸排序）。

Objective:目標： minimize the total cost Σᵢ Σⱼ Cᵢⱼ·Xᵢⱼ, where Xᵢⱼ = 1 if Pᵢ is assigned to Jⱼ and 0 otherwise, subject to each person/job used once and the feasibility (precedence) rule.把總成本 Σᵢ Σⱼ Cᵢⱼ·Xᵢⱼ 最小化。其中 Pᵢ 被指派去做 Jⱼ 時 Xᵢⱼ = 1，否則為 0。限制是每個人、每個工作都只用一次，而且要符合上面那條 precedence 規則。

The concrete instance (n = 4)實際例子（n = 4）

Four people P₁…P₄, four jobs J₁…J₄. The cost matrix (Table 6-1) — rows are people, columns are jobs:四個人 P₁…P₄、四個工作 J₁…J₄。成本矩陣（Table 6-1），列是人、行是工作：

The job precedence (Fig. 6-21) is two simple chains — J1 must precede J3, and J2 must precede J4:工作的 precedence（Fig. 6-21）就是兩條簡單的鏈——J1 一定要排在 J3 前面，J2 一定要排在 J4 前面：

J1 J2 | | v v J3 J4

J1 and J2 are incomparable; so are J3 and J4. The legal (topologically sorted) job sequences are therefore:J1 和 J2 之間沒關係，J3 和 J4 之間也沒關係。所以合法的（topological order）工作序列有這幾種：

J1, J2, J3, J4 J1, J2, J4, J3 J1, J3, J2, J4 J2, J1, J3, J4 J2, J1, J4, J3

An assignment is feasible iff the jobs handed to P₁, P₂, P₃, P₄ (in that order) form one of these sequences. For example P1→J1, P2→J2, P3→J3, P4→J4 is feasible (sequence J1,J2,J3,J4). The question: among all feasible assignments, which is cheapest?一個分法合法，當且僅當 P₁, P₂, P₃, P₄（照這個順序）拿到的工作剛好是上面其中一條序列。舉例來說 P1→J1, P2→J2, P3→J3, P4→J4 就合法（對應序列 J1,J2,J3,J4）。問題是：在所有合法分法裡，哪一個最便宜？

Watch it solved看它怎麼解

Two stages. First we reduce the cost matrix (subtract a constant from every row and every column so each has a zero) — the total subtracted is a guaranteed lower bound on any assignment. Then we branch: a decision tree picks P₁'s job, then P₂'s, and so on, carrying a running cost and using the bound to prune. Step through.分兩個階段。第一步先把成本矩陣做 reduction（每一列、每一行都減掉一個常數，讓每列每行都出現一個 0），減掉的總量就是任何分法的下界（lower bound）。接著branch：用一棵決策樹先選 P₁ 的工作、再選 P₂ 的，一路下去，沿途記錄累積成本，並用下界來剪枝（prune）。一步步看下去。

Why reduction keeps the same answer (the key idea):為什麼 reduction 不會改變答案（核心觀念）： every assignment picks exactly one cell in each row and exactly one cell in each column. So if you subtract a constant c from a whole row, every possible assignment loses exactly c from its total (each one used precisely one cell of that row) — and the same is true for subtracting c from a whole column. Since every assignment's cost drops by the very same amount, their ranking is untouched: the cheapest assignment in the reduced matrix is the cheapest in the original. And because every assignment must pay back all the subtracted constants (one cell per row and per column, each carrying its row's and column's subtraction), the sum of all subtracted constants is a cost no assignment can avoid — that is exactly why the total subtracted (here 54) is a valid lower bound on the original cost.每個分法都剛好在每一列挑一格、在每一行挑一格。所以你只要從整列減掉一個常數 c，每一種分法的總成本都剛好少 c（因為每種分法都正好用了那一列的某一格）——整行減 c 也完全一樣。既然每個分法都降一樣多，它們的排名根本沒變：reduced matrix 裡最便宜的分法，就是原矩陣裡最便宜的那個。再來，因為每個分法都得把這些被減掉的常數「補回來」（每列一格、每行一格，各自背著那列、那行被減的量），所以所有被減常數的總和，是任何分法都躲不掉的成本——這正是為什麼減掉的總量（這裡是 54）會是原成本的一個合法下界（lower bound）。
People: linear order人：線性順序 Jobs: partial order工作：partial order Bound: row+col reduction = 54下界：列+行 reduction = 54 Optimal cost: 70最佳成本：70 Strategy: best-first B&B策略：best-first B&B

Two different "lower bounds" — how they fit together兩個不同的「下界」——它們怎麼配在一起

It is easy to get confused here, because the page uses two lower bounds for two different jobs:這邊很容易搞混，因為頁面其實用了兩個下界，各自做不同的事：

  • The reduction bound (54) is a single global lower bound on the whole problem: no feasible assignment can cost less than 54. It is the standard textbook bound (Table 6-2) and it certifies, before any search, that we are not chasing an impossibly cheap answer.Reduction 下界（54）是對整個問題的單一全域下界：沒有任何合法分法會低於 54。這是課本的標準做法（Table 6-2），它在搜尋開始前先保證：我們不是在追一個根本不存在的超便宜解。
  • The node bound (running cost) is a per-node lower bound used inside the tree. At a node we have already chosen some people's jobs; their running cost (a sum of the original Cᵢⱼ values, e.g. 29 + 30 = 59) is a lower bound on every full assignment that extends this node, because the people still unplaced can only add non-negative cost. This is what actually drives best-first and the pruning.節點下界（running cost）是樹裡用的每個節點各自的下界。在某個節點上，我們已經幫部分人選好工作了；他們的 running cost（把那些原始 Cᵢⱼ 加起來，例如 29 + 30 = 59）就是「任何把這個節點補完的完整分法」的下界，因為還沒安排的人只會再加非負成本、不會變少。真正在驅動 best-first 與剪枝的，就是這個下界。

Honest reconciliation: in this small instance the running-cost bound does all the real pruning, so the search below uses original Cᵢⱼ costs and the number 54 is not re-used per node. The reduced matrix's role here is illustrative: it shows the global floor (54) and the technique. (A stronger solver would, at each node, add a reduction bound for the remaining sub-matrix on top of the running cost to prune harder; this page keeps it simple.)老實說清楚：在這個小例子裡，真正在剪枝的是 running-cost 下界，所以下面的搜尋用的是原始 Cᵢⱼ 成本，那個 54 並沒有被每個節點拿來重算。reduced matrix 在這裡主要是示範用：讓你看到全域下限（54）和這個技巧長怎樣。（更強的解法會在每個節點，把剩下子矩陣的 reduction 下界加在 running cost 之上來剪得更兇；這頁為了好懂就先不做。）

Verify optimality: just 5 legal assignments驗證最佳：合法分法只有 5 種

Because there are only five legal job sequences, we can list every feasible assignment and its cost — then you can check by hand that 70 really is the minimum (no faith in pruning required):因為合法的工作序列只有 5 種，我們可以把每一個合法分法和它的成本都列出來——你就能親手確認 70 真的是最小值（不用相信剪枝也行）：

Job sequence (P1,P2,P3,P4) assignment cost J2, J1, J4, J3 P1→J2 P2→J1 P3→J4 P4→J3 19+32+ 9+10 = 70 ★ optimal J2, J1, J3, J4 P1→J2 P2→J1 P3→J3 P4→J4 19+32+ 7+15 = 73 J1, J2, J4, J3 P1→J1 P2→J2 P3→J4 P4→J3 29+30+ 9+10 = 78 J1, J2, J3, J4 P1→J1 P2→J2 P3→J3 P4→J4 29+30+ 7+15 = 81 J1, J3, J2, J4 P1→J1 P2→J3 P3→J2 P4→J4 29+26+21+15 = 91

The cheapest is 70. Notice all five costs sit at or above the global lower bound 54 (as they must), and the best-first search reaches the 70 row first, then only needs to confirm nothing else can beat it.最便宜的是 70。注意這 5 個成本全都 ≥ 全域下界 54（本來就該如此），而 best-first 搜尋會最先走到 70 這一列，接著只要確認其他人都贏不過它就好。
