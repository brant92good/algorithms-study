The problemе•ҸйЎҢжҳҜд»Җйәј

You have n people and n jobs, and you must give each person exactly one job and each job exactly one person (a one-to-one assignment). Assigning person Pбөў to job Jвұј costs Cбөўвұј вҖ” these costs are given in a table. The naive goal would be "pick the cheapest one-to-one assignment," but here there is a twist.дҪ жңү n еҖӢдәәе’Ң n еҖӢе·ҘдҪңпјҢжҜҸеҖӢдәәеүӣеҘҪеҲҶеҲ°дёҖеҖӢе·ҘдҪңгҖҒжҜҸеҖӢе·ҘдҪңеүӣеҘҪеҲҶзөҰдёҖеҖӢдәәпјҲдёҖе°ҚдёҖзҡ„ assignmentпјүгҖӮжҠҠ Pбөў жҢҮжҙҫеҺ»еҒҡ Jвұј зҡ„жҲҗжң¬жҳҜ CбөўвұјпјҢйҖҷдәӣжҲҗжң¬йғҪеҲ—еңЁиЎЁиЈЎгҖӮзӣҙиҰәдёҠдҪ жңғжғігҖҢжҢ‘зёҪжҲҗжң¬жңҖе°Ҹзҡ„дёҖе°ҚдёҖеҲҶжі•е°ұеҘҪгҖҚпјҢдҪҶйҖҷйЎҢжңүеҖӢиҪүжҠҳгҖӮ

The jobs carry a partial order вҖ” a precedence relation. Some jobs must come before others; some pairs of jobs are unrelated (incomparable). The people, by contrast, are linearly ordered: PвӮҒ < PвӮӮ < вҖҰ < PвӮҷ. The rule that makes an assignment feasible is that it must respect the job order:йҖҷдәӣе·ҘдҪңд№Ӣй–“жңү partial orderпјҲеҒҸеәҸпјүпјҢд№ҹе°ұжҳҜдёҖзЁ®е…ҲеҫҢй—ңдҝӮпјҡжңүдәӣе·ҘдҪңдёҖе®ҡиҰҒжҺ’еңЁеҲҘзҡ„е·ҘдҪңеүҚйқўпјҢжңүдәӣе·ҘдҪңд№Ӣй–“еүҮжІ’й—ңдҝӮпјҲз„Ўжі•жҜ”ијғпјүгҖӮзӣёе°Қең°пјҢдәәжҳҜз·ҡжҖ§жҺ’еҘҪзҡ„пјҡPвӮҒ < PвӮӮ < вҖҰ < PвӮҷгҖӮдёҖеҖӢеҲҶжі•иҰҒз®—еҗҲжі•пјҲfeasibleпјүпјҢе°ұеҫ—йҒөе®Ҳе·ҘдҪңзҡ„й ҶеәҸпјҡ

If person Pбөў comes before person Pвұј (i.e. Pбөў < Pвұј), then the job given to Pбөў must come before-or-equal to the job given to Pвұј in the job order. In short: a feasible assignment lists the jobs in a topological order of the precedence relation as you read down the person list PвӮҒ, PвӮӮ, вҖҰ, PвӮҷ.еҰӮжһң Pбөў жҺ’еңЁ Pвұј еүҚйқўпјҲд№ҹе°ұжҳҜ Pбөў < PвұјпјүпјҢйӮЈ Pбөў жӢҝеҲ°зҡ„е·ҘдҪңпјҢеңЁе·ҘдҪңй ҶеәҸдёҠд№ҹиҰҒжҺ’еңЁ Pвұј жӢҝеҲ°зҡ„е·ҘдҪңеүҚйқўжҲ–еҗҢзӯүгҖӮз°Ўе–®и¬ӣпјҡдҪ з…§ PвӮҒ, PвӮӮ, вҖҰ, PвӮҷ з”ұдёҠеҫҖдёӢи®ҖпјҢд»–еҖ‘жӢҝеҲ°зҡ„е·ҘдҪңиҰҒеүӣеҘҪжҳҜ precedence й—ңдҝӮзҡ„дёҖеҖӢ topological orderпјҲжӢ“жЁёжҺ’еәҸпјүгҖӮ

Watch it solvedзңӢе®ғжҖҺйәји§Ј

Two stages. First we reduce the cost matrix (subtract a constant from every row and every column so each has a zero) вҖ” the total subtracted is a guaranteed lower bound on any assignment. Then we branch: a decision tree picks PвӮҒ's job, then PвӮӮ's, and so on, carrying a running cost and using the bound to prune.
