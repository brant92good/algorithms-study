Graph Traversal

Breadth-First Search

Explore a graph level by level using a queue — visiting everything one step away, then two steps away, and so on.

What is it?

A graph is just a set of nodes (the circles, e.g. A, B, C) connected by edges (the lines). To traverse a graph means to visit every node, following edges. The question BFS answers: starting from one node, in what order do we reach the rest?

BFS uses a queue — a waiting line that is FIFO (First In, First Out): whatever you add first is what you take out first, like a line at a checkout. The loop is simple:

1. Put the start node in the queue. 2. Dequeue a node (take it from the front) and mark it visited. 3. Enqueue (add to the back) each of its neighbors you haven't seen yet. 4. Repeat until the queue is empty.

Because the queue is FIFO, every node at distance 1 from the start comes out before any node at distance 2, which all come out before distance 3, and so on. BFS sweeps the graph in rings of increasing distance. A direct consequence: in an unweighted graph (every edge counts as 1 step), the first time BFS reaches a node, it has reached it by the fewest possible edges — so BFS finds shortest paths measured in number of edges.
