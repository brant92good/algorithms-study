Hill Climbing

A greedy variant of depth-first search: at every step, expand only the locally best-looking child by an evaluation function f(n) — fast, but it sees only one step ahead and can stall at a local optimum.

📄 Source: A05 · Searching Strategies §5 (Hill Climbing, the 8-puzzle, Fig. 6-15).

The idea

Many problems are solved by exploring a solution-space tree: the root is the start state, each child is a state you can reach in one move, and somewhere down the tree sits a goal state. The searching strategy just decides the order in which you expand nodes.

Plain depth-first search (DFS) dives down one branch, expanding children in a fixed arbitrary order, and backtracks on dead ends. Hill climbing keeps the dive-deep spirit of DFS but adds judgement: instead of taking children in arbitrary order, it scores each child with an evaluation function f(n) and always expands the locally best one (lowest f). It is the search equivalent of always walking uphill toward higher ground.

Key intuition — and its trap. Hill climbing has only a local view: it compares a node's children to each other and commits to the best of those, never reconsidering. If the best child still leads nowhere good, hill climbing is stuck — it has climbed a small hill (a local optimum) with no way to see that a taller peak lies across the valley. (Best-first search fixes exactly this by keeping a global view of all open nodes.)

When to use it. When you want a quick, low-memory guided search and a decent (not necessarily optimal) solution is acceptable. It expands one node per level along a single path, so it is cheap — but it gives no optimality guarantee and can fail to reach a goal at all. For guaranteed-optimal search you need bounding (branch-and-bound, A*).

The evaluation function (8-puzzle)

The 8-puzzle is a 3×3 board holding tiles 1–8 and one blank; you slide a tile into the blank to reach a goal arrangement. The slides score each board with:

f(n) = d(n) + w(n)

  • d(n) — the depth of node n (how many moves made so far),
  • w(n) — the number of misplaced tiles in n (tiles not yet in their goal cell).

Lower f is better. At each step hill climbing generates the children, computes f for each, and descends into the child with the smallest f — never looking at any other node again.
