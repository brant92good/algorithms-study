Hill Climbing

A greedy variant of depth-first search: at every step, expand only the locally best-looking child by an evaluation function f(n) — fast, but it sees only one step ahead and can stall at a local optimum.它是 depth-first search 的貪婪版：每一步只用 evaluation function f(n) 挑出當下看起來最好的那個子節點去展開——很快，但它只看得到眼前一步，可能會卡在 local optimum（局部最佳）。

The idea核心想法

Many problems are solved by exploring a solution-space tree: the root is the start state, each child is a state you can reach in one move, and somewhere down the tree sits a goal state. The searching strategy just decides the order in which you expand nodes.很多問題都是靠走一棵 solution-space tree（解空間樹）來解的：根節點是起始狀態，每個子節點是你走一步能到的狀態，而樹的某個地方藏著一個 goal 狀態。所謂搜尋策略，講白了就是決定你用什麼順序展開節點。

Plain depth-first search (DFS) dives down one branch, expanding children in a fixed arbitrary order, and backtracks on dead ends. Hill climbing keeps the dive-deep spirit of DFS but adds judgement: instead of taking children in arbitrary order, it scores each child with an evaluation function f(n) and always expands the locally best one (lowest f). It is the search equivalent of always walking uphill toward higher ground.純 depth-first search（DFS）會一路往一條分支鑽下去，子節點照固定但隨意的順序展開，撞到死路就退回去（backtrack）。Hill climbing 保留了 DFS 那種一直往深鑽的精神，但多了一點判斷力：它不照隨意的順序拿子節點，而是用 evaluation function f(n) 幫每個子節點打分，永遠展開當下最好的那個（f 最小）。換成爬山的比喻，就是每一步都往更高的地方走。

Key intuition — and its trap.關鍵直覺——還有它的陷阱。 Hill climbing has only a local view: it compares a node's children to each other and commits to the best of those, never reconsidering. If the best child still leads nowhere good, hill climbing is stuck — it has climbed a small hill (a local optimum) with no way to see that a taller peak lies across the valley. (Best-first search fixes exactly this by keeping a global view of all open nodes.)Hill climbing 只有局部視野：它拿一個節點的子節點互相比，選出這幾個裡面最好的就下去，之後絕不回頭。如果那個最好的子節點還是通往爛地方，它就卡住了——它爬上了一座小山頭（local optimum），卻完全看不到山谷對面其實有更高的峰。（Best-first search 修的正是這個，靠的是對所有 open 節點保有全域視野。）

When to use it.什麼時候用它。 When you want a quick, low-memory guided search and a decent (not necessarily optimal) solution is acceptable. It expands one node per level along a single path, so it is cheap — but it gives no optimality guarantee and can fail to reach a goal at all. For guaranteed-optimal search you need bounding (branch-and-bound, A*).當你想要一個快又省記憶體的引導式搜尋，而且找到一個還可以（不一定最佳）的解就能接受時。它沿著單一一條路徑、每層只展開一個節點，所以很省——但它不保證最佳，甚至有可能根本走不到目標。如果你要保證最佳解，就得加上界限（branch-and-bound、A*）。

The evaluation function (8-puzzle)evaluation function（拿 8-puzzle 當例子）

The 8-puzzle is a 3×3 board holding tiles 1–8 and one blank; you slide a tile into the blank to reach a goal arrangement. One move = slide a tile that is adjacent to the blank into the blank cell — equivalently, the blank itself steps one square up, down, left, or right and swaps with the tile that was there. Every child board is just the parent after one such blank move. The slides score each board with:8-puzzle 是一個 3×3 的格子，上面有 1–8 的方塊跟一個 blank；你把方塊推進 blank，想辦法排成目標的樣子。一個步（move）就是：把跟 blank 相鄰的某個方塊推進 blank 的格子——換句話說，就是 blank 自己往上、下、左或右走一格，跟那格原本的方塊對調。每個子盤面其實就是父盤面做完這樣一次 blank 移動後的樣子。每個盤面的分數是這樣算的：

f(n) = d(n) + w(n)

  • d(n) — the depth of node n (how many moves made so far),——節點 n 的 depth（深度），也就是到目前為止走了幾步，
  • w(n) — the number of misplaced tiles in n (tiles not yet in their goal cell).——n 裡面 擺錯位置的方塊有幾個（還沒到自己目標格子的方塊）。

Lower f is better. At each step hill climbing generates the children, computes f for each, and descends into the child with the smallest f — never looking at any other node again.f 越小越好。每一步 hill climbing 都把子節點生出來，算出各自的 f，然後鑽進 f 最小的那個——其他節點從此再也不看一眼。

How the children are ordered. The children below are generated in a fixed order — here the blank moves up, then left, then right, then down (whichever of those are legal). That fixed order is what "ties broken by order" means: when two children share the smallest f, hill climbing keeps the one generated first. So in Step 1 both the first and second child have f = 4, and it takes the first one (blank up) only because it was generated earlier — not because it is better.子節點是照什麼順序排的。下面的子節點是用一個固定的順序生出來的——這裡 blank 依序往 up、left、right、down 走（其中合法的才會出現）。這個固定順序就是「同分時照順序決定」的意思：當兩個子節點的 f 一樣小時，hill climbing 留下先生出來的那個。所以在 Step 1 第一跟第二個子節點都是 f = 4，它選第一個（blank 往上）純粹是因為它先被生出來，不是因為它比較好。

Heads-up — this run is a lucky one.提醒——這趟剛好很順。 Here the greedy descent walks straight to the goal, so you are watching hill climbing succeed. That is not guaranteed: on other start boards the lowest-f child can lead into a region where every child has a worse (or no better) f and no goal is reachable — that is the local optimum from the trap above, and because hill climbing never reconsiders the nodes it abandoned, it simply stalls there. The success you see below is the happy case, not the rule.這裡的貪婪下降剛好一路直達目標，所以你看到的是 hill climbing 成功的樣子。但這不是保證的：換成別的起始盤面，f 最小的子節點有可能把你帶進一個區域，那裡每個子節點的 f 都更差（或沒更好）、又走不到目標——這就是上面講的那個 local optimum 陷阱，而且因為 hill climbing 從不回頭看它丟掉的節點，它就只能卡死在那。下面看到的順利只是運氣好的情況，不是常態。
