2-Variable Linear Programming2-Variable Linear Programming

Given a list of straight-line constraints, find the single point (x, y) that makes a linear objective as small as possible — the lowest point of a convex "valley".給你一堆直線形式的限制條件，要找出那一個點 (x, y)，讓某個線性目標值越小越好——也就是一個凸「山谷」的最低點。

The problem這題在問什麼

You have two unknowns, x and y — together they pick a single point in the plane. You want to make a linear objective as small as possible. "Linear" means the objective is just a weighted sum like a·x + b·y; you minimize it. That choice isn't free: it's fenced in by a list of n linear constraints, each an inequality of the form aᵢ·x + bᵢ·y ≤ cᵢ. The general statement is:你有兩個未知數 x 跟 y，它們合起來就是平面上的一個點。你要讓某個線性目標越小越好。「線性」的意思就是目標只是個加權和，像 a·x + b·y 這樣，你要把它最小化。但這個選擇不是隨便挑：它被 n 條線性限制框住，每一條都是 aᵢ·x + bᵢ·y ≤ cᵢ 這種不等式。一般的寫法是：

Minimize  a·x + b·y    subject to   aᵢ·x + bᵢ·y ≤ cᵢ ,   i = 1 … n

Some vocabulary, defined from zero:幾個名詞，從頭講清楚：

  • A constraint aᵢx + bᵢy ≤ cᵢ is a half-plane — it splits the plane with a straight line and keeps everything on one side.一條限制 aᵢx + bᵢy ≤ cᵢ 就是一個 half-plane（半平面）——用一條直線把平面切開，只留下其中一邊。
  • The feasible region is the set of points satisfying all n constraints at once: the intersection of all the half-planes. It is a convex polygon (possibly unbounded).feasible region（可行區域）就是同時滿足全部 n 條限制的點集合：所有 half-plane 的交集。它是一個凸多邊形（可能是無界的）。
  • The optimum is the feasible point with the smallest objective value. We call it (x₀, y₀).optimum（最佳解）就是可行區域裡目標值最小的那個點，我們叫它 (x₀, y₀)。

The canonical, easy-to-picture version.最好想像的標準版本。 The slides reduce the general problem to a tidy special case (substituting y' = a·x + b·y), where you simply minimize y and every constraint is a "must lie on or above this line":投影片把一般問題化簡成一個乾淨的特例（用 y' = a·x + b·y 代換），這時你只要最小化 y，而且每條限制都變成「必須落在這條線上或上方」：

Minimize  y    subject to   y ≥ aᵢ·x + bᵢ ,   i = 1 … n

Each rule y ≥ aᵢx + bᵢ says the point must sit on or above that line. With n rules, the floor you must stay above at each x is the highest of all the lines there — the pointwise maximum, called the upper envelope:每條規則 y ≥ aᵢx + bᵢ 都在說：點必須落在那條線上或上方。有 n 條規則時，每個 x 處你必須踩在上面的那個地板，就是當地最高的那條線——也就是逐點取最大值，叫做 upper envelope（上包絡）：

F(x) = max { a₁x+b₁, a₂x+b₂, …, aₙx+bₙ }

Because it is the max of straight lines, F is a convex "valley": it descends, reaches one bottom, and climbs again — it can never dip twice. So the whole problem collapses to: find the lowest point of F. The answer is that x₀, with y₀ = F(x₀).因為它是一堆直線取最大值，F 是一個 convex（凸）的「山谷」：它先往下走，到一個最低點，再往上爬——絕不會凹兩次。所以整題就濃縮成一句話：找出 F 的最低點。答案就是那個 x₀，搭配 y₀ = F(x₀)。

Tiny concrete instance.來個小例子。 Three constraints y ≥ 2, y ≥ x, y ≥ −x + 4. At x = 1 the three lines give 2, 1, 3, so F(1) = max{2,1,3} = 3. Sweeping x, the valley bottoms out at x = 2 where lines y = x and y = −x + 4 cross at height 2 — there the optimum is (x₀, y₀) = (2, 2).三條限制 y ≥ 2、y ≥ x、y ≥ −x + 4。在 x = 1 時三條線給出 2、1、3，所以 F(1) = max{2,1,3} = 3。讓 x 掃過去，山谷在 x = 2 觸底——這裡 y = x 跟 y = −x + 4 剛好在高度 2 交會，最佳解就是 (x₀, y₀) = (2, 2)。

Four constraint lines below. Step through: first they stack up, then the gold curve traces the upper envelope F(x), then a slope test at a median test point xₘ tells us which way the bottom lies (so the losing line of each pair on the wrong side can be pruned), and we land on the lowest point (x₀, y₀).下面有四條限制線。一步步看：先把它們疊上來，接著金色曲線描出 upper envelope F(x)，然後在一個中位測試點 xₘ 做斜率測試，告訴我們谷底在哪一邊（這樣每一對裡站錯邊的那條輸家就可以被砍掉），最後落在最低點 (x₀, y₀)。

Time:時間： O(n) F(x) = upper envelope Prune:修剪： ~n/4 per round
