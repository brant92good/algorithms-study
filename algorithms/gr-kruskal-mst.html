Greedy

Kruskal's Minimum Spanning Tree

Connect every node of a weighted graph using the cheapest total wiring — by greedily picking the lightest edge that never closes a loop.

What is it?

Imagine a graph: dots (nodes) joined by lines (edges), and each edge has a weight — think of it as the cost or length of a wire. A spanning tree is a set of edges that touches every node and keeps the whole thing connected, with no cycles (no loops). If there are n nodes, a spanning tree always has exactly n − 1 edges — just enough to link everything, not one more.

Many spanning trees are possible. The minimum spanning tree (MST) is the one whose edge weights add up to the smallest total. That is what Kruskal's algorithm finds.

The greedy rule: sort all edges from cheapest to most expensive, then walk through them in that order. For each edge, ask one question — does adding it create a cycle? If no, take it (it safely links two pieces that weren't connected yet). If yes, skip it (its endpoints are already connected, so this edge is redundant). Keep going until you've taken n − 1 edges.

How do we detect a cycle? Think of each node as belonging to a group of already-connected nodes. At the start every node is alone in its own group. When you add an edge, you merge the two groups it joins into one. An edge would form a cycle exactly when both its endpoints are already in the same group — there's already a path between them, so a new edge would just close a loop. (The bookkeeping that tracks and merges these groups is called union-find.)

Why is greedy safe here? Always grabbing the cheapest edge that doesn't make a cycle never paints you into a corner — the lightest edge crossing between two groups is guaranteed to belong to some minimum spanning tree. So the locally cheapest safe choice is also globally optimal.
