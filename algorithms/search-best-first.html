Best-First Search

Keep every unexplored node in one big list, and always expand the one that looks most promising — combining the depth of DFS with the breadth of BFS, but with a global view.

What is it?

Many hard problems become "search a tree of partial solutions": each node is a state, its children are the states you can reach in one move, and a goal node is a solved state. The strategy decides the order you expand nodes.

Hill climbing (the strategy before this one) is greedy and local: it expands a node, looks only at that node's own children, jumps to the best-looking child, and forgets everything else. That is fast but it can stall — if every child of the current node looks bad, hill climbing is stuck even though a better node may be sitting elsewhere in the tree.

Best-first search fixes this. It keeps an OPEN LIST of all nodes discovered but not yet expanded — across the whole tree, not just the current branch. At each step it expands the single node with the best (lowest) estimated cost anywhere in that list. That is the global view: it can abandon a branch that went sour and jump back to a promising node it saw moves ago.

We use the 8-puzzle: a 3×3 board with tiles 1–8 and one blank; slide tiles into the blank to reach the goal 12384765. Each node is scored by

f(n) = d(n) + w(n)

where d(n) = depth (moves made so far) and w(n) = number of tiles not yet in their goal position. Lower f is better. Same scoring as hill climbing — only the selection rule changes (pick the global best, not the local best).

The scheme (4 steps):

  1. Step 1. Form a one-element list containing just the root node.
  2. Step 2. Remove the first element from the list and expand it. If one of its descendants is a goal node, stop; otherwise add the descendants to the list.
  3. Step 3. Sort the entire list by the estimation function f.
  4. Step 4. If the list is empty, report failure. Otherwise go to Step 2.

The sort in Step 3 is the whole trick: after sorting, the first element in Step 2 is always the globally best-looking node, no matter which branch it came from.
