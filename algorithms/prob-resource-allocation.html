You have a fixed pile of resource units and several projects competing for them. Split the pile so the total profit across all projects is as large as possible.你手上有固定數量的資源單位，好幾個 project 都在搶。怎麼分這堆資源，才能讓所有 project 的總 profit 最大？

📄 Source: Alg07 · Ch 7 §9 (Resource Allocation), slides 19–21.

🧠 Solved by: Dynamic Programming

The problem這題在問什麼

You are given m identical units of some resource — money, machines, staff, anything that comes in whole, interchangeable chunks — and n projects that all want some. Handing a project more units earns more profit, but you only have m units total, so you cannot fully fund everyone. You must decide how many units to give each project. The thing you are asked to find: the allocation that maximizes the sum of profits over all n projects.你有 m 個一模一樣的資源 unit——錢、機器、人力，任何以整塊、可互換的形式出現的東西——還有 n 個 project 都想分一點。給一個 project 越多 unit，賺的 profit 越多，但你總共只有 m 個 unit，沒辦法每個都餵飽。你要決定每個 project 各給幾個 unit。要找的就是：讓全部 n 個 project 的 profit 總和最大的那種分配。

How profitable each choice is comes from a table p(i, j) = the profit earned when project i receives j units. Giving a project 0 units earns 0. Here is the deck's concrete instance — m = 3 units to distribute among n = 4 projects:每種選擇能賺多少，看一張表 p(i, j) = project i 拿到 j 個 unit 時賺的 profit。給一個 project 0 個 unit 就賺 0。下面是投影片裡的具體例子——m = 3 個 unit 要分給 n = 4 個 project：

Read it cell by cell: p(1,2) = 8 means "2 units to project 1 earns 8 profit"; p(2,1) = 5 means "1 unit to project 2 earns 5." A valid allocation hands out at most 3 units in total. For example, "2 units to project 1, 1 unit to project 2, 0 to projects 3 and 4" uses all 3 units and earns 8 + 5 = 13. "All 3 units to project 1 alone" uses 3 units and earns p(1,3) = 9. The question is: over every way to split 3 units among the 4 projects, which gives the highest total?一格一格讀：p(1,2) = 8 是「給 project 1 兩個 unit 賺 8 profit」；p(2,1) = 5 是「給 project 2 一個 unit 賺 5」。一個合法的分配總共最多發出 3 個 unit。舉例來說，「project 1 給 2 個、project 2 給 1 個、project 3 和 4 給 0」用掉全部 3 個 unit，賺 8 + 5 = 13。「3 個 unit 全給 project 1」用掉 3 個 unit，賺 p(1,3) = 9。問題是：在所有把 3 個 unit 分給這 4 個 project 的方法裡，哪種總和最高？

Brute force would enumerate every split — wasteful as m and n grow. The smarter route asks a smaller question, one project at a time: "using only the first i projects and q units of budget, what is the best profit?" Call that f(i, q). The key decision when we bring in project i is how many units k to give it (0, 1, … up to q); whatever is left goes to the earlier projects, and we keep whichever k wins:暴力法會把每種分法都列出來——當 m 和 n 一變大就很浪費。聰明一點的做法是一次只問一個 project 的小問題：「只用前 i 個 project、預算 q 個 unit，最佳 profit 是多少？」把它叫做 f(i, q)。輪到 project i 時的關鍵決定是要給它幾個 unit k（0、1、… 一直到 q）；剩下的給前面的 project，哪個 k 贏就留哪個：

f(i, q) = max over k = 0..q of p(i, k) + f(i-1, q - k)

with base row f(1, q) = p(1, q) (only project 1 available, so just take its profit). The answer is f(n, m) = f(4, 3) — the best profit using all 4 projects and the full budget of 3 units. We compute it by filling the table f(i, q) one row at a time below.base 列是 f(1, q) = p(1, q)（只有 project 1 可用，直接拿它的 profit）。答案是 f(n, m) = f(4, 3)——用全部 4 個 project、整個 3 個 unit 預算能拿到的最佳 profit。下面我們就一列一列把表 f(i, q) 填出來，把它算出來。

Primary source: Alg07 · Ch 7 §9 (Resource Allocation), slides 19–21 — the profit table and the DP recurrence, with the optimal split earning 8 + 5 = 13. The "best split over a decision" pattern here is the same one CLRS develops for rod-cutting and matrix-chain ordering (Ch. 15). The deck also redraws this DP as a layered ("multistage") graph where each path corresponds to one allocation, following Horowitz & Sahni, Fundamentals of Computer Algorithms, Ch. 7 — but the table above is fully self-contained, so we solve it that way here.主要出處：Alg07 · Ch 7 §9（Resource Allocation），slides 19–21——profit 表和那條 DP 遞迴式，最佳分法賺 8 + 5 = 13。這裡「在一個決定上取最佳分法」的模式，跟 CLRS 在 rod-cutting 和 matrix-chain ordering（Ch. 15）發展的是同一套。投影片也把這個 DP 改畫成一張分層（multistage）的 graph，每條路徑對應一種分配方式，出自 Horowitz & Sahni, Fundamentals of Computer Algorithms, Ch. 7——不過上面那張表本身就夠完整了，所以我們這裡直接用填表的方式解。

The method: this is one instance of Dynamic Programming — define a subproblem, write a recurrence over one decision, fill a table. Sibling problems in this bucket: Traveling Salesperson (DP / Held–Karp), 0/1 Knapsack, and Longest Common Subsequence.用什麼解：這是 Dynamic Programming 的一個例子——定義子問題、針對一個決定寫遞迴式、填表。同一類的問題：Traveling Salesperson (DP / Held–Karp)、0/1 Knapsack 和 Longest Common Subsequence。
