Dynamic Programming · Ch7

0/1 Knapsack (DP)

Pack the most valuable load into a fixed-capacity bag when each item is all-or-nothing — by building a table of answers to smaller bags.

What is it?

You have a knapsack (a bag) that can carry at most M units of weight. There are n items; item i has weight wᵢ and profit (value) pᵢ. You want to choose a set of items whose total weight fits in the bag and whose total profit is as large as possible.

The "0/1" means each item is taken whole or not at all — you can't slice an item to take half of it. So for every item the decision is a single yes/no: xᵢ = 1 (take it) or xᵢ = 0 (skip it).

The core idea. Look at the items one at a time and split on the first decision. For the very first item you either skip it (the bag is unchanged) or take it (the bag now has wᵢ less room, and you've banked pᵢ profit). Either way, what's left is the same kind of problem on the remaining items — a subproblem. Dynamic programming solves every subproblem once and stores the answers in a table so they're never recomputed.

Write gᵢ(y) for the best profit obtainable using only items after index i (that is, items i+1 … n) with a bag of capacity y. Splitting on the next item gives the recurrence:

gᵢ(y) = max{ gi+1(y) ,  gi+1(y − wi+1) + pi+1 }

The left term is skip item i+1; the right term is take it (only allowed when y ≥ wi+1). The base case is gₙ(y) = 0 for every y (no items left ⇒ no profit), and the final answer is g₀(M).

Tiny example. Items (w,p) = (1,1), (3,4), (4,5), (5,7) and capacity M = 7. We build a table of gᵢ(y), one row per i (from the empty base row g₄ up to g₀) and one column per capacity y = 0…7.

Watch it run

Each row is computed from the row below it. The gold cell is the one being filled; the amber cells are its two sources — skip = same column one row down, take = column y−w one row down, plus p.

Time: O(nM) Space: O(nM) Class: pseudo-polynomial

Quick check 0 / 2 answered

Capacity left is y = 7 and the next item has weight 3, profit 4. From the row below you read gᵢ₊₁(7) = 4 and gᵢ₊₁(4) = 5. What is gᵢ(7)?
In gᵢ(y) = max{ gᵢ₊₁(y), gᵢ₊₁(y−wᵢ₊₁)+pᵢ₊₁ }, what is the base case gₙ(y) for every capacity y?
