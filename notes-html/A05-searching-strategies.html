Overview總覽

This chapter surveys searching strategies that explore a solution-space tree (also called a state-space tree). Many hard problems can be cast as "search a tree of candidate (partial) solutions," and the searching strategy decides the order in which nodes are expanded.這一章在講各種 searching strategies（搜尋策略），重點是怎麼去走一棵 solution-space tree（解空間樹，也叫 state-space tree）。很多難題其實都可以改寫成「在一棵候選（部分）解的樹裡面搜尋」，而搜尋策略決定的就是節點展開的順序。

The chapter is organized in two halves:整章分成兩半：

  1. Search strategies for decision / feasibility problems (find a solution, or decide if one exists): - Satisfiability problem (tree of 2ⁿ assignments) - Hamiltonian circuit problem - Sum-of-subsets problem - Depth-first search (DFS) — guided by a stack - Breadth-first search (BFS) - Hill climbing — a greedy variant of DFS (8-puzzle) - Best-first search — combines DFS + BFS, global view (8-puzzle)給 decision／feasibility 問題用的搜尋策略（找出某一個解，或判斷到底存不存在解）： - Satisfiability problem（2ⁿ 種 assignment 構成的樹） - Hamiltonian circuit problem - Sum-of-subsets problem - Depth-first search (DFS) — 用一個 stack 來帶路 - Breadth-first search (BFS) - Hill climbing — DFS 的 greedy 版本（8-puzzle） - Best-first search — 結合 DFS + BFS，有全域視野（8-puzzle）

  2. Search strategies for optimization problems (find the best solution): - Branch-and-bound — the master strategy for optimization (multi-stage graph) - Personnel assignment problem (cost matrix, partial order of jobs, topological sort) - Travelling salesperson problem (reduced cost matrices, lower bounds, decision tree) - 0/1 knapsack problem (upper/lower bounds via fractional relaxation) - A* algorithm — best-first search with f(n) = g(n) + h(n), h(n) ≤ h*(n) - Channel routing problem (HCG, vertical constraint graph, max cliques, A*)給 optimization 問題用的搜尋策略（找出最好的那個解）： - Branch-and-bound — 解最佳化問題的招牌策略（multi-stage graph） - Personnel assignment problem（cost matrix、jobs 的 partial order、topological sort） - Travelling salesperson problem（reduced cost matrix、lower bound、decision tree） - 0/1 knapsack problem（用 fractional relaxation 算 upper／lower bound） - A* algorithm — best-first search 加上 f(n) = g(n) + h(n)、h(n) ≤ h*(n) - Channel routing problem（HCG、vertical constraint graph、max cliques、A*）

Key reason DFS/BFS/hill-climbing/best-first cannot solve optimization problems directly: they stop at the first goal node found, which need not be optimal. Branch-and-bound and A* add bounding so the optimal goal is guaranteed.為什麼 DFS／BFS／hill-climbing／best-first 沒辦法直接拿來解最佳化問題？關鍵在於：它們碰到第一個 goal node 就停了，但那個不一定是最佳解。Branch-and-bound 跟 A* 多加了 bounding，所以才能保證拿到最佳的 goal。

1. The Satisfiability (SAT) Problem1. Satisfiability (SAT) 問題

Idea想法

Given a Boolean formula in n variables x₁, x₂, …, xₙ, decide whether there is a truth assignment making it true. There are 2ⁿ possible assignments, which can be laid out as a binary tree of depth n: at level i we branch on xᵢ = true vs xᵢ = false (Tree Representation of Eight Assignments shows n = 3 → 2³ = 8 leaves).給你一個有 n 個變數 x₁, x₂, …, xₙ 的布林式，問：有沒有一組 truth assignment 能讓它變成 true？總共有 2ⁿ 種 assignment，可以排成一棵深度 n 的 binary tree：在第 i 層就分岔成 xᵢ = true 跟 xᵢ = false（Tree Representation of Eight Assignments 畫的就是 n = 3 → 2³ = 8 片葉子的情況）。

Problem問題

Depth-First Search (DFS)Depth-First Search (DFS)

  • A stack can be used to guide the depth-first search. 5. Hill Climbing (a variant of DFS) — the 8-Puzzle5. Hill Climbing（DFS 的變形）—— 8-Puzzle

Idea想法

Hill climbing is a variant of depth-first search in which, instead of expanding children in fixed order, the method selects the locally optimal node to expand (the child that looks best by an evaluation function). It is greedy and local — it commits to the locally best child and can get stuck.Hill climbing 是 depth-first search 的變形：它不是照固定順序展開小孩，而是每次挑當下看起來最好的那個節點展開（也就是用某個 evaluation function 評分後最好的那個小孩）。它是 greedy 而且只看局部的 —— 一旦認定了局部最好的小孩就鑽下去，所以有可能會卡住。

8-puzzle evaluation function8-puzzle 的 evaluation function

For the 8-puzzle (a 3×3 board with tiles 1–8 and one blank, slide tiles to reach a goal configuration):8-puzzle 是一個 3×3 的盤面，上面有 1–8 的方塊跟一個空格，你要靠滑動方塊把盤面排成目標樣式：

f(n) = d(n) + w(n)
   d(n) = depth of node n  (number of moves made so far)
   w(n) = number of misplaced tiles in node n

Lower f(n) is better. At each step expand the child minimizing f.f(n) 越小越好。每一步就展開讓 f 最小的那個小孩。

6. Best-First Search6. Best-First Search

Idea想法

Combines depth-first search and breadth-first search. Rather than committing to a local best (hill climbing) or a rigid level order (BFS), best-first selects the node with the best estimated cost among all open nodes anywhere in the tree. Because it can jump to the globally most promising node, this method has a global view.把 depth-first search 跟 breadth-first search 結合起來。它不像 hill climbing 死守局部最好、也不像 BFS 硬照層序走，而是從樹上所有還沒展開的節點裡，挑估計成本最好的那個，不管它在哪。因為它可以直接跳到全域最有希望的節點，所以這個方法是有全域視野的。

  • Fig. 6-16 — An 8-Puzzle Problem Solved by a Best-First Search Scheme (same evaluation function f(n) = d(n) + w(n) as hill climbing, but chosen globally).Fig. 6-16 — An 8-Puzzle Problem Solved by a Best-First Search Scheme（evaluation function 跟 hill climbing 一樣是 f(n) = d(n) + w(n)，差別在於這次是從全域去挑）。

Best-First Search Scheme (explicit Step 1–Step 4)Best-First Search 流程（Step 1–Step 4）

Step 1. Form a one-element list consisting of the root node.Step 1. 建一個只有 root 一個元素的 list。

Step 2. Remove the first element from the list. Expand it. If one of the descendants of this element is a goal node, then stop; otherwise, add the descendants into the list.Step 2. 從 list 拿掉第一個元素，把它展開。如果它的後代裡有一個是 goal node，就停；否則把這些後代全部加進 list。

Step 3. Sort the entire list by the values of some estimation function.Step 3. 用某個 estimation function 的值把整個 list 排序。

Step 4. If the list is empty, then failure. Otherwise, go to Step 2.Step 4. 如果 list 空了，就失敗；否則回到 Step 2。

The sort in Step 3 is what makes the first element in Step 2 always the globally best-looking node.就是 Step 3 那個排序，讓 Step 2 每次拿到的第一個元素，永遠是全域看起來最好的那個節點。

7. The Branch-and-Bound Strategy7. Branch-and-Bound 策略

Idea想法

Branch-and-bound can be used to solve optimization problems — and is needed because DFS, BFS, hill climbing, and best-first search cannot (they halt at the first goal, not the best one).Branch-and-bound 可以拿來解 optimization 問題 —— 而且非用它不可，因為 DFS、BFS、hill climbing、best-first search 都做不到（它們碰到第一個 goal 就停了，不是最佳的那個）。

Two ingredients: - Branching — split the solution space into subsets (subtrees), as in any tree search. - Bounding — compute, for each node, a bound on the best solution obtainable in its subtree. If a node's bound is worse than the best complete solution already found, prune the entire subtree.兩個要件： - Branching（分支） —— 把解空間切成一塊塊子集合（subtree），跟一般 tree search 一樣。 - Bounding（定界） —— 對每個節點算一個 bound，代表它那棵 subtree 裡最好能拿到的解。如果某個節點的 bound 比你目前已經找到的最佳完整解還差，那整棵 subtree 直接砍掉（prune）。

For a minimization problem you maintain a lower bound per node and an upper bound = best feasible solution found so far; prune any node whose lower bound ≥ current best.如果是最小化問題，你會給每個節點維護一個 lower bound，再維護一個 upper bound = 目前找到的最佳可行解；只要某節點的 lower bound ≥ 目前最佳值，就把它砍掉。

8. The Personnel Assignment Problem (Branch-and-Bound)8. Personnel Assignment 問題（Branch-and-Bound）

Problem statement問題描述

  • A linearly ordered set of persons P = {P₁, P₂, …, Pₙ} with P₁ < P₂ < … < Pₙ.一組線性排序（linearly ordered）的人 P = {P₁, P₂, …, Pₙ}，滿足 P₁ < P₂ < … < Pₙ。
  • A partially ordered set of jobs J = {J₁, J₂, …, Jₙ}.一組偏序（partially ordered）的工作 J = {J₁, J₂, …, Jₙ}。
  • Constraint: if Pᵢ and Pⱼ are assigned to jobs f(Pᵢ) and f(Pⱼ), then f(Pᵢ) ≤ f(Pⱼ) whenever Pᵢ ≤ Pⱼ (the assignment must respect the partial order of jobs — a feasible assignment).限制：如果 Pᵢ 跟 Pⱼ 分別被指派到工作 f(Pᵢ) 跟 f(Pⱼ)，那麼只要 Pᵢ ≤ Pⱼ，就必須 f(Pᵢ) ≤ f(Pⱼ)（這個指派要符合 jobs 的 partial order，才算一個可行（feasible）的指派）。
  • Cᵢⱼ = cost of assigning person Pᵢ to job Jⱼ.Cᵢⱼ = 把人 Pᵢ 指派到工作 Jⱼ 的成本。
  • Decision variable: Xᵢⱼ = 1 if Pᵢ is assigned to Jⱼ, else 0.決策變數：Pᵢ 被指派到 Jⱼ 時 Xᵢⱼ = 1，否則為 0。

Objective:目標：

Minimize   Σᵢ Σⱼ Cᵢⱼ · Xᵢⱼ

11. The A* Algorithm11. A* 演算法

Idea想法

A* is used to solve optimization problems with the best-first strategy, but with a special cost function that guarantees optimality. If a feasible solution (goal node) is obtained, then it is optimal and we can stop.A* 是拿 best-first 策略來解 optimization 問題，但配上一個特別的 cost function 來保證最佳性。只要拿到一個可行解（goal node），它就一定是最佳的，可以直接停。

Cost functionCost function

For each node n:對每個節點 n：

f(n) = g(n) + h(n)
   g(n): actual cost from the root to node n.
   h(n): estimated cost from node n to a goal node.
   h*(n): the "real" (true minimum) cost from node n to a goal node.

Admissibility condition: the heuristic must never overestimate:Admissibility 條件：heuristic 絕對不能高估：

h(n) ≤ h*(n)
⇒  f(n) = g(n) + h(n) ≤ g(n) + h*(n) = f*(n)

i.e. f(n) is a lower bound on the best total cost of any solution through n. This is what makes the first goal reached optimal.也就是說 f(n) 是「任何經過 n 的解，其最佳總成本」的一個 lower bound。正是這點，讓第一個碰到的 goal 就是最佳的。

Stopping rule停止條件

Stop iff the selected (expanded) node is also a goal node. (Reaching a goal node in the list is not enough — it must be the node with the smallest f, i.e. the one selected for expansion.)當且僅當你選中（要展開）的那個節點剛好也是 goal node，才停。（光是 list 裡出現一個 goal node 還不夠 —— 它必須是 f 最小、被挑出來要展開的那個。）
