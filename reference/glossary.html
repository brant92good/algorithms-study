Ch6 · Searching StrategiesCh6 · Searching 策略

Solution tree
Tree whose root-to-leaf paths enumerate all candidate solutions; each level fixes one decision. Searching = walking this tree.把所有可能的解攤成一棵樹：從根走到葉子的每一條路就是一個候選解，每往下一層就等於做一個決定。所謂 searching，其實就是在走這棵樹。
DFS (depth-first search)
Expand the deepest unexpanded node first. Implemented with a stack. Backtracks on dead ends.一路往最深還沒展開的節點鑽下去，用 stack 來實作。走到死路就退回上一步（backtrack），換別條路再試。
BFS (breadth-first search)
Expand the shallowest unexpanded node first. Implemented with a queue.跟 DFS 相反：先展開最淺、還沒展開的節點，一層一層往外擴。用 queue 來實作。
Hill climbing
DFS variant: among the children of the current node, expand the locally best one (greedy). No global view — can stall in a local optimum.DFS 的變形：在目前節點的小孩裡，挑當下看起來最好的那個展開（很 greedy）。問題是它只看眼前、沒有全局視野，所以很容易卡在 local optimum 出不來。
Best-first search
Expand the node with the best estimate among all open nodes (global view). Combines DFS + BFS reach.每次都從目前所有 open 節點裡，挑估計值最好的那個來展開——這就是全局視野。等於把 DFS 跟 BFS 能碰到的範圍都涵蓋進來。
Evaluation function f(n)
Estimated promise of node n. 8-puzzle: f(n)=d(n)+w(n), d=depth, w=# misplaced tiles.用來估節點 n「有多值得展開」的分數。以 8-puzzle 為例：f(n)=d(n)+w(n)，d 是深度，w 是放錯位置的方塊數。
Branch-and-bound
Strategy for optimization problems: prune any subtree whose bound can't beat the best solution found. (DFS/BFS/hill/best-first solve only feasibility, not optimization.)專門拿來解optimization問題的招：邊展開搜尋樹，邊算每個子樹的 bound，只要某棵子樹再怎麼好都贏不了目前最佳解，就直接整棵砍掉。（DFS／BFS／hill／best-first 只能判斷有沒有解，沒辦法找最佳解。）
Lower bound (reduced cost matrix)
Subtract the row min from each row, then the column min from each column, so every row & column has a 0. The total subtracted is a lower bound on any tour cost.做法：每一列先減掉該列的最小值，每一行再減掉該行的最小值，弄到每列每行都至少出現一個 0。你總共減掉的數字加起來，就是任何 tour 成本的 lower bound（再怎麼省也不可能比這更低）。
A* algorithm
Best-first for optimization with f(n)=g(n)+h(n): g=cost root→n, h=estimate n→goal. If h(n)≤h*(n) (never overestimates), the first goal popped is optimal.就是拿來解 optimization 的 best-first，評估函數用 f(n)=g(n)+h(n)：g 是 root→n 已經花的成本，h 是估計 n→goal 還要多少。只要 h(n)≤h*(n)（永遠不高估，也就是 admissible），那第一個被拿出來的 goal 就保證是最佳解。

Prune-and-SearchPrune-and-Search

Prune-and-search
Each iteration discards a fixed fraction f of the data, then recurses on the rest. T(n)=T((1−f)n)+O(n^k) ⇒ O(n^k).核心想法是：每一輪都固定丟掉一定比例 f 的資料，剩下的再遞迴下去處理。T(n)=T((1−f)n)+O(n^k) ⇒ O(n^k)。
Selection problem
Find the k-th smallest of n elements. Median = the (n/2)-th smallest.在 n 個元素裡找出第 k 小的那個。中位數（median）就是第 (n/2) 小的元素。
Median of medians
Split into groups of 5, take each group's median, take the median of those as pivot p. Guarantees ≥ n/4 pruned per step ⇒ selection in O(n).每 5 個分一組，先各取各組的 median，再取「這些 median 的 median」當 pivot p。這樣選 pivot 能保證每一步至少砍掉 ≥ n/4 的資料，所以 selection 做得到 O(n)。

Ch7 · Dynamic ProgrammingCh7 · Dynamic Programming

Multistage graph
DAG with vertices partitioned into ordered stages V₁…V_k; edges go stage i→i+1. Goal: min-cost path s→t.一張 DAG，把頂點分成一層一層有順序的 stage V₁…V_k，邊只能從 stage i 連到 i+1。要做的事：找出 s→t 成本最小的那條路。
Principle of optimality
If a sequence of decisions D₁…Dₙ is optimal, then every suffix (last k decisions) is itself optimal. Lets us write a recurrence.如果整串決策 D₁…Dₙ 是最佳的，那它的每個後段（最後 k 個決策）本身一定也是最佳的。有了這個性質，我們才寫得出 recurrence。
Forward reasoning
Express decision xᵢ via optimal sequences for x_{i+1}…xₙ. (Relations formed forward are solved backward.)把決策 xᵢ 用後面 x_{i+1}…xₙ 的最佳序列來表達。（關係式是往前建的，但解的時候要倒著回來算。）
Backward reasoning
Express decision xᵢ via optimal sequences for x₁…x_{i−1}.反過來，把決策 xᵢ 用前面 x₁…x_{i−1} 的最佳序列來表達。
0/1 knapsack (DP)
gᵢ(y)=max{ g_{i+1}(y), g_{i+1}(y−w_{i+1})+p_{i+1} }; answer g₀(M).每個 item 就兩種選擇：不拿、或拿。gᵢ(y)=max{ g_{i+1}(y), g_{i+1}(y−w_{i+1})+p_{i+1} }；最後答案就是 g₀(M)。

Ch8 · NP-CompletenessCh8 · NP-Completeness

P
Problems solvable by a deterministic polynomial-time algorithm.就是那種「有deterministic（一步一步、不用猜）的多項式時間演算法可以解掉」的問題。
NP
Decision problems solvable by a nondeterministic polynomial algorithm = guess a certificate, check it in poly time.能用nondeterministic（可以亂猜）多項式演算法解的decision問題。換個方式講：先猜一個 certificate（解的證據），再在多項式時間內檢查它對不對。
Reduction A → B
A solvable in poly time using a poly-time algorithm for B. "A is no harder than B."意思是：只要手上有解 B 的多項式時間演算法，就能拿來在多項式時間內解掉 A。白話講就是「A 不會比 B 難」。
NP-hard
Every NP problem reduces to it. (May not be in NP — e.g. the halting problem.)代表「每個 NP 問題都能 reduce 到它」，所以它至少跟 NP 裡最難的一樣難。注意它自己不一定在 NP 裡，像 halting problem 就是。
NP-complete (NPC)
NP-hard and in NP. If any NPC problem ∈ P then P = NP.兩個條件都要滿足：既是 NP-hard，又剛好在 NP 裡。這群是 NP 裡最難的。重點是：只要有任何一個 NPC 問題被證明 ∈ P，那就直接推出 P = NP 了。
Nondeterministic algorithm
Two phases: choice() guesses; then a deterministic check. NP if the check is polynomial.分兩段：先用 choice() 猜一個答案，再老老實實做一次 deterministic 檢查。只要那個檢查是多項式時間，這問題就屬於 NP。
Cook's theorem
SAT is NP-complete — the first one. Every NP problem reduces to SAT.證明了 SAT 是 NP-complete，而且是史上第一個。它的厲害之處在於：每個 NP 問題都能 reduce 到 SAT，等於替後面所有 NPC 證明開了路。
Decision vs optimization
Decision answers Yes/No ("is there a tour ≤ C?"). Solve optimization by binary-searching C with the decision algorithm.Decision 版只回答 Yes／No（「有沒有成本 ≤ C 的 tour？」）。想解 optimization 也不難：拿 decision 演算法當工具，對 C 做 binary search，就能逼出最佳值。
