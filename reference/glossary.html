Glossary

Shared vocabulary for the Algorithms final. Every lesson uses these exact terms.

Ch6 · Searching Strategies

Solution tree
Tree whose root-to-leaf paths enumerate all candidate solutions; each level fixes one decision. Searching = walking this tree.
DFS (depth-first search)
Expand the deepest unexpanded node first. Implemented with a stack. Backtracks on dead ends.
BFS (breadth-first search)
Expand the shallowest unexpanded node first. Implemented with a queue.
Hill climbing
DFS variant: among the children of the current node, expand the locally best one (greedy). No global view — can stall in a local optimum.
Best-first search
Expand the node with the best estimate among all open nodes (global view). Combines DFS + BFS reach.
Evaluation function f(n)
Estimated promise of node n. 8-puzzle: f(n)=d(n)+w(n), d=depth, w=# misplaced tiles.
Branch-and-bound
Strategy for optimization problems: prune any subtree whose bound can't beat the best solution found. (DFS/BFS/hill/best-first solve only feasibility, not optimization.)
Lower bound (reduced cost matrix)
Subtract the row min from each row, then the column min from each column, so every row & column has a 0. The total subtracted is a lower bound on any tour cost.
A* algorithm
Best-first for optimization with f(n)=g(n)+h(n): g=cost root→n, h=estimate n→goal. If h(n)≤h*(n) (never overestimates), the first goal popped is optimal.

Prune-and-Search

Prune-and-search
Each iteration discards a fixed fraction f of the data, then recurses on the rest. T(n)=T((1−f)n)+O(n^k) ⇒ O(n^k).
Selection problem
Find the k-th smallest of n elements. Median = the (n/2)-th smallest.
Median of medians
Split into groups of 5, take each group's median, take the median of those as pivot p. Guarantees ≥ n/4 pruned per step ⇒ selection in O(n).

Ch7 · Dynamic Programming

Multistage graph
DAG with vertices partitioned into ordered stages V₁…V_k; edges go stage i→i+1. Goal: min-cost path s→t.
Principle of optimality
If a sequence of decisions D₁…Dₙ is optimal, then every suffix (last k decisions) is itself optimal. Lets us write a recurrence.
Forward reasoning
Express decision xᵢ via optimal sequences for x_{i+1}…xₙ. (Relations formed forward are solved backward.)
Backward reasoning
Express decision xᵢ via optimal sequences for x₁…x_{i−1}.
0/1 knapsack (DP)
gᵢ(y)=max{ g_{i+1}(y), g_{i+1}(y−w_{i+1})+p_{i+1} }; answer g₀(M).

Ch8 · NP-Completeness

P
Problems solvable by a deterministic polynomial-time algorithm.
NP
Decision problems solvable by a nondeterministic polynomial algorithm = guess a certificate, check it in poly time.
Reduction A → B
A solvable in poly time using a poly-time algorithm for B. "A is no harder than B."
NP-hard
Every NP problem reduces to it. (May not be in NP — e.g. the halting problem.)
NP-complete (NPC)
NP-hard and in NP. If any NPC problem ∈ P then P = NP.
Nondeterministic algorithm
Two phases: choice() guesses; then a deterministic check. NP if the check is polynomial.
Cook's theorem
SAT is NP-complete — the first one. Every NP problem reduces to SAT.
Decision vs optimization
Decision answers Yes/No ("is there a tour ≤ C?"). Solve optimization by binary-searching C with the decision algorithm.
